Channels & Producer/Consumer

When one part of your app produces work faster than another can consume it, a channel is the clean, thread-safe, async pipe between them — with backpressure built in.

#concurrency #channels #backpressure #async

The idea in plain English

The producer/consumer pattern decouples whoever creates work from whoever processes it via a queue in between. This smooths bursts, lets you scale producers and consumers independently, and keeps each side simple.

System.Threading.Channels is .NET's modern, async-first implementation. Producers await writer.WriteAsync(item) and consumers await foreach over the reader — all thread-safe, no manual locks. It replaces the older BlockingCollection for async code.

The killer feature is backpressure: a bounded channel has a capacity, and when it's full, WriteAsync waits rather than letting the producer race ahead and blow up memory. This naturally throttles a fast producer to the consumer's pace — the safe default for any real pipeline.

Channels come in single/multi producer and consumer flavors and let you pick the behavior when full (wait, drop oldest, drop newest). They're the building block for ingestion pipelines, work queues, and event fan-out inside a process.

Real-world example

🏢 How this works in production

An ingestion service that receives events over HTTP and writes them to a database in batches is a textbook channel: the HTTP handlers are producers writing to a bounded channel, and a background consumer drains it, batching writes. If the database slows, the bounded channel fills, WriteAsync backs up, and the pressure propagates to the producers (who return 503) instead of the service running out of memory — exactly the graceful degradation you want.

A bounded channel between producer and consumer

In C#

C# 
// Bounded channel = built-in backpressure.
var channel = Channel.CreateBounded<Event>(new BoundedChannelOptions(1000)
{
    FullMode = BoundedChannelFullMode.Wait     // producer awaits when full
});

// Producer (e.g. an HTTP handler):
await channel.Writer.WriteAsync(evt);          // waits if the channel is full

// Consumer (a background service): drains and batches.
await foreach (var e in channel.Reader.ReadAllAsync(ct))
    await SaveAsync(e);

Key takeaways

🎯 Remember these three

  • Channels give a thread-safe, async producer/consumer pipe — no manual locking.
  • A bounded channel provides backpressure: a full channel makes the producer wait, keeping memory bounded.
  • It's the modern async replacement for BlockingCollection and the building block for in-process pipelines.
