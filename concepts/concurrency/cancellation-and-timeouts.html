Cancellation & Timeouts

You can't safely kill a thread, so .NET uses cooperative cancellation: you pass a CancellationToken, and well-behaved code checks it and bows out cleanly.

#concurrency #cancellation #timeouts #async

The idea in plain English

Forcibly aborting a thread leaves locks held and state half-written — so .NET doesn't do it. Instead, cancellation is cooperative: a CancellationTokenSource produces a CancellationToken, you thread that token through your async calls, and each operation periodically checks it (or the framework checks it for you) and throws OperationCanceledException to unwind cleanly.

Timeouts are just cancellation on a timer: new CancellationTokenSource(TimeSpan.FromSeconds(5)) cancels its token after five seconds, so any awaited call observing that token gives up. This is how you stop a slow dependency from hanging a request forever — the single most important resilience habit.

Tokens compose: CreateLinkedTokenSource combines, say, the request's cancellation token with a timeout token, so the operation cancels if either fires. Pass the token everywhere — into HttpClient, EF Core, your own loops — so cancellation propagates end to end.

Honor it: check token.ThrowIfCancellationRequested() in long loops, and treat OperationCanceledException as a normal, expected outcome (not an error to log as a failure).

Real-world example

🏢 How this works in production

ASP.NET Core hands every request a HttpContext.RequestAborted token; pass it down and when the client disconnects, the database query and downstream calls cancel automatically instead of wasting work on a response nobody will read. Combine it with a timeout token and you get the resilience pattern every service needs — a slow dependency can't tie up a request forever, which is the foundation of not cascading failures.

Cooperative cancellation flows through the call chain

In C#

C# 
// Timeout = a token that cancels itself after a delay.
using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(5));

// Pass the token through EVERY async call so cancellation propagates.
var data = await httpClient.GetStringAsync(url, cts.Token);   // gives up after 5s

// Combine a request token with a timeout — cancels if either fires.
using var linked = CancellationTokenSource.CreateLinkedTokenSource(
    requestToken, cts.Token);

// In a long loop, cooperate:

Key takeaways

🎯 Remember these three

  • Cancellation is cooperative — pass a CancellationToken through every async call; code checks it and throws OperationCanceledException.
  • A timeout is cancellation on a timer (CancellationTokenSource(TimeSpan)) — the key to not hanging on slow dependencies.
  • Link tokens to combine timeout + request cancellation; treat cancellation as expected, not an error.
