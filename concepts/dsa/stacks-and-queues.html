Stacks & Queues

Two simple structures defined by where you add and remove: a stack is last-in-first-out, a queue is first-in-first-out. Both are O(1) at the ends — and both are everywhere.

#dsa #stack #queue #data-structures

The idea in plain English

A stack is LIFO (Last In, First Out): you push onto the top and pop from the top, like a stack of plates. The most recently added item is the first one out. Push, pop, and peek are all O(1).

A queue is FIFO (First In, First Out): you enqueue at the back and dequeue from the front, like a line at a checkout. The oldest item is served first. Enqueue and dequeue are O(1).

Their power is matching a structure to a problem's natural order. Stacks model anything with nesting or reversal: the call stack (function calls/recursion), undo/redo, browser back, expression/bracket parsing, and depth-first search. Queues model fair, in-order processing: task scheduling, message buffers, request handling, and breadth-first search (which explores level by level because it uses a queue).

Variants extend the idea: a deque (double-ended queue) allows O(1) at both ends; a priority queue (a heap) dequeues by priority rather than arrival order. Choosing stack vs queue is really choosing whether you want to revisit the newest or the oldest item next.

Real-world example

🏢 How this works in production

These are foundational plumbing: the call stack is how every program tracks function calls and local variables (and stack overflow is literally exhausting it via deep recursion). Message queues (RabbitMQ, SQS) decouple producers from consumers in distributed systems. And the choice between them defines two core algorithms — DFS uses a stack (go deep), BFS uses a queue (go wide) — which is why so many graph problems hinge on picking the right one.

LIFO vs FIFO

In C#

C# 
// Stack — LIFO. Natural for nesting/reversal (DFS, undo, bracket matching).
var stack = new Stack<int>();
stack.Push(1); stack.Push(2);
int top = stack.Pop();           // 2 — last in, first out — O(1)

// Queue — FIFO. Natural for in-order processing (BFS, scheduling, buffers).
var queue = new Queue<int>();
queue.Enqueue(1); queue.Enqueue(2);
int first = queue.Dequeue();     // 1 — first in, first out — O(1)

// DFS uses a stack (go deep); BFS uses a queue (go wide).

Key takeaways

🎯 Remember these three

  • Stack = LIFO (push/pop the top); queue = FIFO (enqueue back, dequeue front) — both O(1) at their ends.
  • Stacks model nesting/reversal (call stack, undo, parsing, DFS); queues model in-order/fair processing (scheduling, buffers, BFS).
  • Variants: deque (both ends) and priority queue/heap (dequeue by priority, not arrival).
