How Gmail Works

Webmail looks simple, but at a billion-plus users it’s a master class in storage at scale, instant search, and real-time delivery.

#email #search-index #storage #case-study

The idea in plain English

Gmail has a few hard problems happening at once:

  • Storage — astronomical volumes of mail, sharded by user across many servers so each user’s mailbox is self-contained and the system scales horizontally. Attachments live in object storage; messages are deduplicated where possible.
  • Search — “instant” search across years of mail is powered by an inverted index (word → list of messages containing it), built per user as mail arrives. You’re not scanning your mailbox; you’re looking up a pre-built index.
  • Real-time delivery — new mail appears without refreshing because the client holds a long-lived connection (long polling / push) and the server pushes updates the instant they land.
  • Labels, not folders — a message is stored once and tagged with many labels (Inbox, Work, Important). A “folder” move is just a label change — far more flexible than copying files between folders.
  • Spam filtering — incoming mail runs through ML classifiers (and reputation/SPF/DKIM checks) before it ever reaches your inbox.

Sending vs receiving are different paths: outbound goes through SMTP to other mail servers; inbound is received, filtered, indexed, stored, and pushed to your open clients. State (your auth) rides on a session; everything syncs across your devices from the same server-side mailbox.

Real-world example

🏢 How this works in production

When someone emails you, Gmail receives it, runs spam/phishing ML, updates your personal inverted index so it’s instantly searchable, applies labels (maybe auto-categorizing it as “Promotions”), and pushes it to your open tab and phone in real time — all in well under a second. Search “invoice from 2019” and it’s an index lookup, not a scan. The label model is why the same message can appear under Inbox and Important and a custom label without ever being duplicated.

Key takeaways

🎯 Remember these three

  • Storage is sharded by user (horizontal scale); attachments go to object storage.
  • Instant search is an inverted index (word → messages), not a mailbox scan; delivery is real-time push over a long-lived connection.
  • Labels (one message, many tags) beat folders; inbound mail is ML-spam-filtered before it lands.

🗣️ When this comes up

“Design Gmail/an email system” → hit four pillars: sharded storage, an inverted index for search, real-time push delivery, and the label model (one message, many tags). Adding spam ML and the SMTP send vs push receive split shows breadth.
