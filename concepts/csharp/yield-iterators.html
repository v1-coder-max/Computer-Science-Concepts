yield & Iterators

yield return lets a method hand back a sequence one element at a time, on demand — the compiler turns it into a state machine so you don’t build the whole list up front.

#yield #iterators #lazy #state-machine

The idea in plain English

Normally a method that returns a sequence builds the whole collection, then returns it. An iterator method — one that uses yield return — is different: it produces values lazily. Each time the consumer asks for the next item (every loop of a foreach), the method runs until the next yield return, hands back that value, and pauses — remembering exactly where it was, including local variables. yield break ends the sequence.

The compiler makes this work by rewriting your method into a state machine that implements IEnumerator<T> (MoveNext() + Current) — the very same machinery as async/await, just for sequences instead of awaits. You write simple sequential code; the compiler generates the suspend/resume bookkeeping.

Why it matters:

  • Deferred execution — nothing runs until you enumerate (this is what powers LINQ).
  • Memory efficiency — you never materialize the whole sequence; stream millions of rows one at a time.
  • Infinite/early-stop sequences — generate forever and let the consumer Take(10) or break.

Real-world example

🏢 How this works in production

Streaming a huge file or query results row-by-row uses iterators so you never load the whole dataset into memory — process line 1, then line 2, holding just one at a time. LINQ’s operators (Where, Select) are iterators, which is exactly why LINQ is lazy and composable. Generators of infinite sequences (Fibonacci, ids, retry delays) rely on yield plus a consumer that stops when it has enough. Anytime you want to produce a sequence without paying for the whole thing up front, yield is the tool.

Produce on demand, pause between

In C#

C# 
// Lazy: produces values on demand — nothing runs until enumerated.
public static IEnumerable<int> EvensUpTo(int max)
{
    for (int i = 0; i <= max; i += 2)
        yield return i;     // hand back i, then PAUSE here until the next MoveNext
}

// Infinite sequence + consumer stops early — only computes what's needed.
public static IEnumerable<long> Fibonacci()
{
    long a = 0, b = 1;
    while (true) { yield return a; (a, b) = (b, a + b); }   // never-ending
}
var first10 = Fibonacci().Take(10).ToList();   // computes exactly 10

// Memory-efficient streaming — one line in memory at a time, not the whole file.
public static IEnumerable<string> ReadLines(string path)
{
    using var reader = new StreamReader(path);
    string? line;
    while ((line = reader.ReadLine()) is not null)
        yield return line;
}
// The compiler rewrites each of these into an IEnumerator<T> state machine.

Key takeaways

🎯 Remember these three

  • yield return produces a sequence lazily, one item at a time; the compiler generates an IEnumerator<T> state machine (same idea as async).
  • It enables deferred execution, memory-efficient streaming, and infinite/early-stop sequences.
  • LINQ operators are iterators — which is why LINQ is lazy and composable.
