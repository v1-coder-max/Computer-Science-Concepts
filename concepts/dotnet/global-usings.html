Global Usings

Declare a using once for the whole project instead of repeating the same stack of imports at the top of every file.

#global-usings #implicit-usings #csharp-10 #boilerplate

The idea in plain English

Every C# file used to start with the same ten using lines — System, System.Collections.Generic, System.Linq, and friends. Global usings (C# 10) let you write a using a single time, prefixed with global, and it applies to every file in the project: 

global using System.Linq;

On top of that, the .NET SDK has implicit usings (<ImplicitUsings>enable</ImplicitUsings>, on by default in new projects). It auto-adds the common namespaces appropriate to your project type — which is exactly why a minimal API’s Program.cs can call WebApplication.CreateBuilder with no using in sight.

Convention is to keep your own global usings in one file (e.g. GlobalUsings.cs) or declare them in the .csproj. They also support the static and alias forms, so project-wide helpers and type aliases are available everywhere without repetition.

Real-world example

🏢 How this works in production

On a large service, teams collect the namespaces that appear in nearly every file — the result type, common DTO namespace, logging, System.Text.Json — into a single GlobalUsings.cs. New files start with the actual code instead of a 12-line import preamble, diffs get quieter, and onboarding is a touch easier because the “ambient” namespaces are declared in one obvious place. It’s a small ergonomic win that compounds across thousands of files.

One declaration, every file

In C#

C# — GlobalUsings.cs 
global using System.Text.Json;
global using System.Collections.Generic;

// Bring static members in everywhere (e.g. Math.Max without the 'Math.'):
global using static System.Math;

// Project-wide alias for a long or ambiguous type:
global using JsonDoc = System.Text.Json.JsonDocument;

Or declare them in the project file (and let the SDK add the usual ones):

XML — .csproj 
<PropertyGroup>
  <ImplicitUsings>enable</ImplicitUsings>  <!-- SDK adds System, Linq, etc. -->
</PropertyGroup>
<ItemGroup>
  <Using Include="Serilog" />             <!-- your own global using -->
</ItemGroup>

Key takeaways

🎯 Remember these three

  • global using applies a namespace import to the entire project — declared once, not per file.
  • ImplicitUsings makes the SDK add the common namespaces for your project type automatically.
  • Keep them in one place (GlobalUsings.cs or the .csproj); they support static and alias forms too.

Interview tip

🗣️ When this comes up

Usually a quick aside: “why doesn’t the minimal-API template have any using statements?” Answer: implicit usings (the SDK adds them) plus C# 10 global usings. A clean detail to mention is keeping them in a single GlobalUsings.cs for discoverability.
