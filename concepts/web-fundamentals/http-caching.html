HTTP Caching

The cheapest request is the one you never make. HTTP caching lets browsers, CDNs, and proxies reuse responses instead of re-downloading them.

#caching #cache-control #etag #performance

The idea in plain English

HTTP caching works on two ideas — freshness and validation.

  • Freshness — the server stamps a response with Cache-Control: max-age=N. For N seconds, the cache serves the stored copy without contacting the server at all. Fastest path.
  • Validation — once stale, the cache doesn’t blindly re-download. It asks “is my copy still good?” using a validator: an ETag (a content fingerprint) with If-None-Match, or Last-Modified with If-Modified-Since. If unchanged, the server replies 304 Not Modified with no body — saving the bandwidth.

The Cache-Control directives people confuse:

  • no-cache — you may store it, but must revalidate before each use (not “don’t cache”!).
  • no-store — never store it (for sensitive data).
  • private vs public — only the user’s browser may cache vs shared caches/CDNs may too.

For assets that can change, the trick is cache busting: cache aggressively (a year) but put a content hash in the filename (app.9f3a.js), so a new build = a new URL = a guaranteed fresh fetch.

Real-world example

🏢 How this works in production

Every fast site does this. YouTube/Netflix serve images, JS, and CSS with Cache-Control: max-age=31536000, immutable and fingerprinted filenames, so repeat visits load instantly from the browser/CDN cache and a deploy still ships new code (new hash, new URL). API responses that change use ETags so the client revalidates cheaply — a 304 on an unchanged resource is a few header bytes instead of a full payload, which at CDN scale saves enormous bandwidth.

Fresh, or revalidate

On the wire & in C#

HTTP 
HTTP/1.1 200 OK
Cache-Control: public, max-age=31536000, immutable
ETag: "9f3a1c"

# Later, once stale, the browser revalidates:
GET /app.js HTTP/1.1
If-None-Match: "9f3a1c"

HTTP/1.1 304 Not Modified        # unchanged → no body, reuse the cached copy
C# — ASP.NET Core 
app.MapGet("/products", () =>
{
    var body = Serialize(products);
    return Results.Text(body, "application/json")
        .WithHeaders(h => h.CacheControl = new() { Public = true, MaxAge = TimeSpan.FromMinutes(5) });
});

// Static files: long max-age + fingerprinted names (asp-append-version) = safe cache busting.

Key takeaways

🎯 Remember these three

  • Freshness (max-age) serves from cache with no network; validation (ETag/Last-Modified) yields a cheap 304 when unchanged.
  • no-cache = revalidate before use (not “don’t cache”); no-store = never store.
  • Cache static assets hard and bust the cache with content-hashed filenames.
