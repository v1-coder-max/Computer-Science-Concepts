How DNS Works

DNS is the internet’s phone book — it turns a human name like example.com into the IP address a computer actually connects to.

#dns #resolution #records #ttl

The idea in plain English

Every request starts with a question: “what IP is example.com?” Answering it is DNS resolution, and it walks a hierarchy, checking caches at each step:

  1. Caches first — browser cache, then OS cache.
  2. Recursive resolver — usually your ISP’s (or 1.1.1.1 / 8.8.8.8). It does the legwork.
  3. Root servers — point to the right TLD server.
  4. TLD servers — .com, .org — point to the domain’s authoritative server.
  5. Authoritative nameserver — holds the real records and returns the IP.

DNS stores several record types:

  • A / AAAA — name → IPv4 / IPv6 address.
  • CNAME — alias one name to another.
  • MX — mail servers; TXT — verification/SPF; NS — which nameservers are authoritative.

Every answer carries a TTL (time-to-live) telling resolvers how long to cache it. High TTL = fast and cheap but slow to change; low TTL = nimble (handy before a migration) but more lookups. That cache is also why DNS changes take time to propagate.

Real-world example

🏢 How this works in production

CDNs lean hard on DNS. When you load Netflix or a Cloudflare-fronted site, GeoDNS/anycast returns a different IP depending on where you are, steering you to the nearest edge server for low latency. Teams planning a server migration drop the record’s TTL to a minute days ahead, so when they flip the IP the change propagates almost immediately instead of being cached for hours. DNS-based load balancing and failover work the same way.

The resolution chain

Looking it up

Bash 
# See the records and TTL for a domain:
dig example.com A          # IPv4 address record
dig example.com MX         # mail servers
nslookup example.com       # quick name → IP

# ANSWER SECTION:
# example.com.  300  IN  A  93.184.216.34     ← 300 = TTL in seconds
C# 
IPAddress[] ips = await System.Net.Dns.GetHostAddressesAsync("example.com");
foreach (var ip in ips) Console.WriteLine(ip);  // resolves via the OS resolver

Key takeaways

🎯 Remember these three

  • DNS maps names to IPs via a cached hierarchy: resolver → root → TLD → authoritative.
  • Key record types: A/AAAA (address), CNAME (alias), MX (mail), TXT, NS.
  • TTL controls caching — low TTL before changes for fast propagation, high TTL for fewer lookups.

Interview tip

🗣️ When this comes up

“What happens when you type a URL and press enter?” starts with DNS. Walk the cache → resolver → root → TLD → authoritative chain, mention TTL/caching, and note that CDNs use GeoDNS to route you to the nearest edge. Knowing A vs CNAME vs MX records rounds it out.
