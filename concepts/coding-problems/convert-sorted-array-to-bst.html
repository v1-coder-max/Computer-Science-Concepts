Convert Sorted Array to BST

Turn a sorted array into a height-balanced BST. Pick the middle as the root and recurse on each half — balance for free.

#trees #bst #recursion #easy #divide-and-conquer #interview

The problem

Given an integer array sorted in ascending order, build a height-balanced binary search tree (the depths of the two subtrees of every node differ by at most 1).

📥 Example

[-10,-3,0,5,9] → a balanced BST rooted at 0 (e.g. [0,-3,9,-10,null,5])

Brute force

Inserting elements one by one into a BST risks an unbalanced (even degenerate) tree and is O(n log n) at best. Choosing the middle each time guarantees balance in O(n).

Optimized approach

For a sorted array, the middle element makes an ideal root: everything left is smaller (left subtree), everything right is larger (right subtree), and both halves have nearly equal size, so the tree stays balanced. Recurse: root = mid, build the left subtree from [lo, mid−1] and the right from [mid+1, hi].

Time O(n), Space O(log n).

C# solution

C# 
public TreeNode SortedArrayToBST(int[] nums)
{
    return Build(0, nums.Length - 1);

    TreeNode Build(int lo, int hi)
    {
        if (lo > hi) return null;
        int mid = lo + (hi - lo) / 2;
        return new TreeNode
        {
            val = nums[mid],
            left = Build(lo, mid - 1),
            right = Build(mid + 1, hi)
        };
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — each element becomes one node.
  • Space: O(log n) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • The middle element as root keeps subtrees balanced.
  • Recurse on the left and right halves.
  • Mirrors binary search’s divide structure.
