Partition List

Thread two separate lists — one for values below x, one for the rest — then splice them. Original order is preserved automatically.

#linked-list #two-pointers #medium

The problem

Given the head of a linked list and a value x, reorder it so all nodes with value < x come before nodes with value ≥ x, preserving the relative order within each group.

📥 Example

[1,4,3,2,5,2], x = 3 → [1,2,2,4,3,5]

Brute force

Repeatedly scanning and moving nodes is messy and slow. Building two output lists in a single pass is clean and O(n).

Optimized approach

Use two dummy heads, less and greater. Walk the list once, appending each node to the appropriate list. Finally connect less's tail to greater's head and terminate the second list.

Time O(n), Space O(1).

C# solution

C# 
public ListNode Partition(ListNode head, int x)
{
    var lessDummy = new ListNode(0);
    var greatDummy = new ListNode(0);
    ListNode less = lessDummy, great = greatDummy;
    for (var cur = head; cur != null; cur = cur.next)
    {
        if (cur.val < x) { less.next = cur; less = cur; }
        else { great.next = cur; great = cur; }
    }
    great.next = null;                 // terminate the second list
    less.next = greatDummy.next;       // splice
    return lessDummy.next;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — two dummy heads.

Pattern & takeaways

🎯 Remember these three

  • Two output lists keep each group's order without comparisons.
  • Always terminate the second list (great.next = null) to avoid a cycle.
  • Dummy heads make the splice a two-line finish.

Interview tip

🗣️ When this comes up

Forgetting to null-terminate the greater list creates a cycle — the classic bug. Mention it. The two-list-then-splice pattern recurs (odd/even, merge).
