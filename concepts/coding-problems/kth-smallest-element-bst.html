Kth Smallest Element in a BST

An in-order traversal of a BST visits values in sorted order — so the kth value you visit is the answer. Stop early.

#trees #bst #in-order #medium #traversal #interview

The problem

Given the root of a BST and an integer k, return the kth smallest value (1-indexed).

📥 Example

root = [3,1,4,null,2], k = 1 → 1  ·  k = 3 → 3

The idea

In a BST, in-order (left → node → right) produces values in ascending order. So traverse in-order and count; the kth node visited is the kth smallest. An iterative in-order with an explicit stack lets you stop the instant you reach k without walking the whole tree.

Time O(h + k), Space O(h).

C# solution

C# 
public int KthSmallest(TreeNode root, int k)
{
    var stack = new Stack<TreeNode>();
    var node = root;
    while (node != null || stack.Count > 0)
    {
        while (node != null) { stack.Push(node); node = node.left; }  // go left
        node = stack.Pop();
        if (--k == 0) return node.val;        // kth smallest
        node = node.right;                    // then right
    }
    return -1;   // k out of range
}

Complexity

⏱️ Complexity

  • Time: O(h + k) — descend to the smallest, then pop k nodes.
  • Space: O(h) for the stack.

Pattern & takeaways

🎯 Remember these three

  • In-order on a BST is sorted — the kth visited node is the kth smallest.
  • Iterative in-order lets you stop at k instead of traversing all n.
  • For many repeated queries, augment nodes with subtree sizes for O(h).

Interview tip

🗣️ When this comes up

The one fact that solves it: in-order traversal of a BST is sorted. Use the iterative version so you can stop at k. If asked about frequent kth queries with updates, mention augmenting nodes with subtree counts.
