Redundant Connection

A tree plus one extra edge has exactly one cycle. Union-Find spots the culprit: the first edge connecting two already-joined nodes.

#graphs #union-find #medium #cycle #dsu #interview

The problem

Given a graph that started as a tree with n nodes and had one extra edge added (so it has one cycle), return that redundant edge — the last one in the input that forms the cycle.

📥 Example

[[1,2],[1,3],[2,3]] → [2,3]

Brute force

For each edge, remove it and check if the rest is still a connected acyclic tree (DFS) — O(n²). Union-Find answers it in near-linear time by detecting the cycle as edges are added.

Optimized approach

Process edges with a Disjoint Set Union. For each edge (u, v), find their roots: if they’re already in the same set, this edge closes a cycle → it’s the redundant one. Otherwise union them. With path compression and union by rank, each operation is effectively O(1).

Time ~O(n·α(n)), Space O(n).

C# solution

C# 
public int[] FindRedundantConnection(int[][] edges)
{
    int n = edges.Length;
    var parent = new int[n + 1];
    for (int i = 0; i <= n; i++) parent[i] = i;

    int Find(int x) => parent[x] == x ? x : (parent[x] = Find(parent[x]));  // path compression

    foreach (var e in edges)
    {
        int ru = Find(e[0]), rv = Find(e[1]);
        if (ru == rv) return e;          // already connected → redundant edge
        parent[ru] = rv;                 // union
    }
    return Array.Empty<int>();
}

Complexity

⏱️ Complexity

  • Time: ~O(n·α(n)) with path compression.
  • Space: O(n) for the parent array.

Pattern & takeaways

🎯 Remember these three

  • Cycle detection while building → Union-Find.
  • Same-root find before union means this edge closes a cycle.
  • Path compression keeps operations effectively constant time.
