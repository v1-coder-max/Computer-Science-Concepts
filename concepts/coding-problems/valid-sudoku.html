Valid Sudoku

Is the 9×9 board valid so far? Track seen digits per row, column, and 3×3 box with sets — any repeat fails.

#matrix #hashing #medium #constraints #sets #interview

The problem

Determine if a partially filled 9×9 Sudoku board is valid: each row, each column, and each of the nine 3×3 sub-boxes must contain the digits 1–9 without repetition. Empty cells are ..

📥 Example

A board with a repeated 8 in a column → false; otherwise → true.

Brute force

Separately scan each row, each column, and each box for duplicates — three passes, more code, easy to get the box indexing wrong. One pass with three sets per constraint is cleaner.

Optimized approach

Use sets keyed by constraint: one per row, one per column, and one per box (box index = (r/3)*3 + c/3). Scan every filled cell once; if its digit already appears in that cell’s row, column, or box set, the board is invalid. Otherwise add it to all three. The board size is fixed (81 cells), so this is effectively constant work.

Time O(1) (fixed 9×9), Space O(1).

C# solution

C# 
public bool IsValidSudoku(char[][] board)
{
    var rows = new HashSet<char>[9];
    var cols = new HashSet<char>[9];
    var boxes = new HashSet<char>[9];
    for (int i = 0; i < 9; i++) { rows[i] = new(); cols[i] = new(); boxes[i] = new(); }

    for (int r = 0; r < 9; r++)
        for (int c = 0; c < 9; c++)
        {
            char d = board[r][c];
            if (d == '.') continue;
            int b = (r / 3) * 3 + c / 3;
            if (!rows[r].Add(d) || !cols[c].Add(d) || !boxes[b].Add(d))
                return false;          // Add returns false if already present
        }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(1) — fixed 81 cells (O(n²) for a general n×n).
  • Space: O(1) — 27 small sets.

Pattern & takeaways

🎯 Remember these three

  • Three constraint sets — row, column, 3×3 box — in one pass.
  • Box index formula: (r/3)*3 + c/3.
  • HashSet.Add returning false is a tidy duplicate check.

Interview tip

🗣️ When this comes up

The only tricky part is the box index formula — get that right and the rest is set membership. One pass beats three separate scans; mention you only validate, not solve (that’s the backtracking follow-up).
