Last Stone Weight

Smash the two heaviest stones; the difference goes back. You always need the current two largest — that’s a max-heap.

#heap #priority-queue #simulation #easy #max-heap #interview

The problem

Each turn, take the two heaviest stones and smash them; if unequal, the difference returns to the pile. Return the weight of the last remaining stone, or 0 if none remain.

📥 Example

[2,7,4,1,8,1] → 1

The idea

The operation always needs the two largest stones, repeatedly — a textbook max-heap. Pop the top two; if they differ, push the difference back. Repeat until ≤ 1 stone remains. C#’s PriorityQueue is a min-heap, so negate the priorities to simulate a max-heap.

Time O(n log n), Space O(n).

C# solution

C# 
public int LastStoneWeight(int[] stones)
{
    // negate priority to turn the min-heap into a max-heap
    var heap = new PriorityQueue<int, int>();
    foreach (int s in stones) heap.Enqueue(s, -s);

    while (heap.Count > 1)
    {
        int a = heap.Dequeue();      // heaviest
        int b = heap.Dequeue();      // second heaviest
        if (a != b) heap.Enqueue(a - b, -(a - b));
    }
    return heap.Count == 1 ? heap.Peek() : 0;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — up to n smashes, each O(log n).
  • Space: O(n) — the heap.

Pattern & takeaways

🎯 Remember these three

  • “Repeatedly take the largest” → max-heap simulation.
  • Negate priorities to get a max-heap from C#’s min-heap PriorityQueue.
  • Push the difference back only when the two stones differ.

Interview tip

🗣️ When this comes up

Recognize “repeatedly grab the largest two” as a max-heap. The C#-specific detail interviewers note: PriorityQueue is a min-heap, so negate priorities (or supply a reverse comparer) to simulate max behavior.
