Remove Nth Node From End

Delete the nth-from-last node in one pass. Two pointers held n apart turn “from the end” into “from the front,” and a dummy head handles deleting the first node.

#linked-list #two-pointers #dummy-head #medium

The problem

Given the head of a linked list, remove the nth node from the end and return the head. Do it in one traversal.

📥 Example

1→2→3→4→5, n = 2 → 1→2→3→5 (remove the 4)

Brute force

Walk once to get the length L, then walk again to node L − n to unlink. Correct, but two passes. With a fixed-gap pointer pair we can do it in one.

Optimized approach

Use a dummy node before the head (so removing the first node is uniform). Advance a fast pointer n + 1 steps. Then move fast and a slow pointer (starting at dummy) together until fast hits the end. Now slow sits exactly before the target, so slow.next = slow.next.next unlinks it.

Time O(L), Space O(1).

C# solution

C# 
public ListNode RemoveNthFromEnd(ListNode head, int n)
{
    var dummy = new ListNode { next = head };
    ListNode fast = dummy, slow = dummy;

    for (int i = 0; i <= n; i++) fast = fast.next;   // open a gap of n+1
    while (fast != null) { fast = fast.next; slow = slow.next; }

    slow.next = slow.next.next;     // unlink the nth-from-end node
    return dummy.next;
}

Complexity

⏱️ Complexity

  • Time: O(L) — a single traversal of the list.
  • Space: O(1) — two pointers and a dummy.

Pattern & takeaways

🎯 Remember these three

  • A fixed-gap two-pointer converts “nth from end” into a one-pass problem.
  • A dummy head makes deleting the first node the same code as any other.
  • Stop slow before the target so you can relink its next.

Interview tip

🗣️ When this comes up

Two pointers + dummy is the expected one-pass answer. The off-by-one in the gap (n vs n+1) is the classic slip — set it up so slow lands on the predecessor, and dry-run the “remove the head” edge case out loud.
