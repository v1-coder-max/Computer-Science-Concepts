Maximum Product Subarray

Like Kadane’s, but products — and a single negative can flip your biggest value into your smallest. The fix: carry both the running max and the running min.

#dynamic-programming #arrays #kadane #medium

The problem

Given an integer array nums, find the contiguous subarray with the largest product and return that product.

📥 Example

[2,3,-2,4] → 6 ([2,3])  ·  [-2,3,-4] → 24 (the whole array)

Why max alone fails

With sums, a running max is enough (Kadane). With products, a negative number turns the largest product into the smallest and vice versa — so a very negative running min can become the new maximum after multiplying by another negative. You must track both extremes.

Optimized approach

Keep curMax and curMin — the largest and smallest products of a subarray ending at the current index. For each number n, the candidates are n, curMax × n, and curMin × n; the new curMax is their max and curMin their min (swap intuition: a negative n exchanges the roles). Track a global best.

Time O(n), Space O(1).

C# solution

C# 
public int MaxProduct(int[] nums)
{
    int curMax = nums[0], curMin = nums[0], best = nums[0];

    for (int i = 1; i < nums.Length; i++)
    {
        int n = nums[i];
        // a negative n swaps the roles of max and min
        int a = curMax * n, b = curMin * n;
        curMax = Math.Max(n, Math.Max(a, b));
        curMin = Math.Min(n, Math.Min(a, b));
        best = Math.Max(best, curMax);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — two rolling extremes.

Pattern & takeaways

🎯 Remember these three

  • Products need both max and min because negatives flip sign.
  • Each step’s candidates: n, curMax·n, curMin·n.
  • It’s Kadane’s with a twist — restart implicitly via the lone n candidate (handles zeros too).

Interview tip

🗣️ When this comes up

The whole trick is “track the min too, because negatives flip.” Say that and the rest writes itself. Mention that including the bare n candidate cleanly handles zeros (which reset the running products).
