Word Break

Can a string be chopped into a sequence of dictionary words? A reachability DP: position i is reachable if some earlier reachable point connects to it via a valid word.

#dynamic-programming #strings #reachability #medium

The problem

Given a string s and a dictionary wordDict, return true if s can be segmented into a space-separated sequence of one or more dictionary words (words may be reused).

📥 Example

s = "leetcode", dict = ["leet","code"] → true  ·  s = "catsandog", dict = ["cats","dog","sand","and","cat"] → false

Brute force

Recursively try every split point — exponential, because the same suffixes get re-solved many times. Those overlapping subproblems are the DP signal; memoize or build a table.

Optimized approach

Let dp[i] = “the prefix s[0..i) can be fully segmented.” Base dp[0] = true (empty prefix). For each end i, look for a split j < i where dp[j] is true and s[j..i) is a dictionary word — if found, dp[i] = true. The answer is dp[n]. Put the dictionary in a HashSet for O(1) lookups.

Time O(n²) (×word length for substring), Space O(n).

C# solution

C# — bottom-up DP 
public bool WordBreak(string s, IList<string> wordDict)
{
    var words = new HashSet<string>(wordDict);
    var dp = new bool[s.Length + 1];
    dp[0] = true;                                  // empty prefix is segmentable

    for (int i = 1; i <= s.Length; i++)
        for (int j = 0; j < i; j++)
            if (dp[j] && words.Contains(s.Substring(j, i - j)))
            {
                dp[i] = true;
                break;                             // one valid split is enough
            }

    return dp[s.Length];
}

Complexity

⏱️ Complexity

  • Time: O(n²) split points (times substring/hash cost).
  • Space: O(n) dp array plus the dictionary set.

Pattern & takeaways

🎯 Remember these three

  • Model as reachability over prefix boundaries: dp[i] from some valid dp[j].
  • A HashSet dictionary gives O(1) word checks.
  • The recursion + memo (top-down) is equivalent; the “count all segmentations” variant sums instead of OR-ing.

Interview tip

🗣️ When this comes up

Recognize the overlapping subproblems and reach straight for DP. It mirrors Coin Change (here “words” instead of “coins,” boolean reachability instead of min count). If asked to return the actual segmentation(s), switch to backtracking with memoization.
