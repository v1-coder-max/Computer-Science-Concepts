Degree of an Array

The degree is the highest frequency of any value. The shortest subarray with that degree starts at that value's first occurrence and ends at its last.

#arrays #hash-map #easy

The problem

The degree of an array is the maximum frequency of any element. Find the length of the smallest contiguous subarray with the same degree as the whole array.

📥 Example

[1,2,2,3,1] → 2 (the subarray [2,2])

Brute force

Trying every subarray and computing its degree is O(n³) or worse. One pass with per-value bookkeeping solves it.

Optimized approach

For each value track its first index, running count, and the span (current index − first index + 1). Whenever a value's count sets a new degree, record its span; ties take the minimum span.

Time O(n), Space O(n).

C# solution

C# 
public int FindShortestSubArray(int[] nums)
{
    var first = new Dictionary<int, int>();
    var count = new Dictionary<int, int>();
    int degree = 0, best = 0;
    for (int i = 0; i < nums.Length; i++)
    {
        int x = nums[i];
        if (!first.ContainsKey(x)) first[x] = i;
        count[x] = count.GetValueOrDefault(x) + 1;
        int span = i - first[x] + 1;
        if (count[x] > degree) { degree = count[x]; best = span; }
        else if (count[x] == degree) best = Math.Min(best, span);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(n) — first-index and count maps.

Pattern & takeaways

🎯 Remember these three

  • The minimal subarray spans a top-frequency value's first to last occurrence.
  • Track first index, count, and span together in one pass.
  • On a tie in degree, keep the shorter span.
