Reverse Integer

Reverse a signed integer’s digits — and return 0 if the result overflows 32 bits. The trick is checking the bound before each digit push.

#math #integers #medium #overflow #digits #interview

The problem

Given a signed 32-bit integer x, return x with its digits reversed. If reversing overflows the 32-bit signed range, return 0.

📥 Example

123 → 321  ·  -123 → -321  ·  1534236469 → 0 (overflow)

The idea

Pop digits off with x % 10 and build the result as result * 10 + digit (this handles the sign automatically in C#). The catch is overflow: before doing result * 10 + digit, check whether result is already past int.MaxValue / 10 (or below int.MinValue / 10) — if so, the next push would overflow, so return 0. Using a long accumulator and a final range check is the simplest safe version.

Time O(log x), Space O(1).

C# solution

C# 
public int Reverse(int x)
{
    long result = 0;
    while (x != 0)
    {
        result = result * 10 + x % 10;   // % keeps the sign in C#
        x /= 10;
        if (result > int.MaxValue || result < int.MinValue) return 0;
    }
    return (int)result;
}

Complexity

⏱️ Complexity

  • Time: O(log x) — one step per digit.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Build with result * 10 + x % 10 — sign handled by C# modulo.
  • Guard overflow: a long accumulator + range check is simplest.
  • For pure-int solutions, compare against int.MaxValue / 10 before multiplying.

Interview tip

🗣️ When this comes up

The whole problem is the overflow guard — say so. A long accumulator with a final range check is the cleanest; the strict-int version checks result against int.MaxValue/10 before each multiply.
