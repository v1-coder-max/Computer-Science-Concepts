Number of Longest Increasing Subsequence

Extend LIS while also tallying how many ways each best length is reached; a strictly longer extension resets the count, an equal one adds to it.

#dynamic-programming #medium

The problem

Given an integer array, return the number of longest strictly increasing subsequences.

📥 Example

[1,3,5,4,7] → 2 ([1,3,4,7] and [1,3,5,7])

Brute force

Enumerating all increasing subsequences is exponential. An O(n²) DP carries both length and count.

Optimized approach

For each i, keep len[i] (longest LIS ending at i) and count[i] (how many achieve it). For each earlier j with nums[j] < nums[i]: if len[j]+1 > len[i], set len[i] and copy count[j]; if equal, add count[j]. Sum counts of indices with the global max length.

Time O(n²), Space O(n).

C# solution

C# 
public int FindNumberOfLIS(int[] nums)
{
    int n = nums.Length;
    var len = new int[n];
    var count = new int[n];
    Array.Fill(len, 1);
    Array.Fill(count, 1);
    int best = 1;
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < i; j++)
        {
            if (nums[j] < nums[i])
            {
                if (len[j] + 1 > len[i]) { len[i] = len[j] + 1; count[i] = count[j]; }
                else if (len[j] + 1 == len[i]) count[i] += count[j];
            }
        }
        best = Math.Max(best, len[i]);
    }
    int total = 0;
    for (int i = 0; i < n; i++) if (len[i] == best) total += count[i];
    return total;
}

Complexity

⏱️ Complexity

  • Time: O(n²) — all pairs (i, j).
  • Space: O(n) — length and count arrays.

Pattern & takeaways

🎯 Remember these three

  • Carry both length and count per index.
  • A longer extension resets count; an equal one adds to it.
  • Sum the counts of all indices achieving the global max length.

Interview tip

🗣️ When this comes up

The reset-vs-accumulate rule for the count array is the crux: strictly longer overwrites, equal length adds. Summing counts at the max length gives the answer.
