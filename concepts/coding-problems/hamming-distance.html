Hamming Distance

Differing bits are exactly the set bits of x ^ y. Count them and you have the Hamming distance.

#bit-manipulation #easy

The problem

Given two integers x and y, return their Hamming distance — the number of bit positions where they differ.

📥 Example

x = 1, y = 4 → 2 (0001 vs 0100)

Brute force

Comparing the two numbers bit by bit works, but x ^ y collapses 'where they differ' into a single value whose set bits you simply count.

Optimized approach

Compute x ^ y: a 1 marks each differing position. Count the set bits — repeatedly clearing the lowest set bit with z & (z − 1) loops once per differing bit (Brian Kernighan's method).

Time O(1), Space O(1).

C# solution

C# 
public int HammingDistance(int x, int y)
{
    int z = x ^ y, count = 0;
    while (z != 0)
    {
        z &= z - 1;   // clear the lowest set bit
        count++;
    }
    return count;
}

Complexity

⏱️ Complexity

  • Time: O(1) — at most 32 iterations.
  • Space: O(1) — constant.

Pattern & takeaways

🎯 Remember these three

  • Differing bits = set bits of x ^ y.
  • z & (z − 1) clears the lowest set bit, looping once per 1.
  • BitOperations.PopCount does the same in one call.
