Single Number

Every value appears twice except one. XOR the whole array and the pairs cancel themselves out — what remains is the loner. O(n) time, O(1) space, no hash set.

#bit-manipulation #xor #arrays #easy

The problem

Given a non-empty array where every element appears exactly twice except for one that appears once, find that single one. Aim for linear time and constant extra space.

📥 Example

[4,1,2,1,2] → 4  ·  [2,2,1] → 1

Brute force

A HashSet (add on first sight, remove on second; whatever’s left is the answer) or a frequency map works in O(n) time but O(n) space. We can do it with no extra memory using a bitwise trick.

Optimized approach

XOR has three magic properties: a ^ a = 0 (a value cancels itself), a ^ 0 = a (identity), and it’s commutative & associative (order doesn’t matter). XOR every element together: each duplicated pair cancels to 0, and the lone value XORed with all those zeros is itself. One pass, a single accumulator.

Time O(n), Space O(1).

C# solution

C# 
public int SingleNumber(int[] nums)
{
    int result = 0;
    foreach (int n in nums)
        result ^= n;        // pairs cancel to 0; the unique value remains
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single XOR pass.
  • Space: O(1) — one integer accumulator.

Pattern & takeaways

🎯 Remember these three

  • XOR cancels pairs: a ^ a = 0, a ^ 0 = a — the core bit trick.
  • Order-independence (commutative/associative) is why one accumulator suffices.
  • Variants: “appears 3×” uses bit-count mod 3; “two singles” splits by a differing bit.
