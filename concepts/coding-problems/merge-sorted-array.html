Merge Sorted Array

Merge nums2 into nums1 in place. The trick: fill from the back, writing the largest elements first so you never overwrite unread data.

#arrays #two-pointers #easy #in-place #merge #interview

The problem

Given sorted nums1 (length m+n, with m values then n zeros) and sorted nums2 (length n), merge nums2 into nums1 as one sorted array, in place.

📥 Example

nums1=[1,2,3,0,0,0], m=3, nums2=[2,5,6], n=3 → [1,2,2,3,5,6]

Brute force

Copy both into a new array, sort, and write back — O((m+n) log(m+n)) and extra space, ignoring that both inputs are already sorted. Merging front-to-back in place would overwrite unread nums1 values, so we go backwards.

Optimized approach

Use three pointers: i = m−1 (end of real nums1 values), j = n−1 (end of nums2), and k = m+n−1 (write position). Compare nums1[i] and nums2[j] and place the larger at k, stepping that pointer and k back. The free space is always at or ahead of k, so nothing unread is clobbered.

Time O(m+n), Space O(1).

C# solution

C# 
public void Merge(int[] nums1, int m, int[] nums2, int n)
{
    int i = m - 1, j = n - 1, k = m + n - 1;
    while (j >= 0)                          // until nums2 is fully placed
    {
        if (i >= 0 && nums1[i] > nums2[j]) nums1[k--] = nums1[i--];
        else                              nums1[k--] = nums2[j--];
    }
}

Complexity

⏱️ Complexity

  • Time: O(m+n) — each element placed once.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Merge sorted arrays in place by writing from the back.
  • Largest-first fill keeps the write pointer ahead of unread data.
  • Stop when nums2 is exhausted — remaining nums1 is already in place.
