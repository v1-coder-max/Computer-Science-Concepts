Subarray Sum Equals K

Count subarrays that sum to k. Prefix sums + a hash map of how often each prefix sum has appeared turns it into a one-pass complement lookup.

#arrays #prefix-sum #hashing #medium #counting #interview

The problem

Given an integer array nums and an integer k, return the number of contiguous subarrays whose sum equals k (values may be negative).

📥 Example

nums=[1,1,1], k=2 → 2  ·  nums=[1,2,3], k=3 → 2

Brute force

Check every subarray’s sum — O(n²) (or O(n³) without running sums). Negative numbers rule out a sliding window. Prefix sums plus a hash map cut it to O(n).

Optimized approach

A subarray (i, j] sums to k iff prefix[j] − prefix[i] == k, i.e. prefix[i] == prefix[j] − k. Sweep, maintaining the running prefix sum and a map from prefix-sum value → how many times it has occurred. At each index, add count[prefix − k] to the answer, then record the current prefix. Seed the map with {0: 1} to count subarrays starting at index 0.

Time O(n), Space O(n).

C# solution

C# 
public int SubarraySum(int[] nums, int k)
{
    var count = new Dictionary<int, int> { [0] = 1 };   // empty prefix
    int prefix = 0, result = 0;
    foreach (int n in nums)
    {
        prefix += n;
        if (count.TryGetValue(prefix - k, out int c)) result += c;   // complement
        count[prefix] = count.GetValueOrDefault(prefix) + 1;
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(n) — the prefix-sum frequency map.

Pattern & takeaways

🎯 Remember these three

  • Prefix sum + hash map of prefix frequencies = O(n) subarray-sum counting.
  • Look up the complement prefix − k before inserting the current prefix.
  • Seed {0:1} so subarrays starting at index 0 are counted.

Interview tip

🗣️ When this comes up

It’s “Two Sum on prefix sums.” The seed {0:1} and looking up the complement before inserting are the two details that decide correctness. Stress that negatives kill the sliding-window approach.
