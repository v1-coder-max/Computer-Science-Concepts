Lowest Common Ancestor of a Binary Tree

The LCA in a general binary tree — no ordering to exploit. A post-order recursion returns matches upward; the node that sees both is the answer.

#trees #recursion #dfs #medium #lca #interview

The problem

Given the root of a binary tree and two nodes p and q, return their lowest common ancestor (the deepest node that has both as descendants; a node is its own descendant).

📥 Example

root=[3,5,1,6,2,0,8,null,null,7,4], p=5, q=1 → 3; p=5, q=4 → 5

Brute force

Find the root-to-p and root-to-q paths, then walk both to the last common node — O(n) time but two traversals and O(n) path storage. A single post-order recursion is cleaner.

Optimized approach

Recurse: if the current node is null or equals p or q, return it. Recurse into both children. If both sides return non-null, the current node is where p and q split — it’s the LCA. If only one side is non-null, propagate that side up (the LCA is somewhere above or is that node). One post-order pass.

Time O(n), Space O(h).

C# solution

C# 
public TreeNode LowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q)
{
    if (root == null || root == p || root == q) return root;

    TreeNode left  = LowestCommonAncestor(root.left, p, q);
    TreeNode right = LowestCommonAncestor(root.right, p, q);

    if (left != null && right != null) return root;   // p and q split here
    return left ?? right;                              // propagate the found side
}

Complexity

⏱️ Complexity

  • Time: O(n) — single traversal.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • General-tree LCA = post-order recursion returning matches up.
  • Both subtrees non-null ⇒ the current node is the LCA.
  • Contrast with the BST version, which uses ordering for O(h).

🗣️ When this comes up

This is the very common general-tree LCA (vs the easier BST one). The “both children return non-null ⇒ here” logic is the whole trick — narrate how matches bubble up.
