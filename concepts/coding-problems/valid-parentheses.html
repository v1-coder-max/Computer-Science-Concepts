Valid Parentheses

Are the brackets correctly opened, closed, and nested? The textbook reason the stack exists — last opened must be first closed.

#stack #strings #matching #easy

The problem

Given a string s of just ()[]{}, return true if every bracket is closed by the same type in the correct order (properly nested), else false.

📥 Example

"()[]{}" → true  ·  "(]" → false  ·  "([)]" → false  ·  "{[]}" → true

Why a stack

Nesting is last-in-first-out: the most recently opened bracket must be the next one closed. That’s exactly a stack. Push every opening bracket. On a closing bracket, the top of the stack must be its matching opener — if not (or the stack is empty), it’s invalid. At the end the stack must be empty (no unclosed brackets). There’s no meaningful brute force; the stack is the solution.

Time O(n), Space O(n).

C# solution

C# 
public bool IsValid(string s)
{
    var pairs = new Dictionary<char, char> { [')'] = '(', [']'] = '[', ['}'] = '{' };
    var stack = new Stack<char>();

    foreach (char c in s)
    {
        if (pairs.ContainsKey(c))    // c is a closing bracket
        {
            // top must be the matching opener (and the stack must be non-empty)
            if (stack.Count == 0 || stack.Pop() != pairs[c]) return false;
        }
        else
        {
            stack.Push(c);           // opening bracket
        }
    }
    return stack.Count == 0;         // no unclosed openers left
}

Complexity

⏱️ Complexity

  • Time: O(n) — each character is pushed/popped at most once.
  • Space: O(n) — worst case all opening brackets on the stack.

Pattern & takeaways

🎯 Remember these three

  • Nesting / matching → stack: the most-recent-opener-must-close-first rule is LIFO.
  • Map closers to openers for clean, type-aware matching.
  • Two failure cases: a closer with the wrong/empty top, and leftover openers at the end.

Interview tip

🗣️ When this comes up

The moment you see “matched/nested,” say stack. Cover both invalid cases explicitly (mismatch/empty on a closer; non-empty stack at the end). It’s also the warm-up for harder stack problems like “longest valid parentheses” and expression evaluation.
