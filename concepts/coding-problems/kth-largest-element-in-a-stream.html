Kth Largest Element in a Stream

Numbers keep arriving; report the kth largest each time. A size-k min-heap keeps exactly the top k, and its root is the answer.

#heap #priority-queue #design #easy #streaming #interview

The problem

Design a class initialized with k and a stream; Add(val) inserts a value and returns the kth largest element seen so far.

📥 Example

k=3, [4,5,8,2]; Add(3)→4; Add(5)→5; Add(10)→5; Add(9)→8; Add(4)→8

The idea

Keep a min-heap of size k. On each add, push the value and, if the heap exceeds k, pop the smallest. The heap then holds the k largest values seen, and its root (the minimum of those k) is exactly the kth largest. Each add is O(log k); resorting the whole stream would be far slower.

Time O(log k) per add, Space O(k).

C# solution

C# 
public class KthLargest
{
    private readonly int _k;
    private readonly PriorityQueue<int, int> _heap = new();   // min-heap

    public KthLargest(int k, int[] nums)
    {
        _k = k;
        foreach (int n in nums) Add(n);
    }

    public int Add(int val)
    {
        _heap.Enqueue(val, val);
        if (_heap.Count > _k) _heap.Dequeue();   // keep only the top k
        return _heap.Peek();                      // root = kth largest
    }
}

Complexity

⏱️ Complexity

  • Time: O(log k) per add; O(n log k) to initialize from n values.
  • Space: O(k) — the bounded heap.

Pattern & takeaways

🎯 Remember these three

  • “Kth largest over a stream” → a size-k min-heap.
  • Evict the smallest whenever size exceeds k; the root is the answer.
  • O(log k) per add beats re-sorting the whole stream.
