Valid Perfect Square

Is there an integer whose square equals n? Binary-search the root; if its square lands exactly on n, yes.

#binary-search #math #easy

The problem

Given a positive integer num, return true if it is a perfect square (the square of an integer), without using any built-in square-root function.

📥 Example

16 → true (4²); 14 → false

Brute force

Trying every i until i*i ≥ num is O(√n). Binary search on the root cuts that to O(log n).

Optimized approach

Binary-search a candidate root in [1, num]. Compare mid*mid to num (using long to avoid overflow): equal means perfect square; too big, search left; too small, search right.

Time O(log n), Space O(1).

C# solution

C# 
public bool IsPerfectSquare(int num)
{
    long lo = 1, hi = num;
    while (lo <= hi)
    {
        long mid = lo + (hi - lo) / 2;
        long sq = mid * mid;
        if (sq == num) return true;
        if (sq < num) lo = mid + 1;
        else hi = mid - 1;
    }
    return false;
}

Complexity

⏱️ Complexity

  • Time: O(log n) — binary search on the root.
  • Space: O(1) — a few longs.

Pattern & takeaways

🎯 Remember these three

  • Binary-search the answer (the root), not an array.
  • Use long for mid*mid — it overflows int near the top.
  • A clever alternative: n is a perfect square iff it equals the sum of the first k odd numbers.

Interview tip

🗣️ When this comes up

The two things interviewers check: searching the root space with binary search, and using long to avoid the mid*mid overflow. Mention the odd-numbers identity as a bonus.
