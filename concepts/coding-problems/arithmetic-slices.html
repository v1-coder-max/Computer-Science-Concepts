Arithmetic Slices

Each index that continues the previous common difference adds one more arithmetic slice than the index before it. Sum those contributions.

#dynamic-programming #arrays #medium

The problem

Count the contiguous subarrays of length at least 3 that form an arithmetic sequence (equal consecutive differences).

📥 Example

[1,2,3,4] → 3 ([1,2,3], [2,3,4], [1,2,3,4])

Brute force

Checking every subarray for a constant difference is O(n²). A running DP makes it O(n).

Optimized approach

Let cur be the number of arithmetic slices ending at the current index. If the latest difference equals the previous one, cur = prevCur + 1; otherwise cur = 0. Add cur to the total each step.

Time O(n), Space O(1).

C# solution

C# 
public int NumberOfArithmeticSlices(int[] nums)
{
    int total = 0, cur = 0;
    for (int i = 2; i < nums.Length; i++)
    {
        if (nums[i] - nums[i - 1] == nums[i - 1] - nums[i - 2])
            cur++;            // extend the run
        else
            cur = 0;          // run broken
        total += cur;
    }
    return total;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — two running counters.

Pattern & takeaways

🎯 Remember these three

  • Slices ending at i = (slices ending at i−1) + 1 when the difference continues.
  • Reset to 0 when the common difference breaks.
  • Summing the running count totals all arithmetic subarrays.

Interview tip

🗣️ When this comes up

The recurrence — a continuing run adds one more slice than the previous index — is the insight. It collapses the DP to two variables, like Kadane's algorithm.
