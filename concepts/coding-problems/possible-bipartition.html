Possible Bipartition

Model dislikes as edges and try to 2-color the graph: each person opposite everyone they dislike. A conflict means it's impossible.

#graphs #bfs #dfs #medium

The problem

Given n people and a list of mutual dislikes, return whether everyone can be split into two groups so no pair of people who dislike each other share a group.

📥 Example

n = 4, dislikes = [[1,2],[1,3],[2,4]] → true

Brute force

Trying all 2^n group assignments is exponential. Bipartite 2-coloring with BFS/DFS is O(V+E).

Optimized approach

Build the dislike graph. 2-color it with BFS/DFS: assign each unvisited person a color, then color every disliked neighbor the opposite color. If a neighbor already has the same color, the graph isn't bipartite → return false.

Time O(V+E), Space O(V+E).

C# solution

C# 
public bool PossibleBipartition(int n, int[][] dislikes)
{
    var adj = new List<int>[n + 1];
    for (int i = 1; i <= n; i++) adj[i] = new List<int>();
    foreach (var d in dislikes) { adj[d[0]].Add(d[1]); adj[d[1]].Add(d[0]); }
    var color = new int[n + 1];   // 0 = uncolored
    for (int start = 1; start <= n; start++)
    {
        if (color[start] != 0) continue;
        var q = new Queue<int>();
        q.Enqueue(start); color[start] = 1;
        while (q.Count > 0)
        {
            int u = q.Dequeue();
            foreach (int v in adj[u])
            {
                if (color[v] == 0) { color[v] = -color[u]; q.Enqueue(v); }
                else if (color[v] == color[u]) return false;
            }
        }
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(V+E) — visit each node and edge once.
  • Space: O(V+E) — adjacency lists and colors.

Pattern & takeaways

🎯 Remember these three

  • Two groups with no internal dislikes ⇔ a bipartite graph.
  • 2-color via BFS/DFS; a same-color neighbor means failure.
  • Loop over components — the graph may be disconnected.
