Spiral Matrix

Read a matrix in a spiral. Keep four shrinking boundaries — top, bottom, left, right — and walk each edge in turn.

#matrix #arrays #medium #simulation #boundaries #interview

The problem

Given an m × n matrix, return all its elements in spiral order (right across the top, down the right side, left across the bottom, up the left, inward).

📥 Example

[[1,2,3],[4,5,6],[7,8,9]] → [1,2,3,6,9,8,7,4,5]

Brute force

There’s no real brute force — you must visit each cell. The challenge is the bookkeeping. A direction-vector simulation with a visited grid works but uses O(m·n) extra space; boundary tracking is cleaner and O(1) extra.

Optimized approach

Track top, bottom, left, right. Repeat: traverse the top row left→right then top++; the right column top→bottom then right−−; if rows remain, the bottom row right→left then bottom−−; if columns remain, the left column bottom→top then left++. Stop when the boundaries cross. The two guards prevent re-reading a row/column in thin matrices.

Time O(m·n), Space O(1) (excluding output).

C# solution

C# 
public IList<int> SpiralOrder(int[][] matrix)
{
    var result = new List<int>();
    int top = 0, bottom = matrix.Length - 1, left = 0, right = matrix[0].Length - 1;
    while (top <= bottom && left <= right)
    {
        for (int c = left; c <= right; c++) result.Add(matrix[top][c]);    top++;
        for (int r = top; r <= bottom; r++) result.Add(matrix[r][right]);  right--;
        if (top <= bottom)
        { for (int c = right; c >= left; c--) result.Add(matrix[bottom][c]); bottom--; }
        if (left <= right)
        { for (int r = bottom; r >= top; r--) result.Add(matrix[r][left]);  left++; }
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — each cell read once.
  • Space: O(1) beyond the output.

Pattern & takeaways

🎯 Remember these three

  • Track four boundaries and shrink after each edge.
  • Guard the bottom and left passes for single-row/column rings.
  • Direction-vector + visited is the equivalent but uses extra space.

Interview tip

🗣️ When this comes up

Boundary tracking is cleaner than direction vectors and uses no extra space. The two “if rows/cols remain” guards are the detail that breaks on thin matrices — call them out.
