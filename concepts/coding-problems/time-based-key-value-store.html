Time Based Key-Value Store

Store versioned values and fetch the one at or before a query time. Per key, keep entries in timestamp order and binary search for the floor.

#binary-search #design #hash-map #medium #versioning #interview

The problem

Design Set(key, value, timestamp) and Get(key, timestamp) returning the value with the largest stored timestamp ≤ the query (or "" if none). Sets arrive in increasing timestamp order.

📥 Example

set("foo","bar",1); get("foo",1)→"bar"; get("foo",3)→"bar"; set("foo","baz",4); get("foo",4)→"baz"

Brute force

Scanning a key’s history linearly for the largest timestamp ≤ query is O(n) per get. Because sets arrive in increasing time, each key’s list is already sorted — so binary search the floor in O(log n).

Optimized approach

Map each key to a list of (timestamp, value) appended in increasing time (already sorted). For Get, binary search for the rightmost entry whose timestamp ≤ the query (a floor / upper-bound search) and return its value, or "" if all timestamps exceed the query.

Set O(1), Get O(log n).

C# solution

C# 
public class TimeMap
{
    private readonly Dictionary<string, List<(int t, string v)>> _map = new();

    public void Set(string key, string value, int timestamp)
    {
        if (!_map.ContainsKey(key)) _map[key] = new();
        _map[key].Add((timestamp, value));         // appended in increasing time
    }

    public string Get(string key, int timestamp)
    {
        if (!_map.TryGetValue(key, out var list)) return "";
        int lo = 0, hi = list.Count - 1;
        string result = "";
        while (lo <= hi)
        {
            int mid = lo + (hi - lo) / 2;
            if (list[mid].t <= timestamp) { result = list[mid].v; lo = mid + 1; }  // go later
            else hi = mid - 1;
        }
        return result;
    }
}

Complexity

⏱️ Complexity

  • Time: O(1) Set, O(log n) Get.
  • Space: O(total entries).

Pattern & takeaways

🎯 Remember these three

  • Per-key sorted history + binary search for the floor timestamp.
  • Increasing-time inserts keep the list sorted for free.
  • It’s an upper-bound / floor search — keep the candidate as you go right.

Interview tip

🗣️ When this comes up

Recognize “value at or before a time” as a floor binary search over a sorted-by-time list. The free-sortedness from increasing inserts is the detail that makes Set O(1).
