Minimum Size Subarray Sum

The shortest subarray whose sum reaches a target. With positive numbers, a grow-then-shrink sliding window nails it in one pass.

#sliding-window #arrays #medium #two-pointers #positive #interview

The problem

Given positive integers nums and a target, return the minimal length of a contiguous subarray with sum ≥ target, or 0 if none exists.

📥 Example

target=7, nums=[2,3,1,2,4,3] → 2 (the subarray [4,3])

Brute force

Check every subarray’s sum — O(n²) (or O(n) per start with running sums). Because all values are positive, growing the window only increases the sum and shrinking only decreases it, which is exactly what a sliding window exploits for O(n).

Optimized approach

Expand the window by moving right and adding to a running sum. Whenever the sum is ≥ target, record the window length and shrink from left (subtracting) as long as the sum stays ≥ target — this finds the minimal window ending here. Continue to the end. Positivity guarantees the monotonic grow/shrink behavior.

Time O(n), Space O(1).

C# solution

C# 
public int MinSubArrayLen(int target, int[] nums)
{
    int left = 0, sum = 0, best = int.MaxValue;
    for (int right = 0; right < nums.Length; right++)
    {
        sum += nums[right];
        while (sum >= target)                  // shrink while still valid
        {
            best = Math.Min(best, right - left + 1);
            sum -= nums[left++];
        }
    }
    return best == int.MaxValue ? 0 : best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index enters and leaves once.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Positive numbers make a monotonic sliding window valid.
  • Grow to satisfy the target, shrink to minimize length.
  • For arrays with negatives, fall back to prefix sums + a deque/binary search.

Interview tip

🗣️ When this comes up

Call out the positivity assumption — it’s what makes the window monotonic. If the interviewer allows negatives, the window breaks and you switch to prefix sums.
