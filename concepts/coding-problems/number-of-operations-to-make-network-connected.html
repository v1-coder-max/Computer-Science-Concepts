Number of Operations to Make Network Connected

Spare cables (those between already-connected computers) can be moved to bridge gaps. If you have at least components − 1 spares, that many moves connect everything.

#graphs #union-find #medium

The problem

Given n computers and a list of cables, you may unplug any cable and reconnect it elsewhere. Return the minimum moves to connect all computers, or −1 if impossible.

📥 Example

n = 6, connections = [[0,1],[0,2],[0,3],[1,2],[1,3]] → 2

Brute force

Searching reconnection sequences is unnecessary; the answer is a simple function of the component count.

Optimized approach

You need at least n − 1 cables; if there are fewer, return −1. Use union-find to count connected components c. Connecting c components needs c − 1 moves, and any redundant cable (joining an already-connected pair) supplies a spare — there are always enough when cables ≥ n − 1.

Time O(E·α), Space O(n).

C# solution

C# 
public int MakeConnected(int n, int[][] connections)
{
    if (connections.Length < n - 1) return -1;   // not enough cables
    var parent = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
    int Find(int x) { while (parent[x] != x) { parent[x] = parent[parent[x]]; x = parent[x]; } return x; }
    int components = n;
    foreach (var c in connections)
    {
        int ra = Find(c[0]), rb = Find(c[1]);
        if (ra != rb) { parent[ra] = rb; components--; }
    }
    return components - 1;
}

Complexity

⏱️ Complexity

  • Time: O(E·α) — near-constant union/find.
  • Space: O(n) — the parent array.

Pattern & takeaways

🎯 Remember these three

  • With < n − 1 cables it's impossible (−1).
  • Answer = components − 1 (each move merges two groups).
  • Redundant cables are the spares you reuse — you always have enough.

🗣️ When this comes up

Two facts close it: fewer than n−1 cables is hopeless, and otherwise the answer is components−1. Union-find counts components; the spare-cable argument explains why it's always achievable.
