Swim in Rising Water

Cross a grid where you can only step into water at or below the current level — minimize the highest elevation on your path. A Dijkstra-style min-heap on the running max.

#graphs #dijkstra #grid #hard #minimax-path #interview

The problem

Given an n × n grid of elevations, at time t every cell with elevation ≤ t is swimmable. Starting top-left, return the least time to reach the bottom-right (4-directional moves, instant within a time).

📥 Example

[[0,2],[1,3]] → 3

Brute force

Binary search the time t and BFS/DFS to test reachability at each t — O(n² log(max)). It works, but a single Dijkstra-like pass is cleaner and direct.

Optimized approach

This is a minimax path: minimize the maximum elevation crossed. Use a min-heap keyed by the path’s running maximum elevation. Pop the cell with the smallest such max, update the answer with its elevation, and push neighbors with max(currentMax, neighborElevation). The first time you pop the bottom-right cell, its key is the answer. It’s Dijkstra with “max along path” replacing “sum.”

Time O(n² log n), Space O(n²).

C# solution

C# 
public int SwimInWater(int[][] grid)
{
    int n = grid.Length;
    var visited = new bool[n, n];
    var pq = new PriorityQueue<(int r, int c), int>();
    pq.Enqueue((0, 0), grid[0][0]);
    int[][] dirs = { new[]{1,0}, new[]{-1,0}, new[]{0,1}, new[]{0,-1} };

    while (true)
    {
        pq.TryDequeue(out var cell, out int t);
        var (r, c) = cell;
        if (visited[r, c]) continue;
        visited[r, c] = true;
        if (r == n - 1 && c == n - 1) return t;       // first arrival = answer
        foreach (var d in dirs)
        {
            int nr = r + d[0], nc = c + d[1];
            if (nr >= 0 && nc >= 0 && nr < n && nc < n && !visited[nr, nc])
                pq.Enqueue((nr, nc), Math.Max(t, grid[nr][nc]));
        }
    }
}

Complexity

⏱️ Complexity

  • Time: O(n² log n) — each cell settled once via the heap.
  • Space: O(n²) for the heap and visited grid.

Pattern & takeaways

🎯 Remember these three

  • “Minimize the max along a path” = minimax path → Dijkstra with max.
  • Heap key is the path’s running maximum, not a sum.
  • Binary-search-the-answer + flood fill is the equivalent alternative.
