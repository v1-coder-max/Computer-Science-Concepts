String to Integer (atoi)

Parse a string to a 32-bit int like C’s atoi: skip spaces, read an optional sign, gather digits, and clamp on overflow before it happens.

#strings #math #medium #parsing #overflow #interview

The problem

Implement MyAtoi(s): skip leading whitespace, read an optional +/-, then consecutive digits, ignoring the rest. Clamp the result to the 32-bit signed range [-2³¹, 2³¹−1].

📥 Example

" -42" → -42  ·  "4193 with words" → 4193  ·  "2147483648" → 2147483647

Brute force

Parsing into a long or BigInteger then clamping is easy but sidesteps the real challenge: handling overflow with bounded integer types and the messy leading-whitespace/sign rules.

Optimized approach

Step through the string: (1) skip leading spaces, (2) read an optional sign, (3) accumulate digits into result. Before each result = result * 10 + digit, check whether it would overflow int — if result exceeds int.MaxValue / 10, or equals it with a digit that pushes past the bound — and clamp to int.MaxValue/int.MinValue accordingly. Stop at the first non-digit.

Time O(n), Space O(1).

C# solution

C# 
public int MyAtoi(string s)
{
    int i = 0, n = s.Length;
    while (i < n && s[i] == ' ') i++;             // 1) whitespace
    int sign = 1;
    if (i < n && (s[i] == '+' || s[i] == '-')) sign = s[i++] == '-' ? -1 : 1;  // 2) sign

    int result = 0;
    while (i < n && char.IsDigit(s[i]))            // 3) digits
    {
        int d = s[i++] - '0';
        if (result > int.MaxValue / 10 || (result == int.MaxValue / 10 && d > 7))
            return sign == 1 ? int.MaxValue : int.MinValue;   // clamp
        result = result * 10 + d;
    }
    return result * sign;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one scan.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Four phases: whitespace, sign, digits, clamp.
  • Check overflow before the multiply, not after.
  • Use the int.MaxValue/10 and last-digit comparison for a pure-int solution.

Interview tip

🗣️ When this comes up

It’s 90% edge cases: whitespace, sign, non-digits, and overflow. Walk the four phases methodically and show the pre-multiply overflow guard — that’s exactly what the interviewer is probing.
