Minimum Absolute Difference in BST

In-order traversal of a BST yields sorted values, so the closest pair is always two neighbors in that order.

#trees #bst #dfs #easy

The problem

Given a BST, return the minimum absolute difference between the values of any two different nodes.

📥 Example

BST [4,2,6,1,3] → 1

Brute force

Comparing all pairs is O(n²). The BST's in-order order means only adjacent values can be closest, dropping it to O(n).

Optimized approach

Traverse in order, remembering the previous value visited. At each node, the difference with the previous in-order value is a candidate for the minimum; track the smallest.

Time O(n), Space O(h).

C# solution

C# 
public int GetMinimumDifference(TreeNode root)
{
    int best = int.MaxValue;
    int? prev = null;
    void InOrder(TreeNode node)
    {
        if (node == null) return;
        InOrder(node.left);
        if (prev.HasValue) best = Math.Min(best, node.val - prev.Value);
        prev = node.val;
        InOrder(node.right);
    }
    InOrder(root);
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single in-order pass.
  • Space: O(h) — recursion = height.

Pattern & takeaways

🎯 Remember these three

  • In-order of a BST is sorted — the closest pair is adjacent.
  • Carry the previous value to compare consecutive elements.
  • Works for any 'minimum difference in a BST' variant, including the count-of-modes one.
