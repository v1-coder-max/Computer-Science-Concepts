Predict the Winner

Identical to Stone Game but for arbitrary values: track the best score difference each player can force on every subarray.

#dynamic-programming #game-theory #medium

The problem

Two players alternately take a number from either end of an array, adding it to their score. Return whether player 1 can end with a score ≥ player 2's, assuming optimal play.

📥 Example

[1,5,2] → false; [1,5,233,7] → true

Brute force

Plain minimax recursion is exponential; memoized interval DP is O(n²).

Optimized approach

Let dp[i][j] be the maximum score difference the current player can secure on i..j. Taking the left yields nums[i] − dp[i+1][j]; the right yields nums[j] − dp[i][j−1]; take the max. Player 1 wins (or ties) iff dp[0][n−1] ≥ 0.

Time O(n²), Space O(n²).

C# solution

C# 
public bool PredictTheWinner(int[] nums)
{
    int n = nums.Length;
    var dp = new int[n, n];
    for (int i = 0; i < n; i++) dp[i, i] = nums[i];
    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++)
        {
            int j = i + len - 1;
            dp[i, j] = Math.Max(nums[i] - dp[i + 1, j], nums[j] - dp[i, j - 1]);
        }
    return dp[0, n - 1] >= 0;
}

Complexity

⏱️ Complexity

  • Time: O(n²) — fill each interval once.
  • Space: O(n²) — the n×n table (reducible to O(n)).

Pattern & takeaways

🎯 Remember these three

  • Same score-difference interval DP as Stone Game.
  • Win or tie ⇔ full-range difference ≥ 0.
  • Subtracting the opponent's optimal result encodes the minimax cleanly.
