Can Place Flowers

Greedily plant in any empty plot whose neighbors are also empty; count how many fit and compare to n.

#greedy #arrays #easy

The problem

A flowerbed (array of 0s and 1s) cannot have two adjacent flowers. Return whether n new flowers can be planted.

📥 Example

[1,0,0,0,1], n = 1 → true; n = 2 → false

Brute force

Trying combinations is unnecessary — greedily planting at the earliest valid spot is optimal.

Optimized approach

Scan the bed. At each 0 whose left and right are also 0 (treating the edges as empty), plant there (set it to 1) and increment a counter. If the count reaches n, return true.

Time O(m), Space O(1).

C# solution

C# 
public bool CanPlaceFlowers(int[] flowerbed, int n)
{
    int count = 0, len = flowerbed.Length;
    for (int i = 0; i < len; i++)
    {
        if (flowerbed[i] == 0 &&
            (i == 0 || flowerbed[i - 1] == 0) &&
            (i == len - 1 || flowerbed[i + 1] == 0))
        {
            flowerbed[i] = 1;       // plant greedily
            count++;
        }
    }
    return count >= n;
}

Complexity

⏱️ Complexity

  • Time: O(m) — single pass over the bed.
  • Space: O(1) — a counter.

Pattern & takeaways

🎯 Remember these three

  • Greedily plant at the first valid empty plot — it's optimal.
  • Treat the array edges as empty neighbors.
  • Planting in place lets the neighbor check stay correct as you go.

Interview tip

🗣️ When this comes up

The greedy works because planting as early as possible never blocks a later opportunity. Handle the boundary plots (treat out-of-range neighbors as empty) carefully.
