Unique Paths II

Unique Paths, now with obstacles. Same right/down DP — but an obstacle cell contributes zero paths.

#dynamic-programming #2d-dp #grid #medium #obstacles #interview

The problem

Given an m × n grid where 1 marks an obstacle and 0 is free, count the paths from top-left to bottom-right moving only right or down (start/end may be checked for obstacles).

📥 Example

[[0,0,0],[0,1,0],[0,0,0]] → 2

Brute force

Recursive branching that abandons paths hitting obstacles is exponential. The grid DP handles obstacles by simply zeroing those cells.

Optimized approach

Use dp[r][c] = dp[r−1][c] + dp[r][c−1] as in Unique Paths, but set dp[r][c] = 0 whenever the cell is an obstacle (no path can pass through). Initialize the first row/column to 1 only up to the first obstacle (everything after it is unreachable). If the start cell is an obstacle, the answer is 0. A rolling 1-D array works the same way.

Time O(m·n), Space O(n).

C# solution

C# 
public int UniquePathsWithObstacles(int[][] grid)
{
    int n = grid[0].Length;
    var dp = new int[n];
    dp[0] = grid[0][0] == 1 ? 0 : 1;             // start may be blocked

    foreach (var row in grid)
        for (int c = 0; c < n; c++)
        {
            if (row[c] == 1) dp[c] = 0;          // obstacle: no paths through
            else if (c > 0) dp[c] += dp[c - 1];  // top (old dp[c]) + left
        }
    return dp[n - 1];
}

Complexity

⏱️ Complexity

  • Time: O(m·n).
  • Space: O(n) rolling array.

Pattern & takeaways

🎯 Remember these three

  • Same recurrence as Unique Paths; zero out obstacle cells.
  • First row/column propagate 1 only until the first obstacle.
  • Check the start cell — a blocked start means 0 paths.
