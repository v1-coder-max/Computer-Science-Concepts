Minimum Number of Arrows to Burst Balloons

Fewest arrows to pop all balloons (intervals on a line). Sort by end and shoot at each interval’s end — one arrow clears all overlapping balloons.

#greedy #intervals #sorting #medium #activity-selection #interview

The problem

Each balloon is an interval [start, end]. An arrow shot at x bursts every balloon with start ≤ x ≤ end. Return the minimum number of arrows to burst them all.

📥 Example

[[10,16],[2,8],[1,6],[7,12]] → 2

Brute force

Trying arrow positions or all subsets is exponential. The greedy choice — shoot at the earliest interval end — provably covers the most balloons per arrow.

Optimized approach

Sort balloons by end coordinate. Shoot the first arrow at the first balloon’s end; it bursts every balloon whose start ≤ that end. Skip all such covered balloons. When you reach a balloon that starts after the current arrow position, you need a new arrow at that balloon’s end. Repeat. This is the activity-selection greedy: committing to the earliest end maximizes overlap coverage.

Time O(n log n), Space O(1).

C# solution

C# 
public int FindMinArrowShots(int[][] points)
{
    if (points.Length == 0) return 0;
    Array.Sort(points, (a, b) => a[1].CompareTo(b[1]));   // by end

    int arrows = 1;
    long arrowAt = points[0][1];
    foreach (var p in points)
        if (p[0] > arrowAt)            // starts after the current arrow
        {
            arrows++;
            arrowAt = p[1];            // shoot a new arrow at this end
        }
    return arrows;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — sort dominates.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Sort by end; shoot at the earliest end to maximize coverage.
  • A new arrow is needed only when a balloon starts past the last arrow.
  • It’s the activity-selection greedy (like Non-overlapping Intervals).

Interview tip

🗣️ When this comes up

Sorting by end (not start) and shooting at each end is the greedy that minimizes arrows — the same proof as interval scheduling. Use a long for the arrow position to avoid edge-case overflow on extreme inputs.
