Find the Town Judge

The judge is trusted by all n−1 others and trusts nobody. Counting in- and out-degrees pinpoints exactly that node.

#graphs #counting #easy

The problem

In a town of n people, the judge is trusted by everyone else and trusts no one. Given trust pairs [a, b] (a trusts b), return the judge's label, or -1.

📥 Example

n = 3, trust = [[1,3],[2,3]] → 3

Brute force

Building adjacency lists and scanning each person's trust list works, but a single net-degree counter is simpler and just as fast.

Optimized approach

Track each person's net trust = (people trusting them) − (people they trust). Every trust [a, b] does net[b]++ and net[a]--. The judge is the unique person with net trust n − 1.

Time O(V+E), Space O(V).

C# solution

C# 
public int FindJudge(int n, int[][] trust)
{
    var net = new int[n + 1];
    foreach (var t in trust) { net[t[0]]--; net[t[1]]++; }
    for (int i = 1; i <= n; i++)
        if (net[i] == n - 1) return i;
    return -1;
}

Complexity

⏱️ Complexity

  • Time: O(V+E) — one pass over trust, one over people.
  • Space: O(V) — the net-degree array.

Pattern & takeaways

🎯 Remember these three

  • Judge ⇔ in-degree n − 1 and out-degree 0.
  • A single net-degree counter folds both conditions into one number.
  • Exactly one node can have net n − 1, so the judge is unique.
