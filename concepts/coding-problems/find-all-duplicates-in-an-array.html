Find All Duplicates in an Array

With values in 1..n, the array can store its own seen-set: flip the sign at index v−1 to mark v; a sign already flipped means a duplicate.

#arrays #in-place #medium

The problem

Given an array of n integers where each value is in [1, n] and appears once or twice, return all values that appear twice — in O(n) time and O(1) extra space.

📥 Example

[4,3,2,7,8,2,3,1] → [2, 3]

Brute force

A hash set finds duplicates in O(n) time but O(n) space. The value range lets you encode 'seen' inside the array instead.

Optimized approach

For each value v, look at index |v|−1. If it is already negative, v has been seen before — record it. Otherwise negate it to mark v as seen.

Time O(n), Space O(1).

C# solution

C# 
public IList<int> FindDuplicates(int[] nums)
{
    var res = new List<int>();
    foreach (int x in nums)
    {
        int idx = Math.Abs(x) - 1;
        if (nums[idx] < 0) res.Add(idx + 1);   // seen before
        else nums[idx] = -nums[idx];           // mark seen
    }
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — marks stored in the input.

Pattern & takeaways

🎯 Remember these three

  • A 1..n value range lets the array double as a seen-set via sign flips.
  • An already-negative slot means its index+1 is a duplicate.
  • Use Math.Abs when reading values you may have negated.
