Minimum Operations to Reduce X to Zero

Removing from the ends to reach x is the same as keeping a middle subarray that sums to total − x; maximize its length to minimize removals.

#sliding-window #two-pointers #medium

The problem

You may repeatedly remove the leftmost or rightmost element, subtracting it from x. Return the minimum operations to make x exactly 0, or −1.

📥 Example

nums = [1,1,4,2,3], x = 5 → 2 (remove 3 then 2)

Brute force

Exploring all end-removal sequences is exponential. Reformulating as a middle-subarray sum gives a sliding window in O(n).

Optimized approach

Let goal = total − x. Find the longest contiguous subarray summing to goal with a sliding window (all values are positive). The answer is n − (that length), or −1 if no such subarray exists.

Time O(n), Space O(1).

C# solution

C# 
public int MinOperations(int[] nums, int x)
{
    int total = nums.Sum(), goal = total - x;
    if (goal < 0) return -1;
    int sum = 0, left = 0, best = -1;
    for (int right = 0; right < nums.Length; right++)
    {
        sum += nums[right];
        while (sum > goal && left <= right) sum -= nums[left++];
        if (sum == goal) best = Math.Max(best, right - left + 1);
    }
    return best < 0 ? -1 : nums.Length - best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each pointer moves at most n times.
  • Space: O(1) — a few counters.

Pattern & takeaways

🎯 Remember these three

  • Removing from the ends ⇔ keeping a middle subarray of sum total − x.
  • Maximize the kept length to minimize removals.
  • Positive values make the sliding window valid.
