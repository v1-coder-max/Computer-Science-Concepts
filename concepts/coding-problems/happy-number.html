Happy Number

Replace a number by the sum of the squares of its digits, repeat. It’s “happy” if it reaches 1 — otherwise it loops. Detect the loop with fast/slow pointers.

#math #two-pointers #easy #cycle #floyd #interview

The problem

A number is happy if repeatedly summing the squares of its digits eventually reaches 1. Return true if n is happy.

📥 Example

19 → true (1+81=82 → 68 → 100 → 1)  ·  2 → false

Brute force

Track every number seen in a HashSet; if you revisit one, there’s a cycle → false; if you hit 1, true. Correct and simple, but O(n) extra space. Floyd’s cycle detection removes the set.

Optimized approach

The sequence of digit-square sums is like a linked list that either reaches 1 or enters a cycle — exactly the setup for Floyd’s tortoise and hare. Advance slow one step and fast two steps through the transformation; if they meet at a value that isn’t 1, there’s a cycle → not happy. If fast reaches 1, it’s happy. O(1) extra space.

Time O(log n) per step, Space O(1).

C# solution

C# 
public bool IsHappy(int n)
{
    int Next(int x)
    {
        int sum = 0;
        while (x > 0) { int d = x % 10; sum += d * d; x /= 10; }
        return sum;
    }

    int slow = n, fast = Next(n);
    while (fast != 1 && slow != fast)
    {
        slow = Next(slow);
        fast = Next(Next(fast));
    }
    return fast == 1;
}

Complexity

⏱️ Complexity

  • Time: O(log n) digits per step, bounded total steps.
  • Space: O(1) with Floyd (O(n) with a set).

Pattern & takeaways

🎯 Remember these three

  • The transformation forms an implicit linked list → cycle detection.
  • HashSet is the easy O(n)-space solution; Floyd makes it O(1).
  • Happy ⇔ the sequence reaches 1 rather than looping.
