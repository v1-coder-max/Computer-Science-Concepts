Majority Element

Find the value that appears more than n/2 times — in O(1) space. Boyer-Moore voting: a candidate and a counter that cancels out minorities.

#arrays #boyer-moore #easy #voting #o1 #interview

The problem

Given an array nums of size n, return the majority element (the one that appears more than ⌊n/2⌋ times). You may assume it always exists.

📥 Example

[2,2,1,1,1,2,2] → 2  ·  [3,2,3] → 3

Brute force

A hash map counting frequencies finds it in O(n) time but O(n) space; sorting and taking the middle element is O(n log n). Boyer-Moore voting gets O(n) time and O(1) space.

Optimized approach

Keep a candidate and a count. Scan: when count is 0, adopt the current value as the candidate; if the current value equals the candidate, increment count, else decrement it. Because the majority element occurs more than half the time, every cancellation pairs it with a non-majority element, so it’s the last candidate standing.

Time O(n), Space O(1).

C# solution

C# 
public int MajorityElement(int[] nums)
{
    int candidate = 0, count = 0;
    foreach (int n in nums)
    {
        if (count == 0) candidate = n;          // adopt a new candidate
        count += n == candidate ? 1 : -1;       // vote or cancel
    }
    return candidate;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — a candidate and a counter.

Pattern & takeaways

🎯 Remember these three

  • Majority (> n/2) → Boyer-Moore voting in O(1) space.
  • Reset the candidate whenever the count hits zero.
  • Minority elements can’t outlast a strict majority’s votes.

Interview tip

🗣️ When this comes up

Mention the hash-map and sort solutions, then deliver Boyer-Moore for O(1) space and explain why it works (strict majority survives cancellation). It’s a classic “can you beat O(n) space” filter.
