Lowest Common Ancestor of a BST

Find the deepest node that is an ancestor of both targets. In a BST you don’t search — the ordering tells you which way to walk, and the split point is the answer.

#trees #bst #binary-search #medium

The problem

Given a BST and two nodes p and q, return their lowest common ancestor — the deepest node that has both as descendants (a node can be a descendant of itself).

📥 Example

BST [6,2,8,0,4,7,9], p = 2, q = 8 → 6;   p = 2, q = 4 → 2

The idea

Exploit the BST ordering. Starting at the root: if both targets are smaller than the current node, the LCA must be in the left subtree. If both are larger, go right. The first node where the targets split — one on each side, or one equals the node — is the lowest common ancestor. No full search, just a walk down one path.

Time O(h), Space O(1) iterative (h = tree height).

C# solution

C# 
public TreeNode LowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q)
{
    var node = root;
    while (node != null)
    {
        if (p.val < node.val && q.val < node.val)      node = node.left;   // both smaller
        else if (p.val > node.val && q.val > node.val) node = node.right;  // both larger
        else return node;   // they split here (or one equals node) → LCA
    }
    return null;
}

Complexity

⏱️ Complexity

  • Time: O(h) — one root-to-split walk (O(log n) balanced, O(n) skewed).
  • Space: O(1) iterative.

Pattern & takeaways

🎯 Remember these three

  • In a BST, ordering replaces searching — the LCA is the first split node.
  • Both-smaller → left; both-larger → right; otherwise you’re standing on the answer.
  • For a general binary tree (no ordering), it’s the harder return-from-both-subtrees recursion.

Interview tip

🗣️ When this comes up

Stress that the BST version is O(h) and needs no extra space — it’s essentially binary search. Be ready to contrast with the general-tree LCA (post-order recursion returning matches up the tree), which interviewers love to ask as the follow-up.
