Flood Fill

Bucket-fill a region of same-colored pixels from a starting cell. A simple DFS/BFS — with one guard so it doesn’t loop when the new color equals the old.

#graphs #dfs #grid #easy #flood-fill #interview

The problem

Given an image grid, a start pixel (sr, sc), and a newColor, recolor the start pixel and all 4-directionally connected pixels of the same original color to newColor. Return the image.

📥 Example

image=[[1,1,1],[1,1,0],[1,0,1]], sr=1, sc=1, newColor=2 → the connected 1-region becomes 2.

Brute force

There’s no real brute force beyond the flood itself; the only pitfall is forgetting the equal-color guard, which causes infinite recursion.

Optimized approach

Record the original color at the start. If it already equals newColor, return immediately (otherwise you’d loop forever). Otherwise DFS/BFS from the start: recolor the current cell and recurse into the four neighbors that still hold the original color. Each cell is recolored once, so it terminates after touching the region.

Time O(m·n), Space O(m·n).

C# solution

C# 
public int[][] FloodFill(int[][] image, int sr, int sc, int color)
{
    int original = image[sr][sc];
    if (original == color) return image;          // guard: avoid infinite loop
    int m = image.Length, n = image[0].Length;

    void Dfs(int r, int c)
    {
        if (r < 0 || c < 0 || r >= m || c >= n || image[r][c] != original) return;
        image[r][c] = color;
        Dfs(r+1,c); Dfs(r-1,c); Dfs(r,c+1); Dfs(r,c-1);
    }
    Dfs(sr, sc);
    return image;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) worst case.
  • Space: O(m·n) recursion in the worst case.

Pattern & takeaways

🎯 Remember these three

  • Standard grid DFS/BFS flood fill.
  • Guard against newColor == originalColor to avoid looping.

Interview tip

🗣️ When this comes up

Quick and friendly, but the equal-color guard is the one detail that separates a correct answer from an infinite loop. It’s the foundation of all grid flood problems.
