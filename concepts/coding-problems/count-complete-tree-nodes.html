Count Complete Tree Nodes

Counting by traversal is O(n), but a complete tree's structure lets you skip whole subtrees with a height comparison.

#trees #binary-search #medium

The problem

Given a complete binary tree, count its nodes. Aim for better than O(n).

📥 Example

A perfect tree of height h has exactly 2^h − 1 nodes — recognized in O(h).

Brute force

A full traversal counts every node in O(n) and ignores the completeness guarantee.

Optimized approach

Compute the leftmost and rightmost path heights. If they're equal, the subtree is perfect → 2^h − 1 nodes in O(h). Otherwise recurse into both children. Each level does O(log n) height work, giving O(log²n).

Time O(log²n), Space O(log n).

C# solution

C# 
public int CountNodes(TreeNode root)
{
    if (root == null) return 0;
    int Left(TreeNode n) { int h = 0; while (n != null) { h++; n = n.left; } return h; }
    int Right(TreeNode n) { int h = 0; while (n != null) { h++; n = n.right; } return h; }
    int l = Left(root), r = Right(root);
    if (l == r) return (1 << l) - 1;                 // perfect subtree
    return 1 + CountNodes(root.left) + CountNodes(root.right);
}

Complexity

⏱️ Complexity

  • Time: O(log²n) — log n levels × log n height checks.
  • Space: O(log n) — recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Equal left/right heights ⇒ a perfect subtree of 2^h − 1 nodes.
  • Only the path to the last leaf needs real recursion.
  • Beats the naive O(n) traversal using the completeness property.
