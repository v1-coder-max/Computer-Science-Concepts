4Sum

All unique quadruples that sum to a target. Sort, fix two numbers with nested loops, and two-pointer the remaining pair — skipping duplicates throughout.

#two-pointers #arrays #medium #k-sum #dedupe #interview

The problem

Given an array nums and a target, return all unique quadruples [a,b,c,d] with a+b+c+d == target. The solution set must not contain duplicate quadruples.

📥 Example

nums = [1,0,-1,0,-2,2], target = 0 → [[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]

Brute force

Four nested loops over all quadruples — O(n⁴) — plus a set to dedupe. Sorting and reusing the 3Sum/2Sum two-pointer reduces it to O(n³).

Optimized approach

It’s the general k-Sum pattern. Sort the array. Fix the first index i and second index j with two loops, then run the two-pointer 2Sum on the remainder to find pairs hitting target − nums[i] − nums[j]. Skip equal values at every level to avoid duplicate quadruples. Use a 64-bit sum to dodge overflow.

Time O(n³), Space O(1) (excluding output).

C# solution

C# 
public IList<IList<int>> FourSum(int[] nums, int target)
{
    Array.Sort(nums);
    int n = nums.Length;
    var result = new List<IList<int>>();

    for (int i = 0; i < n - 3; i++)
    {
        if (i > 0 && nums[i] == nums[i - 1]) continue;          // dedupe i
        for (int j = i + 1; j < n - 2; j++)
        {
            if (j > i + 1 && nums[j] == nums[j - 1]) continue;  // dedupe j
            int lo = j + 1, hi = n - 1;
            while (lo < hi)
            {
                long sum = (long)nums[i] + nums[j] + nums[lo] + nums[hi];
                if (sum == target)
                {
                    result.Add(new List<int> { nums[i], nums[j], nums[lo], nums[hi] });
                    while (lo < hi && nums[lo] == nums[lo + 1]) lo++;   // dedupe lo
                    while (lo < hi && nums[hi] == nums[hi - 1]) hi--;   // dedupe hi
                    lo++; hi--;
                }
                else if (sum < target) lo++;
                else hi--;
            }
        }
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n³) — two fixed indices × a linear two-pointer scan.
  • Space: O(1) beyond the output.

Pattern & takeaways

🎯 Remember these three

  • k-Sum reduces to fix (k−2) indices + two-pointer the pair.
  • Sort first; skip equal values at every level to dedupe.
  • Use a 64-bit sum to avoid overflow on large inputs.

Interview tip

🗣️ When this comes up

Present it as the recursive k-Sum template — 4Sum is 3Sum is 2Sum with extra fixed loops. The duplicate-skipping at each level is the part that trips people; narrate it.
