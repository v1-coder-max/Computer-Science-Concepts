Pacific Atlantic Water Flow

Find cells whose water can reach both oceans. The trick: search backwards from each ocean’s border, climbing uphill, then intersect.

#graphs #dfs #grid #medium #reverse-search #interview

The problem

Given a grid of heights, water flows to lower-or-equal neighbors. The Pacific touches the top/left edges, the Atlantic the bottom/right. Return cells from which water can reach both oceans.

📥 Example

A 5×5 height grid → the list of coordinates that drain to both oceans.

Brute force

From every cell, DFS to see if water can reach each ocean — O((m·n)²), since each search scans much of the grid. The reverse trick removes the per-cell repetition.

Optimized approach

Instead of asking “can this cell reach the ocean,” ask “which cells can the ocean reach if water flowed uphill.” From all Pacific border cells, DFS/BFS to neighbors with height ≥ current, marking a pacific set; do the same from the Atlantic border into an atlantic set. Any cell in both sets can drain to both oceans.

Time O(m·n), Space O(m·n).

C# solution

C# 
public IList<IList<int>> PacificAtlantic(int[][] heights)
{
    int m = heights.Length, n = heights[0].Length;
    var pac = new bool[m, n]; var atl = new bool[m, n];

    void Dfs(int r, int c, bool[,] seen, int prev)
    {
        if (r < 0 || c < 0 || r >= m || c >= n || seen[r, c] || heights[r][c] < prev) return;
        seen[r, c] = true;
        Dfs(r+1,c,seen,heights[r][c]); Dfs(r-1,c,seen,heights[r][c]);
        Dfs(r,c+1,seen,heights[r][c]); Dfs(r,c-1,seen,heights[r][c]);
    }
    for (int c = 0; c < n; c++) { Dfs(0, c, pac, int.MinValue); Dfs(m-1, c, atl, int.MinValue); }
    for (int r = 0; r < m; r++) { Dfs(r, 0, pac, int.MinValue); Dfs(r, n-1, atl, int.MinValue); }

    var result = new List<IList<int>>();
    for (int r = 0; r < m; r++) for (int c = 0; c < n; c++)
        if (pac[r, c] && atl[r, c]) result.Add(new List<int> { r, c });
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — two floods over the grid.
  • Space: O(m·n) for the two reachability grids.

Pattern & takeaways

🎯 Remember these three

  • Flip the question: search from the oceans, flowing uphill.
  • Two floods + a set intersection beats per-cell searches.
  • The uphill condition (height ≥ current) reverses normal downhill flow.

Interview tip

🗣️ When this comes up

The whole insight is reversing the search direction — start from the borders and flood uphill. Without it you re-search from every cell. Then intersect the two reachable sets.
