Minimum Falling Path Sum

Each cell's cheapest path is its own value plus the cheapest of the up-to-three cells directly above. Fill row by row and read the bottom.

#dynamic-programming #matrix #medium

The problem

Given an n×n grid, a falling path starts anywhere in the top row and at each step moves to the cell directly below or diagonally below-left/right. Return the minimum sum of such a path.

📥 Example

[[2,1,3],[6,5,4],[7,8,9]] → 13 (1 → 4 → 8... → 13)

Brute force

Enumerating all falling paths is exponential. Each cell's best path only depends on the row above, which is a clean O(n²) DP.

Optimized approach

Let dp[r][c] be the minimum falling-path sum ending at (r, c). It equals grid[r][c] plus the minimum of dp[r−1][c−1], dp[r−1][c], and dp[r−1][c+1] (whichever exist). The answer is the smallest value in the bottom row.

Time O(n²), Space O(n).

C# solution

C# 
public int MinFallingPathSum(int[][] matrix)
{
    int n = matrix.Length;
    var dp = (int[])matrix[0].Clone();
    for (int r = 1; r < n; r++)
    {
        var next = new int[n];
        for (int c = 0; c < n; c++)
        {
            int best = dp[c];
            if (c > 0) best = Math.Min(best, dp[c - 1]);
            if (c < n - 1) best = Math.Min(best, dp[c + 1]);
            next[c] = matrix[r][c] + best;
        }
        dp = next;
    }
    return dp.Min();
}

Complexity

⏱️ Complexity

  • Time: O(n²) — fill each of n² cells once.
  • Space: O(n) — one rolling row.

Pattern & takeaways

🎯 Remember these three

  • Each cell depends only on the three cells above it.
  • Roll a single row to drop space from O(n²) to O(n).
  • The answer is the minimum of the final row, not a fixed corner.
