Combination Sum II

Like Combination Sum, but each number is used once and the input has duplicates. Sort, recurse on i+1, and skip same-level repeats.

#backtracking #recursion #dedupe #medium #pruning #interview

The problem

Given candidates (with duplicates) and a target, return all unique combinations summing to target where each candidate is used at most once.

📥 Example

candidates = [10,1,2,7,6,1,5], target = 8 → [[1,1,6],[1,2,5],[1,7],[2,6]]

The idea

Two changes from Combination Sum: each element is used at most once, so recurse with i + 1 (not i); and the input has duplicates, so sort and skip a repeat at the same level (i > start && candidates[i] == candidates[i-1]) to avoid duplicate combinations. Sorting also lets you break once a candidate exceeds the remaining target.

Time O(2n) worst, Space O(n).

C# solution

C# 
public IList<IList<int>> CombinationSum2(int[] candidates, int target)
{
    Array.Sort(candidates);
    var result = new List<IList<int>>();
    Backtrack(0, target, new List<int>());
    return result;

    void Backtrack(int start, int remaining, List<int> path)
    {
        if (remaining == 0) { result.Add(new List<int>(path)); return; }
        for (int i = start; i < candidates.Length; i++)
        {
            if (candidates[i] > remaining) break;                      // prune
            if (i > start && candidates[i] == candidates[i - 1]) continue; // dedupe
            path.Add(candidates[i]);
            Backtrack(i + 1, remaining - candidates[i], path);        // each used once
            path.RemoveAt(path.Count - 1);
        }
    }
}

Complexity

⏱️ Complexity

  • Time: O(2n) worst case over combinations explored.
  • Space: O(n) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • i + 1 for single use; i for reuse (vs Combination Sum).
  • Sort + same-level skip removes duplicate combinations.
  • Sorting also enables the > remaining early break.

Interview tip

🗣️ When this comes up

Contrast the two knobs explicitly: i vs i+1 controls reuse; sort + same-level skip controls duplicate outputs. Getting both right on one problem is what the interviewer is checking.
