Search in a Binary Search Tree

A BST turns search into a guided walk: at each node, go left if the target is smaller, right if larger.

#trees #bst #easy

The problem

Given the root of a BST and a value, return the subtree rooted at the node with that value, or null if absent.

📥 Example

BST [4,2,7,1,3], target 2 → the subtree [2,1,3]

Brute force

A full traversal that checks every node is O(n) and ignores the BST property — wasteful when the ordering lets you discard half the tree at each step.

Optimized approach

Walk from the root. If the node equals the target, return it. If the target is smaller, go left; if larger, go right. Falling off the tree means the value isn't present.

Time O(h), Space O(1).

C# solution

C# 
public TreeNode SearchBST(TreeNode root, int val)
{
    while (root != null && root.val != val)
        root = val < root.val ? root.left : root.right;
    return root;
}

Complexity

⏱️ Complexity

  • Time: O(h) — h = height; O(log n) if balanced.
  • Space: O(1) — iterative, no stack.

Pattern & takeaways

🎯 Remember these three

  • The BST ordering lets each comparison discard a whole subtree.
  • Iterate instead of recursing for O(1) space.
  • Height drives the cost: O(log n) balanced, O(n) degenerate.
