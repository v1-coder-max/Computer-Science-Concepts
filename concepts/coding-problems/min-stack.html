Min Stack

Design a stack that also returns its minimum in O(1). The trick: remember the running minimum with each element, so it pops away naturally.

#stack #design #o1 #medium

The problem

Design a stack supporting Push(x), Pop(), Top(), and GetMin() — all in O(1) time.

📥 Example

Push(-2); Push(0); Push(-3); GetMin() → -3; Pop(); Top() → 0; GetMin() → -2

The naive trap

Scanning the stack for the minimum on every GetMin() is O(n) — too slow. The challenge is keeping GetMin() at O(1) even as elements are popped (the min can change when the element holding it leaves).

Optimized approach

Store, alongside each value, the minimum of the stack at the moment it was pushed. Then GetMin() is just the min recorded at the top, and when you pop, the previous element’s recorded min becomes current automatically — no rescanning. Use a stack of (value, minSoFar) pairs (or two parallel stacks).

All operations O(1); Space O(n).

C# solution

C# 
public class MinStack
{
    // Each frame stores the value AND the stack's minimum at push time.
    private readonly Stack<(int val, int min)> _stack = new();

    public void Push(int x)
    {
        int min = _stack.Count == 0 ? x : Math.Min(x, _stack.Peek().min);
        _stack.Push((x, min));
    }

    public void Pop()      => _stack.Pop();
    public int  Top()      => _stack.Peek().val;
    public int  GetMin()   => _stack.Peek().min;   // O(1) — no scanning
}

Complexity

⏱️ Complexity

  • Time: O(1) for every operation.
  • Space: O(n) — one min stored per element.

Pattern & takeaways

🎯 Remember these three

  • Carry auxiliary state (the running min) with each element so it pops away correctly.
  • Storing it per-frame avoids any rescan — GetMin() reads the top’s recorded min.
  • A pair-stack (or two parallel stacks) is the clean implementation.

Interview tip

🗣️ When this comes up

The interviewer is checking whether you keep GetMin() at O(1) through pops. The winning idea: store the min alongside each value. Mention the two-parallel-stacks variant and the optimization of only pushing to the min-stack when a new min appears.
