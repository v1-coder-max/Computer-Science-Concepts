Delete and Earn

Picking a value earns all its copies but forbids the adjacent values. Bucket earnings by value and it becomes House Robber on a number line.

#dynamic-programming #arrays #medium

The problem

Each time you take a value v, you earn v and must delete every v−1 and v+1. Maximize the total points earned from the array.

📥 Example

[2,2,3,3,3,4] → 9 (take all 3s for 9, deleting 2s and 4)

Brute force

Subset enumeration is exponential. Since taking v only conflicts with v±1, bucketing by value reduces it to a 1-D adjacency DP.

Optimized approach

Build earn[v] = v × count(v). Now choosing value v forbids v−1 and v+1 — exactly the no-adjacent rule of House Robber. Run that DP over values 0..maxV: dp[v] = max(dp[v−1], dp[v−2] + earn[v]).

Time O(n + maxV), Space O(maxV).

C# solution

C# 
public int DeleteAndEarn(int[] nums)
{
    int maxV = nums.Max();
    var earn = new int[maxV + 1];
    foreach (int x in nums) earn[x] += x;
    int prev2 = 0, prev1 = 0;
    for (int v = 0; v <= maxV; v++)
    {
        int cur = Math.Max(prev1, prev2 + earn[v]);
        prev2 = prev1; prev1 = cur;
    }
    return prev1;
}

Complexity

⏱️ Complexity

  • Time: O(n + maxV) — bucket then sweep the value range.
  • Space: O(maxV) — the earnings array.

Pattern & takeaways

🎯 Remember these three

  • Bucket earnings by value: taking v forbids v ± 1 — that's House Robber.
  • dp[v] = max(skip, take v + dp[v−2]) over the value range.
  • Recognizing the reduction is the whole problem; the DP is then standard.
