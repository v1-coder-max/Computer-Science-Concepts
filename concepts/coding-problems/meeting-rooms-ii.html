Meeting Rooms II

How many rooms at once? The answer is the peak number of overlapping meetings. A min-heap of end times tracks rooms in use.

#intervals #heap #sweep #medium #concurrency #interview

The problem

Given meeting intervals, return the minimum number of conference rooms required so no two simultaneous meetings share a room.

📥 Example

[[0,30],[5,10],[15,20]] → 2

The idea

Rooms needed = maximum concurrency. Sort meetings by start, and keep a min-heap of end times for rooms currently busy. For each meeting, if the earliest-ending room (heap top) is free by its start, reuse it (pop); otherwise allocate a new room. Push this meeting’s end. The heap’s maximum size is the answer. Equivalently, sweep sorted starts and ends, +1 on a start and −1 on an end, and track the peak.

Time O(n log n), Space O(n).

C# solution

C# 
public int MinMeetingRooms(int[][] intervals)
{
    if (intervals.Length == 0) return 0;
    Array.Sort(intervals, (a, b) => a[0] - b[0]);    // by start

    var ends = new PriorityQueue<int, int>();         // min-heap of end times
    foreach (var iv in intervals)
    {
        if (ends.Count > 0 && ends.Peek() <= iv[0])
            ends.Dequeue();                           // a room freed up → reuse
        ends.Enqueue(iv[1], iv[1]);
    }
    return ends.Count;                                // rooms in use at the peak
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — sort plus heap operations.
  • Space: O(n) — the heap of end times.

Pattern & takeaways

🎯 Remember these three

  • Rooms = peak overlap; a min-heap of end times models busy rooms.
  • Reuse the earliest-freeing room when the next meeting starts after it ends.
  • A start/end sweep with a running counter is the equivalent O(n log n).

Interview tip

🗣️ When this comes up

Reframe as “maximum concurrent meetings.” Present either the min-heap of end times or the two-pointer start/end sweep — both O(n log n). The heap version reads most naturally as “reuse the soonest-free room.”
