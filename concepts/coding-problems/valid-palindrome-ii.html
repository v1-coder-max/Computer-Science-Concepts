Valid Palindrome II

Almost a palindrome: you get one free deletion. Two pointers close in, and the first mismatch forces a single, decisive choice.

#two-pointers #strings #greedy #easy

The problem

Given a string, return true if it can be made a palindrome by removing at most one character.

📥 Example

"abca" → true (delete 'c' → "aba")

Brute force

Try deleting each character in turn and palindrome-check the result — O(n²). Almost all of that work is wasted, because a palindrome only breaks at a specific spot.

Optimized approach

Walk two pointers inward while characters match. At the first mismatch, your one deletion must remove either the left or the right character — so check whether the substring with the left skipped, or the right skipped, is a palindrome.

Time O(n), Space O(1).

C# solution

C# 
public bool ValidPalindrome(string s)
{
    int l = 0, r = s.Length - 1;
    while (l < r)
    {
        if (s[l] != s[r])
            return IsPalin(s, l + 1, r) || IsPalin(s, l, r - 1);
        l++; r--;
    }
    return true;
}
private bool IsPalin(string s, int l, int r)
{
    while (l < r) { if (s[l] != s[r]) return false; l++; r--; }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one scan, plus one linear palindrome check.
  • Space: O(1) — only pointers.

Pattern & takeaways

🎯 Remember these three

  • A palindrome fails at exactly one place — branch there, do not retry every deletion.
  • On mismatch, test skip-left OR skip-right; either passing is enough.
  • The helper check makes the one-deletion logic obvious and bug-free.

Interview tip

🗣️ When this comes up

The insight to verbalize: you only ever need one deletion, applied at the first mismatch. Then it is just two ordinary palindrome checks — say that and the O(n) bound follows.
