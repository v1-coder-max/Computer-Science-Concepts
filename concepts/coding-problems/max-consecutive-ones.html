Max Consecutive Ones

Track the current streak of 1s; a zero resets it. The longest streak you ever see is the answer.

#arrays #sliding-window #easy

The problem

Given a binary array, return the maximum number of consecutive 1s.

📥 Example

[1,1,0,1,1,1] → 3

Brute force

There is barely a brute force here — but the natural generalization is the sliding-window 'flip at most k zeros' problem, of which this is the k = 0 case.

Optimized approach

Keep a running count of consecutive 1s. On a 1, increment and update the best; on a 0, reset the run to zero. One pass, one variable.

Time O(n), Space O(1).

C# solution

C# 
public int FindMaxConsecutiveOnes(int[] nums)
{
    int cur = 0, best = 0;
    foreach (int x in nums)
    {
        cur = x == 1 ? cur + 1 : 0;
        best = Math.Max(best, cur);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single scan.
  • Space: O(1) — two counters.

Pattern & takeaways

🎯 Remember these three

  • A running streak with a reset-on-zero solves consecutive-run questions.
  • This is the k = 0 case of Max Consecutive Ones III (flip k zeros).
  • Update the best inside the loop, not only at the end.
