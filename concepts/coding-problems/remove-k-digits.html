Remove K Digits

Delete k digits to make the smallest possible number. A monotonic increasing stack greedily removes a larger digit whenever a smaller one follows.

#stack #greedy #monotonic-stack #medium #strings #interview

The problem

Given a non-negative integer as a string num and an integer k, remove k digits so the remaining number is the smallest possible. Return it as a string (no leading zeros, "0" if empty).

📥 Example

num="1432219", k=3 → "1219"  ·  num="10200", k=1 → "200"

Brute force

Trying all C(n, k) digit removals is combinatorial. The greedy insight — remove a digit that’s larger than the digit after it — lets a single stack pass build the optimum.

Optimized approach

To minimize, a digit should be removed when it’s larger than the digit to its right (a descent makes the number bigger). Use a stack: for each digit, while the budget k > 0 and the stack top is greater than the current digit, pop it (spend a removal). Push the current digit. If removals remain after the pass, trim from the end. Finally strip leading zeros; an empty result is "0".

Time O(n), Space O(n).

C# solution

C# 
public string RemoveKdigits(string num, int k)
{
    var stack = new List<char>();           // used as a stack
    foreach (char d in num)
    {
        while (k > 0 && stack.Count > 0 && stack[^1] > d) { stack.RemoveAt(stack.Count - 1); k--; }
        stack.Add(d);
    }
    stack.RemoveRange(stack.Count - k, k);  // trim any leftover removals
    int start = 0;
    while (start < stack.Count && stack[start] == '0') start++;   // strip leading zeros
    var result = new string(stack.ToArray())[start..];
    return result.Length == 0 ? "0" : result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each digit pushed/popped once.
  • Space: O(n) for the stack.

Pattern & takeaways

🎯 Remember these three

  • Greedy: remove a digit when it’s larger than the next one.
  • A monotonic increasing stack enforces this in one pass.
  • Handle leftover k (trim end) and leading zeros.

Interview tip

🗣️ When this comes up

The greedy “remove at a descent” rule plus a monotonic stack is the pattern. Don’t forget the two cleanup steps — leftover removals and leading zeros — which are common test cases.
