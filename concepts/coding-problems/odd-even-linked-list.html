Odd Even Linked List

Reorder so all odd-position nodes come before all even-position ones, keeping each group’s order. Weave two sublists with pointers.

#linked-list #pointers #medium #in-place #partition #interview

The problem

Given a singly linked list, group all nodes at odd indices together followed by the nodes at even indices (1-indexed positions, not values), preserving relative order. Do it in O(1) space.

📥 Example

1→2→3→4→5 → 1→3→5→2→4

Brute force

Collecting values into two lists and rebuilding is O(n) space. Relinking the existing nodes into an odd chain and an even chain achieves O(1) space.

Optimized approach

Keep two pointers: odd (starts at head) and even (starts at head.next), plus a saved evenHead. Repeatedly link odd.next = even.next and advance odd, then even.next = odd.next and advance even — this splices the list into an odd-position chain and an even-position chain in place. Finally connect odd.next = evenHead to join them.

Time O(n), Space O(1).

C# solution

C# 
public ListNode OddEvenList(ListNode head)
{
    if (head == null || head.next == null) return head;

    ListNode odd = head, even = head.next, evenHead = even;
    while (even != null && even.next != null)
    {
        odd.next = even.next; odd = odd.next;     // extend odd chain
        even.next = odd.next; even = even.next;   // extend even chain
    }
    odd.next = evenHead;                          // join odd then even
    return head;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — pointer relinking.

Pattern & takeaways

🎯 Remember these three

  • Split positions into an odd chain and an even chain.
  • Save evenHead to join the chains at the end.
  • It’s positional, not value-based — don’t sort.

Interview tip

🗣️ When this comes up

Clarify it’s by position, not value. The two-pointer weave is the in-place trick; saving evenHead before you start is the detail people forget.
