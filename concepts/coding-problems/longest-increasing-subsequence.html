Longest Increasing Subsequence

How long is the longest strictly increasing subsequence? A clean O(n²) DP — and a clever O(n log n) “patience” trick that binary-searches a list of tails.

#dynamic-programming #binary-search #subsequence #medium

The problem

Given an integer array nums, return the length of the longest strictly increasing subsequence (elements keep their relative order but need not be contiguous).

📥 Example

[10,9,2,5,3,7,101,18] → 4 (e.g. 2, 3, 7, 18 or 2, 5, 7, 101)

O(n²) DP

Let dp[i] = length of the longest increasing subsequence ending at index i. Then dp[i] = 1 + max(dp[j]) over all j < i with nums[j] < nums[i] (or 1 if none). The answer is max(dp). Each element looks back at all earlier ones — O(n²).

O(n log n) — patience sorting

Maintain a list tails where tails[k] is the smallest possible tail of an increasing subsequence of length k+1. For each number, binary-search the first tail ≥ it: replace it (a better, smaller tail) or append (extends the longest run). The length of tails is the answer. tails isn’t a real subsequence, but its length is correct.

Time O(n²) DP / O(n log n) tails; Space O(n).

C# solution

C# — O(n log n) tails 
public int LengthOfLIS(int[] nums)
{
    var tails = new List<int>();
    foreach (int x in nums)
    {
        int lo = 0, hi = tails.Count;          // find first tail >= x
        while (lo < hi)
        {
            int mid = (lo + hi) / 2;
            if (tails[mid] < x) lo = mid + 1;
            else                hi = mid;
        }
        if (lo == tails.Count) tails.Add(x);   // x extends the longest run
        else                   tails[lo] = x;  // x is a smaller tail for that length
    }
    return tails.Count;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) with the tails+binary-search method (O(n²) for the plain DP).
  • Space: O(n) for the dp or tails array.

Pattern & takeaways

🎯 Remember these three

  • DP definition: longest run ending at i, looking back at smaller earlier elements.
  • The tails array holds the smallest tail per length — binary search to place each value.
  • tails.Count is the answer even though tails isn’t an actual subsequence.

Interview tip

🗣️ When this comes up

Get the O(n²) DP out first (it shows you understand the recurrence), then offer the O(n log n) patience method if pushed. Be honest that the tails array gives the right length but isn’t the actual subsequence — interviewers probe that.
