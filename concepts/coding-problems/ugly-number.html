Ugly Number

Keep dividing the number by 2, then 3, then 5. If you reach 1, those were its only prime factors — it's ugly.

#math #easy

The problem

An ugly number is a positive integer whose prime factors are limited to 2, 3, and 5. Return whether a given number is ugly.

📥 Example

6 = 2·3 → true; 14 = 2·7 → false

Brute force

Full factorization is unnecessary. You only care about three specific primes, so divide them out greedily.

Optimized approach

Reject non-positive numbers. Then, for each prime in {2, 3, 5}, divide it out as long as it divides the number. If the result is 1, the number had no other prime factors and is ugly.

Time O(log n), Space O(1).

C# solution

C# 
public bool IsUgly(int n)
{
    if (n <= 0) return false;
    foreach (int p in new[] { 2, 3, 5 })
        while (n % p == 0) n /= p;
    return n == 1;
}

Complexity

⏱️ Complexity

  • Time: O(log n) — each division shrinks n.
  • Space: O(1) — constant.

Pattern & takeaways

🎯 Remember these three

  • Divide out 2, 3, 5 fully; reaching 1 means ugly.
  • Non-positive numbers are never ugly — guard first.
  • Ugly Number II (the n-th ugly number) is a different, DP/heap problem.

Interview tip

🗣️ When this comes up

Note you only divide out three known primes, not factor fully. Watch the n ≤ 0 guard, and distinguish this from Ugly Number II (generating the sequence).
