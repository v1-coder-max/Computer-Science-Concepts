Keys and Rooms

Rooms hold keys to other rooms. Starting locked in room 0, can you open them all? That is plain graph reachability.

#graphs #dfs #bfs #medium

The problem

Each room i contains a list of keys to other rooms. Starting in room 0, return true if you can visit every room.

📥 Example

[[1],[2],[3],[]] → true; [[1,3],[3,0,1],[2],[0]] → true

Brute force

There is no brute-force shortcut beyond a graph traversal; the task is recognizing it as one. The keys are directed edges and room 0 is the source.

Optimized approach

Treat rooms as nodes and keys as directed edges. DFS (or BFS) from room 0, marking rooms visited. If the visited count reaches the number of rooms, every room is reachable.

Time O(V+E), Space O(V).

C# solution

C# 
public bool CanVisitAllRooms(IList<IList<int>> rooms)
{
    var visited = new HashSet<int> { 0 };
    var stack = new Stack<int>();
    stack.Push(0);
    while (stack.Count > 0)
    {
        int room = stack.Pop();
        foreach (int key in rooms[room])
            if (visited.Add(key)) stack.Push(key);
    }
    return visited.Count == rooms.Count;
}

Complexity

⏱️ Complexity

  • Time: O(V+E) — visit each room and key once.
  • Space: O(V) — the visited set and stack.

Pattern & takeaways

🎯 Remember these three

  • Keys are directed edges; the question is reachability from node 0.
  • A visited set prevents cycles and revisits.
  • visited.Count == rooms.Count is the all-reachable test.
