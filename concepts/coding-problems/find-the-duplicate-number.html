Find the Duplicate Number

One number repeats in an array of n+1 values from 1..n. Treat values as next-pointers and the duplicate becomes a cycle entrance — Floyd finds it.

#arrays #two-pointers #medium #floyd #cycle #interview

The problem

Given an array nums of n+1 integers, each in [1, n], exactly one value is repeated (possibly many times). Find it without modifying the array and in O(1) extra space.

📥 Example

[1,3,4,2,2] → 2  ·  [3,1,3,4,2] → 3

Brute force

A HashSet detects the repeat in O(n) time but O(n) space; sorting modifies the array and is O(n log n). Both violate a constraint. The pointer-cycle view satisfies both — O(n) time, O(1) space, read-only.

Optimized approach

Read each index i → nums[i] as a linked-list jump. Since values are in [1, n] and there are n+1 of them, two indices point to the same value — that creates a cycle, and the cycle’s entrance is the duplicate. Apply Floyd’s tortoise and hare: find a meeting point, then reset one pointer to the start and advance both by one until they meet — that meeting node is the duplicate.

Time O(n), Space O(1).

C# solution

C# 
public int FindDuplicate(int[] nums)
{
    int slow = nums[0], fast = nums[0];
    do { slow = nums[slow]; fast = nums[nums[fast]]; } while (slow != fast);  // meet in cycle

    slow = nums[0];
    while (slow != fast) { slow = nums[slow]; fast = nums[fast]; }            // find entrance
    return slow;
}

Complexity

⏱️ Complexity

  • Time: O(n).
  • Space: O(1) — and the array is left unmodified.

Pattern & takeaways

🎯 Remember these three

  • Map index → value to turn it into a cycle-detection problem.
  • The duplicate is the cycle entrance — Floyd’s two-phase trick.
  • Satisfies the read-only + O(1)-space constraints a HashSet can’t.
