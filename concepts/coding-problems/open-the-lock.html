Open the Lock

Fewest turns to reach a target combination on a 4-wheel lock, dodging deadends. Each combination is a node with 8 neighbors — it’s a BFS shortest path.

#graphs #bfs #shortest-path #medium #state-space #interview

The problem

A lock has 4 wheels (digits 0–9, wrapping). From "0000", each move turns one wheel by one. Given deadends (forbidden states) and a target, return the minimum moves to open it, or -1.

📥 Example

deadends=["0201","0101","0102","1212","2002"], target="0202" → 6

Brute force

DFS exploring all move sequences is exponential and doesn’t naturally give the shortest. Because every move costs 1, BFS reaches the target in the fewest moves and visits each state once.

Optimized approach

Treat each 4-digit combination as a graph node; its 8 neighbors turn one wheel up or down (with 0↔9 wrap). Run BFS from "0000", skipping deadends and visited states, counting levels. The first time you dequeue the target, the level count is the minimum number of moves. If "0000" is a deadend, return −1 immediately.

Time O(10⁴ · 8), Space O(10⁴).

C# solution

C# 
public int OpenLock(string[] deadends, string target)
{
    var dead = new HashSet<string>(deadends);
    if (dead.Contains("0000")) return -1;
    if (target == "0000") return 0;

    var visited = new HashSet<string> { "0000" };
    var q = new Queue<string>(); q.Enqueue("0000");
    int moves = 0;

    while (q.Count > 0)
    {
        moves++;
        for (int i = q.Count; i > 0; i--)
        {
            string cur = q.Dequeue();
            for (int w = 0; w < 4; w++)
                foreach (int d in new[] { 1, 9 })            // +1 or -1 (mod 10)
                {
                    var arr = cur.ToCharArray();
                    arr[w] = (char)('0' + ((arr[w] - '0' + d) % 10));
                    string next = new string(arr);
                    if (dead.Contains(next) || !visited.Add(next)) continue;
                    if (next == target) return moves;
                    q.Enqueue(next);
                }
        }
    }
    return -1;
}

Complexity

⏱️ Complexity

  • Time: O(10⁴ · 8) — bounded state space.
  • Space: O(10⁴) for visited and queue.

Pattern & takeaways

🎯 Remember these three

  • Model the puzzle as a state graph; BFS gives min moves.
  • Each state has 8 neighbors (4 wheels × 2 directions, mod 10).
  • Skip deadends and mark visited to avoid revisiting.
