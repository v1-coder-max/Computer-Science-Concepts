Palindrome Partitioning

Cut a string into pieces that are all palindromes, every possible way. Backtracking over prefix cut points: take a palindromic prefix, recurse on the remainder.

#backtracking #strings #palindrome #medium

The problem

Given a string s, partition it so that every substring of the partition is a palindrome. Return all possible such partitionings.

📥 Example

s = "aab" → [["a","a","b"], ["aa","b"]]

The idea

At each step you decide where to make the next cut. From the current start index, try every prefix s[start..end]; if that prefix is a palindrome, add it to the current partition and recurse from end + 1; then remove it and try a longer prefix. When start reaches the end of the string, the current list is a valid full partition — record it. Checking each prefix with a two-pointer palindrome test keeps it simple.

Time O(n · 2ⁿ), Space O(n) recursion depth.

C# solution

C# — backtracking 
public IList<IList<string>> Partition(string s)
{
    var result = new List<IList<string>>();
    Backtrack(0, new List<string>());
    return result;

    void Backtrack(int start, List<string> path)
    {
        if (start == s.Length) { result.Add(new List<string>(path)); return; }
        for (int end = start; end < s.Length; end++)
        {
            if (!IsPalindrome(start, end)) continue;     // prune non-palindromic prefixes
            path.Add(s.Substring(start, end - start + 1));   // choose
            Backtrack(end + 1, path);                        // explore the rest
            path.RemoveAt(path.Count - 1);                   // un-choose
        }
    }

    bool IsPalindrome(int l, int r)
    {
        while (l < r) if (s[l++] != s[r--]) return false;
        return true;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n · 2ⁿ) — up to 2ⁿ⁻¹ partitions, each checked/copied in O(n).
  • Space: O(n) recursion depth (excluding output).

Pattern & takeaways

🎯 Remember these three

  • Partitioning = backtracking over prefix cut points with a validity test per piece.
  • Prune early: only recurse when the prefix is actually a palindrome.
  • Precompute an isPalindrome[i][j] DP table to speed up repeated checks.

Interview tip

🗣️ When this comes up

Frame it as “choose the next cut, recurse on the rest” — the partition template. Mention precomputing palindrome checks with a DP table if asked to optimize, and note the related “minimum cuts” variant is a pure DP.
