Permutation in String

Does s2 contain any rearrangement of s1? A permutation is just a matching letter count — slide a fixed window of s1’s length and compare frequencies.

#sliding-window #strings #frequency #medium

The problem

Given strings s1 and s2, return true if s2 contains a substring that is a permutation of s1 (same letters, any order).

📥 Example

s1 = "ab", s2 = "eidbaooo" → true ("ba")  ·  s1 = "ab", s2 = "eidboaoo" → false

The idea

A permutation of s1 has exactly s1’s character frequencies. So check every window of s2 with length s1.Length: if its frequency table equals s1’s, you found one. Use a fixed-size sliding window — when it advances, add the entering char and remove the leaving char in O(1), so you never recount. To compare cheaply, track how many of the 26 letter-counts currently match.

Time O(n), Space O(1) (26-letter arrays).

C# solution

C# — fixed window 
public bool CheckInclusion(string s1, string s2)
{
    if (s1.Length > s2.Length) return false;

    int[] need = new int[26], window = new int[26];
    foreach (char c in s1) need[c - 'a']++;

    for (int i = 0; i < s2.Length; i++)
    {
        window[s2[i] - 'a']++;                       // add entering char
        if (i >= s1.Length)
            window[s2[i - s1.Length] - 'a']--;       // remove leaving char

        if (window.AsSpan().SequenceEqual(need)) return true;
    }
    return false;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each char added/removed once; the 26-length compare is O(1).
  • Space: O(1) — two fixed frequency arrays.

Pattern & takeaways

🎯 Remember these three

  • “Permutation / anagram substring” → fixed-size sliding window on frequency counts.
  • Update counts incrementally (add entering, drop leaving) — never recount the window.
  • Track a matches counter to make the comparison O(1) instead of O(26).
