Cousins in Binary Tree

Cousins live on the same level but under different parents. One BFS records each node's depth and parent, then it's a comparison.

#trees #bfs #dfs #easy

The problem

Given a binary tree and two distinct values, return whether their nodes are cousins (same depth, different parents).

📥 Example

In a tree where 4 and 5 sit on level 2 under different parents → true

Brute force

Two separate searches for depth and parent work, but one traversal capturing both is cleaner.

Optimized approach

BFS (or DFS) while tracking each node's depth and parent. The two values are cousins iff their depths match but their parents differ.

Time O(n), Space O(h).

C# solution

C# 
public bool IsCousins(TreeNode root, int x, int y)
{
    int dx = 0, dy = 0;
    TreeNode px = null, py = null;
    void Dfs(TreeNode node, TreeNode parent, int depth)
    {
        if (node == null) return;
        if (node.val == x) { dx = depth; px = parent; }
        if (node.val == y) { dy = depth; py = parent; }
        Dfs(node.left, node, depth + 1);
        Dfs(node.right, node, depth + 1);
    }
    Dfs(root, null, 0);
    return dx == dy && px != py;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) — recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Cousins ⇔ equal depth AND different parents.
  • Capture depth and parent together in one traversal.
  • Two children of the same parent are siblings, not cousins.

Interview tip

🗣️ When this comes up

The definition is the whole problem: same depth, different parent. Track both in one pass and don't forget the different-parent half (siblings aren't cousins).
