Subarray Product Less Than K

Every new right endpoint adds a batch of valid subarrays. A shrinking window keeps the running product under k so you can count them in one pass.

#sliding-window #two-pointers #arrays #medium

The problem

Given an array of positive integers and an integer k, count the contiguous subarrays whose product is strictly less than k.

📥 Example

nums = [10,5,2,6], k = 100 → 8

Brute force

Checking every subarray's product is O(n²). Because the values are positive, the product is monotonic as the window grows — perfect for a sliding window.

Optimized approach

Expand the right edge, multiplying into the product. While the product is ≥ k, shrink from the left. Each right position then contributes (window length) new subarrays — all the ones ending at that index.

Time O(n), Space O(1).

C# solution

C# 
public int NumSubarrayProductLessThanK(int[] nums, int k)
{
    if (k <= 1) return 0;
    int prod = 1, left = 0, count = 0;
    for (int right = 0; right < nums.Length; right++)
    {
        prod *= nums[right];
        while (prod >= k) prod /= nums[left++];
        count += right - left + 1;   // subarrays ending at right
    }
    return count;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each pointer advances at most n times.
  • Space: O(1) — a few counters.

Pattern & takeaways

🎯 Remember these three

  • Positive values make the product monotonic — the window trick applies.
  • A window of length L ending at right contributes exactly L subarrays.
  • Guard k <= 1: no positive product can be below it.

Interview tip

🗣️ When this comes up

The counting step — adding right - left + 1 — is the crux; explain it as 'all subarrays ending here'. Also note why positivity is required for the window to work.
