Number of 1 Bits

Count the 1-bits (Hamming weight). The slick way: n &= n − 1 clears the lowest set bit, so the loop runs once per 1-bit.

#bit-manipulation #integers #easy #kernighan #popcount #interview

The problem

Given an unsigned integer, return the number of 1 bits it has (its Hamming weight).

📥 Example

11 (1011) → 3  ·  128 (10000000) → 1

The idea

Subtracting 1 from n flips its lowest set bit to 0 and all bits below it to 1. AND-ing with the original (n & (n − 1)) therefore clears exactly the lowest set bit. Loop until n is 0, counting iterations — you do one iteration per 1-bit, not per total bit. (Or just call BitOperations.PopCount.)

Time O(set bits), Space O(1).

C# solution

C# 
public int HammingWeight(uint n)
{
    int count = 0;
    while (n != 0)
    {
        n &= n - 1;      // clear the lowest set bit
        count++;
    }
    return count;
}
// Built-in: System.Numerics.BitOperations.PopCount(n)

Complexity

⏱️ Complexity

  • Time: O(number of set bits) — faster than checking all 32.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • n & (n − 1) clears the lowest set bit — the Kernighan trick.
  • Loop count = number of 1-bits, not bit width.
  • Use BitOperations.PopCount when you don’t need the manual version.
