Sort List

Sort a linked list in O(n log n). Merge sort fits lists perfectly — split by the middle, sort halves, merge — and a bottom-up version uses O(1) extra space.

#linked-list #sorting #merge-sort #medium #divide-and-conquer #interview

The problem

Given the head of a linked list, sort it in ascending order and return the new head. Aim for O(n log n) time.

📥 Example

4→2→1→3 → 1→2→3→4  ·  -1→5→3→4→0 → -1→0→3→4→5

Brute force

Copy values to an array, sort, and rewrite — O(n log n) time but O(n) extra space. Merge sort on the list itself avoids random access (which lists lack) and can run in O(1) auxiliary space bottom-up.

Optimized approach

Top-down merge sort: find the middle with slow/fast pointers, split into two halves, recursively sort each, and merge the two sorted lists (the Merge Two Sorted Lists routine). That’s O(n log n) time and O(log n) recursion stack. The bottom-up variant merges runs of size 1, 2, 4, … iteratively for O(1) auxiliary space. Linked lists are ideal for merge sort because merging only relinks pointers.

Time O(n log n), Space O(log n) top-down (O(1) bottom-up).

C# solution

C# 
public ListNode SortList(ListNode head)
{
    if (head == null || head.next == null) return head;

    // split at the middle
    ListNode slow = head, fast = head.next;
    while (fast != null && fast.next != null) { slow = slow.next; fast = fast.next.next; }
    ListNode mid = slow.next; slow.next = null;

    ListNode left = SortList(head), right = SortList(mid);
    return Merge(left, right);

    ListNode Merge(ListNode a, ListNode b)
    {
        var dummy = new ListNode(); var tail = dummy;
        while (a != null && b != null)
        {
            if (a.val <= b.val) { tail.next = a; a = a.next; }
            else                { tail.next = b; b = b.next; }
            tail = tail.next;
        }
        tail.next = a ?? b;
        return dummy.next;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n log n).
  • Space: O(log n) recursion (O(1) with bottom-up merging).

Pattern & takeaways

🎯 Remember these three

  • Merge sort is the natural O(n log n) list sort.
  • Split at the middle (slow/fast), sort halves, merge.
  • Bottom-up merging of doubling runs gives O(1) auxiliary space.

Interview tip

🗣️ When this comes up

Explain why merge sort (not quicksort) suits linked lists — sequential access and O(1) merging. Mention the bottom-up O(1)-space version if they push on the recursion stack.
