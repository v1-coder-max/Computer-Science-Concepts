Number of Provinces

Count groups of connected cities given an adjacency matrix. It’s connected components — DFS or union-find over the matrix.

#graphs #union-find #dfs #medium #components #interview

The problem

Given an n × n matrix isConnected where isConnected[i][j] = 1 means cities i and j are directly connected, return the number of provinces (maximal connected groups).

📥 Example

[[1,1,0],[1,1,0],[0,0,1]] → 2

Brute force

Manually tracing transitive connections is error-prone. Both DFS (mark a city and all it reaches) and union-find (merge connected cities) give a clean O(n²) count over the adjacency matrix.

Optimized approach

Union-Find: start with n provinces; for every pair (i, j) with isConnected[i][j] == 1, union them — each successful merge reduces the count by one. Alternatively, DFS from each unvisited city, marking everything reachable as one province and incrementing a counter per new DFS. Either is O(n²) since the matrix has n² entries.

Time O(n²), Space O(n).

C# solution

C# 
public int FindCircleNum(int[][] isConnected)
{
    int n = isConnected.Length;
    var parent = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
    int count = n;

    int Find(int x) => parent[x] == x ? x : (parent[x] = Find(parent[x]));

    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            if (isConnected[i][j] == 1)
            {
                int a = Find(i), b = Find(j);
                if (a != b) { parent[a] = b; count--; }
            }
    return count;
}

Complexity

⏱️ Complexity

  • Time: O(n²·α(n)) — scan the matrix with union-find.
  • Space: O(n) for the parent array.

Pattern & takeaways

🎯 Remember these three

  • It’s connected components on an adjacency matrix.
  • Union-Find: n minus successful merges = provinces.
  • DFS-per-unvisited-city is the equivalent O(n²) approach.

Recognize the adjacency-matrix phrasing as plain connected components. Union-find or DFS both score; the “n − merges” counter is the slick union-find form.
