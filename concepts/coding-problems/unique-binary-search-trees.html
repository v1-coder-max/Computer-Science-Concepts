Unique Binary Search Trees

How many structurally distinct BSTs hold 1..n? Pick each value as the root and multiply the ways for its left and right subtrees — the Catalan numbers.

#dynamic-programming #math #medium #catalan #counting #interview

The problem

Given n, return the number of structurally unique BSTs that store values 1..n.

📥 Example

n=3 → 5  ·  n=4 → 14

Brute force

Generating all trees and counting is exponential. The counting structure — root choice splits into independent left/right subproblems — yields a clean O(n²) DP.

Optimized approach

Let dp[k] = number of unique BSTs with k nodes. Choosing root i (1-indexed) puts i−1 nodes on the left and k−i on the right; those subtrees are independent, so the count is dp[i−1] · dp[k−i]. Sum over all roots: dp[k] = Σ dp[i−1]·dp[k−i], with dp[0] = dp[1] = 1. This is exactly the Catalan number recurrence.

Time O(n²), Space O(n).

C# solution

C# 
public int NumTrees(int n)
{
    var dp = new int[n + 1];
    dp[0] = dp[1] = 1;
    for (int nodes = 2; nodes <= n; nodes++)
        for (int root = 1; root <= nodes; root++)
            dp[nodes] += dp[root - 1] * dp[nodes - root];   // left × right
    return dp[n];
}

Complexity

⏱️ Complexity

  • Time: O(n²).
  • Space: O(n).

Pattern & takeaways

🎯 Remember these three

  • Each root splits into independent left/right subproblems → multiply.
  • Sum over all root choices → the Catalan recurrence.
  • Closed form exists: C(2n, n)/(n+1).
