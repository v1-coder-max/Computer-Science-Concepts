Count Good Nodes in Binary Tree

A node is “good” if nothing on the path from the root is larger than it. Carry the running max down each branch and count.

#trees #dfs #recursion #medium #path-max #interview

The problem

Given the root of a binary tree, a node X is good if no node on the path from the root to X has a value greater than X. Return the number of good nodes.

📥 Example

[3,1,4,3,null,1,5] → 4

The idea

“Good” depends only on the maximum value seen so far along the current root-to-node path. So DFS down the tree carrying maxSoFar: a node is good when node.val >= maxSoFar. Update the max (max(maxSoFar, node.val)) before recursing into the children. Sum the good count from both subtrees plus this node.

Time O(n), Space O(h).

C# solution

C# 
public int GoodNodes(TreeNode root)
{
    return Dfs(root, int.MinValue);

    int Dfs(TreeNode node, int maxSoFar)
    {
        if (node == null) return 0;
        int good = node.val >= maxSoFar ? 1 : 0;     // this node good?
        int nextMax = Math.Max(maxSoFar, node.val);
        return good + Dfs(node.left, nextMax) + Dfs(node.right, nextMax);
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — visit each node once.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Thread path state (the running max) down the recursion.
  • Decide the node locally, then aggregate children’s counts.
  • Same “carry context down” shape as path-sum and good-path problems.
