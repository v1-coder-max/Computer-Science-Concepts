Binary Tree Paths

Walk down the tree carrying the path so far; whenever you hit a leaf, you have a complete route to record.

#trees #dfs #backtracking #easy

The problem

Given a binary tree, return all root-to-leaf paths as strings like "1->2->5".

📥 Example

Tree [1,2,3,null,5] → ["1->2->5", "1->3"]

Brute force

There is no shortcut around visiting every node; the work is in carrying the partial path cleanly and recording it only at leaves.

Optimized approach

DFS, passing down the path accumulated so far. At a leaf (no children), append the finished path to the result. Backtracking the string is automatic when you pass a new value rather than mutating shared state.

Time O(n), Space O(h).

C# solution

C# 
public IList<string> BinaryTreePaths(TreeNode root)
{
    var res = new List<string>();
    void Dfs(TreeNode node, string path)
    {
        if (node == null) return;
        path += node.val;
        if (node.left == null && node.right == null) { res.Add(path); return; }
        Dfs(node.left, path + "->");
        Dfs(node.right, path + "->");
    }
    Dfs(root, "");
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) — recursion depth = tree height.

Pattern & takeaways

🎯 Remember these three

  • Record paths only at leaves, not at every node.
  • Passing a fresh string down avoids manual backtracking.
  • A leaf is a node with no left and no right child.

Interview tip
