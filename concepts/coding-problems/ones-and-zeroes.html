Ones and Zeroes

Each string is an item costing its zeros and ones; it's a knapsack with two capacities instead of one.

#dynamic-programming #knapsack #medium

The problem

Given an array of binary strings and budgets of m zeros and n ones, return the size of the largest subset whose total zeros ≤ m and total ones ≤ n.

📥 Example

strs = ["10","0001","111001","1","0"], m = 5, n = 3 → 4

Brute force

Trying all 2^L subsets is exponential. A 2-D knapsack DP runs in O(L·m·n).

Optimized approach

Let dp[i][j] be the most strings using ≤ i zeros and ≤ j ones. For each string with z zeros and o ones, update backwards: dp[i][j] = max(dp[i][j], dp[i−z][j−o] + 1) for i ≥ z, j ≥ o. The reverse iteration enforces 0/1 (each string used once).

Time O(L·m·n), Space O(m·n).

C# solution

C# 
public int FindMaxForm(string[] strs, int m, int n)
{
    var dp = new int[m + 1, n + 1];
    foreach (var s in strs)
    {
        int zeros = s.Count(c => c == '0');
        int ones = s.Length - zeros;
        for (int i = m; i >= zeros; i--)
            for (int j = n; j >= ones; j--)
                dp[i, j] = Math.Max(dp[i, j], dp[i - zeros, j - ones] + 1);
    }
    return dp[m, n];
}

Complexity

⏱️ Complexity

  • Time: O(L·m·n) — L strings × the 2-D table.
  • Space: O(m·n) — the m×n table.

Pattern & takeaways

🎯 Remember these three

  • A 0/1 knapsack with two capacities (zeros and ones).
  • Iterate the table backwards so each string is used at most once.
  • Each item adds 1 to the subset size, weighted by its zero/one cost.
