Find Eventual Safe States

A node is safe if it can't lead into a cycle. Color nodes during DFS — unvisited, in-progress, safe, unsafe — to detect cycles and propagate safety.

#graphs #dfs #cycle-detection #medium

The problem

In a directed graph, a node is eventually safe if every path starting from it leads to a terminal node (one with no outgoing edges). Return all safe nodes in ascending order.

📥 Example

graph = [[1,2],[2,3],[5],[0],[5],[],[]] → [2,4,5,6]

Brute force

Testing reachability of cycles from each node independently repeats work. Three-color DFS resolves every node in O(V+E).

Optimized approach

DFS with colors: 0 = unvisited, 1 = visiting (on the current path), 2 = safe, 3 = unsafe. A node is safe iff all its neighbors are safe; encountering a visiting node (back edge) means a cycle → unsafe. Memoize results so each node is settled once.

Time O(V+E), Space O(V).

C# solution

C# 
public IList<int> EventualSafeNodes(int[][] graph)
{
    int n = graph.Length;
    var color = new int[n];   // 0=unknown,1=visiting,2=safe,3=unsafe
    bool Safe(int u)
    {
        if (color[u] == 2) return true;
        if (color[u] == 1 || color[u] == 3) return false;
        color[u] = 1;
        foreach (int v in graph[u])
            if (!Safe(v)) { color[u] = 3; return false; }
        color[u] = 2;
        return true;
    }
    var res = new List<int>();
    for (int i = 0; i < n; i++) if (Safe(i)) res.Add(i);
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(V+E) — each node and edge processed once.
  • Space: O(V) — the color array + recursion.

Pattern & takeaways

🎯 Remember these three

  • Safe ⇔ no path leads into a cycle.
  • Three-color DFS detects cycles (a visiting node = back edge).
  • Memoizing safe/unsafe makes it linear.
