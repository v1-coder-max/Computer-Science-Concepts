Accounts Merge

Merge accounts that share any email. Union-Find over emails: connect emails within each account, then group by their root.

#graphs #union-find #hashing #medium #dsu #interview

The problem

Each account is a name followed by emails. Two accounts belong to the same person if they share any email. Merge them: output each person’s name with all their emails sorted. (Different people may share a name.)

📥 Example

Two "John" accounts sharing an email merge into one; a third "John" with no shared email stays separate.

Brute force

Pairwise comparing accounts for shared emails is O(n²·k). Modeling emails as nodes and unioning them captures transitive sharing in near-linear time.

Optimized approach

Treat each email as a node. For every account, union all its emails together (and map each email to its owner name). Transitive sharing — A shares with B, B with C — is handled automatically by the disjoint-set structure. Finally, group emails by their root, sort each group, and prepend the owner’s name. Path compression keeps it near-linear.

Time ~O(N·α + sorting), Space O(N).

C# solution

C# 
public IList<IList<string>> AccountsMerge(IList<IList<string>> accounts)
{
    var parent = new Dictionary<string, string>();
    var owner = new Dictionary<string, string>();
    string Find(string x) { while (parent[x] != x) { parent[x] = parent[parent[x]]; x = parent[x]; } return x; }

    foreach (var acc in accounts)
        for (int i = 1; i < acc.Count; i++)
        {
            parent.TryAdd(acc[i], acc[i]); owner[acc[i]] = acc[0];
            if (i > 1) { parent.TryAdd(acc[i - 1], acc[i - 1]); parent[Find(acc[i])] = Find(acc[i - 1]); }
        }

    var groups = new Dictionary<string, List<string>>();
    foreach (var email in parent.Keys)
        (groups.TryGetValue(Find(email), out var g) ? g : groups[Find(email)] = new()).Add(email);

    var result = new List<IList<string>>();
    foreach (var (root, emails) in groups)
    {
        emails.Sort();
        var row = new List<string> { owner[root] };
        row.AddRange(emails);
        result.Add(row);
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: ~O(N·α(N)) plus O(N log N) for sorting emails.
  • Space: O(N) for the maps.

Pattern & takeaways

🎯 Remember these three

  • Model emails as nodes; union within each account.
  • Transitive merges fall out of the disjoint-set structure.
  • Group by root, sort, and attach the owner name.
