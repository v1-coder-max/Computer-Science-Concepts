Multiply Strings

Multiply two numbers given as strings, no big-integer types. Simulate grade-school multiplication, using the fact that digits i and j land at positions i+j and i+j+1.

#math #strings #medium #simulation #carry #interview

The problem

Given two non-negative integers num1 and num2 as strings, return their product as a string. You may not convert them to native integer types.

📥 Example

num1="123", num2="456" → "56088"

Brute force

You can’t parse to int/long for large inputs (overflow), and that’s disallowed anyway. Simulating multiplication on digit arrays handles arbitrary length.

Optimized approach

The product of an m-digit and n-digit number has at most m + n digits. Use a result array of that size. Multiply every digit pair: num1[i] * num2[j] contributes to result positions i + j (carry) and i + j + 1 (units). Add into those slots and propagate carries. Finally, build the string, skipping a single leading zero (unless the product is 0).

Time O(m·n), Space O(m+n).

C# solution

C# 
public string Multiply(string num1, string num2)
{
    if (num1 == "0" || num2 == "0") return "0";
    int m = num1.Length, n = num2.Length;
    var res = new int[m + n];

    for (int i = m - 1; i >= 0; i--)
        for (int j = n - 1; j >= 0; j--)
        {
            int mul = (num1[i] - '0') * (num2[j] - '0');
            int sum = mul + res[i + j + 1];        // add into the units position
            res[i + j + 1] = sum % 10;
            res[i + j] += sum / 10;                // carry to the higher position
        }

    var sb = new System.Text.StringBuilder();
    foreach (int d in res)
        if (!(sb.Length == 0 && d == 0)) sb.Append(d);   // skip leading zero
    return sb.ToString();
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — every digit pair multiplied.
  • Space: O(m+n) for the result array.

Pattern & takeaways

🎯 Remember these three

  • Product length ≤ m + n digits — size the array accordingly.
  • Digits i, j map to positions i+j (carry) and i+j+1 (units).
  • Skip a single leading zero when building the final string.
