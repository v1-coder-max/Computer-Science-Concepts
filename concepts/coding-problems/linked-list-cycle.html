Linked List Cycle

Does a linked list loop back on itself? Floyd's tortoise and hare — a slow and a fast pointer — answer it in O(1) space, no hash set needed.

#linked-list #two-pointers #floyd #easy

The problem

Given the head of a linked list, return true if there is a cycle (some node’s next points back to an earlier node), else false.

📥 Example

3 → 2 → 0 → -4 → (back to 2) → true  ·  1 → 2 → null → false

Brute force

Walk the list and store every visited node in a HashSet; if you ever revisit a node, there’s a cycle. Correct and O(n) time — but O(n) space. The interview wants O(1) space.

Optimized approach

Use two pointers: slow advances one node per step, fast advances two. If the list ends (fast hits null), there’s no cycle. If there is a cycle, the fast pointer keeps lapping the loop and must eventually collide with slow — like a faster runner catching a slower one on a circular track. A meeting means a cycle exists.

Time O(n), Space O(1).

C# solution

C# 
public bool HasCycle(ListNode head)
{
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null)
    {
        slow = slow.next;        // +1
        fast = fast.next.next;   // +2
        if (slow == fast) return true;   // they met → cycle
    }
    return false;                // fast reached the end → no cycle
}

Complexity

⏱️ Complexity

  • Time: O(n) — slow visits each node at most once before they meet.
  • Space: O(1) — just two pointers, no extra structure.

Pattern & takeaways

🎯 Remember these three

  • Fast & slow pointers detect cycles in O(1) space — the canonical Floyd trick.
  • Guard fast != null && fast.next != null before the double hop.
  • To find the cycle’s start, reset one pointer to head and advance both by one — they meet at the entry.

Interview tip

🗣️ When this comes up

Mention the HashSet solution to show you see it, then upgrade to tortoise and hare for O(1) space — that contrast is what they’re grading. Be ready for the follow-up “where does the cycle begin?”, which is the same trick with the reset-to-head step.
