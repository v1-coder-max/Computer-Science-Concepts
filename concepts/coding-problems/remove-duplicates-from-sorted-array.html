Remove Duplicates from Sorted Array

Dedupe a sorted array in place. A slow write pointer trails a fast scan and only advances when a genuinely new value appears.

#two-pointers #arrays #easy #in-place #sorted #interview

The problem

Given a sorted array nums, remove duplicates in place so each unique value appears once, and return the count k of unique elements (the first k slots must hold them in order).

📥 Example

[0,0,1,1,1,2,2,3,3,4] → k=5, nums starts 0,1,2,3,4

Brute force

Copy uniques into a new array or use a set — O(n) extra space and loses the in-place requirement. Because the array is sorted, duplicates are adjacent, so a two-pointer in-place pass suffices.

Optimized approach

Keep a write index w (last position of a unique value). Scan with a read index r; whenever nums[r] differs from nums[w], it’s new, so advance w and copy it there. Since the array is sorted, equal values are consecutive, so one comparison against the last kept value is enough. Return w + 1.

Time O(n), Space O(1).

C# solution

C# 
public int RemoveDuplicates(int[] nums)
{
    if (nums.Length == 0) return 0;
    int w = 0;                              // last unique position
    for (int r = 1; r < nums.Length; r++)
        if (nums[r] != nums[w])             // a new value
            nums[++w] = nums[r];
    return w + 1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Slow/fast pointers compact in place when the array is sorted.
  • Compare each new value against the last kept one only.
  • Return writeIndex + 1 as the unique count.
