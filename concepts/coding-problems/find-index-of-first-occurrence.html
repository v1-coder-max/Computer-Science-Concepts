Find the Index of the First Occurrence

Find where needle first appears in haystack (strStr). A clear window check is fine for interviews; KMP makes it linear.

#strings #two-pointers #easy #pattern-matching #kmp #interview

The problem

Given strings haystack and needle, return the index of the first occurrence of needle in haystack, or -1 if it isn’t present.

📥 Example

haystack="sadbutsad", needle="sad" → 0  ·  needle="xyz" → -1

Brute force

For each start index in the haystack, compare the next m characters with the needle — O(n·m) worst case (e.g. "aaaa…" with "aaab"). Correct and usually fine, but it re-compares characters after each failure.

Optimized approach

KMP avoids re-comparing. Precompute the needle’s failure function (longest proper prefix that is also a suffix for each position). On a mismatch, instead of restarting, jump the needle pointer to the last matched prefix length — so the haystack pointer never moves backward. That yields O(n + m) total. The build of the failure table is itself O(m).

Time O(n + m), Space O(m).

C# solution

C# 
public int StrStr(string haystack, string needle)
{
    int n = haystack.Length, m = needle.Length;
    if (m == 0) return 0;

    var lps = new int[m];                 // failure function
    for (int i = 1, len = 0; i < m;)
    {
        if (needle[i] == needle[len]) lps[i++] = ++len;
        else if (len > 0) len = lps[len - 1];
        else lps[i++] = 0;
    }

    for (int i = 0, j = 0; i < n;)
    {
        if (haystack[i] == needle[j]) { i++; j++; if (j == m) return i - m; }
        else if (j > 0) j = lps[j - 1];   // reuse matched prefix
        else i++;
    }
    return -1;
}

Complexity

⏱️ Complexity

  • Time: O(n + m) with KMP (O(n·m) brute).
  • Space: O(m) for the failure table.

Pattern & takeaways

🎯 Remember these three

  • Brute force is O(n·m); state it, then offer KMP for O(n+m).
  • KMP’s failure function records reusable matched prefixes.
  • On mismatch the text pointer never backtracks.

Interview tip

🗣️ When this comes up

The brute force is acceptable, but mentioning KMP and what the failure function buys you (no text backtracking → linear) shows real string-algorithm depth. You rarely need to code KMP flawlessly, but explain it.
