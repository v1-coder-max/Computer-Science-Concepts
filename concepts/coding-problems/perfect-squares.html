Perfect Squares

Fewest perfect squares (1, 4, 9, …) that sum to n. A min-count DP: dp[i] = 1 + min(dp[i − square]) over all squares ≤ i.

#dynamic-programming #bfs #medium #knapsack #min-count #interview

The problem

Given an integer n, return the least number of perfect-square numbers (1, 4, 9, 16, …) that sum to n.

📥 Example

n=12 → 3 (4+4+4)  ·  n=13 → 2 (4+9)

Brute force

Recursively subtract each square and take the min — exponential with heavy overlap. The bounded sum lets a DP (or BFS shortest path) solve it in polynomial time.

Optimized approach

Like Coin Change with “coins” being perfect squares. Let dp[i] = fewest squares summing to i, with dp[0] = 0. For each i from 1 to n, try every square s ≤ i: dp[i] = min(dp[i], 1 + dp[i − s]). The answer is dp[n]. (A BFS over remaining values, where edges subtract a square, finds the same minimum as a shortest path.)

Time O(n·√n), Space O(n).

C# solution

C# 
public int NumSquares(int n)
{
    var dp = new int[n + 1];
    Array.Fill(dp, int.MaxValue);
    dp[0] = 0;

    for (int i = 1; i <= n; i++)
        for (int s = 1; s * s <= i; s++)
            dp[i] = Math.Min(dp[i], 1 + dp[i - s * s]);   // use square s*s

    return dp[n];
}

Complexity

⏱️ Complexity

  • Time: O(n·√n) — √n squares per amount.
  • Space: O(n).

Pattern & takeaways

🎯 Remember these three

  • It’s Coin Change with square denominations.
  • dp[i] = 1 + min(dp[i − s²]) over squares.
  • BFS shortest-path on remaining value is an equivalent solution.
