Binary Tree Level Order Traversal

Read a tree row by row, top to bottom. A queue plus the trick of processing one level’s worth of nodes per pass gives you BFS that groups by depth.

#trees #bfs #queue #medium

The problem

Given the root of a binary tree, return its node values grouped by level — a list of lists, top level first, left to right within each level.

📥 Example

[3,9,20,null,null,15,7] → [[3], [9,20], [15,7]]

Why BFS

DFS goes deep first, mixing levels. To keep levels separate you need breadth-first search: a queue that always holds the nodes of the current frontier. The key detail is to record the queue’s size before draining it, so you dequeue exactly one level per outer iteration and start a fresh sublist.

Approach

Seed the queue with the root. While it’s non-empty: capture size = queue.Count, then dequeue that many nodes — collecting their values into one row and enqueuing their children for the next level.

Time O(n), Space O(n) (queue holds up to one level, ~n/2 nodes at the widest).

C# solution

C# 
public IList<IList<int>> LevelOrder(TreeNode root)
{
    var result = new List<IList<int>>();
    if (root == null) return result;

    var q = new Queue<TreeNode>();
    q.Enqueue(root);
    while (q.Count > 0)
    {
        int size = q.Count;                 // freeze this level's width
        var level = new List<int>();
        for (int i = 0; i < size; i++)
        {
            var n = q.Dequeue();
            level.Add(n.val);
            if (n.left  != null) q.Enqueue(n.left);
            if (n.right != null) q.Enqueue(n.right);
        }
        result.Add(level);
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — every node enqueued and dequeued once.
  • Space: O(n) — the queue holds up to a full level (~n/2 at the bottom).

Pattern & takeaways

🎯 Remember these three

  • Level grouping → BFS with a queue; freeze the level size each iteration.
  • This template extends to zigzag order, right-side view, and level averages with tiny tweaks.
  • Use a queue for breadth, a stack/recursion for depth — pick by what the output needs.

Interview tip

🗣️ When this comes up

The whole problem hinges on capturing size before the inner loop — say it out loud so the interviewer sees you know why the levels stay separate. From here, zigzag and right-side-view are one-line variations.
