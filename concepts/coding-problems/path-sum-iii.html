Path Sum III

Paths can start at any node, so brute force is quadratic. A prefix-sum map — the same trick as Subarray Sum Equals K — makes it linear.

#trees #dfs #prefix-sum #medium

The problem

Count the number of downward paths (parent to child, not necessarily through the root) in a binary tree whose node values sum to a given target.

📥 Example

A tree summing to 8 → 3 qualifying paths

Brute force

Starting a fresh sum-search at every node is O(n²) in a balanced tree (worse if skewed). Tracking prefix sums removes the repeated work.

Optimized approach

DFS while maintaining the running sum from the root. A path ending at the current node sums to target whenever some earlier prefix equals (running − target) — so keep a map of prefix-sum counts, add map[running − target], then recurse and undo the count on the way back up.

Time O(n), Space O(n).

C# solution

C# 
public int PathSum(TreeNode root, int targetSum)
{
    var prefix = new Dictionary<long, int> { [0] = 1 };
    int Dfs(TreeNode node, long run)
    {
        if (node == null) return 0;
        run += node.val;
        int count = prefix.GetValueOrDefault(run - targetSum);
        prefix[run] = prefix.GetValueOrDefault(run) + 1;
        count += Dfs(node.left, run) + Dfs(node.right, run);
        prefix[run]--;                 // backtrack
        return count;
    }
    return Dfs(root, 0);
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(n) — the prefix-sum map.

Pattern & takeaways

🎯 Remember these three

  • The same prefix-sum + hash map idea as Subarray Sum Equals K, on a tree.
  • Decrement the map when leaving a node so sibling branches stay correct.
  • Seed the map with {0: 1} so paths starting at the root are counted.
