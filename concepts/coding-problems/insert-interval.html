Insert Interval

The list is already sorted and non-overlapping — slot in one new interval. Because it’s sorted, you can do it in three clean phases, no re-sort needed.

#intervals #arrays #merge #medium

The problem

Given a list of sorted, non-overlapping intervals and a newInterval, insert it and merge any overlaps, returning the still-sorted, non-overlapping result.

📥 Example

[[1,3],[6,9]], new = [2,5] → [[1,5],[6,9]]

The idea

Since the input is already sorted, walk once and split into three phases:

  1. Before: intervals that end before newInterval starts — copy them as-is.
  2. Overlap: intervals that touch newInterval — absorb them by widening newInterval to the min start / max end; then emit it once.
  3. After: intervals that start after newInterval ends — copy the rest.

Time O(n), Space O(n) — no sort because the list was already ordered.

C# solution

C# 
public int[][] Insert(int[][] intervals, int[] newInterval)
{
    var result = new List<int[]>();
    int i = 0, n = intervals.Length;

    // 1) intervals strictly before newInterval
    while (i < n && intervals[i][1] < newInterval[0])
        result.Add(intervals[i++]);

    // 2) overlapping intervals → widen newInterval
    while (i < n && intervals[i][0] <= newInterval[1])
    {
        newInterval[0] = Math.Min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.Max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.Add(newInterval);

    // 3) intervals strictly after
    while (i < n) result.Add(intervals[i++]);

    return result.ToArray();
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single linear pass, no sorting.
  • Space: O(n) for the output.

Pattern & takeaways

🎯 Remember these three

  • A pre-sorted input means O(n) — exploit the order, don’t re-sort.
  • Three phases: before → merge overlaps → after.
  • Overlap test: interval.start ≤ newInterval.end; widen with min-start / max-end.

Interview tip

🗣️ When this comes up

The trap is reaching for a sort — point out the input is already sorted and do the three-phase O(n) sweep. Naming the phases up front keeps your pointer logic from tangling, which is where most bugs hide.
