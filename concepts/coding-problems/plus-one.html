Plus One

Add one to a number stored as a digit array. Walk from the least significant digit, handling carry — with one special case when every digit is 9.

#math #arrays #easy #carry #digits #interview

The problem

Given a non-empty array of decimal digits (most significant first) representing a non-negative integer, increment it by one and return the resulting digit array.

📥 Example

[1,2,3] → [1,2,4]  ·  [9,9,9] → [1,0,0,0]

Brute force

Converting the array to an integer, adding one, and converting back fails for numbers larger than 64 bits. Operating on the digit array directly handles arbitrary length.

Optimized approach

Scan from the last digit. If a digit is less than 9, increment it and return — no further carry. If it’s 9, set it to 0 and continue the carry left. If you fall off the front (every digit was 9), the number rolled over (like 999→1000), so prepend a 1 to a zero-filled array.

Time O(n), Space O(1) (O(n) only for the all-nines new array).

C# solution

C# 
public int[] PlusOne(int[] digits)
{
    for (int i = digits.Length - 1; i >= 0; i--)
    {
        if (digits[i] < 9) { digits[i]++; return digits; }   // no carry out
        digits[i] = 0;                                       // 9 → 0, carry left
    }
    // every digit was 9 → length grows by one
    var result = new int[digits.Length + 1];
    result[0] = 1;
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — at most one pass.
  • Space: O(1), or O(n) for the all-nines rollover.

Pattern & takeaways

🎯 Remember these three

  • Process digits from the least significant end with carry.
  • Stop at the first digit < 9 — no further carry.
  • All-nines is the only case that lengthens the array.

Interview tip

🗣️ When this comes up

Avoid the integer-conversion trap (overflow on big inputs) and operate on digits. The all-nines rollover is the one edge case to mention up front.
