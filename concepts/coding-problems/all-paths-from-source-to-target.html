All Paths From Source to Target

Because the graph is a DAG, no cycles can trap you — DFS can simply extend the current path, record it at the target, and backtrack.

#graphs #dfs #backtracking #medium

The problem

Given a directed acyclic graph given as an adjacency list, return all paths from node 0 to node n − 1.

📥 Example

graph = [[1,2],[3],[3],[]] → [[0,1,3],[0,2,3]]

Brute force

Enumerating paths is inherently exponential in the worst case — there can be exponentially many. DFS with backtracking enumerates them efficiently without revisiting state.

Optimized approach

DFS from node 0, carrying the current path. At node n − 1, record a copy of the path. Otherwise recurse into each neighbor. No visited set is needed because a DAG has no cycles.

Time O(2^n · n), Space O(n).

C# solution

C# 
public IList<IList<int>> AllPathsSourceTarget(int[][] graph)
{
    var res = new List<IList<int>>();
    int target = graph.Length - 1;
    void Dfs(int node, List<int> path)
    {
        path.Add(node);
        if (node == target) res.Add(new List<int>(path));
        else foreach (int next in graph[node]) Dfs(next, path);
        path.RemoveAt(path.Count - 1);   // backtrack
    }
    Dfs(0, new List<int>());
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(2^n · n) — exponentially many paths possible.
  • Space: O(n) — recursion + current path.

Pattern & takeaways

🎯 Remember these three

  • A DAG means no cycles, so no visited set is needed for path enumeration.
  • Record a copy of the path at the target; backtrack on return.
  • Output size can be exponential — that's inherent, not a flaw.

Interview tip

🗣️ When this comes up

Two things to state: a DAG removes the need for a visited set, and you must snapshot the path when recording (mutable lists bite here). The exponential output is expected.
