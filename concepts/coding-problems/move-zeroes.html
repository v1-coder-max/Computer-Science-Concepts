Move Zeroes

Push all zeros to the end, keeping the non-zeros in order. A write pointer packs the non-zeros, then you zero-fill the tail.

#arrays #two-pointers #easy #in-place #stable #interview

The problem

Given an array nums, move all 0s to the end while keeping the relative order of the non-zero elements. Do it in place.

📥 Example

[0,1,0,3,12] → [1,3,12,0,0]

Brute force

Build a new array of non-zeros then pad with zeros — O(n) space. Or repeatedly shift on each zero — O(n²). A single write-pointer pass is O(n) and in place.

Optimized approach

Walk the array with a read index; whenever nums[r] is non-zero, place it at nums[w] and advance w (swap to keep values, or copy then zero-fill). After the pass, indices w..end are all zeros. A swap-based variant does it in one loop while preserving order.

Time O(n), Space O(1).

C# solution

C# 
public void MoveZeroes(int[] nums)
{
    int w = 0;
    for (int r = 0; r < nums.Length; r++)
        if (nums[r] != 0)
        {
            (nums[w], nums[r]) = (nums[r], nums[w]);   // swap keeps order, sinks zeros
            w++;
        }
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Stable in-place compaction with a slow write pointer.
  • Swapping preserves non-zero order while sinking zeros.
  • Same template as Remove Element / dedupe.
