Subsets II

Subsets, but the input has duplicates and the output must be unique. Sort, then skip a repeat at the same level unless it follows its twin.

#backtracking #recursion #dedupe #medium #subsets #interview

The problem

Given an integer array nums that may contain duplicates, return all possible subsets without duplicate subsets.

📥 Example

[1,2,2] → [[],[1],[1,2],[1,2,2],[2],[2,2]]

The idea

Use the subsets backtracking template, but prevent duplicate subsets. First sort so equal values are adjacent. Then in the choice loop, skip a value if it equals the previous one and the previous one wasn’t chosen at this level (i > start && nums[i] == nums[i-1]). That allows [2,2] (consecutive picks) but blocks generating the same subset from a later identical element.

Time O(n·2n), Space O(n).

C# solution

C# 
public IList<IList<int>> SubsetsWithDup(int[] nums)
{
    Array.Sort(nums);
    var result = new List<IList<int>>();
    Backtrack(0, new List<int>());
    return result;

    void Backtrack(int start, List<int> path)
    {
        result.Add(new List<int>(path));
        for (int i = start; i < nums.Length; i++)
        {
            if (i > start && nums[i] == nums[i - 1]) continue;  // skip dup at this level
            path.Add(nums[i]);
            Backtrack(i + 1, path);
            path.RemoveAt(path.Count - 1);
        }
    }
}

Complexity

⏱️ Complexity

  • Time: O(n·2n) — up to 2n subsets, each O(n) to copy.
  • Space: O(n) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Sort first so duplicates are adjacent and skippable.
  • Skip with i > start && nums[i] == nums[i-1] — same level only.
  • This same-level skip is the universal “dedupe in backtracking” trick.

Interview tip

🗣️ When this comes up

The whole problem is the dedupe rule. Explain why it’s i > start (not i > 0): you only skip a duplicate that starts a sibling branch, not one that legitimately extends the path.
