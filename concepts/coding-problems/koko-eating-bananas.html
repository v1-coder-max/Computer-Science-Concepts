Koko Eating Bananas

Find the slowest eating speed that still finishes in time. The answer isn’t in an array — but feasibility is monotonic in speed, so you binary search on the answer itself.

#binary-search #search-the-answer #monotonic #medium

The problem

Koko has piles of bananas and h hours. At speed k bananas/hour, eating a pile of size p takes ⌈p / k⌉ hours (she finishes a pile before moving on). Return the minimum integer k that lets her eat all piles within h hours.

📥 Example

piles = [3,6,7,11], h = 8 → 4

The idea

There’s no array to search — but think about the speed. If speed k works, any speed > k also works; if k is too slow, anything slower is too. That monotonic yes/no boundary is exactly what binary search needs. So search the speed range 1 … max(piles): for a candidate k, compute the hours needed; if it fits in h, try slower (move hi down), else faster (move lo up). This is “binary search on the answer.”

Time O(n · log(max pile)), Space O(1).

C# solution

C# 
public int MinEatingSpeed(int[] piles, int h)
{
    int lo = 1, hi = piles.Max();
    while (lo < hi)
    {
        int k = lo + (hi - lo) / 2;
        if (HoursNeeded(k) <= h) hi = k;     // feasible → search slower
        else                     lo = k + 1; // too slow → search faster
    }
    return lo;                                // leftmost feasible speed

    long HoursNeeded(int k)
    {
        long hours = 0;
        foreach (int p in piles) hours += (p + k - 1) / k;   // ceil(p / k)
        return hours;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n · log(max pile)) — each feasibility check scans all piles.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Binary search on the answer when feasibility is monotonic, even with no array.
  • Bounds: lowest possible answer to highest; the check(mid) predicate is the core.
  • ⌈p/k⌉ = (p + k − 1) / k with integer division.

Interview tip

🗣️ When this comes up

When a problem asks for a “minimum/maximum value that satisfies a condition,” test for monotonicity — if it’s there, say binary search on the answer. Define the check() predicate explicitly; it’s where the real logic lives.
