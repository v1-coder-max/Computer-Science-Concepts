Median of Two Sorted Arrays

The median of two sorted arrays in O(log(min(m,n))). Binary search a partition of the smaller array so the merged left half is exactly half the elements.

#binary-search #arrays #hard #partition #median #interview

The problem

Given two sorted arrays nums1 and nums2, return the median of the combined arrays in O(log(m+n)) time.

📥 Example

nums1=[1,3], nums2=[2] → 2.0  ·  nums1=[1,2], nums2=[3,4] → 2.5

Brute force

Merge both arrays and pick the middle — O(m+n) time and (naively) O(m+n) space. The logarithmic solution never merges; it binary-searches the correct partition.

Optimized approach

Binary search a cut in the smaller array. A cut i in nums1 forces a cut j in nums2 so the combined left side holds exactly half the elements. The partition is correct when leftMax1 ≤ rightMin2 and leftMax2 ≤ rightMin1 (everything on the left ≤ everything on the right). Adjust the search by comparing the four boundary values. The median is then computed from those boundaries (averaging two for an even total).

Time O(log(min(m,n))), Space O(1).

C# solution

C# 
public double FindMedianSortedArrays(int[] a, int[] b)
{
    if (a.Length > b.Length) (a, b) = (b, a);     // search the smaller array
    int m = a.Length, n = b.Length, total = m + n, half = (total + 1) / 2;
    int lo = 0, hi = m;

    while (lo <= hi)
    {
        int i = (lo + hi) / 2;                    // cut in a
        int j = half - i;                          // cut in b
        int L1 = i == 0 ? int.MinValue : a[i - 1];
        int R1 = i == m ? int.MaxValue : a[i];
        int L2 = j == 0 ? int.MinValue : b[j - 1];
        int R2 = j == n ? int.MaxValue : b[j];

        if (L1 <= R2 && L2 <= R1)                   // correct partition
        {
            int leftMax = Math.Max(L1, L2);
            if (total % 2 == 1) return leftMax;
            return (leftMax + Math.Min(R1, R2)) / 2.0;
        }
        if (L1 > R2) hi = i - 1;                    // cut a smaller
        else lo = i + 1;                           // cut a larger
    }
    return 0.0;
}

Complexity

⏱️ Complexity

  • Time: O(log(min(m,n))) — binary search the smaller array.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Binary-search the partition, not the values; search the smaller array.
  • Valid cut: both left maxes ≤ both right mins.
  • Sentinels (±∞) cleanly handle empty partition sides.
