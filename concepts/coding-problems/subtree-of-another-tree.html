Subtree of Another Tree

Is subRoot an exact subtree of root? Walk every node of the big tree and run the same-tree check there.

#trees #recursion #dfs #easy #same-tree #interview

The problem

Given the roots of two binary trees root and subRoot, return true if there is a subtree of root with the same structure and node values as subRoot.

📥 Example

root = [3,4,5,1,2], subRoot = [4,1,2] → true

The idea

A subtree match means: at some node of root, the tree hanging there is identical to subRoot. So reuse the Same Tree equality check, and try it at every node: traverse root, and at each node test SameTree(node, subRoot). The first match wins; if you exhaust the tree, there’s none. (An O(m+n) trick serializes both trees and does substring matching, but the direct double recursion is the expected answer.)

Time O(m·n), Space O(h).

C# solution

C# 
public bool IsSubtree(TreeNode root, TreeNode subRoot)
{
    if (root == null) return false;
    if (SameTree(root, subRoot)) return true;          // match anchored here?
    return IsSubtree(root.left, subRoot)               // else try children
        || IsSubtree(root.right, subRoot);
}

private bool SameTree(TreeNode a, TreeNode b)
{
    if (a == null && b == null) return true;
    if (a == null || b == null || a.val != b.val) return false;
    return SameTree(a.left, b.left) && SameTree(a.right, b.right);
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — a same-tree check (O(n)) at each of m nodes.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Subtree = identical tree anchored at some node; reuse the same-tree equality test.
  • Compose existing helpers instead of writing new traversal logic.
  • Serialization + substring search gives an O(m+n) alternative.
