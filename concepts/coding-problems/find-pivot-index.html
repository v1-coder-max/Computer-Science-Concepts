Find Pivot Index

Find the index where the left sum equals the right sum. With the total known, a running left sum settles it in one pass.

#arrays #prefix-sum #easy #balance #one-pass #interview

The problem

Given an array nums, return the leftmost index where the sum of all elements to its left equals the sum of all to its right, or -1. Out-of-bounds sides count as 0.

📥 Example

[1,7,3,6,5,6] → 3  ·  [1,2,3] → -1

Brute force

For each index, sum its left and right sides separately — O(n²). Precomputing the total and tracking a running left sum makes the right side total − left − nums[i], giving O(n).

Optimized approach

Compute the total sum. Scan with a running leftSum. At index i, the right side is total − leftSum − nums[i]. If leftSum == rightSum, return i. Otherwise add nums[i] to leftSum and continue. The first match is the pivot.

Time O(n), Space O(1).

C# solution

C# 
public int PivotIndex(int[] nums)
{
    int total = nums.Sum(), leftSum = 0;
    for (int i = 0; i < nums.Length; i++)
    {
        int rightSum = total - leftSum - nums[i];
        if (leftSum == rightSum) return i;
        leftSum += nums[i];
    }
    return -1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass after the total.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Precompute the total, then derive the right side in O(1).
  • Return the leftmost balancing index.
  • A core prefix-sum balance pattern (also “equilibrium index”).
