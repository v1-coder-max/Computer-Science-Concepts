Perfect Number

Sum the proper divisors by checking up to √n and adding both d and n/d; equality with n means perfect.

#math #easy

The problem

A perfect number equals the sum of its positive divisors excluding itself. Return whether a given number is perfect.

📥 Example

28 = 1 + 2 + 4 + 7 + 14 → true

Brute force

Summing divisors by scanning to n is O(n). Scanning to √n and adding divisor pairs is O(√n).

Optimized approach

Iterate d from 2 to √n. When d divides n, add both d and n/d (avoiding double-counting a square root). Include 1 as a proper divisor. The number is perfect iff the sum equals n (and n > 1).

Time O(√n), Space O(1).

C# solution

C# 
public bool CheckPerfectNumber(int num)
{
    if (num <= 1) return false;
    int sum = 1;                 // 1 is a proper divisor
    for (int d = 2; (long)d * d <= num; d++)
    {
        if (num % d == 0)
        {
            sum += d;
            if (d != num / d) sum += num / d;
        }
    }
    return sum == num;
}

Complexity

⏱️ Complexity

  • Time: O(√n) — scan divisors to √n.
  • Space: O(1) — a running sum.

Pattern & takeaways

🎯 Remember these three

  • Add divisors in pairs (d, n/d) up to √n.
  • Don't double-count the square root.
  • Exclude n itself (start the sum at 1).
