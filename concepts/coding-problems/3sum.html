3Sum

Find every unique triplet that sums to zero. The leap from Two Sum: sort the array, fix one number, and two-pointer the rest — while carefully skipping duplicates.

#arrays #two-pointers #sorting #medium

The problem

Given an integer array nums, return all unique triplets [a, b, c] such that a + b + c == 0. The result must not contain duplicate triplets.

📥 Example

[-1, 0, 1, 2, -1, -4] → [[-1, -1, 2], [-1, 0, 1]]

Brute force

Three nested loops over all triplets, dedup with a set. Time O(n³), Space O(n). Too slow.

Optimized approach

Sort the array first. Then fix each element nums[i] and reduce the rest to a Two Sum on a sorted array: a left pointer just after i and a right pointer at the end. If the three sum to 0, record it; if the sum is too small move left right (bigger), if too big move right left (smaller). Skip duplicate values for i, left, and right so triplets aren’t repeated.

Time O(n²) (one fixed element × the linear two-pointer scan), Space O(1) beyond the output (ignoring the sort).

C# solution

C# 
public IList<IList<int>> ThreeSum(int[] nums)
{
    Array.Sort(nums);                       // enables the two-pointer scan + easy dedup
    var result = new List<IList<int>>();

    for (int i = 0; i < nums.Length - 2; i++)
    {
        if (i > 0 && nums[i] == nums[i - 1]) continue;   // skip duplicate "fixed" values
        if (nums[i] > 0) break;                          // sorted: no triplet can sum to 0

        int left = i + 1, right = nums.Length - 1;
        while (left < right)
        {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum < 0) left++;
            else if (sum > 0) right--;
            else
            {
                result.Add(new[] { nums[i], nums[left], nums[right] });
                left++; right--;
                while (left < right && nums[left] == nums[left - 1]) left++;     // skip dups
                while (left < right && nums[right] == nums[right + 1]) right--;
            }
        }
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n²) — the sort is O(n log n); the fixed element × two-pointer scan dominates.
  • Space: O(1) beyond the output (or O(n) depending on the sort implementation).

Pattern & takeaways

🎯 Remember these three

  • Sort + two pointers turns an O(n³) triplet search into O(n²).
  • Sorting also makes duplicate-skipping trivial (skip equal neighbors for i, left, right).
  • Early exits help: once nums[i] > 0 in a sorted array, no triplet can reach 0.

🗣️ When this comes up
