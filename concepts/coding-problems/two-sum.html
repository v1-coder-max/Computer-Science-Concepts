Two Sum

Given an array and a target, return the indices of the two numbers that add up to it. The “hello world” of coding interviews — and a perfect intro to trading space for time.

#arrays #hashing #easy #two-pass-vs-one-pass

The problem

Given an integer array nums and an integer target, return the indices of the two numbers that add up to target. Exactly one solution exists, and you may not use the same element twice.

📥 Example

nums = [2, 7, 11, 15], target = 9 → [0, 1]  (because 2 + 7 = 9)

Brute force

Check every pair with two nested loops: for each i, scan every j > i and test nums[i] + nums[j] == target. Correct but quadratic.

Time O(n²), Space O(1).

Optimized approach

The key insight: for each number x, the partner you need is the complement target − x. Keep a hash map of values you’ve already seen → their index. As you walk the array once, check whether the complement is already in the map; if so, you’ve found the pair. One pass, constant-time lookups.

Time O(n), Space O(n) — we trade memory (the map) for speed.

C# solution

C# 
public int[] TwoSum(int[] nums, int target)
{
    var seen = new Dictionary<int, int>();      // value -> index
    for (int i = 0; i < nums.Length; i++)
    {
        int complement = target - nums[i];       // the partner we need
        if (seen.TryGetValue(complement, out int j))
            return new[] { j, i };               // found it — one pass
        seen[nums[i]] = i;                        // remember this value's index
    }
    return Array.Empty<int>();                    // problem guarantees a solution
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single pass with O(1) hash lookups.
  • Space: O(n) — the worst case stores nearly every element in the map.

Pattern & takeaways

🎯 Remember these three

  • The reusable trick: for pair-sum problems, look up the complement in a hash map.
  • One pass beats two — insert after checking so you never pair an element with itself.
  • It’s the classic space-for-time trade: O(n) memory buys you O(n) time over O(n²).

Interview tip

🗣️ When this comes up

State the brute force first (shows you can reason), then improve to the hash-map one-pass and call out the space-for-time trade explicitly. Watch the edge: insert into the map after the complement check so an element isn’t matched with itself.
