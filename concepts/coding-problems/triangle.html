Triangle

Minimum top-to-bottom path sum in a triangle, stepping to adjacent numbers. Work bottom-up, collapsing each row into the one below.

#dynamic-programming #arrays #medium #bottom-up #paths #interview

The problem

Given a triangle (a list of rows), find the minimum path sum from top to bottom, where each step moves to an adjacent number on the row below.

📥 Example

[[2],[3,4],[6,5,7],[4,1,8,3]] → 11 (2→3→5→1)

Brute force

Recursively branch left/right from each cell — exponential. A DP over the triangle reuses subresults; going bottom-up needs only one row of state.

Optimized approach

Process rows from the bottom up. Keep a dp array seeded with the last row. For each higher row, replace dp[i] with triangle[row][i] + min(dp[i], dp[i+1]) — the cell’s value plus the cheaper of its two children below. After collapsing all rows, dp[0] holds the minimum path sum. Only one row of memory is needed.

Time O(n²), Space O(n).

C# solution

C# 
public int MinimumTotal(IList<IList<int>> triangle)
{
    int n = triangle.Count;
    var dp = new int[n + 1];                          // extra slot avoids bounds checks
    for (int row = n - 1; row >= 0; row--)
        for (int i = 0; i <= row; i++)
            dp[i] = triangle[row][i] + Math.Min(dp[i], dp[i + 1]);
    return dp[0];
}

Complexity

⏱️ Complexity

  • Time: O(n²) — total cells in the triangle.
  • Space: O(n) — one rolling row.

Pattern & takeaways

🎯 Remember these three

  • Going bottom-up needs only one row of state.
  • Each cell adds the min of its two children.
  • Top-down works too but needs careful edge handling at row ends.

Interview tip

🗣️ When this comes up

Bottom-up is cleaner than top-down here (no boundary special cases) and uses O(n) space. The “add the min child” recurrence is the core; mention the in-place variant.
