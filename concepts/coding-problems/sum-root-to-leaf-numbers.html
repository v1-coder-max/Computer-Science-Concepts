Sum Root to Leaf Numbers

Each path spells a number, digit by digit from the root. Carry the running value down and add it up at the leaves.

#trees #dfs #medium

The problem

Each root-to-leaf path in a binary tree of digits (0–9) spells a number. Return the total sum of all such numbers.

📥 Example

Tree [1,2,3] → paths 12 and 13 → 25

Brute force

Collecting every path string and parsing it works but allocates strings needlessly. Carrying a running integer is cleaner and avoids parsing.

Optimized approach

DFS with a running value cur. At each node, update cur = cur * 10 + node.val. At a leaf, add cur to the total; otherwise recurse into both children.

Time O(n), Space O(h).

C# solution

C# 
public int SumNumbers(TreeNode root)
{
    int Dfs(TreeNode node, int cur)
    {
        if (node == null) return 0;
        cur = cur * 10 + node.val;
        if (node.left == null && node.right == null) return cur;
        return Dfs(node.left, cur) + Dfs(node.right, cur);
    }
    return Dfs(root, 0);
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) — recursion stack = height.

Pattern & takeaways

🎯 Remember these three

  • Build the number with cur × 10 + node.val as you descend.
  • Add to the sum only at leaves; internal nodes just pass cur down.
  • Returning sums up the tree avoids any external accumulator.

Interview tip

🗣️ When this comes up

Carrying the running number is cleaner than concatenating strings — say so. The base case (leaf returns cur) and the multiply-by-ten step are the two things to get exactly right.
