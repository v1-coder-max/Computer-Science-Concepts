Binary Search

On a sorted array, you don’t scan — you halve. Each comparison throws away half the remaining candidates, turning O(n) into O(log n).

#binary-search #arrays #divide-and-conquer #easy

The problem

Given a sorted array nums and a target, return its index, or -1 if it isn’t present.

📥 Example

nums = [-1,0,3,5,9,12], target = 9 → 4  ·  target = 2 → -1

Brute force

A linear scan checks every element — O(n). It works, but it ignores the sorted order. Sortedness lets us rule out half the array with a single comparison.

Optimized approach

Track a window [lo, hi]. Look at the middle element. If it equals the target, done. If it’s smaller, the target can only be to the right, so move lo = mid + 1. If it’s larger, move hi = mid - 1. Repeat until the window is empty. Compute the midpoint as lo + (hi - lo) / 2 to avoid integer overflow on large indices.

Time O(log n), Space O(1).

C# solution

C# 
public int Search(int[] nums, int target)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;   // overflow-safe midpoint
        if (nums[mid] == target) return mid;
        if (nums[mid] < target)  lo = mid + 1;  // target is to the right
        else                     hi = mid - 1;  // target is to the left
    }
    return -1;
}
// Built-in: Array.BinarySearch(nums, target) returns the index (or a negative
// bitwise-complement of the insertion point if absent).

Complexity

⏱️ Complexity

  • Time: O(log n) — the window halves every iteration.
  • Space: O(1) iterative (O(log n) stack if written recursively).

Pattern & takeaways

🎯 Remember these three

  • Binary search needs a sorted / monotonic space; each step discards half.
  • Use lo + (hi - lo) / 2 and be consistent about your loop invariant (here, inclusive [lo, hi] with lo <= hi).
  • The idea generalizes: “binary search on the answer” solves min/max feasibility problems.

Interview tip

🗣️ When this comes up

Get the boundaries right out loud: inclusive vs half-open, lo <= hi vs lo < hi. Most binary-search bugs are off-by-one in the update step. Mention the overflow-safe midpoint — it’s a small detail interviewers notice.
