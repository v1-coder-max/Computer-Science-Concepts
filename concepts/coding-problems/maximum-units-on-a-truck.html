Maximum Units on a Truck

Every box slot is equally valuable, so fill them with the boxes carrying the most units — sort by units per box, descending.

#greedy #sorting #easy

The problem

Given box types [count, unitsPerBox] and a truck that holds truckSize boxes total, return the maximum units you can load.

📥 Example

[[1,3],[2,2],[3,1]], truckSize = 4 → 8 (1×3 + 2×2 + 1×1)

Brute force

Trying combinations is exponential; the greedy choice (richest boxes first) is provably optimal.

Optimized approach

Sort box types by units per box descending. Load greedily: from each type take as many boxes as fit (min(count, remaining)), adding their units, until the truck is full.

Time O(n log n), Space O(1).

C# solution

C# 
public int MaximumUnits(int[][] boxTypes, int truckSize)
{
    Array.Sort(boxTypes, (a, b) => b[1] - a[1]);
    int units = 0;
    foreach (var box in boxTypes)
    {
        int take = Math.Min(box[0], truckSize);
        units += take * box[1];
        truckSize -= take;
        if (truckSize == 0) break;
    }
    return units;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — dominated by the sort.
  • Space: O(1) — in-place sort.

Pattern & takeaways

🎯 Remember these three

  • Each box slot is interchangeable, so fill with the densest boxes first.
  • Sort by units per box, then greedily take min(count, remaining).
  • An exchange argument proves the greedy is optimal.

Interview tip

🗣️ When this comes up

Justify the greedy with an exchange argument: swapping a low-unit box for a higher-unit one never hurts. Sort descending by units and fill until full.
