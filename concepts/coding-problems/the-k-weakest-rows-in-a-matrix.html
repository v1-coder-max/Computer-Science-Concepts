The K Weakest Rows in a Matrix

Each row's strength is its soldier count (1s come first); rank rows by (count, index) and take the k weakest.

#heap #binary-search #sorting #easy

The problem

A matrix has soldiers (1s) before civilians (0s) in each row. Return the indices of the k weakest rows, ordered by soldier count then row index.

📥 Example

Rows with soldier counts [2,4,1,2,5], k=2 → [2,0]

Brute force

Counting soldiers per row and sorting all rows is O(m log m). Binary search per row finds the count in O(log n).

Optimized approach

Compute each row's soldier count (binary search the 1/0 boundary, or sum). Pair (count, index) and select the k smallest by count then index — via sorting or a size-k max-heap.

Time O(m log m), Space O(m).

C# solution

C# 
public int[] KWeakestRows(int[][] mat, int k)
{
    int Soldiers(int[] row)
    {
        int lo = 0, hi = row.Length;
        while (lo < hi) { int mid = (lo + hi) / 2; if (row[mid] == 1) lo = mid + 1; else hi = mid; }
        return lo;
    }
    return Enumerable.Range(0, mat.Length)
        .OrderBy(i => Soldiers(mat[i]))
        .ThenBy(i => i)
        .Take(k)
        .ToArray();
}

Complexity

⏱️ Complexity

  • Time: O(m log m) — sort the rows by strength.
  • Space: O(m) — the index/count list.

Pattern & takeaways

🎯 Remember these three

  • Strength = soldier count; rank by (count, index).
  • Binary-search the 1→0 boundary to count soldiers in O(log n).
  • A size-k heap avoids fully sorting when k ≪ m.

Two nice touches: binary-searching each row's soldier boundary (since 1s precede 0s), and using a size-k heap instead of a full sort when k is much smaller than the row count.
