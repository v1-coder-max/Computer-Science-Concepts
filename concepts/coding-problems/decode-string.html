Decode String

Expand strings like 3[a2[c]] with arbitrary nesting. Two stacks — one for counts, one for partial strings — handle the brackets.

#stack #strings #medium #nested #parsing #interview

The problem

Decode a string where k[encoded] means the encoded part repeats k times (nesting allowed). Return the expanded string.

📥 Example

"3[a2[c]]" → "accaccacc"  ·  "2[abc]3[cd]ef" → "abcabccdcdcdef"

Brute force

Recursive descent parsing works, but iterative two-stack handling is cleaner and avoids deep recursion on heavy nesting. There’s no naive O(n²) trap here — the difficulty is the nested structure, which stacks resolve directly.

Optimized approach

Scan the string with a count stack and a string stack. On a digit, build the multi-digit number. On [, push the current number and current partial string, then reset them. On ], pop a count k and a previous string, and set current = previous + current·k. On a letter, append it to the current string. The stacks restore the enclosing context each time a bracket closes.

Time O(output length), Space O(output length).

C# solution

C# 
public string DecodeString(string s)
{
    var counts = new Stack<int>();
    var strings = new Stack<string>();
    string current = "";
    int k = 0;

    foreach (char c in s)
    {
        if (char.IsDigit(c)) k = k * 10 + (c - '0');     // multi-digit count
        else if (c == '[')
        {
            counts.Push(k); strings.Push(current);
            k = 0; current = "";
        }
        else if (c == ']')
        {
            int repeat = counts.Pop();
            current = strings.Pop() + string.Concat(Enumerable.Repeat(current, repeat));
        }
        else current += c;
    }
    return current;
}

Complexity

⏱️ Complexity

  • Time: O(total output length).
  • Space: O(output length) across the stacks.

Pattern & takeaways

🎯 Remember these three

  • Nested brackets → two stacks (counts + partial strings).
  • Reset count/current on [; combine on ].
  • Build multi-digit numbers before the bracket opens.

🗣️ When this comes up

The two-stack pattern (save context on open, restore-and-combine on close) generalizes to many nested-structure problems. Handle multi-digit counts and you’re done.
