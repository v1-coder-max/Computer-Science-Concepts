Generate Parentheses

List every valid arrangement of n pairs of parentheses. Backtracking with two simple rules keeps you on only valid paths — you never generate-then-filter.

#backtracking #recursion #pruning #medium

The problem

Given n pairs of parentheses, generate all combinations of well-formed (balanced) parentheses.

📥 Example

n = 3 → ["((()))","(()())","(())()","()(())","()()()"]

The idea

Build the string one character at a time, tracking how many ( and ) you’ve placed. Two constraints keep every partial string valid by construction:

  • You may add ( while open < n (haven’t used all openers).
  • You may add ) while close < open (never close more than you’ve opened).

When the string reaches length 2n, it’s a complete valid combination — record it. The rules prune all invalid branches before they’re ever built.

Time/Space O(4ⁿ / √n) (the nth Catalan number of results).

C# solution

C# — backtracking 
public IList<string> GenerateParenthesis(int n)
{
    var result = new List<string>();
    var sb = new System.Text.StringBuilder();
    Backtrack(0, 0);
    return result;

    void Backtrack(int open, int close)
    {
        if (sb.Length == 2 * n) { result.Add(sb.ToString()); return; }

        if (open < n)                       // can open another
        {
            sb.Append('(');
            Backtrack(open + 1, close);
            sb.Length--;                    // un-choose
        }
        if (close < open)                   // can close one
        {
            sb.Append(')');
            Backtrack(open, close + 1);
            sb.Length--;
        }
    }
}

Complexity

⏱️ Complexity

  • Time: O(4ⁿ / √n) — bounded by the number of valid strings (Catalan number).
  • Space: O(n) recursion depth, plus the output.

Pattern & takeaways

🎯 Remember these three

  • Prune with validity rules instead of generating all 2²ⁿ strings and filtering.
  • Two counters, two conditions: open < n and close < open.
  • Same choose/explore/un-choose skeleton as subsets and permutations.

Interview tip

🗣️ When this comes up

The signal you want to send is pruning: enforce validity as you build, never generate-and-check. State the two counter rules crisply — that’s the entire solution, and it shows you understand backtracking constraints.
