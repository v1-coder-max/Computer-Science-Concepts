Paint Fence

Each new post either differs from the last (k−1 choices) or matches the last but not the one before — leading to a clean two-term recurrence.

#dynamic-programming #medium

The problem

Paint n posts using k colors so that no three consecutive posts have the same color. Return the number of ways.

📥 Example

n = 3, k = 2 → 6

Brute force

Enumerating all colorings is k^n. The 'no three in a row' rule yields an O(n) DP.

Optimized approach

Let dp[i] be the ways to paint i posts. A new post can differ from the previous (×(k−1)) or match it (only valid if the one before differed). This gives dp[i] = (k−1)·(dp[i−1] + dp[i−2]), with dp[1]=k, dp[2]=k·k.

Time O(n), Space O(1).

C# solution

C# 
public int NumWays(int n, int k)
{
    if (n == 0) return 0;
    if (n == 1) return k;
    int prev2 = k, prev1 = k * k;       // dp[1], dp[2]
    for (int i = 3; i <= n; i++)
    {
        int cur = (k - 1) * (prev1 + prev2);
        prev2 = prev1; prev1 = cur;
    }
    return prev1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass, two rolling values.
  • Space: O(1) — constant.

Pattern & takeaways

🎯 Remember these three

  • dp[i] = (k−1)·(dp[i−1] + dp[i−2]) from differ-or-match cases.
  • Base cases dp[1]=k, dp[2]=k².
  • Two rolling variables drop space to O(1).
