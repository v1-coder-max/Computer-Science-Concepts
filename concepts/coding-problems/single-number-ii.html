Single Number II

XOR cancels pairs, not triples. Instead, count set bits at each position: bits belonging to the unique number leave a remainder mod 3.

#bit-manipulation #arrays #medium

The problem

Every element appears exactly three times except one, which appears once. Find that single element in O(n) time and O(1) space.

📥 Example

[2,2,3,2] → 3

Brute force

A hash map of counts is O(n) space. The bit-counting trick achieves O(1) by exploiting the 'three times' structure.

Optimized approach

For each of the 32 bit positions, sum that bit across all numbers. Triples contribute a multiple of 3, so count % 3 isolates the unique number's bit at that position. Reassemble those bits into the answer.

Time O(n), Space O(1).

C# solution

C# 
public int SingleNumber(int[] nums)
{
    int result = 0;
    for (int bit = 0; bit < 32; bit++)
    {
        int sum = 0;
        foreach (int x in nums) sum += (x >> bit) & 1;
        if (sum % 3 != 0) result |= 1 << bit;   // unique number's bit
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — 32 passes over n (constant factor).
  • Space: O(1) — a few integers.

Pattern & takeaways

🎯 Remember these three

  • XOR only handles pairs; for triples, count bits mod 3.
  • Each position is independent — reconstruct the answer bit by bit.
  • A two-variable 'ones/twos' bitmask gives the same result in one pass.
