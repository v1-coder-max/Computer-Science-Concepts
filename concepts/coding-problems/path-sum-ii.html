Path Sum II

List all root-to-leaf paths summing to a target. DFS while carrying the current path and remaining sum, recording a copy at each matching leaf.

#trees #backtracking #dfs #medium #root-to-leaf #interview

The problem

Given a binary tree and a targetSum, return all root-to-leaf paths whose node values sum to targetSum.

📥 Example

root=[5,4,8,11,null,13,4,7,2,null,null,5,1], targetSum=22 → [[5,4,11,2],[5,8,4,5]]

Brute force

Enumerating all paths and filtering by sum is the same traversal but with redundant path copies. Backtracking shares one path buffer and only copies on a hit.

Optimized approach

DFS with a running path list and the remaining target. At each node, append its value and subtract from the target. At a leaf, if the remaining target equals 0, record a copy of the path. Recurse into children, then backtrack by removing the node from the path so siblings reuse the buffer. This is Path Sum extended to collect the actual paths.

Time O(n) traversal (plus O(paths·length) to build output), Space O(h).

C# solution

C# 
public IList<IList<int>> PathSum(TreeNode root, int targetSum)
{
    var result = new List<IList<int>>();
    var path = new List<int>();
    Dfs(root, targetSum);
    return result;

    void Dfs(TreeNode node, int remaining)
    {
        if (node == null) return;
        path.Add(node.val);
        remaining -= node.val;
        if (node.left == null && node.right == null && remaining == 0)
            result.Add(new List<int>(path));        // copy at a matching leaf
        Dfs(node.left, remaining);
        Dfs(node.right, remaining);
        path.RemoveAt(path.Count - 1);              // backtrack
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) to traverse (plus output size).
  • Space: O(h) recursion + path buffer.

Pattern & takeaways

🎯 Remember these three

  • DFS + backtracking over a shared path buffer.
  • Copy the path only at a qualifying leaf.
  • It’s Path Sum that returns the paths, not just a boolean.

Interview tip

🗣️ When this comes up

The key efficiency point: share one path buffer and copy only on a hit, removing the node on the way back up. It’s the bridge between tree DFS and backtracking.
