Valid Parenthesis String

Parentheses with * wildcards (open, close, or empty). Track the range of possible open counts as you scan — if it can stay ≥ 0 and reach 0, it’s valid.

#greedy #strings #ranges #medium #wildcard #interview

The problem

Given a string of (, ), and * (where * can be (, ), or empty), return true if it can be a valid parenthesization.

📥 Example

"(*))" → true  ·  "(*)" → true  ·  ")(" → false

The idea

Track a range [low, high] of how many open parens are currently unmatched. On (, both rise; on ), both fall; on *, low falls (treat as )) and high rises (treat as (). Clamp low at 0 (you’d never keep an invalid negative). If high ever goes negative there are too many ) → invalid. At the end the string is valid iff low == 0 (some assignment balances).

Time O(n), Space O(1).

C# solution

C# 
public bool CheckValidString(string s)
{
    int low = 0, high = 0;       // range of possible open-paren counts
    foreach (char c in s)
    {
        if (c == '(')      { low++; high++; }
        else if (c == ')') { low--; high--; }
        else               { low--; high++; }   // '*' : ) or empty or (

        if (high < 0) return false;             // too many ')'
        if (low < 0) low = 0;                    // clamp: can't go below zero
    }
    return low == 0;             // some assignment balances exactly
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — two counters.

Pattern & takeaways

🎯 Remember these three

  • Carry a range of possible open counts, not a single value.
  • * widens the band; clamp low at 0, fail if high < 0.
  • Valid iff low == 0 at the end — zero is achievable.

Interview tip

🗣️ When this comes up

The elegant trick is tracking a range instead of branching on every *. Explain the clamp (low ≥ 0) and the fail condition (high < 0). It beats the O(n²) DP and the two-pass counting approach.
