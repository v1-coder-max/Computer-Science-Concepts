Rotate Array

Rotate an array right by k with O(1) extra space. The elegant trick: reverse all, then reverse the two parts.

#arrays #in-place #medium #reversal #rotation #interview

The problem

Given an array nums, rotate it to the right by k steps in place.

📥 Example

[1,2,3,4,5,6,7], k=3 → [5,6,7,1,2,3,4]

Brute force

Rotating one step k times is O(n·k). Copying into a new array at shifted positions is O(n) time but O(n) space. The reversal trick is O(n) time and O(1) space.

Optimized approach

First normalize k %= n. Then: reverse the entire array, reverse the first k elements, and reverse the remaining n−k. The full reverse puts the last k at the front (but reversed); the two partial reverses fix their internal order. Three linear reversals, all in place.

Time O(n), Space O(1).

C# solution

C# 
public void Rotate(int[] nums, int k)
{
    int n = nums.Length;
    k %= n;
    Reverse(0, n - 1);      // whole array
    Reverse(0, k - 1);      // first k
    Reverse(k, n - 1);      // rest

    void Reverse(int lo, int hi)
    {
        while (lo < hi) { (nums[lo], nums[hi]) = (nums[hi], nums[lo]); lo++; hi--; }
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — three reversals.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Rotation via reverse-all then reverse-parts is O(1) space.
  • Always take k %= n first (k may exceed n).
  • The cyclic-replacement method is an alternative O(1)-space approach.

Interview tip

🗣️ When this comes up

The reversal trick is the expected O(1)-space answer — derive why it works (the full reverse swaps the blocks, the partials fix order). Don’t forget k %= n.
