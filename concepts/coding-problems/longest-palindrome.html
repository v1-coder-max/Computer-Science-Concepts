Longest Palindrome

You are not finding a palindrome inside the string — you are assembling the longest one from its letters. Pairs go on the outside, one leftover can sit in the middle.

#hash-map #greedy #counting #easy

The problem

Given a string of letters, return the length of the longest palindrome you can build using those letters (case-sensitive).

📥 Example

"abccccdd" → 7 (e.g. "dccaccd")

Brute force

Generating permutations to test is hopeless. The structure of a palindrome — mirror-image halves with an optional center — makes this a pure counting problem.

Optimized approach

Count each character. Every character contributes its number of pairs × 2 to the length. If any character has an odd count, you can place one single character in the center, adding one more.

Time O(n), Space O(1).

C# solution

C# 
public int LongestPalindrome(string s)
{
    var count = new Dictionary<char, int>();
    foreach (char c in s) count[c] = count.GetValueOrDefault(c) + 1;
    int length = 0; bool hasOdd = false;
    foreach (int v in count.Values)
    {
        length += v / 2 * 2;        // contribute full pairs
        if (v % 2 == 1) hasOdd = true;
    }
    return length + (hasOdd ? 1 : 0);
}

Complexity

⏱️ Complexity

  • Time: O(n) — count then sum.
  • Space: O(1) — counts bounded by the alphabet.

Pattern & takeaways

🎯 Remember these three

  • A palindrome is pairs plus one optional center — count, don't search.
  • Add count / 2 * 2 per character, not the raw count.
  • A single odd character anywhere lets you add exactly one to the total.

Interview tip

🗣️ When this comes up

The aha is that this is counting, not searching. Be precise that at most one odd-count character contributes a center — adding one for every odd count is the common mistake.
