Set Mismatch

One number got copied over another. Sign-mark each value's slot: an already-marked slot reveals the duplicate, and the one slot left unmarked reveals the missing value.

#arrays #in-place #easy

The problem

An array should contain 1..n but one number is duplicated (replacing a missing one). Return [duplicate, missing].

📥 Example

[1,2,2,4] → [2, 3]

Brute force

Counting frequencies with a hash map works in O(n) space. The value range allows an in-place O(1) solution.

Optimized approach

Walk the array; for each value flip the sign at index |v|−1. A slot already negative pinpoints the duplicate. Afterward, the single index that is still positive is the missing number's slot.

Time O(n), Space O(1).

C# solution

C# 
public int[] FindErrorNums(int[] nums)
{
    int dup = -1;
    foreach (int x in nums)
    {
        int idx = Math.Abs(x) - 1;
        if (nums[idx] < 0) dup = idx + 1;
        else nums[idx] = -nums[idx];
    }
    int missing = -1;
    for (int i = 0; i < nums.Length; i++)
        if (nums[i] > 0) { missing = i + 1; break; }
    return new[] { dup, missing };
}

Complexity

⏱️ Complexity

  • Time: O(n) — two passes.
  • Space: O(1) — marks stored in the input.

Pattern & takeaways

🎯 Remember these three

  • Sign marking finds the duplicate (already negative) in one pass.
  • The lone still-positive index is the missing value.
  • Math (sum and sum-of-squares) gives an alternative O(1) phrasing.
