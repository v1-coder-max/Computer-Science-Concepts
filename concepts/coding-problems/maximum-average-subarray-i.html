Maximum Average Subarray I

A fixed-length window slides by adding the new element and dropping the one that left, so each step is O(1).

#sliding-window #arrays #easy

The problem

Given an array and an integer k, find the contiguous subarray of length k with the maximum average.

📥 Example

[1,12,-5,-6,50,3], k = 4 → 12.75

Brute force

Summing every length-k window from scratch is O(n·k). Reusing the previous sum makes each slide O(1).

Optimized approach

Compute the first window's sum, then slide: sum += nums[i] − nums[i−k]. Track the maximum sum and divide by k at the end (compare sums to avoid floating point until then).

Time O(n), Space O(1).

C# solution

C# 
public double FindMaxAverage(int[] nums, int k)
{
    int sum = 0;
    for (int i = 0; i < k; i++) sum += nums[i];
    int best = sum;
    for (int i = k; i < nums.Length; i++)
    {
        sum += nums[i] - nums[i - k];   // slide the window
        best = Math.Max(best, sum);
    }
    return best / (double)k;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each element added and removed once.
  • Space: O(1) — a couple of sums.

Pattern & takeaways

🎯 Remember these three

  • A fixed-size window updates in O(1) by add-new, drop-old.
  • Compare sums (integers) and divide once at the end to avoid float error.
  • This is the simplest member of the sliding-window family.

Interview tip

🗣️ When this comes up

Keep the running sum as an integer and divide only at the end — comparing averages directly invites floating-point bugs. The O(1) slide is the core technique.
