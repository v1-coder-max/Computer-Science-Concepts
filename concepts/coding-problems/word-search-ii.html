Word Search II

Searching each word separately is wasteful. Put all words in a trie and DFS the board once, pruning the moment a path leaves the trie.

#trie #backtracking #grid #hard #dfs #interview

The problem

Given an m × n board of letters and a list of words, return all words that can be formed from adjacent (4-directional) cells without reusing a cell within a word.

📥 Example

board with "oath","eat", words=["oath","pea","eat","rain"] → ["oath","eat"]

The idea

Running Word Search once per word repeats work over shared prefixes. Instead, build a trie of all the words, then DFS from each cell, walking the trie in lockstep with the board. If the current letter isn’t a child in the trie, prune immediately. When you hit a trie node marked as a word end, record that word. Mark cells visited during a path and restore on backtrack. The trie collapses all the words’ shared prefixes into one search.

Time O(m·n·4·3L−1), Space O(total chars).

C# solution

C# 
public IList<string> FindWords(char[][] board, string[] words)
{
    var root = new Node();
    foreach (var w in words)                       // build the trie
    {
        var n = root;
        foreach (char c in w) n = n.Next[c - 'a'] ??= new Node();
        n.Word = w;
    }

    var result = new List<string>();
    int rows = board.Length, cols = board[0].Length;
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            Dfs(r, c, root);
    return result;

    void Dfs(int r, int c, Node node)
    {
        if (r < 0 || c < 0 || r >= rows || c >= cols) return;
        char ch = board[r][c];
        if (ch == '#' || node.Next[ch - 'a'] == null) return;   // visited or pruned
        node = node.Next[ch - 'a'];
        if (node.Word != null) { result.Add(node.Word); node.Word = null; } // dedupe
        board[r][c] = '#';
        Dfs(r+1,c,node); Dfs(r-1,c,node); Dfs(r,c+1,node); Dfs(r,c-1,node);
        board[r][c] = ch;                                       // restore
    }
}

public class Node { public Node[] Next = new Node[26]; public string Word; }

Complexity

⏱️ Complexity

  • Time: O(m·n·4·3L−1) — DFS bounded by trie depth L.
  • Space: O(total characters in all words) for the trie.

Pattern & takeaways

🎯 Remember these three

  • Use a trie to search all words at once and prune dead prefixes.
  • Null the stored word after collecting it to dedupe automatically.
  • Mark-and-restore cells exactly like single Word Search.

Interview tip

🗣️ When this comes up

The leap from Word Search is “trie of all words, one DFS.” Stress the pruning win and the dedupe-by-nulling trick. This is the canonical example of a trie turning many searches into one.
