Valid Palindrome

Is the string the same forwards and backwards, ignoring case and punctuation? The cleanest answer walks inward from both ends — the two-pointer technique.

#strings #two-pointers #easy

The problem

Given a string s, return true if it is a palindrome considering only alphanumeric characters and ignoring case.

📥 Example

"A man, a plan, a canal: Panama" → true  ·  "race a car" → false

Brute force

Filter out non-alphanumeric characters, lowercase, build the reversed string, and compare. Time O(n), Space O(n) for the extra strings. Simple, but allocates.

Optimized approach

Use two pointers, left at the start and right at the end. Skip any non-alphanumeric character on either side, then compare the two characters case-insensitively. If they ever differ, it’s not a palindrome; otherwise move both inward until they cross. No extra string needed.

Time O(n), Space O(1).

C# solution

C# 
public bool IsPalindrome(string s)
{
    int left = 0, right = s.Length - 1;
    while (left < right)
    {
        while (left < right && !char.IsLetterOrDigit(s[left]))  left++;   // skip junk
        while (left < right && !char.IsLetterOrDigit(s[right])) right--;
        if (char.ToLowerInvariant(s[left]) != char.ToLowerInvariant(s[right]))
            return false;                                                 // mismatch
        left++; right--;                                                  // converge
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each character is visited at most once.
  • Space: O(1) — two index variables, no extra string.

Pattern & takeaways

🎯 Remember these three

  • The two-pointer converge-from-both-ends pattern is the go-to for palindrome and sorted-pair problems.
  • Skipping invalid characters in inner while loops avoids building a cleaned copy (O(1) space).
  • Use char.IsLetterOrDigit + ToLowerInvariant for correct, culture-safe comparison.

Interview tip

🗣️ When this comes up

Reach for two pointers and emphasize the O(1)-space version (skip-in-place rather than building a filtered string). Confirm the rules up front — which characters count, case sensitivity — before coding; that clarity is part of what’s being assessed.
