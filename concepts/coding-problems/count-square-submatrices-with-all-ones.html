Count Square Submatrices with All Ones

The same DP as Maximal Square, but instead of the largest side you sum every cell's value — because a cell with side k ends exactly k squares.

#dynamic-programming #matrix #medium

The problem

Given a binary matrix, count how many square submatrices are made entirely of 1s.

📥 Example

A 3×4 grid of 1s with a couple of 0s → 15 total squares

Brute force

Checking every possible square is O((m·n)·min(m,n)²). The Maximal Square recurrence reuses neighbors and counts all squares in O(m·n).

Optimized approach

Let dp[r][c] be the side of the largest all-1 square ending at (r, c) = 1 + min(top, left, top-left) for a 1-cell. A cell with value k is the corner of exactly k squares (1×1 up to k×k), so the answer is the sum of all dp values.

Time O(m·n), Space O(m·n).

C# solution

C# 
public int CountSquares(int[][] matrix)
{
    int rows = matrix.Length, cols = matrix[0].Length, total = 0;
    var dp = new int[rows][];
    for (int r = 0; r < rows; r++)
    {
        dp[r] = new int[cols];
        for (int c = 0; c < cols; c++)
        {
            if (matrix[r][c] == 0) { dp[r][c] = 0; continue; }
            dp[r][c] = (r == 0 || c == 0)
                ? 1
                : 1 + Math.Min(dp[r - 1][c], Math.Min(dp[r][c - 1], dp[r - 1][c - 1]));
            total += dp[r][c];
        }
    }
    return total;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — fill each cell once.
  • Space: O(m·n) — the DP grid (reducible to one row).

Pattern & takeaways

🎯 Remember these three

  • A cell with side k is the corner of exactly k squares.
  • Reuse the Maximal Square recurrence; just sum instead of taking the max.
  • Space drops to O(cols) with a rolling row.
