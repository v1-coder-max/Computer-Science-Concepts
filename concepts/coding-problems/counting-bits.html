Counting Bits

Count set bits for every number 0..n in O(n). The DP: dp[i] = dp[i >> 1] + (i & 1) — drop the last bit and reuse.

#bit-manipulation #dynamic-programming #easy #popcount #recurrence #interview

The problem

Given n, return an array ans of length n+1 where ans[i] is the number of 1-bits in i.

📥 Example

n = 5 → [0,1,1,2,1,2]

The idea

Counting each number independently is O(n log n). Instead, reuse earlier results: i >> 1 is i with its last bit removed, so it has the same set bits except possibly that last one. Thus dp[i] = dp[i >> 1] + (i & 1) — the count of the halved number plus 1 if i is odd. Since i >> 1 < i, it’s already computed.

Time O(n), Space O(n).

C# solution

C# 
public int[] CountBits(int n)
{
    var dp = new int[n + 1];
    for (int i = 1; i <= n; i++)
        dp[i] = dp[i >> 1] + (i & 1);   // count of i/2, plus i's last bit
    return dp;
}

Complexity

⏱️ Complexity

  • Time: O(n) — O(1) per value via the recurrence.
  • Space: O(n) for the output array.

Pattern & takeaways

🎯 Remember these three

  • i >> 1 shares all bits of i except the last.
  • Recurrence: dp[i] = dp[i >> 1] + (i & 1).
  • An alternative DP uses dp[i] = dp[i & (i−1)] + 1.

Interview tip

🗣️ When this comes up

Spotting the reuse (i >> 1 is already solved) is what turns O(n log n) into O(n). Either recurrence — drop the high bit or the low bit — works; explain why the subproblem is smaller.
