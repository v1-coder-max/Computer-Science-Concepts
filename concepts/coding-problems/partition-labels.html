Partition Labels

Split a string into as many pieces as possible so no letter spans two pieces. Extend the current cut to the last occurrence of any letter seen.

#greedy #strings #two-pointers #medium #intervals #interview

The problem

Given a string s, partition it into as many parts as possible so each letter appears in at most one part. Return the sizes of the parts in order.

📥 Example

"ababcbacadefegdehijhklij" → [9,7,8]

The idea

A part must contain the last occurrence of every letter it includes. Precompute each letter’s last index. Then scan, tracking end = the farthest last-index among letters seen so far in the current part. When the scan index reaches end, the part is complete — close it and start the next. It’s an interval-merge in disguise.

Time O(n), Space O(1) (26 letters).

C# solution

C# 
public IList<int> PartitionLabels(string s)
{
    var last = new int[26];
    for (int i = 0; i < s.Length; i++) last[s[i] - 'a'] = i;   // last index per letter

    var result = new List<int>();
    int start = 0, end = 0;
    for (int i = 0; i < s.Length; i++)
    {
        end = Math.Max(end, last[s[i] - 'a']);   // extend part to cover this letter
        if (i == end)                            // part is self-contained
        {
            result.Add(end - start + 1);
            start = i + 1;
        }
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — two linear passes.
  • Space: O(1) — 26 last-index slots.

Pattern & takeaways

🎯 Remember these three

  • A part must reach the last occurrence of each of its letters.
  • Greedily extend end; cut when the index catches up to it.
  • It’s interval merging where each letter’s span is [first, last].

Interview tip

🗣️ When this comes up

Recognize it as “each letter defines a [first,last] interval; merge overlapping ones.” The last-index precompute plus the catch-up cut is the clean O(n). Saying the interval framing shows real understanding.
