Squares of a Sorted Array

Square a sorted array and keep it sorted — in O(n). The largest squares come from the ends, so fill the result from the back with two pointers.

#two-pointers #arrays #easy #sorted #merge #interview

The problem

Given an array nums sorted in non-decreasing order, return an array of the squares of each number, also sorted non-decreasing.

📥 Example

[-4,-1,0,3,10] → [0,1,9,16,100]

Brute force

Square every element then sort — O(n log n). But the input is already sorted, so the squares are largest at the two ends; a two-pointer merge gives O(n).

Optimized approach

Negatives square to large values too, so the biggest square is at one of the two ends. Use pointers left and right; compare nums[left]² with nums[right]², place the larger at the back of the result, and move that pointer inward. Filling the result from the end keeps it sorted with no extra sort.

Time O(n), Space O(n).

C# solution

C# 
public int[] SortedSquares(int[] nums)
{
    int n = nums.Length;
    var result = new int[n];
    int left = 0, right = n - 1;
    for (int pos = n - 1; pos >= 0; pos--)
    {
        int ls = nums[left] * nums[left], rs = nums[right] * nums[right];
        if (ls > rs) { result[pos] = ls; left++; }
        else         { result[pos] = rs; right--; }
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(n) for the output.

Pattern & takeaways

🎯 Remember these three

  • Largest squares live at the ends of a sorted array.
  • Fill the result back to front with opposite-end pointers.
  • Beats the O(n log n) square-then-sort approach.
