Subsets

Produce every possible subset (the power set). For each element you make one binary choice — take it or leave it — and backtracking walks the whole decision tree.

#backtracking #recursion #combinatorics #medium

The problem

Given an array of distinct integers nums, return all possible subsets (the power set). The result must not contain duplicate subsets; order doesn’t matter.

📥 Example

[1,2,3] → [[], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]] — 2³ = 8 subsets.

The idea

There are exactly 2ⁿ subsets because each of the n elements is independently either in or out. That maps to a binary decision tree of depth n. Backtracking explores it: at index i, choose to include nums[i], recurse, then un-choose (remove it) and recurse without it. Every node of the recursion corresponds to a valid subset, so record the current path at each call.

Time O(n · 2ⁿ), Space O(n) recursion depth (plus output).

C# solution

C# — backtracking 
public IList<IList<int>> Subsets(int[] nums)
{
    var result = new List<IList<int>>();
    Backtrack(0, new List<int>());
    return result;

    void Backtrack(int start, List<int> path)
    {
        result.Add(new List<int>(path));        // every node is a valid subset
        for (int i = start; i < nums.Length; i++)
        {
            path.Add(nums[i]);                  // choose
            Backtrack(i + 1, path);             // explore
            path.RemoveAt(path.Count - 1);      // un-choose (backtrack)
        }
    }
}

Complexity

⏱️ Complexity

  • Time: O(n · 2ⁿ) — 2ⁿ subsets, each up to length n to copy.
  • Space: O(n) recursion depth (output is O(n · 2ⁿ)).

Pattern & takeaways

🎯 Remember these three

  • The backtracking template is choose → explore → un-choose.
  • Passing a start index avoids duplicate/permuted subsets.
  • The same skeleton generates permutations, combinations, and combination-sum — only the choices and pruning change.

Interview tip

🗣️ When this comes up

Subsets is the gateway to all backtracking problems — nail the choose/explore/un-choose loop and the start index. Mention the bitmask alternative (iterate 0…2ⁿ-1, bit j = include nums[j]) to show range.
