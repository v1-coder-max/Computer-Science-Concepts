Maximum Subarray (Kadane’s)

Find the contiguous subarray with the largest sum. The elegant O(n) answer — Kadane’s algorithm — is a gateway to dynamic-programming intuition.

#arrays #dynamic-programming #kadane #medium

The problem

Given an integer array nums (which may contain negatives), find the contiguous subarray with the largest sum and return that sum.

📥 Example

[-2, 1, -3, 4, -1, 2, 1, -5, 4] → 6  (the subarray [4, -1, 2, 1])

Brute force

Consider every subarray (start i, end j) and sum it. Time O(n²) (with a running sum) or O(n³) naïvely, Space O(1).

Optimized approach

Kadane’s insight: walk left to right keeping a running sum current of the best subarray ending at the current index. At each element you face one choice — extend the previous subarray, or start fresh here: current = max(num, current + num). If the running sum ever goes negative, it can only hurt what follows, so you drop it and restart. Track the best current ever seen.

Time O(n), Space O(1). That “best ending here” recurrence is dynamic programming in disguise.

C# solution

C# 
public int MaxSubArray(int[] nums)
{
    int current = nums[0];   // best sum ENDING at the current index
    int best = nums[0];      // best sum seen anywhere

    for (int i = 1; i < nums.Length; i++)
    {
        // extend the previous subarray, or start a new one at nums[i]
        current = Math.Max(nums[i], current + nums[i]);
        best = Math.Max(best, current);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single pass.
  • Space: O(1) — just two running integers.

Pattern & takeaways

🎯 Remember these three

  • Kadane = a running sum that resets whenever it turns negative (it can only drag the future down).
  • The recurrence current = max(num, current + num) is DP: best answer ending here.
  • Start best at nums[0] (not 0) so all-negative arrays return the largest single element.

Interview tip

🗣️ When this comes up

Name Kadane’s algorithm and explain the “extend vs restart” choice — interviewers want the O(n) DP intuition, not just the code. The classic trap is initializing best = 0, which breaks on all-negative inputs; start from nums[0].
