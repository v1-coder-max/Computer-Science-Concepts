Isomorphic Strings

Two strings are isomorphic if you can rename characters of one into the other consistently. The catch: the renaming must be a bijection.

#hash-map #strings #easy

The problem

Given strings s and t, return true if the characters of s can be replaced to get t, with each character mapping to exactly one character and no two characters mapping to the same one.

📥 Example

"egg" → "add" is true; "foo" → "bar" is false.

Brute force

Encoding each string as a pattern of first-occurrence indices and comparing works, but it is easy to get subtly wrong. A direct two-way map is clearer and equally fast.

Optimized approach

Walk both strings together, maintaining a map s→t and a set of used targets. Each pair must either confirm an existing mapping or create a fresh one to an unused target; any conflict means not isomorphic.

Time O(n), Space O(1).

C# solution

C# 
public bool IsIsomorphic(string s, string t)
{
    if (s.Length != t.Length) return false;
    var map = new Dictionary<char, char>();
    var used = new HashSet<char>();
    for (int i = 0; i < s.Length; i++)
    {
        char a = s[i], b = t[i];
        if (map.TryGetValue(a, out char m))
        {
            if (m != b) return false;
        }
        else
        {
            if (used.Contains(b)) return false;  // target already taken
            map[a] = b; used.Add(b);
        }
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass over both strings.
  • Space: O(1) — maps bounded by the alphabet size.

Pattern & takeaways

🎯 Remember these three

  • Isomorphism needs a bijection — check the mapping in both directions.
  • A consistent existing mapping or a fresh map to an unused target; anything else fails.
  • Equal lengths is a quick necessary precondition.

Interview tip

🗣️ When this comes up

The trap is checking only one direction (s→t) and missing two characters mapping to the same target. Call out the used set explicitly — that is the part interviewers probe.
