Evaluate Division

Given ratios like a/b = 2, answer queries like a/c. Model variables as a weighted graph; the answer is a path’s product of edge weights.

#graphs #dfs #weighted #medium #ratios #interview

The problem

Given equations a/b = value, answer queries x/y: return the computed ratio, or -1.0 if it can’t be determined.

📥 Example

a/b=2, b/c=3; query a/c → 6.0; query a/e → -1.0; query x/x (unknown) → -1.0

Brute force

There’s no transitive arithmetic without building relationships. Modeling variables as graph nodes with weighted edges (and the reciprocal back-edge) makes each query a graph traversal.

Optimized approach

Build a graph: each equation a/b = v adds edge a→b with weight v and b→a with weight 1/v. For a query x/y, DFS/BFS from x to y, multiplying edge weights along the path; the accumulated product is the answer. If x or y is unknown or unreachable, return -1. (Weighted union-find that stores each node’s ratio to its root answers queries in near-O(1).)

Time O(Q·(V+E)), Space O(V+E).

C# solution

C# 
public double[] CalcEquation(IList<IList<string>> equations, double[] values,
                              IList<IList<string>> queries)
{
    var graph = new Dictionary<string, List<(string to, double w)>>();
    for (int i = 0; i < equations.Count; i++)
    {
        string a = equations[i][0], b = equations[i][1];
        graph.TryAdd(a, new()); graph.TryAdd(b, new());
        graph[a].Add((b, values[i]));
        graph[b].Add((a, 1.0 / values[i]));
    }

    double Dfs(string cur, string target, double acc, HashSet<string> seen)
    {
        if (cur == target) return acc;
        seen.Add(cur);
        foreach (var (next, w) in graph[cur])
            if (!seen.Contains(next))
            {
                double res = Dfs(next, target, acc * w, seen);
                if (res != -1) return res;
            }
        return -1;
    }

    var result = new double[queries.Count];
    for (int i = 0; i < queries.Count; i++)
    {
        string x = queries[i][0], y = queries[i][1];
        result[i] = (graph.ContainsKey(x) && graph.ContainsKey(y))
            ? Dfs(x, y, 1.0, new()) : -1.0;
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(Q·(V+E)) with per-query DFS.
  • Space: O(V+E) for the graph.

Pattern & takeaways

🎯 Remember these three

  • Variables = nodes; equations = weighted edges (with reciprocals).
  • A query is a path whose weights multiply.
  • Weighted union-find answers queries in near-constant time.
