Maximum Product of Three Numbers

The winner is either the three largest values, or the two most negative times the largest — negatives can multiply into a big positive.

#arrays #math #greedy #easy

The problem

Given an integer array, return the maximum product of any three of its elements.

📥 Example

[-4,-3,-2,1,60] → 720 ((−4)·(−3)·60)

Brute force

Checking all triples is O(n³); sorting and comparing the two candidate products is O(n log n). A single pass tracking five extremes is O(n).

Optimized approach

The answer is max(top1·top2·top3, top1·bottom1·bottom2). Track the three largest and two smallest values as you scan; the second candidate captures the case where two negatives flip to a large positive.

Time O(n), Space O(1).

C# solution

C# 
public int MaximumProduct(int[] nums)
{
    int a1 = int.MinValue, a2 = int.MinValue, a3 = int.MinValue;
    int b1 = int.MaxValue, b2 = int.MaxValue;
    foreach (int x in nums)
    {
        if (x > a1) { a3 = a2; a2 = a1; a1 = x; }
        else if (x > a2) { a3 = a2; a2 = x; }
        else if (x > a3) a3 = x;
        if (x < b1) { b2 = b1; b1 = x; }
        else if (x < b2) b2 = x;
    }
    return Math.Max(a1 * a2 * a3, a1 * b1 * b2);
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — five running extremes.

Pattern & takeaways

🎯 Remember these three

  • Two large negatives can beat three positives — keep the bottom two.
  • Compare exactly two candidates: top-three vs top-one × bottom-two.
  • One pass tracking five values avoids the sort.
