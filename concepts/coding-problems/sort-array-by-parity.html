Sort Array By Parity

Two pointers from both ends: advance past evens on the left and odds on the right, swapping the mismatched pair in between.

#two-pointers #arrays #in-place #easy

The problem

Given an integer array, rearrange it so every even number comes before every odd number (any such order is fine).

📥 Example

[3,1,2,4] → [2,4,3,1] (or any even-then-odd arrangement)

Brute force

Building a new array by partitioning is O(n) time and space. Two pointers do it in place.

Optimized approach

Put l at the start and r at the end. While l < r: if a[l] is even, advance l; if a[r] is odd, retreat r; otherwise (odd at l, even at r) swap and move both.

Time O(n), Space O(1).

C# solution

C# 
public int[] SortArrayByParity(int[] nums)
{
    int l = 0, r = nums.Length - 1;
    while (l < r)
    {
        if (nums[l] % 2 == 0) l++;
        else if (nums[r] % 2 == 1) r--;
        else { (nums[l], nums[r]) = (nums[r], nums[l]); l++; r--; }
    }
    return nums;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each element visited once.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Two pointers partition by a predicate (even vs odd) in place.
  • Only swap when both ends violate the order.
  • It's the Dutch-flag idea with a single split point.

Interview tip

🗣️ When this comes up

Frame it as in-place partitioning. The swap-only-on-double-violation rule keeps it O(1) space, and it generalizes to any two-way partition predicate.
