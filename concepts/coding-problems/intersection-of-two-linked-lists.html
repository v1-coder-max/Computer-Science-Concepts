Intersection of Two Linked Lists

Where do two lists merge? Two pointers that switch to the other list at the end travel equal total distance and meet at the intersection.

#linked-list #two-pointers #easy #intersection #length-align #interview

The problem

Given the heads of two singly linked lists, return the node where they intersect (share the same nodes from there on), or null if they don’t.

📥 Example

Two lists sharing a tail starting at node 8 → return node 8.

Brute force

Store all nodes of one list in a HashSet, then scan the other for the first shared node — O(n+m) time but O(n) space. The pointer-switch trick achieves O(1) space.

Optimized approach

Walk a through list A and b through list B. When a pointer hits the end, redirect it to the other list’s head. After at most one switch each, both pointers have traversed lenA + lenB nodes, so they arrive at the intersection simultaneously (or both reach null together if there’s no intersection). No length precomputation needed.

Time O(n + m), Space O(1).

C# solution

C# 
public ListNode GetIntersectionNode(ListNode headA, ListNode headB)
{
    ListNode a = headA, b = headB;
    while (a != b)
    {
        a = a == null ? headB : a.next;   // switch to the other list at the end
        b = b == null ? headA : b.next;
    }
    return a;   // intersection node, or null
}

Complexity

⏱️ Complexity

  • Time: O(n + m).
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Switching lists at the end equalizes the path length.
  • Both pointers traverse lenA + lenB and meet at the join.
  • The null-handling makes the no-intersection case terminate cleanly.

Interview tip

🗣️ When this comes up

The pointer-switch trick is the elegant O(1)-space answer — explain why both pointers cover the same total distance. Mention the length-difference alignment as the more intuitive alternative.
