Number of Enclaves

Land touching the border can walk off the grid. Flood all of that away, and whatever 1s remain are trapped enclaves.

#graphs #dfs #matrix #medium

The problem

Given a binary grid, return the number of land cells (1s) from which you cannot walk off the boundary in any number of 4-directional moves.

📥 Example

A grid with one interior 1×3 strip → 3 enclaves

Brute force

Checking reachability from every land cell independently repeats work. Flooding from the border once marks all escapable land in O(m·n).

Optimized approach

Flood-fill (DFS/BFS) starting from every land cell on the border, marking them as escapable. Then count the remaining 1s — those are the enclaves that can't reach the edge.

Time O(m·n), Space O(m·n).

C# solution

C# 
public int NumEnclaves(int[][] grid)
{
    int rows = grid.Length, cols = grid[0].Length;
    void Flood(int r, int c)
    {
        if (r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] != 1) return;
        grid[r][c] = 0;
        Flood(r + 1, c); Flood(r - 1, c); Flood(r, c + 1); Flood(r, c - 1);
    }
    for (int r = 0; r < rows; r++) { Flood(r, 0); Flood(r, cols - 1); }
    for (int c = 0; c < cols; c++) { Flood(0, c); Flood(rows - 1, c); }
    int count = 0;
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            if (grid[r][c] == 1) count++;
    return count;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — each cell visited a constant number of times.
  • Space: O(m·n) — recursion/visited worst case.

Pattern & takeaways

🎯 Remember these three

  • Flood from the border to eliminate escapable land first.
  • Remaining 1s are the enclaves — just count them.
  • Same 'sink the border' trick as Surrounded Regions and Pacific Atlantic.
