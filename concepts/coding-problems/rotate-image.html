Rotate Image

Rotate an n × n image 90° clockwise in place. Two simple moves compose the rotation: transpose, then reverse each row.

#matrix #arrays #medium #in-place #transpose #interview

The problem

Given an n × n matrix representing an image, rotate it 90° clockwise in place (no separate output matrix).

📥 Example

[[1,2,3],[4,5,6],[7,8,9]] → [[7,4,1],[8,5,2],[9,6,3]]

Brute force

Write into a new matrix with out[c][n−1−r] = in[r][c] — clear but O(n²) extra space. Composing a transpose with a row-reverse achieves the same rotation in place.

Optimized approach

Transpose the matrix (swap matrix[r][c] with matrix[c][r] for c > r), turning rows into columns. Then reverse each row. Together that’s a 90° clockwise rotation, all with in-place swaps. (Counter-clockwise: transpose then reverse each column instead.)

Time O(n²), Space O(1).

C# solution

C# 
public void Rotate(int[][] matrix)
{
    int n = matrix.Length;
    for (int r = 0; r < n; r++)                  // transpose
        for (int c = r + 1; c < n; c++)
            (matrix[r][c], matrix[c][r]) = (matrix[c][r], matrix[r][c]);

    foreach (var row in matrix)                  // reverse each row
        Array.Reverse(row);
}

Complexity

⏱️ Complexity

  • Time: O(n²) — touch each cell a constant number of times.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • 90° clockwise = transpose + reverse each row.
  • Transpose only the upper triangle (c > r) to avoid double-swaps.
  • Counter-clockwise = transpose + reverse each column.

Interview tip

🗣️ When this comes up

The transpose-then-reverse decomposition is the clean in-place answer — far less error-prone than rotating rings four cells at a time. Be ready to state the counter-clockwise variant.
