Last Stone Weight II

Smashing stones is equivalent to assigning each a ± sign; minimizing the result means splitting the stones into two groups whose sums are as equal as possible.

#dynamic-programming #knapsack #medium

The problem

Repeatedly smash the two heaviest stones (the difference remains). Return the smallest possible weight of the last remaining stone.

📥 Example

[2,7,4,1,8,1] → 1

Brute force

Simulating every smash order is exponential. The ±-sign reframing turns it into subset-sum.

Optimized approach

The final weight is |sum(group A) − sum(group B)|, minimized when each group sums near total/2. Use subset-sum DP to find the largest reachable sum s ≤ total/2; the answer is total − 2·s.

Time O(n·sum), Space O(sum).

C# solution

C# 
public int LastStoneWeightII(int[] stones)
{
    int total = stones.Sum(), half = total / 2;
    var dp = new bool[half + 1];
    dp[0] = true;
    foreach (int stone in stones)
        for (int s = half; s >= stone; s--)
            dp[s] = dp[s] || dp[s - stone];
    for (int s = half; s >= 0; s--)
        if (dp[s]) return total - 2 * s;     // closest split
    return total;
}

Complexity

⏱️ Complexity

  • Time: O(n·sum) — n stones × the sum axis.
  • Space: O(sum) — the 1-D reachability array.

Pattern & takeaways

🎯 Remember these three

  • Smashing ⇔ assigning ± signs ⇔ splitting into two groups.
  • Find the largest reachable subset sum ≤ total/2.
  • Answer = total − 2·bestSum.
