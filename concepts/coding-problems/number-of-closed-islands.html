Number of Closed Islands

Land touching the border can't be a closed island, so flood it away first; then count the remaining land groups.

#graphs #dfs #matrix #medium

The problem

In a grid where 0 is land and 1 is water, a closed island is a maximal group of land not touching the border. Count them.

📥 Example

A 4×4 grid with a 2×2 land block surrounded by water → 1

Brute force

Checking border-connectivity per island repeats work. Flooding the border once, then counting, is O(m·n).

Optimized approach

First flood-fill every land cell connected to the border, marking it water (it can't be closed). Then scan the interior; each remaining land cell starts a new closed island — flood-fill it and increment the count.

Time O(m·n), Space O(m·n).

C# solution

C# 
public int ClosedIsland(int[][] grid)
{
    int rows = grid.Length, cols = grid[0].Length;
    void Flood(int r, int c)
    {
        if (r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] != 0) return;
        grid[r][c] = 1;
        Flood(r + 1, c); Flood(r - 1, c); Flood(r, c + 1); Flood(r, c - 1);
    }
    for (int r = 0; r < rows; r++) { Flood(r, 0); Flood(r, cols - 1); }
    for (int c = 0; c < cols; c++) { Flood(0, c); Flood(rows - 1, c); }
    int count = 0;
    for (int r = 1; r < rows - 1; r++)
        for (int c = 1; c < cols - 1; c++)
            if (grid[r][c] == 0) { count++; Flood(r, c); }
    return count;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — each cell visited a constant number of times.
  • Space: O(m·n) — recursion worst case.

Pattern & takeaways

🎯 Remember these three

  • Border-connected land can't be closed — flood it away first.
  • Then each remaining land group is one closed island.
  • Same 'remove the border' trick as Surrounded Regions and Number of Enclaves.

Interview tip

🗣️ When this comes up

The two-phase approach — sink the border-connected land, then count the rest — is the reusable pattern. Note land is 0 and water is 1 here, the reverse of Number of Islands.
