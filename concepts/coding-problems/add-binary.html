Add Binary

Binary addition is grade-school addition with a twist: the carry triggers at 2 instead of 10.

#math #strings #bit-manipulation #easy

The problem

Given two binary strings a and b, return their sum as a binary string.

📥 Example

"1010" + "1011" → "10101"

Brute force

Converting to integers fails for long strings (overflow), and the point of the exercise is the bitwise simulation.

Optimized approach

Add from the right, digit plus digit plus carry. Each column writes sum % 2 and carries sum / 2 — identical to decimal addition with base 2.

Time O(n), Space O(n).

C# solution

C# 
public string AddBinary(string a, string b)
{
    int i = a.Length - 1, j = b.Length - 1, carry = 0;
    var sb = new StringBuilder();
    while (i >= 0 || j >= 0 || carry > 0)
    {
        int sum = carry;
        if (i >= 0) sum += a[i--] - '0';
        if (j >= 0) sum += b[j--] - '0';
        sb.Append((char)('0' + sum % 2));
        carry = sum / 2;
    }
    return new string(sb.ToString().Reverse().ToArray());
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass over the longer string.
  • Space: O(n) — the output buffer.

Pattern & takeaways

🎯 Remember these three

  • Same carry loop as decimal addition, with the carry boundary at 2.
  • Loop while a digit remains in either string or a carry is pending.
  • Bit operators (XOR for sum, AND-shift for carry) give an alternative carry-free phrasing.
