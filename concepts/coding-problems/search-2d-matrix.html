Search a 2D Matrix

Each row is sorted and every row’s first value exceeds the previous row’s last. That means the matrix is one big sorted array folded into a grid — so binary search it as if it were flat.

#binary-search #matrix #index-mapping #medium

The problem

Given an m × n matrix where each row is sorted left to right and the first integer of each row is greater than the last integer of the previous row, return true if target is present.

📥 Example

[[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 3 → true

The idea

Because of the row-join property, reading the matrix row by row yields a fully sorted sequence of m × n values. So binary search over indices 0 … m·n − 1, mapping a flat index mid back to a cell with row = mid / n, col = mid % n. No need to flatten anything — just translate the index when you read.

Time O(log(m·n)), Space O(1).

C# solution

C# 
public bool SearchMatrix(int[][] matrix, int target)
{
    int rows = matrix.Length, cols = matrix[0].Length;
    int lo = 0, hi = rows * cols - 1;

    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        int val = matrix[mid / cols][mid % cols];     // map flat index → cell
        if (val == target) return true;
        if (val < target) lo = mid + 1;
        else              hi = mid - 1;
    }
    return false;
}

Complexity

⏱️ Complexity

  • Time: O(log(m·n)) — a single binary search over all cells.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Row-join sortedness ⇒ the grid is one sorted array; binary search it directly.
  • Index mapping: row = i / cols, col = i % cols.
  • If rows aren’t joined (only each row/col sorted), use the O(m+n) staircase from the top-right corner.

Interview tip

🗣️ When this comes up

Spot the row-join condition — it’s what licenses the “flatten” view. The index-to-cell mapping (/cols, %cols) is the only twist. Clarify whether the matrix is fully joined or just row/column sorted, since that changes the algorithm.
