Design Add and Search Words

A trie dictionary where search can contain . wildcards. A dot forces a DFS that branches into every child.

#trie #dfs #design #medium #wildcard #interview

The problem

Design a structure with AddWord(word) and Search(word), where word in search may contain . matching any single letter.

📥 Example

add "bad","dad","mad"; search "pad" → false; search ".ad" → true; "b.." → true

The idea

Store words in a trie as usual. Search becomes a DFS over the trie: for a normal letter, follow that single child; for a ., try every existing child and succeed if any branch matches the rest. Reaching the end of the pattern succeeds only if the current node marks a word end. The wildcard is what turns a straight walk into a branching search.

Time O(L) typical, O(26L) worst (all dots); Space O(total chars).

C# solution

C# 
public class WordDictionary
{
    private class Node { public Node[] Next = new Node[26]; public bool End; }
    private readonly Node _root = new();

    public void AddWord(string word)
    {
        var node = _root;
        foreach (char c in word) node = node.Next[c - 'a'] ??= new Node();
        node.End = true;
    }

    public bool Search(string word) => Dfs(_root, 0, word);

    private bool Dfs(Node node, int i, string w)
    {
        if (node == null) return false;
        if (i == w.Length) return node.End;
        if (w[i] == '.')
        {
            foreach (var child in node.Next)          // try every child
                if (Dfs(child, i + 1, w)) return true;
            return false;
        }
        return Dfs(node.Next[w[i] - 'a'], i + 1, w);  // single letter
    }
}

Complexity

⏱️ Complexity

  • Time: O(L) without dots; up to O(26d) with d dots.
  • Space: O(total characters) plus O(L) recursion.

Pattern & takeaways

🎯 Remember these three

  • Store in a trie; the wildcard turns search into a branching DFS.
  • A . recurses into all children; a letter into one.
  • Match succeeds only at pattern end and a word-end node.

Interview tip

🗣️ When this comes up

Build on the plain trie, then explain the one change: a dot recurses into every child. Call out the worst case (all dots) so the interviewer knows you understand the branching cost.
