Missing Number

One number from 0..n is missing. XOR every index with every value and the pairs cancel — what survives is the gap. (Or use the sum formula.)

#bit-manipulation #math #arrays #easy #xor #interview

The problem

Given an array nums containing n distinct numbers from the range [0, n], return the one number missing from the range.

📥 Example

[3,0,1] → 2  ·  [0,1] → 2

The idea

XOR each array value and each index 0..n into one accumulator (seed it with n to cover the top index). Every number that is present appears as both an index and a value, so it cancels (a ^ a = 0); only the missing number is left unpaired. Alternatively, the expected sum n(n+1)/2 minus the actual sum is the missing value — same O(1) space, no overflow worries with the XOR version.

Time O(n), Space O(1).

C# solution

C# 
public int MissingNumber(int[] nums)
{
    int acc = nums.Length;                 // start with index n
    for (int i = 0; i < nums.Length; i++)
        acc ^= i ^ nums[i];                // cancel matching index/value pairs
    return acc;
}
// Sum variant:  n*(n+1)/2 - nums.Sum()

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • XOR pairs cancel — index/value duplicates vanish, the gap survives.
  • Seed with n to include the top index in the XOR.
  • The Gauss sum formula is an equally clean alternative.

Interview tip

🗣️ When this comes up

Offer both: the sum formula (intuitive) and the XOR (no overflow, very clean). Explaining the cancellation — every present number shows up as an index and a value — is the bit-manipulation insight they want.
