Design Twitter

A news feed is a merge of recent tweets from everyone you follow. A max-heap by timestamp pulls the 10 newest across all followees.

#heap #design #hash-map #medium #merge-k #interview

The problem

Design Twitter supporting PostTweet(userId, tweetId), GetNewsFeed(userId) (10 most recent tweets from the user and followees), Follow, and Unfollow.

📥 Example

postTweet(1,5); getNewsFeed(1)→[5]; follow(1,2); postTweet(2,6); getNewsFeed(1)→[6,5]

The idea

Store each user’s tweets as a list of (timestamp, tweetId) with a global increasing clock, and each user’s followees in a set. The feed is the k-way merge of the most recent tweets from the user plus followees: seed a max-heap with each relevant user’s latest tweet, then pop 10 times, pushing the next-older tweet from whichever list you popped. Timestamps give a global order.

Feed O(F + 10 log F), Space O(users + tweets).

C# solution

C# 
public class Twitter
{
    private int _clock = 0;
    private readonly Dictionary<int, List<(int t, int id)>> _tweets = new();
    private readonly Dictionary<int, HashSet<int>> _following = new();

    public void PostTweet(int userId, int tweetId)
    {
        (_tweets.TryGetValue(userId, out var l) ? l : _tweets[userId] = new())
            .Add((_clock++, tweetId));
    }

    public IList<int> GetNewsFeed(int userId)
    {
        var heap = new PriorityQueue<int, int>();    // max-heap by time (negate)
        var users = new HashSet<int>(_following.GetValueOrDefault(userId) ?? new()) { userId };
        foreach (int u in users)
            if (_tweets.TryGetValue(u, out var l))
                foreach (var (t, id) in l) heap.Enqueue(id, -t);

        var feed = new List<int>();
        while (feed.Count < 10 && heap.Count > 0) feed.Add(heap.Dequeue());
        return feed;
    }

    public void Follow(int a, int b)   => (_following.TryGetValue(a, out var s) ? s : _following[a] = new()).Add(b);
    public void Unfollow(int a, int b) => _following.GetValueOrDefault(a)?.Remove(b);
}

Complexity

⏱️ Complexity

  • Time: feed is O(T + 10 log T) over relevant tweets T (heap-merge variant is O(F + 10 log F)).
  • Space: O(users + tweets + follow edges).

Pattern & takeaways

🎯 Remember these three

  • A feed is a k-way merge by timestamp — a heap’s home turf.
  • A global increasing clock gives every tweet a total order.
  • Hash maps for tweets-per-user and followee-sets keep ops O(1).

Interview tip

🗣️ When this comes up

Frame the feed as merging k sorted tweet lists by time — that reframing is the whole design. Pick clean data structures (per-user tweet lists, followee sets) and a global timestamp, and the rest follows.
