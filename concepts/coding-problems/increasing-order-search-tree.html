Increasing Order Search Tree

In-order yields sorted values; relink each visited node as the right child of the previous, leaving a one-sided chain.

#trees #bst #dfs #easy

The problem

Rearrange a BST so it becomes a tree with no left children, in increasing order — each node's right child is the next larger value.

📥 Example

[5,3,6,2,4,null,8] → a right chain 2 → 3 → 4 → 5 → 6 → 8

Brute force

Collecting the in-order values and rebuilding works in O(n) extra space. An in-order pass with a moving pointer does it directly.

Optimized approach

Keep a current pointer starting at a dummy head. In-order traverse; for each node, set its left to null, attach it as current.right, and advance current. The dummy's right is the new root.

Time O(n), Space O(h).

C# solution

C# 
public TreeNode IncreasingBST(TreeNode root)
{
    var dummy = new TreeNode(0);
    var cur = dummy;
    void InOrder(TreeNode node)
    {
        if (node == null) return;
        InOrder(node.left);
        node.left = null;        // detach left
        cur.right = node;        // append to the chain
        cur = node;
        InOrder(node.right);
    }
    InOrder(root);
    return dummy.right;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) — recursion = height.

Pattern & takeaways

🎯 Remember these three

  • In-order traversal produces sorted order for a BST.
  • A moving current pointer threads nodes into a right chain.
  • Null out left pointers so the result is truly one-sided.

Interview tip

🗣️ When this comes up

It's in-order traversal plus pointer rewiring. Using a dummy head avoids special-casing the first node, and nulling left children is the detail people forget.
