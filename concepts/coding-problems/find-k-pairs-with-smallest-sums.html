Find K Pairs with Smallest Sums

Seed a min-heap with each a[i] paired to b[0]; after popping the smallest, push that same a paired with the next b — only promising candidates ever enter the heap.

#heap #two-pointers #medium

The problem

Given two sorted arrays and an integer k, return the k pairs (one element from each) with the smallest sums.

📥 Example

nums1=[1,7,11], nums2=[2,4,6], k=3 → [[1,2],[1,4],[1,6]]

Brute force

Generating all n·m pairs and sorting is O(n·m·log). A heap explores only the frontier for O(k log k).

Optimized approach

Push the initial pairs (a[i], b[0]) into a min-heap keyed by sum (just the first min(n,k) of them). Repeatedly pop the smallest pair (a[i], b[j]), record it, and push (a[i], b[j+1]) — the next candidate along that row. Stop after k pairs.

Time O(k log k), Space O(k).

C# solution

C# 
public IList<IList<int>> KSmallestPairs(int[] nums1, int[] nums2, int k)
{
    var res = new List<IList<int>>();
    if (nums1.Length == 0 || nums2.Length == 0) return res;
    // min-heap of (sum, i, j)
    var heap = new PriorityQueue<(int i, int j), int>();
    for (int i = 0; i < Math.Min(nums1.Length, k); i++)
        heap.Enqueue((i, 0), nums1[i] + nums2[0]);
    while (heap.Count > 0 && res.Count < k)
    {
        var (i, j) = heap.Dequeue();
        res.Add(new List<int> { nums1[i], nums2[j] });
        if (j + 1 < nums2.Length)
            heap.Enqueue((i, j + 1), nums1[i] + nums2[j + 1]);
    }
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(k log k) — at most k heap operations.
  • Space: O(k) — the heap.

Pattern & takeaways

🎯 Remember these three

  • Seed the heap with the row heads (a[i], b[0]).
  • Popping (i, j) pushes the next candidate (i, j+1).
  • Only the frontier of candidates ever enters the heap.

Interview tip

🗣️ When this comes up

It's the merge-k-style heap frontier: each pop spawns exactly one new candidate. Seeding with the first column and advancing the second index keeps the heap small.
