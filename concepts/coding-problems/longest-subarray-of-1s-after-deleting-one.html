Longest Subarray of 1's After Deleting One Element

You must delete one element, so the longest window containing at most one zero (minus the deleted slot) is the answer.

#sliding-window #arrays #medium

The problem

Given a binary array, you must delete exactly one element. Return the length of the longest subarray of 1s in the result.

📥 Example

[1,1,0,1] → 3 (delete the 0)

Brute force

Trying each deletion and measuring runs is O(n²). A sliding window with at most one zero is O(n).

Optimized approach

Slide a window allowing at most one zero; when a second zero appears, shrink from the left past the first zero. The longest such window has length (window − 1) after the mandatory deletion — track the max of (right − left).

Time O(n), Space O(1).

C# solution

C# 
public int LongestSubarray(int[] nums)
{
    int left = 0, zeros = 0, best = 0;
    for (int right = 0; right < nums.Length; right++)
    {
        if (nums[right] == 0) zeros++;
        while (zeros > 1) { if (nums[left] == 0) zeros--; left++; }
        best = Math.Max(best, right - left);   // minus the deleted element
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index enters/leaves once.
  • Space: O(1) — a few counters.

Pattern & takeaways

🎯 Remember these three

  • At most one zero in the window models the single deletion.
  • The answer is right − left (window length minus the deleted slot).
  • It's Max Consecutive Ones III with k = 1, minus one for the forced deletion.

Interview tip

🗣️ When this comes up

It's the flip-one-zero window, but since a deletion is mandatory you report right − left rather than the full window length. State that subtlety explicitly.
