Find All Numbers Disappeared in an Array

Which numbers from 1..n are missing? Mark presence by negating the value at each seen number’s index, then the still-positive slots reveal the gaps.

#arrays #hashing #easy #in-place #marking #interview

The problem

Given an array nums of n integers where each is in [1, n], return all integers in [1, n] that do not appear. Aim for O(n) time and O(1) extra space.

📥 Example

[4,3,2,7,8,2,3,1] → [5,6]

Brute force

A boolean array or HashSet of seen values then a scan is O(n) time but O(n) space. We can instead record presence in place by flipping signs.

Optimized approach

For each value v, treat index |v| − 1 as its bucket and negate the number there (if not already negative) to mark v as seen. Use the absolute value when reading, since earlier marks may have flipped signs. After the pass, any index still holding a positive value was never marked — so index + 1 is missing. Restore signs afterward if the input must stay intact.

Time O(n), Space O(1).

C# solution

C# 
public IList<int> FindDisappearedNumbers(int[] nums)
{
    foreach (int v in nums)
    {
        int idx = Math.Abs(v) - 1;
        if (nums[idx] > 0) nums[idx] = -nums[idx];   // mark v as seen
    }

    var result = new List<int>();
    for (int i = 0; i < nums.Length; i++)
        if (nums[i] > 0) result.Add(i + 1);          // never marked → missing
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — two passes.
  • Space: O(1) — sign marking in place (output excluded).

Pattern & takeaways

🎯 Remember these three

  • Encode “seen” by negating the value at the number’s index.
  • Always read with Math.Abs since marks flip signs.
  • Same index-as-hash idea as First Missing Positive and Find the Duplicate.

🗣️ When this comes up

The sign-marking trick gives O(1) space — the whole point of the question. Emphasize reading with absolute value. It’s the gentle cousin of First Missing Positive.
