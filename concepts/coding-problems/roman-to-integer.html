Roman to Integer

Turn a Roman numeral into a number. The one rule: a symbol smaller than the one to its right is subtracted; otherwise it’s added.

#strings #math #easy #parsing #lookup #interview

The problem

Given a Roman numeral string, convert it to an integer. Symbols: I=1, V=5, X=10, L=50, C=100, D=500, M=1000, with subtractive pairs like IV=4 and IX=9.

📥 Example

"MCMXCIV" → 1994  ·  "LVIII" → 58

Brute force

Special-casing each two-character combination (IV, IX, XL, …) works but is verbose. The general subtractive rule handles them all with one comparison.

Optimized approach

Map each symbol to its value. Scan left to right: if the current symbol’s value is less than the next symbol’s value, subtract it (it’s the small part of a subtractive pair like IV); otherwise add it. One pass, no special pair table.

Time O(n), Space O(1).

C# solution

C# 
public int RomanToInt(string s)
{
    var val = new Dictionary<char, int>
    { ['I']=1, ['V']=5, ['X']=10, ['L']=50, ['C']=100, ['D']=500, ['M']=1000 };

    int total = 0;
    for (int i = 0; i < s.Length; i++)
    {
        int cur = val[s[i]];
        if (i + 1 < s.Length && cur < val[s[i + 1]]) total -= cur;   // subtractive
        else total += cur;
    }
    return total;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — fixed symbol map.

Pattern & takeaways

🎯 Remember these three

  • One rule replaces the special-pair table: subtract if smaller than the right neighbor.
  • A simple symbol→value map drives the scan.
  • Integer-to-Roman is the reverse (greedy largest-symbol-first).
