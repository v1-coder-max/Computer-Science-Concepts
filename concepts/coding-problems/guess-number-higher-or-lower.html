Guess Number Higher or Lower

A guessing game where each wrong guess tells you 'higher' or 'lower'. That is binary search wearing a costume.

#binary-search #easy

The problem

Pick a number from 1..n. A guess(num) API returns 0 if correct, -1 if your guess is too high, and 1 if too low. Return the secret number in the fewest guesses.

📥 Example

n = 10, pick = 6 → returns 6

Brute force

Guessing 1, 2, 3, … is O(n). The higher/lower feedback is exactly the comparison binary search needs, so O(log n) is achievable.

Optimized approach

Maintain a range [lo, hi] and guess the midpoint. The oracle tells you which half to keep: too high → search left, too low → search right, correct → done.

Time O(log n), Space O(1).

C# solution

C# 
public int GuessNumber(int n)
{
    int lo = 1, hi = n;
    while (lo <= hi)
    {
        int mid = lo + (hi - lo) / 2;
        int r = guess(mid);
        if (r == 0) return mid;
        if (r < 0) hi = mid - 1;   // too high
        else lo = mid + 1;         // too low
    }
    return -1;
}

Complexity

⏱️ Complexity

  • Time: O(log n) — each guess halves the range.
  • Space: O(1) — two bounds.

Pattern & takeaways

🎯 Remember these three

  • The higher/lower oracle is the binary-search comparison.
  • Mind the sign convention: -1 means your guess was too high.
  • lo + (hi - lo)/2 keeps the midpoint overflow-safe.
