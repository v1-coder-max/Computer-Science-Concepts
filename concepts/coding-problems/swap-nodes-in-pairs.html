Swap Nodes in Pairs

Swap every two adjacent nodes — by relinking pointers, not values. A dummy head and careful pointer surgery handle it in one pass.

#linked-list #pointers #medium #dummy-head #relink #interview

The problem

Given a linked list, swap every two adjacent nodes and return its head. You must swap the nodes themselves (not just their values).

📥 Example

1→2→3→4 → 2→1→4→3  ·  1→2→3 → 2→1→3

Brute force

Swapping the values of each pair is trivial but violates the “swap nodes” constraint. Relinking pointers is the intended solution and only marginally harder.

Optimized approach

Use a dummy before the head. Keep a prev pointer; while two nodes remain, identify first = prev.next and second = first.next, then relink: prev.next = second, first.next = second.next, second.next = first. Advance prev to first (now the second of the swapped pair) and repeat. A leftover single node stays in place.

Time O(n), Space O(1).

C# solution

C# 
public ListNode SwapPairs(ListNode head)
{
    var dummy = new ListNode { next = head };
    ListNode prev = dummy;

    while (prev.next != null && prev.next.next != null)
    {
        ListNode first = prev.next, second = first.next;
        first.next = second.next;     // first points past second
        second.next = first;          // second points to first
        prev.next = second;           // prev points to second
        prev = first;                 // advance two nodes
    }
    return dummy.next;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node touched once.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Relink nodes, not values, with three pointer assignments.
  • A dummy head anchors the first swap uniformly.
  • It’s the k=2 case of Reverse Nodes in k-Group.

Interview tip

🗣️ When this comes up

Resist the value-swap shortcut — they want pointer relinking. The dummy head and the exact order of the three reassignments are what to get right; it’s Reverse-k-Group with k=2.
