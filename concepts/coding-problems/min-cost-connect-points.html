Min Cost to Connect All Points

Wire up all points for the least total distance. That’s a Minimum Spanning Tree — grow it with Prim’s, always adding the cheapest edge to a new point.

#graphs #mst #prim #medium #spanning-tree #interview

The problem

Given points on a plane, the cost to connect two is their Manhattan distance. Return the minimum total cost to connect all points (so any point reaches any other).

📥 Example

[[0,0],[2,2],[3,10],[5,2],[7,0]] → 20

Brute force

Trying all spanning trees is super-exponential. Even enumerating edge subsets is hopeless. MST algorithms (Prim/Kruskal) solve it greedily and provably optimally.

Optimized approach

Run Prim’s algorithm. Start from any point; keep a min-heap of (cost, point) edges crossing out of the visited set. Repeatedly pop the cheapest edge to an unvisited point, add its cost, mark it visited, and push its edges. Stop when all points are in. Because the graph is complete (every pair connectable), Prim’s with a heap is natural; Kruskal’s with union-find also works.

Time O(n² log n), Space O(n²).

C# solution

C# 
public int MinCostConnectPoints(int[][] points)
{
    int n = points.Length, total = 0, used = 0;
    var visited = new bool[n];
    var pq = new PriorityQueue<int, int>();     // (pointIndex, edgeCost)
    pq.Enqueue(0, 0);

    while (used < n)
    {
        pq.TryDequeue(out int i, out int cost);
        if (visited[i]) continue;               // already in the tree
        visited[i] = true; total += cost; used++;
        for (int j = 0; j < n; j++)
            if (!visited[j])
            {
                int d = Math.Abs(points[i][0] - points[j][0])
                      + Math.Abs(points[i][1] - points[j][1]);
                pq.Enqueue(j, d);
            }
    }
    return total;
}

Complexity

⏱️ Complexity

  • Time: O(n² log n) — complete graph, heap of edges.
  • Space: O(n²) edges in the heap (O(n) with key-based Prim).

Pattern & takeaways

🎯 Remember these three

  • “Connect everything cheaply” → Minimum Spanning Tree.
  • Prim’s grows from a seed via a min-heap of frontier edges.
  • Skip already-visited pops; Kruskal’s + union-find is the alternative.

Interview tip

🗣️ When this comes up

Name it as an MST and pick Prim’s (heap) or Kruskal’s (union-find). The complete-graph nature means Prim’s is a clean fit. The “skip if visited” guard is the small detail that keeps it correct.
