3Sum Closest

The triplet whose sum is closest to a target. Sort, fix one element, and two-pointer the rest toward the target.

#two-pointers #arrays #medium #sorting #closest #interview

The problem

Given an array nums and a target, find three integers whose sum is closest to target and return that sum (exactly one solution).

📥 Example

nums=[-1,2,1,-4], target=1 → 2 (−1 + 2 + 1)

Brute force

Three nested loops over all triples — O(n³). Sorting plus a two-pointer inner scan reduces it to O(n²), the same structure as 3Sum.

Optimized approach

Sort the array. Fix index i, then run two pointers lo, hi on the rest. Compute the triple sum; if it’s closer to the target than the best seen, update the answer. Then move toward the target: if the sum is less than target, lo++ (increase the sum); if more, hi−−. An exact match returns immediately.

Time O(n²), Space O(1).

C# solution

C# 
public int ThreeSumClosest(int[] nums, int target)
{
    Array.Sort(nums);
    int best = nums[0] + nums[1] + nums[2];

    for (int i = 0; i < nums.Length - 2; i++)
    {
        int lo = i + 1, hi = nums.Length - 1;
        while (lo < hi)
        {
            int sum = nums[i] + nums[lo] + nums[hi];
            if (Math.Abs(sum - target) < Math.Abs(best - target)) best = sum;
            if (sum == target) return sum;
            if (sum < target) lo++; else hi--;
        }
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n²) — fixed index × two-pointer scan.
  • Space: O(1) beyond the sort.

Pattern & takeaways

🎯 Remember these three

  • Sort + fix-one + two-pointer is the k-Sum backbone.
  • Track the closest sum by |sum − target|.
  • Move the pointer that pushes the sum toward the target.
