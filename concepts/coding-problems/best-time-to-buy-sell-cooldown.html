Best Time to Buy and Sell Stock with Cooldown

Unlimited trades, but a one-day cooldown after every sale. Model three states — holding, just sold, resting — and step them forward.

#dynamic-programming #state-machine #medium #stocks #cooldown #interview

The problem

Given daily prices, maximize profit with as many transactions as you like, but you cannot buy on the day right after you sell (one-day cooldown). You hold at most one share.

📥 Example

[1,2,3,0,2] → 3 (buy 1, sell 2, cooldown, buy 0, sell 2)

Brute force

Recurse over each day with “buy / sell / skip” branches and a cooldown flag — exponential without memoization. Memoizing on (day, holdingState) works; the state-machine DP below is the clean O(1)-space form.

Optimized approach

Track three running values: hold (max profit while owning a share), sold (max profit on the day you just sold), and rest (max profit not holding and free to buy). Transitions per day: hold = max(hold, rest − price) (keep or buy from rest), sold = hold + price (sell today), rest = max(rest, prevSold) (cooldown then rest). The answer is max(sold, rest) at the end.

Time O(n), Space O(1).

C# solution

C# 
public int MaxProfit(int[] prices)
{
    int hold = int.MinValue, sold = 0, rest = 0;
    foreach (int price in prices)
    {
        int prevSold = sold;
        sold = hold + price;                       // sell today
        hold = Math.Max(hold, rest - price);       // keep, or buy from rest
        rest = Math.Max(rest, prevSold);           // cooldown into rest
    }
    return Math.Max(sold, rest);
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — three rolling states.

Pattern & takeaways

🎯 Remember these three

  • Model trading constraints as a state machine.
  • The cooldown is captured by a separate “just sold” state feeding into rest.
  • Roll the states forward; no transaction count to track.
