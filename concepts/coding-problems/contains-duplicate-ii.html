Contains Duplicate II

Are there two equal values within k indices? Keep a sliding window of the last k values in a set and check membership.

#arrays #hashing #sliding-window #easy #set #interview

The problem

Given an array nums and an integer k, return true if there are two distinct indices i, j with nums[i] == nums[j] and |i − j| ≤ k.

📥 Example

nums=[1,2,3,1], k=3 → true  ·  nums=[1,2,3,1,2,3], k=2 → false

Brute force

Compare each index with the next k indices — O(n·k). A hash map of value→last index, or a bounded set window, drops it to O(n).

Optimized approach

Maintain a HashSet of the values in the current window of size k. As you scan index i: if nums[i] is already in the set, a duplicate within k exists → true. Otherwise add it, and once the window exceeds k, remove the value that fell out (nums[i − k]). The set never holds more than k elements.

Time O(n), Space O(k).

C# solution

C# 
public bool ContainsNearbyDuplicate(int[] nums, int k)
{
    var window = new HashSet<int>();
    for (int i = 0; i < nums.Length; i++)
    {
        if (window.Contains(nums[i])) return true;     // equal within k
        window.Add(nums[i]);
        if (window.Count > k) window.Remove(nums[i - k]);   // evict the oldest
    }
    return false;
}

Complexity

⏱️ Complexity

  • Time: O(n).
  • Space: O(min(n, k)).

Pattern & takeaways

🎯 Remember these three

  • Bound a hash set to the last k values for the distance check.
  • Evict nums[i−k] as the window advances.
  • A value→last-index map is the equivalent one-pass alternative.

Interview tip

The bounded-window set is the clean O(n) answer. Mention the value→index map alternative. It’s a quick screener for combining hashing with a window constraint.
