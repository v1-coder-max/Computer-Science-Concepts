Two Sum IV - Input is a BST

It's Two Sum on a tree: remember each value as you traverse and look for its complement.

#trees #bst #hash-set #easy

The problem

Given a BST and a target, return whether two distinct nodes sum to the target.

📥 Example

BST with values {5,3,6,2,4,7}, target 9 → true (2 + 7 or 5 + 4)

Brute force

Checking all pairs is O(n²). A hash set during traversal finds a complement in O(1).

Optimized approach

Traverse the tree (any order), keeping a set of seen values. At each node, if target − value is already in the set, you've found a pair; otherwise add the value and continue.

Time O(n), Space O(n).

C# solution

C# 
public bool FindTarget(TreeNode root, int k)
{
    var seen = new HashSet<int>();
    bool Dfs(TreeNode node)
    {
        if (node == null) return false;
        if (seen.Contains(k - node.val)) return true;
        seen.Add(node.val);
        return Dfs(node.left) || Dfs(node.right);
    }
    return Dfs(root);
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(n) — the seen set.

Pattern & takeaways

🎯 Remember these three

  • It's the classic Two Sum hash-set idea on a tree.
  • Any traversal order works since you store all seen values.
  • An O(1)-space alternative uses two BST iterators from both ends.
