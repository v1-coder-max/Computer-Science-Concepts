First Missing Positive

The smallest missing positive — in O(n) time and O(1) space. The trick: use the array itself as a hash table, placing each value v at index v−1.

#arrays #hashing #hard #in-place #cyclic-sort #interview

The problem

Given an unsorted array nums, return the smallest positive integer that does not appear. Must run in O(n) time and use O(1) extra space.

📥 Example

[1,2,0] → 3  ·  [3,4,-1,1] → 2  ·  [7,8,9,11,12] → 1

Brute force

A hash set of present values, then scan 1, 2, 3, … for the first absent — O(n) time but O(n) space. The constraints forbid extra space, so we encode presence into the array via index positions.

Optimized approach

The answer must be in [1, n+1]. Cyclic sort: for each position, while nums[i] is in range [1, n] and not already at its correct slot, swap it to index nums[i] − 1. After this, each in-range value v sits at index v−1. Scan: the first index i where nums[i] != i+1 means i+1 is missing; if all match, the answer is n+1.

Time O(n), Space O(1).

C# solution

C# 
public int FirstMissingPositive(int[] nums)
{
    int n = nums.Length;
    for (int i = 0; i < n; i++)
        while (nums[i] >= 1 && nums[i] <= n && nums[nums[i] - 1] != nums[i])
        {
            int j = nums[i] - 1;
            (nums[i], nums[j]) = (nums[j], nums[i]);   // place value at its slot
        }

    for (int i = 0; i < n; i++)
        if (nums[i] != i + 1) return i + 1;            // first missing
    return n + 1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each value moves to its slot at most once.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Answer is bounded by [1, n+1] — ignore out-of-range values.
  • Use the array as a hash via cyclic sort (value v → index v−1).
  • The swap loop is O(n) total despite the inner while.

Interview tip

🗣️ When this comes up

The O(1)-space constraint forces the index-as-hash trick — cyclic sort. Argue the linear time (each swap puts a value in its final slot). It’s a hard but very recognizable pattern.
