Remove Element

Strip every copy of a value from an array in place. A slow write pointer keeps only the elements you want to retain.

#arrays #two-pointers #easy #in-place #filter #interview

The problem

Given an array nums and a value val, remove all instances of val in place and return the new length k; the first k elements must be the retained ones (order can change).

📥 Example

nums=[3,2,2,3], val=3 → k=2, nums starts [2,2]

Brute force

Build a new array of the kept elements — O(n) extra space, and it ignores the in-place requirement. A single two-pointer pass does it with O(1) space.

Optimized approach

Keep a write index w. Scan with a read index; whenever nums[r] != val, copy it to nums[w] and advance w. At the end, w is the count of kept elements. (If removals are rare, swapping the matched element with the last and shrinking avoids extra writes.)

Time O(n), Space O(1).

C# solution

C# 
public int RemoveElement(int[] nums, int val)
{
    int w = 0;
    for (int r = 0; r < nums.Length; r++)
        if (nums[r] != val)
            nums[w++] = nums[r];     // keep this element
    return w;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Read/write two pointers filter in place in O(1) space.
  • The write index doubles as the final length.
  • Swap-with-last is an alternative when matches are few.

Interview tip

🗣️ When this comes up

It’s the simplest case of the in-place read/write pattern — the same one behind Move Zeroes and dedupe. Quick, but it screens for whether you reach for two pointers reflexively.
