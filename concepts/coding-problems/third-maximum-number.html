Third Maximum Number

Track the top three distinct values as you scan; a new value cascades into place. If there is no third distinct, fall back to the maximum.

#arrays #hash-set #easy

The problem

Given an integer array, return the third distinct maximum. If it doesn't exist, return the maximum.

📥 Example

[2,2,3,1] → 1; [1,2] → 2

Brute force

Sorting descending and walking for the third distinct value is O(n log n). A single pass with three running maxima is O(n).

Optimized approach

Keep three values m1 ≥ m2 ≥ m3. For each new number (skipping ones already among them), let it cascade: bigger than m1 shifts all down; bigger than m2 or m3 inserts accordingly. If m3 was never set, return m1.

Time O(n), Space O(1).

C# solution

C# 
public int ThirdMax(int[] nums)
{
    long m1 = long.MinValue, m2 = long.MinValue, m3 = long.MinValue;
    foreach (int x in nums)
    {
        if (x == m1 || x == m2 || x == m3) continue;
        if (x > m1) { m3 = m2; m2 = m1; m1 = x; }
        else if (x > m2) { m3 = m2; m2 = x; }
        else if (x > m3) { m3 = x; }
    }
    return m3 == long.MinValue ? (int)m1 : (int)m3;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — three running maxima.

Pattern & takeaways

🎯 Remember these three

  • Maintain the top-three distinct values, skipping repeats.
  • A new value cascades through m1, m2, m3.
  • Use a sentinel (or long.MinValue) to detect a missing third.

Interview tip

🗣️ When this comes up

Two gotchas to call out: distinctness (skip duplicates) and the fallback to the max when fewer than three distinct values exist. Sentinels keep the comparisons clean.
