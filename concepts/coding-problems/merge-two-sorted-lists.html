Merge Two Sorted Lists

Weave two sorted linked lists into one sorted list. A dummy head plus a tail pointer turns the edge cases into a clean splice — the merge step of merge sort.

#linked-list #two-pointers #merge #easy

The problem

Given the heads of two sorted linked lists l1 and l2, splice them into one sorted list by reusing the existing nodes, and return its head.

📥 Example

1 → 2 → 4  +  1 → 3 → 4 → 1 → 1 → 2 → 3 → 4 → 4

Brute force

Dump all values into an array, sort it, and rebuild a list — O((n+m) log(n+m)) and O(n+m) space. That throws away the fact that both inputs are already sorted. We can merge in linear time.

Optimized approach

Keep a dummy node so you never special-case the empty result, and a tail that always points to the last node of the merged list. Compare the heads of l1 and l2; attach the smaller one to tail, advance that list and tail. When one list runs out, attach whatever remains of the other (it’s already sorted). Return dummy.next.

Time O(n + m), Space O(1).

C# solution

C# 
public ListNode MergeTwoLists(ListNode l1, ListNode l2)
{
    var dummy = new ListNode();   // sentinel so we never special-case the head
    var tail  = dummy;

    while (l1 != null && l2 != null)
    {
        if (l1.val <= l2.val) { tail.next = l1; l1 = l1.next; }
        else                  { tail.next = l2; l2 = l2.next; }
        tail = tail.next;
    }
    tail.next = l1 ?? l2;         // attach the remaining (already sorted) tail
    return dummy.next;
}

Complexity

⏱️ Complexity

  • Time: O(n + m) — each node is visited and linked once.
  • Space: O(1) — splices existing nodes, no new allocations.

Pattern & takeaways

🎯 Remember these three

  • A dummy head removes the “is the result empty?” special case — use it for any list-building problem.
  • The two-pointer merge is the merge step of merge sort; learn it once, reuse everywhere.
  • After one list empties, the other is already sorted — attach it in one move with l1 ?? l2.

Interview tip

🗣️ When this comes up

Lead with the dummy + tail idiom — interviewers love it because it kills the edge cases. The natural follow-up is “merge k sorted lists,” which you solve by pairing this merge with a min-heap or divide-and-conquer.
