Reverse Nodes in k-Group

Reverse the list in chunks of k, leaving any leftover tail under k as-is. Reverse each full group in place and stitch the segments together.

#linked-list #pointers #hard #reverse #in-place #interview

The problem

Given a linked list, reverse the nodes k at a time and return the modified list. If the remaining nodes are fewer than k, leave them as they are.

📥 Example

1→2→3→4→5, k=2 → 2→1→4→3→5

Brute force

Copy values into an array, reverse each k-block, and rebuild — O(n) time but O(n) space and it doesn’t truly relink nodes. The in-place version reverses each group with pointers and connects them.

Optimized approach

Work group by group. First check that k nodes remain (peek ahead); if not, stop — the tail stays untouched. Reverse the next k nodes in place (the standard three-pointer reversal bounded to k steps), then connect the previous group’s tail to the new head and remember the new tail as the connection point for the next group. A dummy head simplifies the first stitch.

Time O(n), Space O(1).

C# solution

C# 
public ListNode ReverseKGroup(ListNode head, int k)
{
    var dummy = new ListNode { next = head };
    ListNode groupPrev = dummy;

    while (true)
    {
        // find the k-th node from groupPrev
        ListNode kth = groupPrev;
        for (int i = 0; i < k && kth != null; i++) kth = kth.next;
        if (kth == null) break;                       // fewer than k left

        ListNode groupNext = kth.next, prev = groupNext, cur = groupPrev.next;
        while (cur != groupNext)                      // reverse this group
        {
            var nxt = cur.next; cur.next = prev; prev = cur; cur = nxt;
        }
        ListNode newTail = groupPrev.next;            // old head = new tail
        groupPrev.next = kth;                         // stitch in reversed head
        groupPrev = newTail;
    }
    return dummy.next;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node reversed once.
  • Space: O(1) — in-place pointer surgery.

Pattern & takeaways

🎯 Remember these three

  • Check k nodes exist before reversing a group; leave the short tail.
  • A dummy head makes the first stitch uniform.
  • Track groupPrev and the new tail to chain segments.

Interview tip

🗣️ When this comes up

It’s the hard linked-list pointer problem — go slow and name the invariants (groupPrev, kth, groupNext). The peek-ahead for k nodes and the segment stitching are where bugs live; dry-run a small example aloud.
