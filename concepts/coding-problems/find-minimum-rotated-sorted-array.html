Find Minimum in Rotated Sorted Array

A sorted array was rotated; find its smallest value in O(log n). The minimum is the one inflection point where order breaks — binary search straight to it.

#binary-search #arrays #rotation #medium

The problem

An ascending array of distinct values was rotated at some pivot. Return its minimum element in logarithmic time.

📥 Example

[3,4,5,1,2] → 1  ·  [4,5,6,7,0,1,2] → 0

The idea

The minimum sits at the single place where a bigger number is immediately followed by a smaller one — the rotation point. Compare mid to the right end: if nums[mid] > nums[hi], the dip is to the right of mid, so move lo = mid + 1. Otherwise the minimum is at mid or to its left, so move hi = mid. The window converges on the minimum.

Time O(log n), Space O(1).

C# solution

C# 
public int FindMin(int[] nums)
{
    int lo = 0, hi = nums.Length - 1;
    while (lo < hi)
    {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] > nums[hi]) lo = mid + 1;   // min is to the right of mid
        else                      hi = mid;       // min is mid or to its left
    }
    return nums[lo];     // lo == hi → the minimum
}

Complexity

⏱️ Complexity

  • Time: O(log n) — halving toward the inflection point.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Compare mid to the right end (not the left) to locate the dip cleanly.
  • Use hi = mid (not mid − 1) so you never skip the minimum.
  • Loop with lo < hi; convergence (lo == hi) lands on the answer.

Interview tip

🗣️ When this comes up

Comparing to the right end is the detail that keeps this bug-free (comparing to the left needs extra cases). State why hi = mid rather than mid − 1. It’s the companion to search-in-rotated, often asked back to back.
