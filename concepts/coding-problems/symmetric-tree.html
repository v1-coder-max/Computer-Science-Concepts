Symmetric Tree

Is the tree a mirror image of itself? Compare the left and right subtrees as mirrors: outer-vs-outer, inner-vs-inner.

#trees #recursion #dfs #easy #mirror #interview

The problem

Given the root of a binary tree, return true if it is symmetric around its center (a mirror of itself).

📥 Example

[1,2,2,3,4,4,3] → true  ·  [1,2,2,null,3,null,3] → false

The idea

Symmetry is a property of two subtrees being mirrors. Define Mirror(a, b): they’re mirrors when their values match and a.left mirrors b.right (outer pair) and a.right mirrors b.left (inner pair). Call Mirror(root.left, root.right). It’s the Same-Tree recursion with the children crossed.

Time O(n), Space O(h).

C# solution

C# 
public bool IsSymmetric(TreeNode root)
{
    return root == null || Mirror(root.left, root.right);

    bool Mirror(TreeNode a, TreeNode b)
    {
        if (a == null && b == null) return true;
        if (a == null || b == null || a.val != b.val) return false;
        return Mirror(a.left, b.right)      // outer pair
            && Mirror(a.right, b.left);     // inner pair
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Frame symmetry as two subtrees being mirrors, not one tree.
  • Cross the children: left↔right, right↔left.
  • It’s Same Tree with the recursion crossed — an iterative queue works too.

Interview tip

🗣️ When this comes up

The trick is realizing you compare two subtrees with crossed children. Contrast with Same Tree (straight pairs) to show you see the difference. An iterative version uses a queue of pairs.
