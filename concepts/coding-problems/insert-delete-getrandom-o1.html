Insert Delete GetRandom O(1)

Insert, remove, and pick a random element — all O(1). The combo: an array for random access plus a value→index map, with a swap-with-last delete.

#design #hash-map #arrays #medium #o1 #interview

The problem

Design a structure supporting Insert(val), Remove(val), and GetRandom() (uniform over current elements), each in average O(1).

📥 Example

insert(1); insert(2); getRandom()→1 or 2; remove(1); getRandom()→2

Brute force

A hash set gives O(1) insert/remove but no O(1) uniform random pick (you’d iterate). A plain array gives O(1) random pick but O(n) remove (shifting). You need both properties at once.

Optimized approach

Keep a dynamic list of values (for O(1) random indexing) and a map from value → its index in the list. Insert: append to the list, record its index. Remove: look up the index, swap that element with the last, update the moved element’s index in the map, then pop the last (O(1), no shifting). GetRandom: index the list with a random position.

Time O(1) average per op, Space O(n).

C# solution

C# 
public class RandomizedSet
{
    private readonly List<int> _vals = new();
    private readonly Dictionary<int, int> _idx = new();   // value → index
    private readonly Random _rng = new();

    public bool Insert(int val)
    {
        if (_idx.ContainsKey(val)) return false;
        _idx[val] = _vals.Count;
        _vals.Add(val);
        return true;
    }

    public bool Remove(int val)
    {
        if (!_idx.TryGetValue(val, out int i)) return false;
        int last = _vals[^1];
        _vals[i] = last; _idx[last] = i;     // move last into the gap
        _vals.RemoveAt(_vals.Count - 1);
        _idx.Remove(val);
        return true;
    }

    public int GetRandom() => _vals[_rng.Next(_vals.Count)];
}

Complexity

⏱️ Complexity

  • Time: O(1) average for all three operations.
  • Space: O(n) — the list and index map.

Pattern & takeaways

🎯 Remember these three

  • Pair an array (random access) with a value→index map.
  • Delete via swap-with-last + pop to avoid O(n) shifting.
  • Update the moved element’s index in the map after the swap.
