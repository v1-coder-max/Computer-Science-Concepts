Sliding Window Maximum

The max of every size-k window. A monotonic decreasing deque of indices keeps the current window’s maximum always at its front.

#sliding-window #deque #monotonic #hard #arrays #interview

The problem

Given an array nums and window size k, return an array of the maximum value in each contiguous window as it slides one step at a time.

📥 Example

nums=[1,3,-1,-3,5,3,6,7], k=3 → [3,3,5,5,6,7]

Brute force

Scan each window for its max — O(n·k). A max-heap improves it to O(n log k) but needs lazy deletion. A monotonic deque achieves true O(n).

Optimized approach

Maintain a deque of indices whose values are decreasing front-to-back. Before pushing index i, pop from the back every index with a value ≤ nums[i] (they can never be the max while i is in the window). Pop from the front any index that has slid out of the window (≤ i − k). The front index always holds the current window’s maximum. Each index enters and leaves the deque once.

Time O(n), Space O(k).

C# solution

C# 
public int[] MaxSlidingWindow(int[] nums, int k)
{
    var deque = new LinkedList<int>();       // indices, values decreasing
    var result = new int[nums.Length - k + 1];

    for (int i = 0; i < nums.Length; i++)
    {
        while (deque.Count > 0 && nums[deque.Last.Value] <= nums[i])
            deque.RemoveLast();              // drop smaller trailing values
        deque.AddLast(i);

        if (deque.First.Value <= i - k)      // front slid out of the window
            deque.RemoveFirst();

        if (i >= k - 1)
            result[i - k + 1] = nums[deque.First.Value];   // front = max
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index added/removed once.
  • Space: O(k) — the deque.

Pattern & takeaways

🎯 Remember these three

  • Window max/min → a monotonic deque of indices.
  • Pop the back when a new value dominates; pop the front when it expires.
  • It beats the O(n log k) heap by avoiding stale-entry cleanup.

Interview tip

🗣️ When this comes up

The monotonic deque is the O(n) answer — explain why trailing smaller values are useless and why the front is always the max. Mention the heap alternative and why the deque is strictly better here.
