Fizz Buzz

The only subtlety is ordering the checks: test divisibility by both 3 and 5 (i.e. 15) before testing each alone.

#math #simulation #easy

The problem

For each integer 1..n, output "Fizz" if divisible by 3, "Buzz" if by 5, "FizzBuzz" if by both, otherwise the number itself.

📥 Example

n = 5 → ["1","2","Fizz","4","Buzz"]

Brute force

There is no brute force — the challenge is clean control flow. Checking 15 first (or concatenating partial labels) avoids the common ordering bug.

Optimized approach

Loop 1..n. Build the label by appending "Fizz" for multiples of 3 and "Buzz" for multiples of 5; if neither applies, use the number. Concatenation neatly handles the both-divisible case without a separate branch.

Time O(n), Space O(1).

C# solution

C# 
public IList<string> FizzBuzz(int n)
{
    var res = new List<string>(n);
    for (int i = 1; i <= n; i++)
    {
        string s = "";
        if (i % 3 == 0) s += "Fizz";
        if (i % 5 == 0) s += "Buzz";
        res.Add(s.Length == 0 ? i.ToString() : s);
    }
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass (output aside).
  • Space: O(1) — constant.

Pattern & takeaways

🎯 Remember these three

  • Concatenating "Fizz"+"Buzz" handles the divisible-by-both case for free.
  • Or test divisibility by 15 first if using explicit branches.
  • It is really a test of clean, bug-free conditionals.
