Contains Duplicate

Does any value appear at least twice? A one-line idea — a set — that shows up inside dozens of harder problems.

#arrays #hashing #easy #set

The problem

Given an integer array nums, return true if any value appears at least twice, and false if every element is distinct.

📥 Example

[1, 2, 3, 1] → true  ·  [1, 2, 3, 4] → false

Brute force

Compare every pair with nested loops. Time O(n²), Space O(1). Too slow for large inputs.

Other approaches

Sort first, then check adjacent elements — duplicates become neighbors. Time O(n log n), Space O(1) (if sorting in place), but it mutates the input.

Optimized approach

Use a HashSet: walk the array and try to add each value. If Add returns false, the value was already there — a duplicate. One pass, O(1) membership checks.

Time O(n), Space O(n).

C# solution

C# 
public bool ContainsDuplicate(int[] nums)
{
    var seen = new HashSet<int>();
    foreach (int n in nums)
        if (!seen.Add(n))      // Add returns false if n was already in the set
            return true;       // duplicate
    return false;
}
// One-liner equivalent: return nums.Length != nums.Distinct().Count();

Complexity

⏱️ Complexity

  • Time: O(n) — one pass, O(1) set operations.
  • Space: O(n) — the set may hold every element.

Pattern & takeaways

🎯 Remember these three

  • A HashSet is the default tool for “seen before?” / uniqueness questions.
  • Add returning false is a clean, allocation-free duplicate check.
  • Sorting is an alternative (O(n log n), no extra space) when you can mutate the input.
