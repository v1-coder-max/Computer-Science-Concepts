Maximum Length of Pair Chain

It's activity selection: sort by end value and greedily extend the chain with the next pair that starts after the current end.

#greedy #intervals #sorting #medium

The problem

A pair [a, b] can follow [c, d] if b < c. Given a set of pairs, return the length of the longest chain you can form.

📥 Example

[[1,2],[2,3],[3,4]] → 2 ([1,2] → [3,4])

Brute force

A longest-increasing-subsequence style DP is O(n²). Sorting by end value and greedily selecting is O(n log n) and optimal.

Optimized approach

Sort pairs by their second value. Greedily build the chain: keep a current end; take the next pair whose start exceeds it, updating the end. This earliest-ending choice maximizes room for future pairs — classic interval scheduling.

Time O(n log n), Space O(1).

C# solution

C# 
public int FindLongestChain(int[][] pairs)
{
    Array.Sort(pairs, (x, y) => x[1] - y[1]);
    int count = 0, curEnd = int.MinValue;
    foreach (var p in pairs)
    {
        if (p[0] > curEnd) { count++; curEnd = p[1]; }  // chain it
    }
    return count;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — dominated by the sort.
  • Space: O(1) — in-place sort.

Pattern & takeaways

🎯 Remember these three

  • Sort by end value — earliest finish leaves the most room.
  • It's the interval-scheduling / activity-selection greedy.
  • Equivalent to non-overlapping intervals: the chain length is what you keep.
