Rotting Oranges

Rot spreads from every rotten orange to its neighbors each minute. “How long until all rot?” is a multi-source BFS: start from all rotten cells at once and count levels.

#graphs #bfs #grid #medium

The problem

In a grid, each cell is 0 (empty), 1 (fresh orange), or 2 (rotten). Every minute, a fresh orange adjacent (4-directionally) to a rotten one becomes rotten. Return the minutes until no fresh orange remains, or -1 if some can never rot.

📥 Example

[[2,1,1],[1,1,0],[0,1,1]] → 4  ·  [[2,1,1],[0,1,1],[1,0,1]] → -1

The idea

Rot expands like ripples — one ring of neighbors per minute. That’s exactly a level-by-level BFS, but with many starting points. Seed the queue with all rotten cells and count the fresh ones. Then process the queue one minute (one level) at a time, rotting fresh neighbors and enqueuing them, incrementing the timer per level. At the end, if any fresh orange is left unreached, return -1.

Time O(rows × cols), Space O(rows × cols).

C# solution

C# — multi-source BFS 
public int OrangesRotting(int[][] grid)
{
    int rows = grid.Length, cols = grid[0].Length, fresh = 0;
    var q = new Queue<(int r, int c)>();

    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            if (grid[r][c] == 2) q.Enqueue((r, c));   // all sources
            else if (grid[r][c] == 1) fresh++;

    int minutes = 0;
    int[][] dirs = { new[]{1,0}, new[]{-1,0}, new[]{0,1}, new[]{0,-1} };

    while (q.Count > 0 && fresh > 0)
    {
        minutes++;
        for (int i = q.Count; i > 0; i--)             // exactly one minute's worth
        {
            var (r, c) = q.Dequeue();
            foreach (var d in dirs)
            {
                int nr = r + d[0], nc = c + d[1];
                if (nr >= 0 && nc >= 0 && nr < rows && nc < cols && grid[nr][nc] == 1)
                {
                    grid[nr][nc] = 2; fresh--;
                    q.Enqueue((nr, nc));
                }
            }
        }
    }
    return fresh == 0 ? minutes : -1;
}

Complexity

⏱️ Complexity

  • Time: O(rows × cols) — each cell enqueued/processed once.
  • Space: O(rows × cols) for the queue.

Pattern & takeaways

🎯 Remember these three

  • “Spread / shortest time over a grid” → multi-source BFS (seed all sources first).
  • Process one level per tick (freeze the queue size) so each layer is one minute.
  • Track fresh remaining to detect the unreachable → -1 case.

Interview tip

🗣️ When this comes up

The signal phrase is “minimum time for something to spread.” Say multi-source BFS and emphasize seeding every source before the loop — that’s what makes the level count equal elapsed minutes. Don’t forget the leftover-fresh → -1 check.
