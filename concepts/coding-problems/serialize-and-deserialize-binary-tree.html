Serialize and Deserialize Binary Tree

Flatten a tree to a string and back. Preorder DFS with explicit null markers records enough structure to reconstruct it exactly.

#trees #dfs #design #hard #serialization #interview

The problem

Design Serialize (tree → string) and Deserialize (string → tree) so that deserializing a serialized tree yields the original tree.

📥 Example

[1,2,3,null,null,4,5] → "1,2,#,#,3,4,#,#,5,#,#" → back to the same tree

The idea

A traversal alone is ambiguous, but a traversal with null markers is not. Serialize with preorder DFS, emitting each value and a sentinel (#) for every null child. To deserialize, read tokens in the same order: a value becomes a node whose left then right are built recursively from the stream; a # returns null. Because preorder writes the root before its children, the rebuild consumes tokens in exactly the right order.

Time O(n), Space O(n).

C# solution

C# 
public string serialize(TreeNode root)
{
    var sb = new System.Text.StringBuilder();
    void Dfs(TreeNode n)
    {
        if (n == null) { sb.Append("#,"); return; }
        sb.Append(n.val).Append(',');
        Dfs(n.left); Dfs(n.right);
    }
    Dfs(root);
    return sb.ToString();
}

public TreeNode deserialize(string data)
{
    var tokens = new Queue<string>(data.Split(',', StringSplitOptions.RemoveEmptyEntries));
    TreeNode Build()
    {
        string t = tokens.Dequeue();
        if (t == "#") return null;
        return new TreeNode { val = int.Parse(t), left = Build(), right = Build() };
    }
    return Build();
}

Complexity

⏱️ Complexity

  • Time: O(n) to serialize and O(n) to deserialize.
  • Space: O(n) for the string and recursion.

Pattern & takeaways

🎯 Remember these three

  • Null markers are what make a single traversal reconstructable.
  • Preorder writes the root first, so the rebuild consumes tokens in order.
  • A queue/stream of tokens keeps the deserialize recursion clean.

Interview tip

🗣️ When this comes up

The insight is that a traversal needs null markers to be reversible. Preorder is easiest for the rebuild. BFS with markers is an equally valid encoding — mention it as an alternative.
