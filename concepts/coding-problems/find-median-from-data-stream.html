Find Median from Data Stream

Track the median as numbers stream in. Split the data into a low half (max-heap) and high half (min-heap); the median sits at their tops.

#heap #design #two-heaps #hard #streaming #interview

The problem

Design a structure with AddNum(num) and FindMedian() returning the median of all values added so far.

📥 Example

add 1; add 2; median→1.5; add 3; median→2

The idea

Keep two heaps: a max-heap lo for the smaller half and a min-heap hi for the larger half. Maintain the invariant that every element in lo ≤ every element in hi, and their sizes differ by at most 1. Then the median is the top of the larger heap (odd count) or the average of both tops (even count). Each add does a couple of heap pushes/pops to rebalance — O(log n).

Add O(log n), FindMedian O(1).

C# solution

C# 
public class MedianFinder
{
    private readonly PriorityQueue<int, int> _lo = new();  // max-heap (negated)
    private readonly PriorityQueue<int, int> _hi = new();  // min-heap

    public void AddNum(int num)
    {
        _lo.Enqueue(num, -num);                 // push to low half
        _hi.Enqueue(_lo.Dequeue(), 0);          // move its max to high half
        // rebalance: hi must not be larger than lo
        if (_hi.Count > _lo.Count)
        {
            int x = _hi.Dequeue();
            _lo.Enqueue(x, -x);
        }
    }

    public double FindMedian() =>
        _lo.Count > _hi.Count ? _lo.Peek() : (_lo.Peek() + _hi.Peek()) / 2.0;
}

Complexity

⏱️ Complexity

  • Time: O(log n) per add, O(1) per median query.
  • Space: O(n) across the two heaps.

Pattern & takeaways

🎯 Remember these three

  • Two heaps: max-heap low half, min-heap high half.
  • Keep sizes balanced (≤ 1 apart) so the median is at the top(s).
  • Push-then-shift across heaps keeps the ordering invariant automatically.

Interview tip

🗣️ When this comes up

“Running median” is the canonical two-heaps question. Nail the invariant (low-half max-heap ≤ high-half min-heap, sizes within 1) and the balance step, and the median read is trivial.
