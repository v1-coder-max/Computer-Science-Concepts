Populating Next Right Pointers in Each Node

Wire each node’s next to its right neighbor on the same level. Using the level above’s already-linked next pointers, do it with O(1) space.

#trees #bfs #medium #linked-level #o1-space #interview

The problem

Given a perfect binary tree, set each node’s next pointer to the node immediately to its right on the same level (or null for the rightmost). The tree already has left, right, and next fields.

📥 Example

[1,2,3,4,5,6,7] → each node points right; rightmost on each level → null.

Brute force

A level-order BFS with a queue connects each level in O(n) time but O(width) = O(n) space. The next pointers let you traverse each level like a linked list, removing the queue.

Optimized approach

Process level by level using the current level’s next chain as a linked list to reach all nodes (no queue). For each node on the current level, link its children: node.left.next = node.right, and node.right.next = node.next?.left (the next node’s left child). A leftmost pointer descends to the start of each level. Because the tree is perfect, every node has two children until the leaves.

Time O(n), Space O(1).

C# solution

C# 
public Node Connect(Node root)
{
    Node leftmost = root;
    while (leftmost != null && leftmost.left != null)
    {
        Node node = leftmost;
        while (node != null)
        {
            node.left.next = node.right;                       // siblings
            if (node.next != null) node.right.next = node.next.left;  // cross-parent
            node = node.next;                                  // walk the level
        }
        leftmost = leftmost.left;                              // drop a level
    }
    return root;
}
// class Node { public int val; public Node left, right, next; }

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(1) — uses the next pointers, no queue.

Pattern & takeaways

🎯 Remember these three

  • Reuse the parent level’s next chain to avoid a queue.
  • Link left.next = right and right.next = next.left.
  • Works in O(1) space because the tree is perfect.

Interview tip

🗣️ When this comes up

The O(1)-space trick — using the next pointers you’ve already set to traverse the next level — is the whole point. For a non-perfect tree (version II), you track the previous node and a dummy per level.
