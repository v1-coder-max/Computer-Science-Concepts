Minimum Window Substring

Find the shortest slice of s that contains every character of t (with multiplicity). A grow-then-shrink window with a “how many needs are satisfied” counter.

#sliding-window #strings #hashing #hard

The problem

Given strings s and t, return the minimum-length substring of s that contains all characters of t (including duplicates), or "" if none exists.

📥 Example

s = "ADOBECODEBANC", t = "ABC" → "BANC"

The idea

Use a variable-size window. Expand right until the window contains all of t; once it’s valid, contract from left as far as possible while it stays valid, recording the smallest valid window. To know validity in O(1), keep a need map of t’s counts and a have counter of how many distinct requirements are currently met; the window is valid when have == required.

Time O(|s| + |t|), Space O(|t|).

C# solution

C# — grow/shrink window 
public string MinWindow(string s, string t)
{
    if (t.Length == 0 || s.Length < t.Length) return "";

    var need = new Dictionary<char, int>();
    foreach (char c in t) need[c] = need.GetValueOrDefault(c) + 1;

    int required = need.Count, have = 0, left = 0;
    int bestLen = int.MaxValue, bestStart = 0;
    var window = new Dictionary<char, int>();

    for (int right = 0; right < s.Length; right++)
    {
        char c = s[right];
        window[c] = window.GetValueOrDefault(c) + 1;
        if (need.ContainsKey(c) && window[c] == need[c]) have++;

        while (have == required)                  // valid → try to shrink
        {
            if (right - left + 1 < bestLen) { bestLen = right - left + 1; bestStart = left; }
            char l = s[left++];
            window[l]--;
            if (need.ContainsKey(l) && window[l] < need[l]) have--;
        }
    }
    return bestLen == int.MaxValue ? "" : s.Substring(bestStart, bestLen);
}

Complexity

⏱️ Complexity

  • Time: O(|s| + |t|) — each character enters and leaves the window once.
  • Space: O(|t|) — the need/window maps over distinct chars.

Pattern & takeaways

🎯 Remember these three

  • The canonical variable window: expand to satisfy, contract to minimize.
  • A have == required counter gives O(1) validity instead of re-scanning counts.
  • Only decrement have when a count drops below its need (not merely changes).

Interview tip

🗣️ When this comes up

This is the hardest standard sliding-window question — narrate the two phases (expand to valid, shrink to minimal) and the have/required bookkeeping. The classic bug is decrementing have on any change instead of only when a count falls below its requirement.
