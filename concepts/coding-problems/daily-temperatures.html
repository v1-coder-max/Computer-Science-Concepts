Daily Temperatures

For each day, how many days until it gets warmer? A monotonic stack of waiting days resolves each one the moment a warmer temperature shows up.

#stack #monotonic-stack #arrays #medium

The problem

Given daily temperatures, return an array where answer[i] is the number of days you must wait after day i for a warmer temperature, or 0 if none ever comes.

📥 Example

[73,74,75,71,69,72,76,73] → [1,1,4,2,1,1,0,0]

Brute force

For each day, scan forward for the first warmer day — O(n²). Many comparisons repeat. A monotonic stack lets each day be resolved exactly once.

Optimized approach

Keep a stack of indices of days still waiting for a warmer day, with temperatures decreasing down the stack. For each new day i: while the day on top of the stack is colder than today, today is its answer — pop it and set answer[popped] = i − popped. Then push i. Every index is pushed and popped once.

Time O(n), Space O(n).

C# solution

C# — monotonic stack 
public int[] DailyTemperatures(int[] temps)
{
    var answer = new int[temps.Length];
    var stack = new Stack<int>();    // indices of days awaiting a warmer day

    for (int i = 0; i < temps.Length; i++)
    {
        // today resolves every colder day still waiting on the stack
        while (stack.Count > 0 && temps[i] > temps[stack.Peek()])
        {
            int prev = stack.Pop();
            answer[prev] = i - prev;
        }
        stack.Push(i);
    }
    return answer;                  // days left on the stack keep their 0
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index pushed and popped at most once.
  • Space: O(n) — the stack in the worst (strictly decreasing) case.

Pattern & takeaways

🎯 Remember these three

  • “Next greater / warmer element” → monotonic stack of indices.
  • Store indices, not values, so you can compute the distance.
  • Unresolved days stay on the stack and keep their default 0.
