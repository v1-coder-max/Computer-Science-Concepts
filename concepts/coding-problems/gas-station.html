Gas Station

Drive a circular route of gas stations. If total gas covers total cost, the start is right after the point where your running tank dips negative.

#greedy #arrays #circular #medium #prefix #interview

The problem

Given gas[i] and cost[i] (gas to travel from station i to i+1), return the starting index to complete the circuit once, or -1 if impossible. The answer is unique if it exists.

📥 Example

gas = [1,2,3,4,5], cost = [3,4,5,1,2] → 3

The idea

Two facts. First, a solution exists iff sum(gas) >= sum(cost). Second, if your running tank goes negative at station i, then no station in [start..i] can be the start (each would also fail by i), so the next viable start is i + 1. Sweep once, accumulating tank += gas[i] − cost[i]; whenever tank < 0, reset tank = 0 and set start = i + 1. If the total is non-negative, that start works.

Time O(n), Space O(1).

C# solution

C# 
public int CanCompleteCircuit(int[] gas, int[] cost)
{
    int total = 0, tank = 0, start = 0;
    for (int i = 0; i < gas.Length; i++)
    {
        int diff = gas[i] - cost[i];
        total += diff;
        tank  += diff;
        if (tank < 0)            // can't reach i+1 from current start
        {
            start = i + 1;
            tank = 0;
        }
    }
    return total >= 0 ? start : -1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single pass.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Feasible iff total gas ≥ total cost.
  • A negative running tank disqualifies the entire prefix — restart after it.
  • The greedy restart finds the unique start in one pass (no nested retries).

Interview tip

🗣️ When this comes up

Lead with the feasibility check (totals), then justify the restart: if you can’t reach i+1, no earlier start could either. That argument is the proof the greedy is correct — say it explicitly.
