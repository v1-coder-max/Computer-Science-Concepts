Add Strings

Add two numbers that are too big for int by simulating addition the way you learned in school: right to left, carrying as you go.

#math #strings #simulation #easy

The problem

Given two non-negative integers as strings num1 and num2, return their sum as a string. You may not convert them to integers directly.

📥 Example

"456" + "77" → "533"

Brute force

Parsing to long works for small inputs but overflows for big numbers — and the problem forbids it precisely to test digit simulation.

Optimized approach

Walk both strings from the right, summing matched digits plus a carry. Append each result digit (modulo 10) and propagate the carry; reverse at the end.

Time O(n), Space O(n).

C# solution

C# 
public string AddStrings(string num1, string num2)
{
    int i = num1.Length - 1, j = num2.Length - 1, carry = 0;
    var sb = new StringBuilder();
    while (i >= 0 || j >= 0 || carry > 0)
    {
        int a = i >= 0 ? num1[i--] - '0' : 0;
        int b = j >= 0 ? num2[j--] - '0' : 0;
        int sum = a + b + carry;
        sb.Append((char)('0' + sum % 10));
        carry = sum / 10;
    }
    return new string(sb.ToString().Reverse().ToArray());
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass over the longer number.
  • Space: O(n) — the result buffer.

Pattern & takeaways

🎯 Remember these three

  • Simulate addition right to left with a carry — no integer conversion.
  • Keep looping while either index is valid or a carry remains.
  • The same template handles add-binary (base 2) and add-two-numbers (linked lists).

Interview tip

🗣️ When this comes up

Lead with the loop condition i >= 0 || j >= 0 || carry — forgetting the trailing carry is the classic bug. Note it generalizes to any base by changing the modulus.
