Unique Paths

How many ways from the top-left to the bottom-right moving only right or down? Each cell’s paths are the sum of the cell above and the cell left.

#dynamic-programming #2d-dp #grid #medium #combinatorics #interview

The problem

Given an m × n grid, a robot starts top-left and may move only right or down. Return the number of distinct paths to the bottom-right corner.

📥 Example

m = 3, n = 7 → 28  ·  m = 3, n = 2 → 3

Brute force

Recursively branch right and down from each cell, counting paths that reach the corner — that explores an exponential tree and recomputes the same cells repeatedly, so it’s O(2m+n). The overlapping subproblems are the DP signal.

Optimized approach

The only ways into a cell are from above or from the left, so dp[r][c] = dp[r−1][c] + dp[r][c−1], with the first row and column all 1 (one straight path). Fill row by row. Since each row needs only the previous row, collapse to a single 1-D array for O(n) space. (It’s also the binomial coefficient C(m+n−2, m−1).)

Time O(m·n), Space O(n).

C# solution

C# 
public int UniquePaths(int m, int n)
{
    var row = new int[n];
    Array.Fill(row, 1);                 // top row: one path to each cell
    for (int r = 1; r < m; r++)
        for (int c = 1; c < n; c++)
            row[c] += row[c - 1];       // above (old row[c]) + left (row[c-1])
    return row[n - 1];
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — fill every cell once.
  • Space: O(n) — one rolling row (O(1) with the math formula).

Pattern & takeaways

🎯 Remember these three

  • Grid path counts: dp[r][c] = above + left.
  • Only the previous row matters → collapse 2-D to 1-D.
  • It equals the binomial coefficient C(m+n−2, m−1).
