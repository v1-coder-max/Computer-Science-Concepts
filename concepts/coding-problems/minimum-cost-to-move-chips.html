Minimum Cost to Move Chips to The Same Position

Moving two steps is free, so all even positions gather for free and all odd positions gather for free; you only pay to merge the smaller group.

#greedy #math #easy

The problem

Chips sit at given positions. Moving a chip by 2 costs 0; moving it by 1 costs 1. Return the minimum total cost to bring all chips to one position.

📥 Example

[2,2,2,3,3] → 2 (move the two 3s left by 1 each)

Brute force

Trying every target position and summing costs is O(n²). The free even-step move collapses it to a parity count.

Optimized approach

Because moving by 2 is free, every chip can reach any same-parity position for nothing. So gather all even-position chips together (free) and all odd-position chips together (free); the cost is moving the smaller group across by 1 — min(evenCount, oddCount).

Time O(n), Space O(1).

C# solution

C# 
public int MinCostToMoveChips(int[] position)
{
    int even = 0, odd = 0;
    foreach (int p in position)
        if (p % 2 == 0) even++; else odd++;
    return Math.Min(even, odd);
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — two counters.

Pattern & takeaways

🎯 Remember these three

  • Free even-step moves mean only parity matters.
  • Answer = min(evenCount, oddCount).
  • The actual positions are irrelevant beyond their parity.
