Binary Search Tree Iterator

Iterate a BST in sorted order with Next/HasNext, using only O(h) space. A stack of leftmost nodes runs the in-order traversal lazily.

#trees #bst #design #medium #in-order #interview

The problem

Implement BSTIterator over a BST: Next() returns the next smallest value, HasNext() reports if more remain. Aim for average O(1) per call and O(h) memory.

📥 Example

BST [7,3,15,null,null,9,20]; next→3,7,9; hasNext→true; next→15,20

Brute force

Flatten the whole tree into a sorted list up front via in-order traversal — O(n) space and O(n) preprocessing. A lazy stack-based iterator uses only O(h) space and produces values on demand.

Optimized approach

Keep a stack holding the current path of leftmost nodes. Initialize by pushing the root and all its left descendants. Next: pop the top (the smallest unvisited), and if it has a right child, push that child and all of its leftmost descendants. HasNext is just “stack not empty.” Each node is pushed and popped once, so Next is amortized O(1), and the stack never exceeds the tree height.

Next amortized O(1), Space O(h).

C# solution

C# 
public class BSTIterator
{
    private readonly Stack<TreeNode> _stack = new();

    public BSTIterator(TreeNode root) => PushLeft(root);

    public int Next()
    {
        var node = _stack.Pop();
        PushLeft(node.right);          // expand the right subtree's left spine
        return node.val;
    }

    public bool HasNext() => _stack.Count > 0;

    private void PushLeft(TreeNode node)
    {
        while (node != null) { _stack.Push(node); node = node.left; }
    }
}

Complexity

⏱️ Complexity

  • Time: O(1) amortized per Next (O(n) total over n calls).
  • Space: O(h) — the stack depth.

Pattern & takeaways

🎯 Remember these three

  • Lazily simulate in-order with an explicit stack.
  • Push the left spine on init and after visiting a right child.
  • O(h) space beats flattening the whole tree to O(n).
