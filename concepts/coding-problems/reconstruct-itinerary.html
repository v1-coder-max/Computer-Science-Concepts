Reconstruct Itinerary

Use every ticket exactly once, preferring the smallest airport code. That’s an Eulerian path — Hierholzer’s DFS builds it post-order, then you reverse.

#graphs #eulerian-path #dfs #hard #hierholzer #interview

The problem

Given a list of airline tickets [from, to], reconstruct the itinerary starting at "JFK" that uses all tickets. If multiple exist, return the lexicographically smallest.

📥 Example

[["JFK","SFO"],["JFK","ATL"],["SFO","ATL"],["ATL","JFK"],["ATL","SFO"]] → JFK ATL JFK SFO ATL SFO

Brute force

Backtracking — try tickets in lexical order, undo when stuck — finds it, but can be exponential on tangled graphs. The Eulerian-path structure lets us avoid the backtracking blowup.

Optimized approach

Build an adjacency list with destinations sorted (use a min-heap or sorted multiset per node so you always take the smallest next code). Run Hierholzer’s algorithm: DFS, consuming edges as you go; when a node has no unused outgoing edges, append it to the route. The route built this way is reversed, so reverse it at the end. Appending post-order naturally handles dead-end detours.

Time O(E log E), Space O(E).

C# solution

C# 
public IList<string> FindItinerary(IList<IList<string>> tickets)
{
    var graph = new Dictionary<string, PriorityQueue<string, string>>();
    foreach (var t in tickets)
    {
        if (!graph.ContainsKey(t[0])) graph[t[0]] = new();
        graph[t[0]].Enqueue(t[1], t[1]);          // smallest code first
    }

    var route = new LinkedList<string>();
    void Dfs(string node)
    {
        var pq = graph.GetValueOrDefault(node);
        while (pq != null && pq.Count > 0)
            Dfs(pq.Dequeue());                     // consume the edge
        route.AddFirst(node);                      // post-order prepend
    }
    Dfs("JFK");
    return route.ToList();
}

Complexity

⏱️ Complexity

  • Time: O(E log E) — sorting destinations per node.
  • Space: O(E) for the graph and recursion.

Pattern & takeaways

🎯 Remember these three

  • “Use every edge once” = Eulerian path → Hierholzer’s.
  • Take the smallest next code (heap) for lexical order.
  • Append nodes post-order and reverse — dead ends sort themselves out.

Interview tip

🗣️ When this comes up

Recognizing it as an Eulerian path is the leap that beats naive backtracking. The post-order-append-then-reverse trick of Hierholzer’s is worth memorizing — it elegantly handles the detour-before-dead-end ordering.
