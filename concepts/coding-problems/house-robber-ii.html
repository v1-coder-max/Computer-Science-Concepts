House Robber II

House Robber, but the houses form a circle — first and last are neighbours. Solve it by running the linear version twice.

#dynamic-programming #arrays #medium #circular #reduction #interview

The problem

Houses are arranged in a circle; adjacent houses can’t both be robbed (and the first and last are adjacent). Return the maximum amount you can rob.

📥 Example

[2,3,2] → 3  ·  [1,2,3,1] → 4

The idea

The circular constraint just says you can’t rob both the first and last house. So either the first house is off-limits or the last house is — never both matter together. Run the linear House Robber on houses[0..n−2] (exclude the last) and on houses[1..n−1] (exclude the first), and take the larger. Handle the single-house edge case directly.

Time O(n), Space O(1).

C# solution

C# 
public int Rob(int[] nums)
{
    if (nums.Length == 1) return nums[0];
    return Math.Max(RobLine(nums, 0, nums.Length - 2),   // exclude last
                    RobLine(nums, 1, nums.Length - 1));  // exclude first

    int RobLine(int[] a, int lo, int hi)
    {
        int prev2 = 0, prev1 = 0;
        for (int i = lo; i <= hi; i++)
        {
            int cur = Math.Max(prev1, a[i] + prev2);
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — two linear passes.
  • Space: O(1) — rolling variables.

Pattern & takeaways

🎯 Remember these three

  • Circular ⇒ first and last can’t both be robbed.
  • Reduce to two linear House Robber runs, take the max.
  • Special-case a single house to avoid an empty range.

Interview tip

🗣️ When this comes up

Don’t invent a new circular DP — reduce to the linear one you already know, run twice. The reduction argument (first xor last excluded) is the insight; the rest is reuse.
