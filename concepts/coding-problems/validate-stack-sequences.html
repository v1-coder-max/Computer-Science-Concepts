Validate Stack Sequences

Just run the machine: push values in order and pop whenever the top matches the next expected pop. A clean finish means it's valid.

#stack #simulation #medium

The problem

Given two sequences pushed and popped (both permutations of distinct values), return whether they could result from a valid sequence of stack pushes and pops.

📥 Example

pushed=[1,2,3,4,5], popped=[4,5,3,2,1] → true

Brute force

Reasoning about all interleavings is unnecessary; a direct simulation is O(n) and definitive.

Optimized approach

Push each value from pushed onto a stack. After each push (and at the end), pop greedily while the top equals the next value in popped. If the stack is empty when finished, the sequence is valid.

Time O(n), Space O(n).

C# solution

C# 
public bool ValidateStackSequences(int[] pushed, int[] popped)
{
    var stack = new Stack<int>();
    int j = 0;
    foreach (int x in pushed)
    {
        stack.Push(x);
        while (stack.Count > 0 && stack.Peek() == popped[j])
        {
            stack.Pop();
            j++;
        }
    }
    return stack.Count == 0;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each value pushed and popped once.
  • Space: O(n) — the simulation stack.

Pattern & takeaways

🎯 Remember these three

  • Simulate directly — push in order, pop greedily when the top matches.
  • An empty stack at the end means the pop order was achievable.
  • Greedy popping is safe because values are distinct.

Interview tip

🗣️ When this comes up

Don't overthink it — simulate. The greedy pop-while-top-matches plus the empty-stack check is the entire solution, and it's provably correct for distinct values.
