Decode Ways

How many ways to decode a digit string into letters (A=1…Z=26)? A Fibonacci-shaped DP: each spot sums the one-digit and two-digit options.

#dynamic-programming #strings #medium #fibonacci #counting #interview

The problem

Given a string s of digits, return the number of ways to decode it into letters, where "1"→A … "26"→Z. Leading zeros make a digit/pair invalid.

📥 Example

"12" → 2 ("AB", "L")  ·  "226" → 3  ·  "06" → 0

The idea

Let dp[i] = ways to decode the prefix of length i. At each position you can take one digit if it’s 1–9 (add dp[i−1]) and/or two digits if they form 10–26 (add dp[i−2]). That recurrence is Fibonacci-like. A leading 0 can’t stand alone and only survives as the second digit of 10 or 20. Roll two variables for O(1) space.

Time O(n), Space O(1).

C# solution

C# 
public int NumDecodings(string s)
{
    if (s[0] == '0') return 0;
    int prev2 = 1, prev1 = 1;          // dp[0]=1 (empty), dp[1]=1

    for (int i = 1; i < s.Length; i++)
    {
        int cur = 0;
        if (s[i] != '0') cur += prev1;                 // one-digit decode
        int two = (s[i - 1] - '0') * 10 + (s[i] - '0');
        if (two >= 10 && two <= 26) cur += prev2;       // two-digit decode
        if (cur == 0) return 0;                        // dead end (e.g. "30")
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — two rolling values.

Pattern & takeaways

🎯 Remember these three

  • Recurrence sums the one-digit and two-digit decodings — Fibonacci-shaped.
  • Zeros are the trap: a 0 only survives inside 10/20.
  • Collapse the table to two rolling variables for O(1) space.
