Summary Ranges

Sweep once, extending a run as long as values keep stepping by exactly one; close it off the moment the chain breaks.

#arrays #two-pointers #easy

The problem

Given a sorted array of distinct integers, return the smallest list of ranges that exactly covers it. A single number is written as "a", a run as "a->b".

📥 Example

[0,1,2,4,5,7] → ["0->2","4->5","7"]

Brute force

There is no real brute force — the array is sorted, so a single linear sweep is already optimal. The work is purely in clean boundary handling.

Optimized approach

Track the start of the current run. At each index, if the next value is not exactly one more (or you are at the end), close the run: emit a single value or an a->b range, then start fresh.

Time O(n), Space O(1).

C# solution

C# 
public IList<string> SummaryRanges(int[] nums)
{
    var res = new List<string>();
    for (int i = 0; i < nums.Length; i++)
    {
        int start = nums[i];
        while (i + 1 < nums.Length && nums[i + 1] == nums[i] + 1) i++;
        res.Add(start == nums[i] ? $"{start}" : $"{start}->{nums[i]}");
    }
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each element visited once.
  • Space: O(1) — output not counted.

Pattern & takeaways

🎯 Remember these three

  • Remember the run's start; extend while values are consecutive.
  • Format a single value differently from a multi-element range.
  • Distinct + sorted means one clean sweep — no extra data structures.

Interview tip

🗣️ When this comes up

It is an easy problem that punishes sloppy boundaries. State the start-tracking invariant and handle the single-element format explicitly so off-by-ones don't creep in.
