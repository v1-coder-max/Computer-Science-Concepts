Jump Game II

Reach the end in the fewest jumps. Treat each jump as a BFS “level”: ride the current reach to its edge, then commit to one more jump.

#greedy #arrays #bfs #medium #windows #interview

The problem

Given an array where nums[i] is the max jump length from index i, return the minimum number of jumps to reach the last index (guaranteed reachable).

📥 Example

[2,3,1,1,4] → 2 (index 0 → 1 → 4)

The idea

Think of indices reachable within j jumps as BFS “level j.” Sweep left to right tracking curEnd (the farthest index reachable with the current number of jumps) and farthest (the farthest reachable using one more jump). When i reaches curEnd, you must spend a jump: increment the count and extend curEnd = farthest. One pass, no per-jump scanning.

Time O(n), Space O(1).

C# solution

C# 
public int Jump(int[] nums)
{
    int jumps = 0, curEnd = 0, farthest = 0;
    for (int i = 0; i < nums.Length - 1; i++)
    {
        farthest = Math.Max(farthest, i + nums[i]);  // best reach from this window
        if (i == curEnd)                              // exhausted current jump
        {
            jumps++;
            curEnd = farthest;                        // commit to the next window
        }
    }
    return jumps;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Each jump is a BFS level = a reachable window.
  • Bump the count only when i hits the current window’s edge.
  • Loop to n − 1: arriving at the last index needs no further jump.

Interview tip

🗣️ When this comes up

The reframe “each jump = one BFS level” is what turns this into O(n) greedy. Track two boundaries and only increment at the window edge. Contrast with Jump Game (reachability) which only needs one.
