Min Cost Climbing Stairs

Each stair costs something; climb 1 or 2 steps. Minimum cost to the top: each step’s cost plus the cheaper of the two steps below.

#dynamic-programming #arrays #easy #rolling #stairs #interview

The problem

Given cost where cost[i] is the price of step i, you may start at index 0 or 1 and climb 1 or 2 steps at a time. Return the minimum cost to reach the top (past the last step).

📥 Example

[10,15,20] → 15  ·  [1,100,1,1,1,100,1,1,100,1] → 6

Brute force

Recursively try both step sizes from each position — exponential with overlap. The subproblems collapse into a simple linear DP.

Optimized approach

Let dp[i] = minimum cost to reach step i. You arrive from step i−1 or i−2, paying that step’s cost: dp[i] = min(dp[i−1] + cost[i−1], dp[i−2] + cost[i−2]). The top is index n. Since only the last two values matter, track two rolling variables for O(1) space.

Time O(n), Space O(1).

C# solution

C# 
public int MinCostClimbingStairs(int[] cost)
{
    int prev2 = 0, prev1 = 0;          // cost to reach steps 0 and 1 (free start)
    for (int i = 2; i <= cost.Length; i++)
    {
        int cur = Math.Min(prev1 + cost[i - 1], prev2 + cost[i - 2]);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1) — two rolling values.

Pattern & takeaways

🎯 Remember these three

  • Reach a step from one or two below, paying that step’s cost.
  • The top is index n, not n−1.
  • Collapse to two rolling variables — it’s Climbing Stairs with costs.
