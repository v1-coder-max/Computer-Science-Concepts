Partition Equal Subset Sum

Can the array split into two equal-sum halves? That’s just asking whether a subset sums to total/2 — a boolean knapsack DP.

#dynamic-programming #subset-sum #medium #knapsack #partition #interview

The problem

Given an array nums of positive integers, return true if it can be partitioned into two subsets with equal sums.

📥 Example

[1,5,11,5] → true ([1,5,5] and [11])  ·  [1,2,3,5] → false

Brute force

Enumerate all subsets and check for one summing to half the total — O(2n). The overlapping subproblems (reachable sums) collapse it into a DP over a bounded sum range.

Optimized approach

If the total is odd, return false immediately. Otherwise the target is total / 2, and the question becomes “can some subset reach target?” — a 0/1 subset-sum. Use a 1-D boolean array dp where dp[s] means sum s is reachable; for each number, update dp[s] |= dp[s − num] iterating s downward so each number is used once. Return dp[target].

Time O(n · sum), Space O(sum).

C# solution

C# 
public bool CanPartition(int[] nums)
{
    int total = nums.Sum();
    if (total % 2 != 0) return false;
    int target = total / 2;

    var dp = new bool[target + 1];
    dp[0] = true;                          // empty subset sums to 0
    foreach (int num in nums)
        for (int s = target; s >= num; s--)   // descend → each num once
            dp[s] = dp[s] || dp[s - num];
    return dp[target];
}

Complexity

⏱️ Complexity

  • Time: O(n · sum/2).
  • Space: O(sum/2) — the 1-D boolean array.

Pattern & takeaways

🎯 Remember these three

  • Equal partition ⇔ a subset sums to total/2.
  • Odd total is an instant false.
  • Iterate the sum downward for 0/1 (use-once) knapsack.

Interview tip

🗣️ When this comes up

The reduction “equal partition = subset sum to half” is the insight. Then it’s a standard 0/1 knapsack — and the reverse iteration (so each item is used once) is the detail interviewers check.
