Alien Dictionary

Given alien words sorted in their language, recover the alphabet order. Adjacent words yield ordering constraints; a topological sort assembles them.

#graphs #topological-sort #hard #ordering #kahn #interview

The problem

Given a list of words sorted lexicographically by the rules of an unknown language, return any valid letter order, or "" if the ordering is invalid.

📥 Example

["wrt","wrf","er","ett","rftt"] → "wertf"

Brute force

There’s no meaningful brute force over permutations of the alphabet (up to 26! orders). The structure — pairwise constraints from sorted words — points straight at building a graph and topologically sorting it.

Optimized approach

Compare each adjacent pair of words; the first differing characters (a, b) give the edge a → b (a comes before b). Edge case: if a word is a prefix of a longer word that precedes it (e.g. "abc" before "ab"), the input is invalid → "". Then run a topological sort (Kahn’s in-degree BFS) over the letter graph; if a cycle prevents finishing all letters, return "".

Time O(total chars), Space O(1) (≤ 26 letters).

C# solution

C# 
public string AlienOrder(string[] words)
{
    var adj = new Dictionary<char, HashSet<char>>();
    var indeg = new Dictionary<char, int>();
    foreach (var w in words) foreach (char c in w)
        { adj.TryAdd(c, new()); indeg.TryAdd(c, 0); }

    for (int i = 0; i + 1 < words.Length; i++)
    {
        string a = words[i], b = words[i + 1];
        if (a.Length > b.Length && a.StartsWith(b)) return "";   // invalid prefix
        for (int j = 0; j < Math.Min(a.Length, b.Length); j++)
            if (a[j] != b[j]) { if (adj[a[j]].Add(b[j])) indeg[b[j]]++; break; }
    }

    var q = new Queue<char>(indeg.Where(kv => kv.Value == 0).Select(kv => kv.Key));
    var sb = new System.Text.StringBuilder();
    while (q.Count > 0)
    {
        char c = q.Dequeue(); sb.Append(c);
        foreach (char next in adj[c]) if (--indeg[next] == 0) q.Enqueue(next);
    }
    return sb.Length == indeg.Count ? sb.ToString() : "";       // cycle ⇒ ""
}

Complexity

⏱️ Complexity

  • Time: O(total characters across all words).
  • Space: O(1) — at most 26 letters and their edges.

Pattern & takeaways

🎯 Remember these three

  • Adjacent words → precedence edges from the first differing char.
  • The prefix edge case ("abc" before "ab") is invalid.
  • Topological sort assembles the order; a cycle means no valid order.

Interview tip

🗣️ When this comes up

Two traps to call out: deriving exactly one edge per adjacent pair (first difference only) and the invalid-prefix case. After that it’s standard Kahn’s topological sort with a cycle check.
