Word Ladder

Transform one word into another, one letter at a time, staying in the dictionary — shortest chain. It’s an unweighted shortest path, so BFS.

#graphs #bfs #shortest-path #hard #word-graph #interview

The problem

Given beginWord, endWord, and a wordList, return the number of words in the shortest transformation sequence (each step changes one letter and must be in the list), or 0 if none exists.

📥 Example

begin="hit", end="cog", list=["hot","dot","dog","lot","log","cog"] → 5 (hit→hot→dot→dog→cog)

Brute force

DFS every transformation chain and keep the shortest — exponential. Since every edge has weight 1, BFS finds the shortest path the first time it reaches endWord, far more efficiently.

Optimized approach

Model words as nodes with edges between words differing by one letter. Run BFS from beginWord, expanding level by level (each level = one transformation). To find neighbors without comparing all pairs, generate wildcard patterns (h*t, *it, …) and group dictionary words by pattern, so neighbor lookup is O(L) per word. The first time endWord appears, its level is the answer.

Time O(N · L²), Space O(N · L).

C# solution

C# 
public int LadderLength(string beginWord, string endWord, IList<string> wordList)
{
    var dict = new HashSet<string>(wordList);
    if (!dict.Contains(endWord)) return 0;

    var q = new Queue<string>();
    q.Enqueue(beginWord);
    int steps = 1;
    while (q.Count > 0)
    {
        for (int i = q.Count; i > 0; i--)
        {
            string word = q.Dequeue();
            if (word == endWord) return steps;
            var chars = word.ToCharArray();
            for (int j = 0; j < chars.Length; j++)
            {
                char original = chars[j];
                for (char c = 'a'; c <= 'z'; c++)
                {
                    chars[j] = c;
                    string next = new string(chars);
                    if (dict.Remove(next)) q.Enqueue(next);   // visit once
                }
                chars[j] = original;
            }
        }
        steps++;
    }
    return 0;
}

Complexity

⏱️ Complexity

  • Time: O(N · L²) — N words, L letters, 26 swaps each.
  • Space: O(N · L) for the set and queue.

Pattern & takeaways

🎯 Remember these three

  • Unweighted shortest path → BFS (level = one step).
  • Generate neighbors by letter swaps / wildcard buckets, not all-pairs.
  • Remove from the set on visit to avoid revisiting.
