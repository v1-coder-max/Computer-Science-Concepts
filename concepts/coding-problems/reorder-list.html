Reorder List

Reweave a list into L0 → Ln → L1 → Ln-1 → …. It looks tricky but it’s just three list skills you already know, composed: find the middle, reverse, merge.

#linked-list #two-pointers #composition #medium

The problem

Given L0 → L1 → … → Ln, rearrange it in place to L0 → Ln → L1 → Ln-1 → L2 → …. You may not change node values, only relink nodes.

📥 Example

1→2→3→4 → 1→4→2→3  ·  1→2→3→4→5 → 1→5→2→4→3

The idea

The target interleaves the front half with the reversed back half. So solve it as a pipeline of three classic sub-problems:

  1. Find the middle with slow/fast pointers.
  2. Reverse the second half (the three-pointer reversal).
  3. Merge the two halves, alternating one node from each.

Time O(n), Space O(1).

C# solution

C# 
public void ReorderList(ListNode head)
{
    if (head?.next == null) return;

    // 1) middle: slow ends at the midpoint
    ListNode slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) { slow = slow.next; fast = fast.next.next; }

    // 2) reverse the second half
    ListNode prev = null, cur = slow.next;
    slow.next = null;                       // cut the list in two
    while (cur != null) { var nxt = cur.next; cur.next = prev; prev = cur; cur = nxt; }

    // 3) merge the two halves alternately
    ListNode first = head, second = prev;
    while (second != null)
    {
        var t1 = first.next; var t2 = second.next;
        first.next = second; second.next = t1;
        first = t1; second = t2;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — each phase is a linear pass.
  • Space: O(1) — all relinking is in place.

Pattern & takeaways

🎯 Remember these three

  • Hard list problems are usually compositions of midpoint + reverse + merge.
  • Cut the list (slow.next = null) before reversing the tail.
  • Save both next pointers before relinking during the merge.
