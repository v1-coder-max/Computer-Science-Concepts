Integer Break

Split n into pieces whose product is as large as possible. A DP tries every first cut; the math shortcut is 'use as many 3s as you can'.

#dynamic-programming #math #medium

The problem

Given an integer n ≥ 2, break it into the sum of at least two positive integers and return the maximum product of those parts.

📥 Example

10 → 36 (3 + 3 + 4)

Brute force

Enumerating all partitions is exponential. Each first cut j leaves a subproblem on n − j, giving an O(n²) DP.

Optimized approach

Let dp[i] be the best product for i. For each first part j < i, the rest is either left whole (i − j) or broken further (dp[i − j]): dp[i] = max over j of max(j·(i−j), j·dp[i−j]). (The O(1) math answer: peel off 3s, never leaving a 1.)

Time O(n²), Space O(n).

C# solution

C# 
public int IntegerBreak(int n)
{
    var dp = new int[n + 1];
    dp[1] = 1;
    for (int i = 2; i <= n; i++)
        for (int j = 1; j < i; j++)
            dp[i] = Math.Max(dp[i], Math.Max(j * (i - j), j * dp[i - j]));
    return dp[n];
}

Complexity

⏱️ Complexity

  • Time: O(n²) — every first-cut for every i.
  • Space: O(n) — the DP array.

Pattern & takeaways

🎯 Remember these three

  • Try every first part j; the rest is whole or further broken.
  • max(j·(i−j), j·dp[i−j]) captures both choices.
  • Math shortcut: break into as many 3s as possible (turn a remainder of 1 into a 2+2).

Interview tip

🗣️ When this comes up

Present the DP, then offer the elegant O(1): 3s maximize the product because 3 beats 2×2-style splits. Knowing both the DP and the math reads as real depth.
