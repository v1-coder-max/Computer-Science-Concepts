Longest Repeating Character Replacement

Make the longest run of one letter using at most k swaps. A sliding window is valid while window length − count of its most common letter ≤ k.

#sliding-window #strings #frequency #medium

The problem

Given a string s of uppercase letters and an integer k, you may replace at most k characters with any letter. Return the length of the longest substring containing a single repeated letter you can achieve.

📥 Example

s = "AABABBA", k = 1 → 4 (e.g. "AABA" → "AAAA")

The idea

A window can become all-one-letter iff the characters you’d need to replace — windowLength − maxFreq, where maxFreq is the count of the most frequent letter in the window — is at most k. Grow the window by moving right; whenever the window becomes invalid (length − maxFreq > k), slide left forward. Track the largest valid window seen.

Time O(n), Space O(1) (26-letter count array).

C# solution

C# — sliding window 
public int CharacterReplacement(string s, int k)
{
    var count = new int[26];
    int left = 0, maxFreq = 0, best = 0;

    for (int right = 0; right < s.Length; right++)
    {
        maxFreq = Math.Max(maxFreq, ++count[s[right] - 'A']);

        // window invalid? shrink from the left
        while ((right - left + 1) - maxFreq > k)
            count[s[left++] - 'A']--;

        best = Math.Max(best, right - left + 1);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index enters and leaves the window once.
  • Space: O(1) — a fixed 26-element frequency array.

Pattern & takeaways

🎯 Remember these three

  • Validity metric: windowLen − maxFreq ≤ k — the chars you must replace.
  • Grow right, shrink left only when invalid; the window never shrinks below the best.
  • You don’t need a perfectly accurate maxFreq after shrinking — the answer only grows when a real new max appears.

Interview tip

🗣️ When this comes up

Define the window’s validity condition out loud first — it’s the whole problem. The subtle point interviewers probe: why you never need to decrease maxFreq when shrinking. Answer: the best window only updates when a genuinely larger maxFreq is found.
