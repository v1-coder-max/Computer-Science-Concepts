Stone Game

Both play optimally, so track the best achievable score difference (current player minus opponent) on each subarray; the first player wins if it's positive.

#dynamic-programming #game-theory #medium

The problem

Two players alternately take a pile from either end of a row; both play optimally. Return whether the first player wins (with an even count and even total, they always can).

📥 Example

[5,3,4,5] → true

Brute force

Minimax recursion over all move sequences is exponential without memoization. Interval DP makes it O(n²).

Optimized approach

Let dp[i][j] be the best score difference the current player can secure on subarray i..j. Taking the left gives nums[i] − dp[i+1][j] (the opponent then plays optimally on the rest); taking the right gives nums[j] − dp[i][j−1]. Take the max. The first player wins iff dp[0][n−1] > 0.

Time O(n²), Space O(n²).

C# solution

C# 
public bool StoneGame(int[] piles)
{
    int n = piles.Length;
    var dp = new int[n, n];
    for (int i = 0; i < n; i++) dp[i, i] = piles[i];
    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++)
        {
            int j = i + len - 1;
            dp[i, j] = Math.Max(piles[i] - dp[i + 1, j], piles[j] - dp[i, j - 1]);
        }
    return dp[0, n - 1] > 0;
}

Complexity

⏱️ Complexity

  • Time: O(n²) — fill each interval once.
  • Space: O(n²) — the n×n table (reducible to O(n)).

Pattern & takeaways

🎯 Remember these three

  • Track the score difference, not absolute scores.
  • dp[i][j] = max(take left − dp[i+1][j], take right − dp[i][j−1]).
  • First player wins iff the full-range difference is positive (always true with even n here).

🗣️ When this comes up

The score-difference formulation is what makes the minimax clean — subtract the opponent's optimal result. (There's also a one-line parity answer, but the DP shows you understand why.)
