House Robber III

Rob houses arranged in a tree, no parent and child both robbed. A post-order DFS returns a pair per node: best if robbed, best if skipped.

#trees #dynamic-programming #dfs #medium #tree-dp #interview

The problem

Houses form a binary tree; you can’t rob two directly-connected (parent-child) houses. Return the maximum amount you can rob.

📥 Example

[3,2,3,null,3,null,1] → 7 (3 + 3 + 1)  ·  [3,4,5,1,3,null,1] → 9

Brute force

A recursion returning a single “best from this node” value recomputes grandchildren repeatedly — exponential. Returning two values (rob / skip) per node memoizes the choice in one post-order pass.

Optimized approach

For each node, compute a pair (rob, skip). If you rob this node, you must skip both children: rob = node.val + left.skip + right.skip. If you skip it, each child takes its own best: skip = max(left.rob, left.skip) + max(right.rob, right.skip). Return the pair up; the answer at the root is max(rob, skip). One post-order traversal, no repeated work.

Time O(n), Space O(h).

C# solution

C# 
public int Rob(TreeNode root)
{
    var (rob, skip) = Dfs(root);
    return Math.Max(rob, skip);

    (int rob, int skip) Dfs(TreeNode node)
    {
        if (node == null) return (0, 0);
        var l = Dfs(node.left);
        var r = Dfs(node.right);
        int rob  = node.val + l.skip + r.skip;                 // rob this node
        int skip = Math.Max(l.rob, l.skip) + Math.Max(r.rob, r.skip);
        return (rob, skip);
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — single post-order pass.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Tree DP: return a (rob, skip) pair per node.
  • Robbing a node forces skipping both children.
  • Returning two states avoids the exponential single-value recursion.

🗣️ When this comes up

The leap from linear House Robber is “return two values per node.” Explain why a single value forces recomputing grandchildren. It’s the canonical intro to tree DP.
