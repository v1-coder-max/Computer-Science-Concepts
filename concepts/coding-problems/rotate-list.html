Rotate List

Rotate a list right by k. Close it into a ring, then break the ring at the right spot — no element-by-element shifting.

#linked-list #pointers #medium #rotation #ring #interview

The problem

Given the head of a linked list, rotate it to the right by k places.

📥 Example

1→2→3→4→5, k=2 → 4→5→1→2→3

Brute force

Move the last node to the front k times — O(n·k). Forming a ring and cutting once is O(n) regardless of k.

Optimized approach

First find the length n and the tail. Connect tail.next = head to form a ring. The new tail is n − (k % n) − 1 steps from the head; walk there, set the node after it as the new head, and break the ring (newTail.next = null). Taking k % n handles rotations larger than the list.

Time O(n), Space O(1).

C# solution

C# 
public ListNode RotateRight(ListNode head, int k)
{
    if (head == null || head.next == null || k == 0) return head;

    int n = 1; ListNode tail = head;
    while (tail.next != null) { tail = tail.next; n++; }   // length + tail

    k %= n;
    if (k == 0) return head;

    tail.next = head;                       // form a ring
    ListNode newTail = head;
    for (int i = 0; i < n - k - 1; i++) newTail = newTail.next;
    ListNode newHead = newTail.next;
    newTail.next = null;                    // break the ring
    return newHead;
}

Complexity

⏱️ Complexity

  • Time: O(n) — count, link, walk.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Form a ring, then cut once at the new tail.
  • New tail is n − (k % n) − 1 from the head.
  • k %= n handles rotations larger than the list.

Interview tip

🗣️ When this comes up

The ring trick avoids per-step shifting. The arithmetic for the new tail (and k %= n) is the only fiddly part — derive it from where the cut must land.
