Same Tree

Are two binary trees identical — same shape and same values? Recurse down both at once, comparing node-for-node.

#trees #recursion #dfs #easy

The problem

Given the roots of two binary trees p and q, return true if they are structurally identical and every corresponding node has the same value.

📥 Example

p = [1,2,3], q = [1,2,3] → true  ·  p = [1,2], q = [1,null,2] → false

The idea

Walk both trees in lockstep. At each pair of nodes there are three outcomes: both null → this branch matches (return true); exactly one null, or values differ → mismatch (return false); otherwise the values match, so the trees are the same here iff their left subtrees match and their right subtrees match. The recursion handles the rest.

Time O(n), Space O(h) recursion depth.

C# solution

C# 
public bool IsSameTree(TreeNode p, TreeNode q)
{
    if (p == null && q == null) return true;     // both empty → match
    if (p == null || q == null) return false;    // exactly one empty → mismatch
    if (p.val != q.val)         return false;    // values differ

    return IsSameTree(p.left, q.left)
        && IsSameTree(p.right, q.right);
}

Complexity

⏱️ Complexity

  • Time: O(n) — visits each node pair once (n = smaller tree’s size).
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Parallel recursion over two trees: compare both nodes at every step.
  • Handle the null cases first — both null, one null — before touching .val.
  • This is the core of subtree-of-another-tree and symmetric-tree.

Interview tip

🗣️ When this comes up

It’s a warm-up that proves you can recurse over two structures at once. Get the null-handling order right (both, then either) and note it’s the helper inside subtree of another tree — interviewers often chain the two.
