Wiggle Subsequence

A wiggle alternates up, down, up, down. Count every direction flip as you scan and you get the longest such subsequence for free.

#greedy #dynamic-programming #arrays #medium

The problem

A wiggle sequence has strictly alternating positive and negative differences between successive elements. Return the length of the longest wiggle subsequence of the array.

📥 Example

[1,7,4,9,2,5] → 6 (the whole array wiggles)

Brute force

An O(n²) DP tracks the best wiggle ending at each index in each direction. But a greedy observation makes it O(n): the answer is just the number of trend changes.

Optimized approach

Scan the differences. Maintain whether the last counted move went up or down; each time the trend flips, extend the subsequence. Equal neighbors are ignored.

Time O(n), Space O(1).

C# solution

C# 
public int WiggleMaxLength(int[] nums)
{
    if (nums.Length < 2) return nums.Length;
    int up = 1, down = 1;
    for (int i = 1; i < nums.Length; i++)
    {
        if (nums[i] > nums[i - 1]) up = down + 1;
        else if (nums[i] < nums[i - 1]) down = up + 1;
    }
    return Math.Max(up, down);
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass over the differences.
  • Space: O(1) — two running lengths.

Pattern & takeaways

🎯 Remember these three

  • The longest wiggle length equals the number of trend reversals plus one.
  • Track two states — last move up, last move down — and update on each flip.
  • Flat steps (equal neighbors) never extend the wiggle.
