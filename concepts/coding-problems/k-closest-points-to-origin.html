K Closest Points to Origin

Find the k nearest points. A size-k max-heap keyed by distance keeps the closest k, evicting the farthest — no need to sort everything.

#heap #priority-queue #quickselect #medium #top-k #interview

The problem

Given an array of points and an integer k, return the k points closest to the origin (Euclidean distance, any order).

📥 Example

points = [[1,3],[-2,2]], k = 1 → [[-2,2]]

The idea

Compare by squared distance (x² + y²) — no square roots needed. Keep a max-heap of size k: push each point, and when the heap exceeds k, pop the farthest. What remains is the closest k. This is O(n log k), better than the O(n log n) full sort when k is small. Quickselect gives O(n) average if you only need the set (unordered).

Time O(n log k), Space O(k).

C# solution

C# 
public int[][] KClosest(int[][] points, int k)
{
    // max-heap of size k: priority is the squared distance, negated
    var heap = new PriorityQueue<int[], int>();
    foreach (var p in points)
    {
        int dist = p[0] * p[0] + p[1] * p[1];
        heap.Enqueue(p, -dist);
        if (heap.Count > k) heap.Dequeue();   // drop the farthest
    }
    var result = new int[k][];
    for (int i = 0; i < k; i++) result[i] = heap.Dequeue();
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n log k) with the heap; O(n) average with Quickselect.
  • Space: O(k) for the heap.

Pattern & takeaways

🎯 Remember these three

  • Compare by squared distance — the sqrt is monotonic and unnecessary.
  • A size-k max-heap keeps the closest k (evict the farthest).
  • Quickselect is O(n) average when order within the k doesn’t matter.

Interview tip

🗣️ When this comes up

Two quick wins: skip the square root (compare squared distances) and use a max-heap so you evict the farthest. If asked for better, Quickselect on the kth distance gives O(n) average.
