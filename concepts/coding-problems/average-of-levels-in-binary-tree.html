Average of Levels in Binary Tree

Process the tree one full level at a time, summing each row and dividing by its size.

#trees #bfs #easy

The problem

Given a binary tree, return a list of the average value of the nodes on each level, from top to bottom.

📥 Example

Tree [3,9,20,null,null,15,7] → [3.0, 14.5, 11.0]

Brute force

A DFS that records depth and accumulates per-level sums works too, but BFS matches the problem's level structure most naturally.

Optimized approach

Run a queue-based BFS. For each level, note its size up front, drain exactly that many nodes while summing their values and enqueuing their children, then record sum / size.

Time O(n), Space O(w).

C# solution

C# 
public IList<double> AverageOfLevels(TreeNode root)
{
    var res = new List<double>();
    var q = new Queue<TreeNode>();
    q.Enqueue(root);
    while (q.Count > 0)
    {
        int size = q.Count;
        double sum = 0;
        for (int i = 0; i < size; i++)
        {
            var node = q.Dequeue();
            sum += node.val;
            if (node.left != null) q.Enqueue(node.left);
            if (node.right != null) q.Enqueue(node.right);
        }
        res.Add(sum / size);
    }
    return res;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node enqueued once.
  • Space: O(w) — w = max level width.

Pattern & takeaways

🎯 Remember these three

  • Snapshot q.Count before draining to isolate one level.
  • Use a double sum to avoid integer-division and overflow surprises.
  • Same level-by-level template as level-order traversal and right-side view.

Interview tip

🗣️ When this comes up

The reusable pattern is 'capture the level size, then loop exactly that many times'. Watch the numeric type — a long/double sum avoids overflow on big values.
