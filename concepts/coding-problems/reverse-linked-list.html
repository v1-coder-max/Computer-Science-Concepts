Reverse Linked List

Flip a singly linked list so the last node becomes the head. The whole trick is three pointers walking the list and re-pointing each next backwards.

#linked-list #pointers #in-place #easy

The problem

Given the head of a singly linked list, reverse it and return the new head. Do it in place — O(1) extra memory.

📥 Example

1 → 2 → 3 → 4 → 5 → null becomes 5 → 4 → 3 → 2 → 1 → null

Brute force

You could push every value onto a list/stack and rebuild a new list in reverse — O(n) time but O(n) extra space, and it allocates new nodes instead of reusing the ones you have. The interview wants the in-place pointer reversal.

Optimized approach

Walk the list with three references: prev (starts null), curr (starts at head), and a saved next. At each step: remember next = curr.next, point curr.next = prev (the reversal), then advance prev = curr and curr = next. When curr falls off the end, prev is the new head.

Time O(n), Space O(1).

C# solution

C# 
public class ListNode { public int val; public ListNode next; }

public ListNode ReverseList(ListNode head)
{
    ListNode prev = null, curr = head;
    while (curr != null)
    {
        ListNode next = curr.next; // save the rest of the list
        curr.next = prev;          // reverse this node's pointer
        prev = curr;               // prev moves up
        curr = next;               // curr moves on
    }
    return prev;                   // prev is the new head
}

// Recursive variant (O(n) stack space):
public ListNode ReverseRec(ListNode head)
{
    if (head == null || head.next == null) return head;
    ListNode newHead = ReverseRec(head.next);
    head.next.next = head;         // make the next node point back
    head.next = null;
    return newHead;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass, each pointer touched once.
  • Space: O(1) iterative (O(n) call stack for the recursive form).

Pattern & takeaways

🎯 Remember these three

  • The prev / curr / next three-pointer dance is the core linked-list pattern.
  • Always save next before you overwrite curr.next, or you lose the rest of the list.
  • prev ends as the new head; reuse nodes in place for O(1) space.
