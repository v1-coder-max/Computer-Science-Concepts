Course Schedule

Can you finish all the courses given their prerequisites? You can — unless the prerequisites form a cycle. This is cycle detection on a directed graph, a.k.a. topological sort.

#graphs #topological-sort #cycle-detection #medium

The problem

There are numCourses courses labelled 0…n-1. Each pair [a, b] means “to take a you must first take b.” Return true if you can complete every course.

📥 Example

2, [[1,0]] → true  ·  2, [[1,0],[0,1]] → false (0 and 1 each need the other)

The idea

Model courses as nodes and each prerequisite b → a as a directed edge. You can finish everything iff the graph is acyclic (a DAG). If there’s a cycle, those courses can never start. So the task reduces to cycle detection — and a clean way to do it is a topological sort.

Kahn's algorithm (BFS)

Compute each node’s in-degree (number of prerequisites). Queue every course with in-degree 0 (no prereqs — can start now). Repeatedly pop one, “take” it, and decrement the in-degree of the courses that depend on it; any that drop to 0 join the queue. If you manage to process all courses this way, there’s no cycle. If some never reach in-degree 0, they’re stuck in a cycle → false.

Time O(V + E), Space O(V + E).

C# solution

C# — Kahn's BFS 
public bool CanFinish(int numCourses, int[][] prerequisites)
{
    var adj = new List<int>[numCourses];
    for (int i = 0; i < numCourses; i++) adj[i] = new List<int>();
    var indeg = new int[numCourses];

    foreach (var p in prerequisites)        // p = [course, prereq]  => prereq -> course
    {
        adj[p[1]].Add(p[0]);
        indeg[p[0]]++;
    }

    var q = new Queue<int>();
    for (int i = 0; i < numCourses; i++)
        if (indeg[i] == 0) q.Enqueue(i);    // can start with no prerequisites

    int taken = 0;
    while (q.Count > 0)
    {
        int course = q.Dequeue();
        taken++;
        foreach (int next in adj[course])
            if (--indeg[next] == 0) q.Enqueue(next);
    }
    return taken == numCourses;             // all taken ⇒ no cycle
}

Complexity

⏱️ Complexity

  • Time: O(V + E) — each course and prerequisite edge processed once.
  • Space: O(V + E) — adjacency list plus in-degree array and queue.

Pattern & takeaways

🎯 Remember these three

  • “Order with dependencies / can it be scheduled?” → topological sort on a DAG.
  • Feasible iff acyclic; a cycle leaves nodes that never reach in-degree 0.
  • Kahn’s BFS counts processed nodes; the DFS variant uses three colors (white/gray/black) to spot a back edge.

Interview tip

🗣️ When this comes up

Say the magic words: “directed graph, feasible iff no cycle, solve with topological sort.” Kahn’s in-degree BFS is the easiest to code bug-free. The follow-up “return a valid order” just means output the dequeue sequence instead of a boolean.
