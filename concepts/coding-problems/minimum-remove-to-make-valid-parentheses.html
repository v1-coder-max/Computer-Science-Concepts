Minimum Remove to Make Valid Parentheses

Track unmatched parentheses by index: a stray ')' is doomed immediately, and any '(' still unmatched at the end must go too.

#stack #strings #medium

The problem

Given a string of letters and parentheses, remove the minimum number of parentheses so the result is valid, and return any such result.

📥 Example

"a)b(c)d" → "ab(c)d"

Brute force

Trying removals combinatorially is exponential. A single stack pass identifies exactly which parentheses are unmatched.

Optimized approach

Scan once, pushing the indices of unmatched '('. On a ')', pop a matching '(' if available, else mark this ')' for removal. Whatever '(' indices remain on the stack are also removed. Rebuild the string skipping marked indices.

Time O(n), Space O(n).

C# solution

C# 
public string MinRemoveToMakeValid(string s)
{
    var arr = s.ToCharArray();
    var stack = new Stack<int>();
    var remove = new HashSet<int>();
    for (int i = 0; i < arr.Length; i++)
    {
        if (arr[i] == '(') stack.Push(i);
        else if (arr[i] == ')')
        {
            if (stack.Count > 0) stack.Pop();
            else remove.Add(i);
        }
    }
    while (stack.Count > 0) remove.Add(stack.Pop());
    var sb = new StringBuilder();
    for (int i = 0; i < arr.Length; i++)
        if (!remove.Contains(i)) sb.Append(arr[i]);
    return sb.ToString();
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass plus a rebuild.
  • Space: O(n) — the index stack and removal set.

Pattern & takeaways

🎯 Remember these three

  • Stack indices of unmatched '(' so you can delete them precisely.
  • An unmatched ')' is hopeless the moment you see it.
  • Leftover '(' indices on the stack get removed at the end.

🗣️ When this comes up

Storing indices (not just counts) is what lets you reconstruct the answer. Two removal sources — stray ')' during the scan and leftover '(' after — must both be handled.
