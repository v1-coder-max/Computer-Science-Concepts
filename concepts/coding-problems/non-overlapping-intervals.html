Non-overlapping Intervals

Remove the fewest intervals so none overlap. Sort by end time and greedily keep each interval that starts after the last one you kept.

#intervals #greedy #sorting #medium #activity-selection #interview

The problem

Given intervals, return the minimum number you must remove so the remaining intervals don’t overlap.

📥 Example

[[1,2],[2,3],[3,4],[1,3]] → 1 (remove [1,3])

The idea

This is classic activity selection: to keep the most non-overlapping intervals (and thus remove the fewest), always keep the interval that ends earliest — it leaves the most room for the rest. Sort by end time, track the last kept end; if the next interval starts >= that end, keep it (advance the end), otherwise it overlaps and must be removed (count it). Removals = total − kept.

Time O(n log n), Space O(1).

C# solution

C# 
public int EraseOverlapIntervals(int[][] intervals)
{
    Array.Sort(intervals, (a, b) => a[1] - b[1]);   // sort by end time
    int removed = 0, lastEnd = int.MinValue;

    foreach (var iv in intervals)
    {
        if (iv[0] >= lastEnd) lastEnd = iv[1];       // non-overlapping → keep
        else removed++;                              // overlaps → remove
    }
    return removed;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — the sort dominates.
  • Space: O(1) beyond the sort.

Pattern & takeaways

🎯 Remember these three

  • Sort by end time (not start) — earliest end leaves the most room.
  • Keep when start ≥ lastEnd; otherwise remove.
  • It’s activity-selection: maximize kept = minimize removed.

Interview tip

🗣️ When this comes up

The crux is sorting by end time — interviewers watch for that. Explain why earliest-finish is optimal (it dominates any other choice). It’s the canonical greedy activity-selection proof.
