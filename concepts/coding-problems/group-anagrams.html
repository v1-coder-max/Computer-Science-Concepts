Group Anagrams

Bucket words that are anagrams of one another. The key idea: give every anagram the same canonical key, then group by it.

#strings #hashing #grouping #medium

The problem

Given an array of strings, group the anagrams together. Return the groups in any order.

📥 Example

["eat","tea","tan","ate","nat","bat"] → [["eat","tea","ate"], ["tan","nat"], ["bat"]]

Brute force

Compare every pair to see if they’re anagrams and union them. Time O(n² · k) for n words of length k — too slow.

Optimized approach

All anagrams share a canonical form. Map each word to a key that’s identical for anagrams, then drop it into a dictionary key → list of words. Two key choices:

  • Sorted string — "eat", "tea", "ate" all sort to "aet". Simple; key cost O(k log k).
  • Character-count signature — a 26-length count turned into a string (e.g. "1#0#0#…1#…1#"). Key cost O(k), faster for long words.

Time O(n · k log k) (sorted key) or O(n · k) (count key); Space O(n · k).

C# solution

C# 
public IList<IList<string>> GroupAnagrams(string[] strs)
{
    var groups = new Dictionary<string, IList<string>>();

    foreach (string word in strs)
    {
        // Canonical key: sorted characters (anagrams share it).
        char[] chars = word.ToCharArray();
        Array.Sort(chars);
        string key = new string(chars);

        if (!groups.TryGetValue(key, out var bucket))
            groups[key] = bucket = new List<string>();
        bucket.Add(word);
    }
    return groups.Values.ToList();
}
// Faster key for long words: a 26-count signature instead of sorting.

Complexity

⏱️ Complexity

  • Time: O(n · k log k) with a sorted key (O(n · k) with a count signature).
  • Space: O(n · k) for the grouping dictionary.

Pattern & takeaways

🎯 Remember these three

  • Hash by a canonical key: map items to a normalized form so equivalents collide.
  • Sorted string is the easy key; a character-count signature is faster for long words.
  • Group-by-key with a dictionary turns an O(n²) pairwise comparison into O(n).

Interview tip

🗣️ When this comes up

The insight to verbalize: anagrams collide under a canonical key. Offer the sorted-key version first, then mention the O(k) count-signature key as an optimization for long strings — that trade-off discussion is what they’re after.
