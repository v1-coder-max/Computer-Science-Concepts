Simplify Path

Canonicalize a Unix path like /a/./b/../c. Split on slashes and let a stack handle ., .., and empty segments.

#stack #strings #medium #parsing #path #interview

The problem

Given an absolute Unix-style path, return its simplified canonical form: a single leading slash, no trailing slash (unless root), no ., no .. (which pops a directory), and no repeated slashes.

📥 Example

"/home//foo/" → "/home/foo"  ·  "/a/./b/../../c/" → "/c"

Brute force

Repeated string find-and-replace of .. and . is error-prone and can be quadratic. Splitting into segments and using a stack handles every case cleanly in one pass.

Optimized approach

Split the path on /. Walk the segments with a stack: skip empty strings and . (current directory); on .., pop one directory if the stack is non-empty (can’t go above root); otherwise push the real directory name. Finally join the stack with / and prepend a leading slash.

Time O(n), Space O(n).

C# solution

C# 
public string SimplifyPath(string path)
{
    var stack = new Stack<string>();
    foreach (var part in path.Split('/'))
    {
        if (part == "" || part == ".") continue;
        if (part == "..") { if (stack.Count > 0) stack.Pop(); }
        else stack.Push(part);
    }
    return "/" + string.Join("/", stack.Reverse());
}

Complexity

⏱️ Complexity

  • Time: O(n) — split and one pass.
  • Space: O(n) for the stack.

Pattern & takeaways

🎯 Remember these three

  • Split on /, then a stack of directory names.
  • Skip "" and .; .. pops (guard against root).
  • Join with single slashes and one leading slash.

Interview tip

🗣️ When this comes up

Splitting then stacking handles all the edge cases (//, trailing slash, .. at root) that string-replacement botches. Enumerate the segment cases up front.
