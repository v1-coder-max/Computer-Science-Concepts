Coin Change

Make an amount with the fewest coins, given coin denominations you can reuse. Greedy fails on some coin sets — the robust answer is bottom-up dynamic programming.

#dynamic-programming #unbounded-knapsack #bottom-up #medium

The problem

Given coin denominations coins (unlimited supply of each) and an amount, return the minimum number of coins needed to make it, or -1 if it can’t be made.

📥 Example

coins = [1,2,5], amount = 11 → 3 (5 + 5 + 1)  ·  coins = [2], amount = 3 → -1

Why not greedy

Always grabbing the biggest coin works for typical currencies but not in general. With coins = [1,3,4], amount = 6, greedy takes 4 + 1 + 1 = 3 coins, but the optimum is 3 + 3 = 2. You must consider every denomination at each amount — that’s DP.

Optimized approach

Let dp[a] = the fewest coins to make amount a. Base: dp[0] = 0. For each amount from 1…amount, try every coin that fits: dp[a] = 1 + min(dp[a - coin]) over all coins ≤ a. Initialize the rest to a sentinel “impossible” (e.g. amount+1); if dp[amount] stays at the sentinel, return -1. This is an unbounded knapsack (coins reusable).

Time O(amount × coins), Space O(amount).

C# solution

C# — bottom-up DP 
public int CoinChange(int[] coins, int amount)
{
    var dp = new int[amount + 1];
    Array.Fill(dp, amount + 1);      // sentinel "impossible" (bigger than any real answer)
    dp[0] = 0;

    for (int a = 1; a <= amount; a++)
        foreach (int coin in coins)
            if (coin <= a)
                dp[a] = Math.Min(dp[a], 1 + dp[a - coin]);

    return dp[amount] > amount ? -1 : dp[amount];
}

Complexity

⏱️ Complexity

  • Time: O(amount × number of coins).
  • Space: O(amount) for the 1-D table.

Pattern & takeaways

🎯 Remember these three

  • Greedy can be wrong for arbitrary coin sets — DP is the safe answer.
  • dp[a] = 1 + min over coins of dp[a-coin]; use a sentinel for “unreachable.”
  • Coins reusable ⇒ unbounded knapsack; “number of ways” is the same table with a sum instead of a min.

Interview tip

🗣️ When this comes up

Pre-empt the greedy temptation with the [1,3,4] counter-example, then go bottom-up. Be explicit about the sentinel and the final “did we reach it?” check — that’s where buggy solutions return 0 instead of -1.
