Sort Colors

Sort an array of just 0s, 1s, and 2s in one pass. The Dutch National Flag partition uses three pointers — low, mid, high.

#arrays #two-pointers #medium #dutch-flag #partition #interview

The problem

Given an array nums with values 0, 1, 2 (representing colors), sort it in place so all 0s come first, then 1s, then 2s — without a library sort.

📥 Example

[2,0,2,1,1,0] → [0,0,1,1,2,2]

Brute force

Counting sort (count each value, then overwrite) works in two passes. The Dutch flag algorithm does it in a single pass with three pointers.

Optimized approach

Maintain low (next 0 position), high (next 2 position), and a scanning mid. If nums[mid] == 0, swap with low and advance both; if == 2, swap with high and decrement high (don’t advance mid — the swapped-in value is unexamined); if == 1, just advance mid. Stop when mid > high.

Time O(n), Space O(1).

C# solution

C# 
public void SortColors(int[] nums)
{
    int low = 0, mid = 0, high = nums.Length - 1;
    while (mid <= high)
    {
        if (nums[mid] == 0)      { (nums[low], nums[mid]) = (nums[mid], nums[low]); low++; mid++; }
        else if (nums[mid] == 2) { (nums[high], nums[mid]) = (nums[mid], nums[high]); high--; }
        else                     mid++;     // a 1 stays in the middle
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — in place.

Pattern & takeaways

🎯 Remember these three

  • Dutch National Flag: three pointers for three groups.
  • After a swap with high, don’t advance mid — recheck it.
  • Counting sort is the simpler two-pass fallback.

Interview tip

🗣️ When this comes up

Name it as Dutch National Flag and explain the “don’t advance mid after a high-swap” subtlety — that’s the bug everyone hits. One pass, O(1) space is the answer they want.
