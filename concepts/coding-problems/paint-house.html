Paint House

Each house's cheapest choice for a color is its own cost plus the cheaper of the previous house's other two colors. One pass down the row.

#dynamic-programming #greedy #medium

The problem

Paint a row of houses, each in one of three colors (red, green, blue), so that no two adjacent houses share a color. Given an n×3 cost matrix, return the minimum total cost.

📥 Example

[[17,2,17],[16,16,5],[14,3,19]] → 10

Brute force

Trying all 3ⁿ colorings is exponential. Each house's best cost per color depends only on the previous house, giving an O(n) DP with three states.

Optimized approach

Carry three running totals — the minimum cost to paint up to the current house ending in each color. For each color at house i, add its cost to the minimum of the other two colors at house i − 1. The answer is the minimum of the three final totals.

Time O(n), Space O(1).

C# solution

C# 
public int MinCost(int[][] costs)
{
    int r = costs[0][0], g = costs[0][1], b = costs[0][2];
    for (int i = 1; i < costs.Length; i++)
    {
        int nr = costs[i][0] + Math.Min(g, b);
        int ng = costs[i][1] + Math.Min(r, b);
        int nb = costs[i][2] + Math.Min(r, g);
        (r, g, b) = (nr, ng, nb);
    }
    return Math.Min(r, Math.Min(g, b));
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass, constant work per house.
  • Space: O(1) — three rolling totals.

Pattern & takeaways

🎯 Remember these three

  • Each color's cost depends only on the previous house's other two colors.
  • Three rolling variables replace an n×3 DP table.
  • Generalizes to k colors (Paint House II) with an O(n·k) — or O(n·log k) — refinement.
