Is Subsequence

Can you spell s by skipping through t left to right? One pointer per string, advancing greedily, answers it in a single pass.

#two-pointers #strings #greedy #easy

The problem

Given strings s and t, return true if s is a subsequence of t (the characters of s appear in order within t, not necessarily contiguously).

📥 Example

s = "abc", t = "ahbgdc" → true

Brute force

Recursive matching (try to match or skip each t character) explores an exponential tree. The greedy choice — always match as early as possible — is provably optimal, so no backtracking is needed.

Optimized approach

Keep a pointer i into s and scan t with j. Whenever t[j] == s[i], advance i. If i reaches the end of s, every character matched in order.

Time O(n), Space O(1).

C# solution

C# 
public bool IsSubsequence(string s, string t)
{
    int i = 0;
    foreach (char c in t)
    {
        if (i < s.Length && s[i] == c) i++;
        if (i == s.Length) return true;
    }
    return i == s.Length;
}

Complexity

⏱️ Complexity

  • Time: O(n) — n = length of t, scanned once.
  • Space: O(1) — two indices.

Pattern & takeaways

🎯 Remember these three

  • Greedily matching the earliest character is optimal — no backtracking needed.
  • Advance the s pointer only on a match; always advance the t pointer.
  • For many queries against the same t, pre-index each character's positions and binary-search instead.
