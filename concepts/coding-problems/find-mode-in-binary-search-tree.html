Find Mode in Binary Search Tree

In-order makes equal values adjacent, so counting runs of the same value reveals the mode without a hash map.

#trees #bst #dfs #easy

The problem

Given a BST that may contain duplicates, return all values that appear most frequently (the modes).

📥 Example

BST [1,null,2,2] → [2]

Brute force

A hash map of counts works in O(n) space. In-order traversal exploits the sorted order for O(1) extra space (besides recursion).

Optimized approach

Traverse in order, tracking the previous value and the current run length. When the run beats the best, reset the mode list; when it ties, append. Equal values are guaranteed adjacent by the BST ordering.

Time O(n), Space O(h).

C# solution

C# 
public int[] FindMode(TreeNode root)
{
    var modes = new List<int>();
    int? prev = null;
    int count = 0, best = 0;
    void InOrder(TreeNode node)
    {
        if (node == null) return;
        InOrder(node.left);
        count = (prev == node.val) ? count + 1 : 1;
        if (count > best) { best = count; modes.Clear(); modes.Add(node.val); }
        else if (count == best) modes.Add(node.val);
        prev = node.val;
        InOrder(node.right);
    }
    InOrder(root);
    return modes.ToArray();
}

Complexity

⏱️ Complexity

  • Time: O(n) — single in-order pass.
  • Space: O(h) — recursion (modes aside).

Pattern & takeaways

🎯 Remember these three

  • In-order makes duplicate values adjacent, so count runs.
  • Reset the mode list on a new max, append on a tie.
  • No hash map needed — the BST ordering does the grouping.
