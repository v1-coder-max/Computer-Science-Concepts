Target Sum

Assign + or − to each number to hit a target — count the ways. A clever algebra step turns it into subset-sum counting.

#dynamic-programming #subset-sum #medium #transform #counting #interview

The problem

Given nums and a target, assign + or − before each number so the expression equals target. Return the number of ways.

📥 Example

nums = [1,1,1,1,1], target = 3 → 5

Brute force

Recursively try +num and −num at each index, counting leaves that hit the target — O(2n). Memoizing on (index, runningSum) helps, but the algebra below makes it a clean subset-sum.

Optimized approach

Let P be the positive subset and N the negative one. Then P − N = target and P + N = total, so P = (total + target) / 2. The problem becomes: count subsets that sum to P. That’s a 1-D DP: dp[s] += dp[s − num] for each num, with dp[0] = 1. (If total + target is odd or negative, the answer is 0.)

Time O(n · P), Space O(P).

C# solution

C# 
public int FindTargetSumWays(int[] nums, int target)
{
    int total = nums.Sum();
    if (Math.Abs(target) > total || (total + target) % 2 != 0) return 0;
    int P = (total + target) / 2;          // subset that must be positive

    var dp = new int[P + 1];
    dp[0] = 1;
    foreach (int num in nums)
        for (int s = P; s >= num; s--)     // descend → each num used once
            dp[s] += dp[s - num];
    return dp[P];
}

Complexity

⏱️ Complexity

  • Time: O(n · P) where P = (total+target)/2.
  • Space: O(P).

Pattern & takeaways

🎯 Remember these three

  • Algebra reduces signed sums to a subset-sum count.
  • Iterate s downward so each number is used once (0/1 knapsack).
  • Guard the parity/range check — odd or out-of-range gives 0.
