Product of Array Except Self

For each index, return the product of all the other elements — in O(n), with no division. The trick is two sweeps: products from the left, then from the right.

#arrays #prefix-suffix #medium #no-division

The problem

Given an integer array nums, return an array answer where answer[i] equals the product of every element except nums[i]. Solve it in O(n) and without using division (which would fail on zeros anyway).

📥 Example

[1, 2, 3, 4] → [24, 12, 8, 6]

Brute force

For each i, multiply all other elements with an inner loop. Time O(n²), Space O(1).

Optimized approach

The product of everything except i is (product of everything to its left) × (product of everything to its right). So:

  1. Left pass: fill answer[i] with the running product of all elements before i.
  2. Right pass: walk back, multiplying each answer[i] by a running product of all elements after i.

Using the output array for the left products and a single variable for the right product keeps it O(1) extra space (the output doesn’t count).

C# solution

C# 
public int[] ProductExceptSelf(int[] nums)
{
    int n = nums.Length;
    var answer = new int[n];

    // Pass 1 (left): answer[i] = product of everything BEFORE i.
    answer[0] = 1;
    for (int i = 1; i < n; i++)
        answer[i] = answer[i - 1] * nums[i - 1];

    // Pass 2 (right): multiply by product of everything AFTER i (running variable).
    int right = 1;
    for (int i = n - 1; i >= 0; i--)
    {
        answer[i] *= right;
        right *= nums[i];
    }
    return answer;
}

Complexity

⏱️ Complexity

  • Time: O(n) — two linear passes.
  • Space: O(1) extra — the output array doesn’t count; only a single right variable.

Pattern & takeaways

🎯 Remember these three

  • The prefix/suffix product pattern: combine a left-to-right accumulation with a right-to-left one.
  • Reuse the output array for prefixes and a single variable for suffixes → O(1) extra space.
  • Division is banned because a single zero breaks it — prefix/suffix sidesteps that cleanly.

Interview tip

🗣️ When this comes up

The interviewer is testing whether you reach for prefix/suffix products rather than division. Mention that division fails on zeros, and that storing prefixes in the output + one suffix variable hits the coveted O(1) extra space.
