Merge Intervals

Collapse overlapping intervals into the fewest non-overlapping ones. The unlock is one word: sort by start. After that it’s a single sweep.

#intervals #sorting #sweep #medium

The problem

Given an array of intervals [start, end], merge all that overlap and return the resulting non-overlapping intervals.

📥 Example

[[1,3],[2,6],[8,10],[15,18]] → [[1,6],[8,10],[15,18]]

The idea

Once intervals are sorted by start, any interval that overlaps a given one must come immediately after it — there’s nothing earlier to worry about. So sweep left to right keeping the last interval in the result. For each next interval: if its start ≤ the last interval’s end, they overlap — extend the last end to max(end, next.end). Otherwise there’s a gap — push the next interval as a new result entry.

Time O(n log n) (the sort), Space O(n) for output.

C# solution

C# 
public int[][] Merge(int[][] intervals)
{
    Array.Sort(intervals, (a, b) => a[0] - b[0]);   // sort by start
    var result = new List<int[]>();

    foreach (var cur in intervals)
    {
        // no overlap with the last merged interval → start a new one
        if (result.Count == 0 || cur[0] > result[^1][1])
            result.Add(new[] { cur[0], cur[1] });
        else                                         // overlap → extend the end
            result[^1][1] = Math.Max(result[^1][1], cur[1]);
    }
    return result.ToArray();
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — dominated by the sort; the sweep is O(n).
  • Space: O(n) for the output (O(log n)–O(n) for the sort itself).

Pattern & takeaways

🎯 Remember these three

  • Almost every interval problem starts with sort by start (sometimes by end).
  • Overlap test: next.start ≤ last.end; merge by max-ing the ends.
  • This sweep generalizes to insert-interval, free-time, and meeting-room counting.
