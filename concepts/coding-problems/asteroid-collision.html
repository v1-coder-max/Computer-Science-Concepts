Asteroid Collision

Asteroids fly left or right; collisions destroy the smaller. A stack of surviving asteroids resolves each impact as it happens.

#stack #simulation #medium #arrays #collision #interview

The problem

Each value in asteroids is an asteroid: sign = direction (positive = right, negative = left), magnitude = size. Right and left movers collide; the smaller explodes (both if equal sizes). Return the survivors.

📥 Example

[5,10,-5] → [5,10]  ·  [8,-8] → []  ·  [10,2,-5] → [10]

Brute force

Repeatedly scan the array applying collisions until stable — O(n²) in the worst case. A single stack pass resolves all collisions in O(n).

Optimized approach

Process asteroids left to right with a stack of survivors. A right-moving asteroid is just pushed. A left-moving one (negative) collides with right-movers on top of the stack: while the top is a positive asteroid smaller than the incoming one, pop it (it explodes); if the top equals it, pop and stop (both gone); if the top is larger, the incoming one explodes. If no right-mover blocks it, push it. The stack ends as the surviving asteroids in order.

Time O(n), Space O(n).

C# solution

C# 
public int[] AsteroidCollision(int[] asteroids)
{
    var stack = new Stack<int>();
    foreach (int a in asteroids)
    {
        bool alive = true;
        while (alive && a < 0 && stack.Count > 0 && stack.Peek() > 0)
        {
            int top = stack.Peek();
            if (top < -a) stack.Pop();           // top explodes, keep checking
            else if (top == -a) { stack.Pop(); alive = false; }  // both explode
            else alive = false;                  // incoming explodes
        }
        if (alive) stack.Push(a);
    }
    var arr = stack.ToArray();
    Array.Reverse(arr);
    return arr;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each asteroid pushed/popped once.
  • Space: O(n) for the stack.

Pattern & takeaways

🎯 Remember these three

  • Collisions resolve against the top of a stack of survivors.
  • Only a left-mover meeting a right-mover collides (signs).
  • Three outcomes per impact: incoming wins, both die, or incoming dies.

Interview tip

🗣️ When this comes up

The stack models “what could still be hit from the left.” Enumerate the three collision outcomes carefully — that case analysis is where bugs creep in.
