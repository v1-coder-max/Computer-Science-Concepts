N-Queens

Place n queens so none attack. Go row by row, and track threatened columns and diagonals in sets for instant validity checks.

#backtracking #recursion #pruning #hard #constraints #interview

The problem

Place n queens on an n × n board so that no two attack each other (same row, column, or diagonal). Return all distinct solutions.

📥 Example

n = 4 → 2 distinct solutions

The idea

Exactly one queen goes per row, so recurse one row at a time, trying each column. A placement is safe if its column and both diagonals are free. Track three sets: used columns, used “↘” diagonals (constant row − col), and used “↙” diagonals (constant row + col). Each check and update is O(1); placing all n rows yields a solution.

Time O(n!), Space O(n).

C# solution

C# 
public IList<IList<string>> SolveNQueens(int n)
{
    var result = new List<IList<string>>();
    var cols  = new HashSet<int>();
    var diag1 = new HashSet<int>();   // row - col
    var diag2 = new HashSet<int>();   // row + col
    var pos = new int[n];
    Place(0);
    return result;

    void Place(int row)
    {
        if (row == n) { result.Add(Render()); return; }
        for (int col = 0; col < n; col++)
        {
            if (cols.Contains(col) || diag1.Contains(row - col) || diag2.Contains(row + col))
                continue;
            cols.Add(col); diag1.Add(row - col); diag2.Add(row + col); pos[row] = col;
            Place(row + 1);
            cols.Remove(col); diag1.Remove(row - col); diag2.Remove(row + col);
        }
    }

    IList<string> Render()
    {
        var board = new List<string>();
        foreach (int c in pos) board.Add(new string('.', c) + "Q" + new string('.', n - c - 1));
        return board;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n!) — pruned far below by the constraint sets.
  • Space: O(n) for the sets and recursion.

Pattern & takeaways

🎯 Remember these three

  • One queen per row reduces the search to column choices.
  • Diagonals are constant row−col and row+col — track them in sets.
  • O(1) constraint checks make the pruning aggressive.

Interview tip

🗣️ When this comes up

The clever bit is the diagonal encoding (row−col, row+col) giving O(1) attack checks. Place one queen per row to shrink the branching, and the rest is standard choose/explore/un-choose.
