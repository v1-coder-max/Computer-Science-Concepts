Assign Cookies

Sort both sides, then give the smallest cookie that satisfies the least-greedy remaining child. Greedy is optimal here.

#greedy #two-pointers #sorting #easy

The problem

Each child has a greed factor and each cookie a size; a child is content if a cookie's size ≥ their greed, one cookie per child. Maximize the number of content children.

📥 Example

greed = [1,2,3], sizes = [1,1] → 1

Brute force

Trying all assignments is factorial. Sorting plus a greedy two-pointer pass is optimal and O(n log n).

Optimized approach

Sort children by greed and cookies by size. Walk both with pointers: if the current cookie satisfies the current child, count it and advance both; otherwise the cookie is too small for everyone left — advance to a bigger cookie.

Time O(n log n), Space O(1).

C# solution

C# 
public int FindContentChildren(int[] g, int[] s)
{
    Array.Sort(g);
    Array.Sort(s);
    int child = 0, cookie = 0;
    while (child < g.Length && cookie < s.Length)
    {
        if (s[cookie] >= g[child]) child++;   // satisfied
        cookie++;
    }
    return child;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — dominated by the two sorts.
  • Space: O(1) — in-place sort.

Pattern & takeaways

🎯 Remember these three

  • Sort both, then match smallest adequate cookie to least greedy child.
  • If a cookie can't satisfy the current child, it can't satisfy any greedier one.
  • The exchange argument proves the greedy choice is optimal.

Interview tip

🗣️ When this comes up

Be ready to justify greediness with an exchange argument: swapping to a smaller satisfying cookie never hurts. The two-pointer sweep after sorting is the implementation.
