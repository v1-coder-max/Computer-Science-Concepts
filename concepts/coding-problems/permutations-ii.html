Permutations II

Permutations when the input has duplicates and the output must be unique. Sort, then skip a duplicate at the same level unless its twin was just used.

#backtracking #recursion #medium #dedupe #permutations #interview

The problem

Given a collection nums that may contain duplicates, return all unique permutations.

📥 Example

[1,1,2] → [[1,1,2],[1,2,1],[2,1,1]]

Brute force

Generate all n! permutations and dedupe with a set — wasteful and memory-heavy. Pruning duplicate branches during backtracking avoids generating repeats at all.

Optimized approach

Backtrack with a used[] array as in standard permutations, but first sort so equal values are adjacent. At each level, skip a value if it equals the previous value and that previous value is currently unused (i > 0 && nums[i] == nums[i-1] && !used[i-1]). That condition ensures duplicates are placed in a fixed left-to-right order, so each distinct permutation is produced once.

Time O(n · n!), Space O(n).

C# solution

C# 
public IList<IList<int>> PermuteUnique(int[] nums)
{
    Array.Sort(nums);
    var result = new List<IList<int>>();
    var used = new bool[nums.Length];
    Backtrack(new List<int>());
    return result;

    void Backtrack(List<int> path)
    {
        if (path.Count == nums.Length) { result.Add(new List<int>(path)); return; }
        for (int i = 0; i < nums.Length; i++)
        {
            if (used[i]) continue;
            if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) continue;   // dedupe
            used[i] = true; path.Add(nums[i]);
            Backtrack(path);
            used[i] = false; path.RemoveAt(path.Count - 1);
        }
    }
}

Complexity

⏱️ Complexity

  • Time: O(n · n!) worst case (fewer with duplicates).
  • Space: O(n) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Sort, then skip nums[i]==nums[i-1] && !used[i-1].
  • The !used[i-1] condition fixes duplicate ordering.
  • It’s Permutations plus the standard backtracking dedupe rule.

Interview tip

🗣️ When this comes up

The exact dedupe condition (!used[i-1]) trips people — explain that it forces equal values to be consumed in order, so you never build the same permutation twice.
