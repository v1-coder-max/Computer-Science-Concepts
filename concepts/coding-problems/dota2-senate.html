Dota2 Senate

Each senator bans the next opponent in line; simulate with two queues, and whoever acts first (smaller index) survives to the next round.

#greedy #queue #medium

The problem

Senators of two parties (R and D) vote in round-robin order. Each can ban the next opposing senator. Return which party ("Radiant" or "Dire") eventually wins.

📥 Example

"RDD" → "Dire"

Brute force

Simulating the string with deletions is O(n²). Two index queues give O(n).

Optimized approach

Put the indices of each party's senators into a queue. Repeatedly compare the two fronts: the senator with the smaller index acts first and bans the other (whose index is dropped); the winner re-queues with index + n for the next round. When one queue empties, the other party wins.

Time O(n), Space O(n).

C# solution

C# 
public string PredictPartyVictory(string senate)
{
    int n = senate.Length;
    var r = new Queue<int>();
    var d = new Queue<int>();
    for (int i = 0; i < n; i++)
        if (senate[i] == 'R') r.Enqueue(i); else d.Enqueue(i);
    while (r.Count > 0 && d.Count > 0)
    {
        int ri = r.Dequeue(), di = d.Dequeue();
        if (ri < di) r.Enqueue(ri + n);   // R bans D, survives
        else d.Enqueue(di + n);           // D bans R, survives
    }
    return r.Count > 0 ? "Radiant" : "Dire";
}

Complexity

⏱️ Complexity

  • Time: O(n) — each senator processed a constant number of times.
  • Space: O(n) — two queues.

Pattern & takeaways

🎯 Remember these three

  • Two index queues model the round-robin order.
  • The smaller front index acts first and bans the opponent.
  • Re-queue survivors with index + n for the next round.

🗣️ When this comes up

The two-queue simulation with the +n re-queue trick captures the round-robin cleanly. Whoever's index is smaller bans first — that greedy is provably optimal for each senator.
