Network Delay Time

A signal spreads from one node along weighted edges — when does the last node receive it? Shortest paths via Dijkstra, then take the max.

#graphs #dijkstra #shortest-path #medium #heap #interview

The problem

Given directed weighted edges times = [u, v, w], n nodes, and a source k, return the time for all nodes to receive a signal from k, or -1 if some node is unreachable.

📥 Example

times = [[2,1,1],[2,3,1],[3,4,1]], n = 4, k = 2 → 2

Brute force

Bellman-Ford relaxes all edges V−1 times — O(V·E), simple but slower. Repeated BFS ignoring weights is just wrong for weighted edges. Dijkstra is the right tool when weights are non-negative.

Optimized approach

Run Dijkstra from k: a min-heap of (distance, node), always expanding the closest unsettled node and relaxing its neighbors. The answer is the maximum of the final shortest distances (the slowest node to hear the signal); if any node is never settled, return -1.

Time O(E log V), Space O(V + E).

C# solution

C# 
public int NetworkDelayTime(int[][] times, int n, int k)
{
    var adj = new List<(int to, int w)>[n + 1];
    for (int i = 1; i <= n; i++) adj[i] = new();
    foreach (var t in times) adj[t[0]].Add((t[1], t[2]));

    var dist = new int[n + 1];
    Array.Fill(dist, int.MaxValue);
    dist[k] = 0;
    var pq = new PriorityQueue<int, int>();    // (node, distance)
    pq.Enqueue(k, 0);

    while (pq.TryDequeue(out int node, out int d))
    {
        if (d > dist[node]) continue;          // stale entry
        foreach (var (to, w) in adj[node])
            if (d + w < dist[to]) { dist[to] = d + w; pq.Enqueue(to, d + w); }
    }

    int ans = 0;
    for (int i = 1; i <= n; i++)
    {
        if (dist[i] == int.MaxValue) return -1;
        ans = Math.Max(ans, dist[i]);
    }
    return ans;
}

Complexity

⏱️ Complexity

  • Time: O(E log V) with a binary-heap Dijkstra.
  • Space: O(V + E) for the graph and heap.

Pattern & takeaways

🎯 Remember these three

  • Single-source shortest paths with non-negative weights → Dijkstra.
  • A min-heap always expands the nearest unsettled node.
  • The “all received” time is the max shortest distance.

Interview tip

🗣️ When this comes up

Recognize “shortest time to reach all” as single-source shortest path → Dijkstra, then max. Skip the stale-entry check and you get TLE on dense graphs — call it out.
