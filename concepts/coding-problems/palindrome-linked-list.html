Palindrome Linked List

Is the list a palindrome — using O(1) space? Find the middle, reverse the second half, and compare it against the first half.

#linked-list #two-pointers #easy #reverse #in-place #interview

The problem

Given the head of a singly linked list, return true if it reads the same forwards and backwards.

📥 Example

1→2→2→1 → true  ·  1→2→3 → false

Brute force

Copy all values into an array and check it with two pointers — O(n) time but O(n) space. The in-place approach below achieves O(1) extra space by reusing the list’s own nodes.

Optimized approach

Use slow/fast pointers to find the middle. Reverse the second half in place. Then walk the first half and the reversed second half together, comparing values — a mismatch means it’s not a palindrome. (Optionally restore the list by reversing the second half back.)

Time O(n), Space O(1).

C# solution

C# 
public bool IsPalindrome(ListNode head)
{
    // 1) find middle
    ListNode slow = head, fast = head;
    while (fast?.next != null) { slow = slow.next; fast = fast.next.next; }

    // 2) reverse second half
    ListNode prev = null, cur = slow;
    while (cur != null) { var nxt = cur.next; cur.next = prev; prev = cur; cur = nxt; }

    // 3) compare halves
    ListNode left = head, right = prev;
    while (right != null)
    {
        if (left.val != right.val) return false;
        left = left.next; right = right.next;
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n) — find, reverse, compare are each linear.
  • Space: O(1) — all in place.

Pattern & takeaways

🎯 Remember these three

  • Combine find-middle + reverse-half + compare for O(1) space.
  • The array copy is the easy O(n)-space fallback.
  • Reverse the half back if the caller needs the list intact.
