Largest Rectangle in Histogram

The biggest rectangle under a histogram. A monotonic increasing stack of bar indices lets each bar settle its widest rectangle exactly when a shorter bar appears.

#stack #monotonic-stack #arrays #hard #area #interview

The problem

Given bar heights heights, return the area of the largest rectangle that fits entirely within the histogram.

📥 Example

[2,1,5,6,2,3] → 10 (heights 5 and 6 over width 2)

Brute force

For each bar, expand left and right while bars are at least as tall, computing its maximal rectangle — O(n²). A monotonic stack does the same boundary-finding in one O(n) pass.

Optimized approach

Keep a stack of indices with increasing heights. When the current bar is shorter than the stack top, that top bar can’t extend further right — pop it and compute its rectangle: height is the popped bar’s height, and the width spans from the new stack top (its left boundary) to the current index (its right boundary). Push the current bar. Process a sentinel 0 at the end to flush the stack. Each bar is pushed and popped once.

Time O(n), Space O(n).

C# solution

C# 
public int LargestRectangleArea(int[] heights)
{
    var stack = new Stack<int>();    // indices of increasing heights
    int best = 0, n = heights.Length;

    for (int i = 0; i <= n; i++)
    {
        int h = i == n ? 0 : heights[i];                 // sentinel flushes the stack
        while (stack.Count > 0 && heights[stack.Peek()] > h)
        {
            int height = heights[stack.Pop()];
            int left = stack.Count == 0 ? -1 : stack.Peek();
            int width = i - left - 1;                    // bounded by left & i
            best = Math.Max(best, height * width);
        }
        stack.Push(i);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index pushed and popped once.
  • Space: O(n) for the stack.

Pattern & takeaways

🎯 Remember these three

  • A monotonic increasing stack finds each bar’s left/right boundaries in O(n).
  • On a pop, width = i − newTop − 1; height = popped bar.
  • A trailing sentinel 0 flushes any bars left on the stack.

Interview tip

🗣️ When this comes up

This is the hardest monotonic-stack staple. The key is what each pop computes — the popped bar’s maximal rectangle, bounded left by the new top and right by the current index. The sentinel is the clean way to flush.
