Binary Tree Maximum Path Sum

A path can bend at a node using both children, but only a single branch can extend to its parent. Return one thing, update another.

#trees #dfs #recursion #hard #global-update #interview

The problem

Given a binary tree, return the maximum path sum of any non-empty path. A path is a sequence of connected nodes; it need not pass through the root.

📥 Example

[-10,9,20,null,null,15,7] → 42 (15 → 20 → 7)

The idea

For each node, the best path that bends here is node.val + max(0, leftGain) + max(0, rightGain) — take a child branch only if it’s positive. That bent path can’t continue to the parent (it already used both sides), so update a global best with it. What you return to the parent is the best single-branch extension: node.val + max(0, max(leftGain, rightGain)).

Time O(n), Space O(h).

C# solution

C# 
public int MaxPathSum(TreeNode root)
{
    int best = int.MinValue;
    Gain(root);
    return best;

    int Gain(TreeNode node)
    {
        if (node == null) return 0;
        int left  = Math.Max(0, Gain(node.left));    // drop negative branches
        int right = Math.Max(0, Gain(node.right));
        best = Math.Max(best, node.val + left + right);   // path bending here
        return node.val + Math.Max(left, right);          // extend one branch up
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — single post-order pass.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Clamp negative gains to 0 — skip branches that only hurt.
  • Update the global best with the bent path; return the single-branch extension.
  • Same return-one/update-another shape as Diameter of Binary Tree.

Interview tip

🗣️ When this comes up

Two distinctions win it: clamping negative child gains to 0, and the difference between the value you record (bent path) and the value you return (one branch). It’s diameter with sums instead of lengths.
