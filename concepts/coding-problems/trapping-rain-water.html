Trapping Rain Water

How much rain sits between the bars? Each column holds min(tallest-to-the-left, tallest-to-the-right) − its own height. Two pointers compute it in one pass.

#two-pointers #arrays #prefix-max #hard

The problem

Given non-negative bar heights, compute how much water is trapped after raining. Water rests on top of a bar up to the lower of the tallest walls on either side.

📥 Example

[0,1,0,2,1,0,1,3,2,1,2,1] → 6

The key insight

Water above column i = min(maxLeft[i], maxRight[i]) − height[i] (clamped at 0). The water is bounded by the shorter of the two tallest walls flanking that column.

Brute force → prefix arrays → two pointers

Recomputing left/right maxima for every column is O(n²). Precomputing maxLeft and maxRight arrays drops it to O(n) time but O(n) space. The two-pointer trick removes the arrays: move from both ends tracking leftMax and rightMax; whichever side has the smaller running max is the binding constraint, so the water there is fixed — add it and step inward.

Time O(n), Space O(1).

C# solution

C# — two pointers 
public int Trap(int[] height)
{
    int left = 0, right = height.Length - 1;
    int leftMax = 0, rightMax = 0, water = 0;
    while (left < right)
    {
        if (height[left] < height[right])     // left wall is the smaller bound
        {
            leftMax = Math.Max(leftMax, height[left]);
            water += leftMax - height[left];   // water fixed by leftMax
            left++;
        }
        else                                  // right wall is the smaller bound
        {
            rightMax = Math.Max(rightMax, height[right]);
            water += rightMax - height[right];
            right--;
        }
    }
    return water;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass with two pointers.
  • Space: O(1) — no prefix arrays.

Pattern & takeaways

🎯 Remember these three

  • Per-column water = min(maxLeft, maxRight) − height.
  • The side with the smaller running max is the binding wall — its water is safe to commit.
  • Two pointers turn the O(n)-space prefix solution into O(1) space.

Interview tip

🗣️ When this comes up

Build up: state the per-column formula, give the prefix-array O(n) solution, then collapse to two pointers for O(1) space. Explaining why the smaller-max side is safe to settle is the part that earns the “hard” credit.
