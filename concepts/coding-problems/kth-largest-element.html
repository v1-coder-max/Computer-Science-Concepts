Kth Largest Element

Find the kth largest value without fully sorting. A size-k min-heap keeps only the top k candidates — its root is your answer — in O(n log k).

#heap #priority-queue #quickselect #medium

The problem

Given an integer array nums and an integer k, return the kth largest element (by value, in sorted order — not the kth distinct).

📥 Example

nums = [3,2,1,5,6,4], k = 2 → 5  ·  nums = [3,2,3,1,2,4,5,5,6], k = 4 → 4

Brute force

Sort descending and take index k-1 — O(n log n). Fine, but it does more work than needed: you don’t care about the order of the other n−k elements. A heap (or Quickselect) does less.

Optimized approach

Maintain a min-heap of size k. Push elements; whenever the heap exceeds k, pop the smallest. After processing everything, the heap holds the k largest values and its root is the smallest of those — exactly the kth largest. Each push/pop is O(log k), so total is O(n log k), and memory is just O(k). In C#, PriorityQueue<TElement,TPriority> is a min-heap by priority.

Time O(n log k), Space O(k). Quickselect averages O(n) (worst O(n²)).

C# solution

C# — min-heap 
public int FindKthLargest(int[] nums, int k)
{
    // PriorityQueue is a MIN-heap: lowest priority dequeues first.
    var heap = new PriorityQueue<int, int>();
    foreach (int n in nums)
    {
        heap.Enqueue(n, n);
        if (heap.Count > k) heap.Dequeue();   // drop the smallest, keep top k
    }
    return heap.Peek();                       // root = kth largest
}

// Quickselect alternative — O(n) average via partition around a pivot:
//   pick pivot, partition; if pivot lands at index (n-k) it's the answer,
//   else recurse into the side that contains it.

Complexity

⏱️ Complexity

  • Time: O(n log k) for the heap; Quickselect is O(n) average, O(n²) worst.
  • Space: O(k) for the heap; O(1) for in-place Quickselect.

Pattern & takeaways

🎯 Remember these three

  • “Top-k / kth largest” → a bounded heap of size k; keep the opposite extreme on the root.
  • For the k largest, use a min-heap (evict the smallest); flip for k smallest.
  • Quickselect gives O(n) average when you need raw speed and can sacrifice worst-case.

Interview tip

🗣️ When this comes up

Say “top-k → heap” immediately, and be precise that the k largest uses a min-heap. If pressed for better than O(n log k), bring up Quickselect and its O(n) average — naming the trade-off (worst-case O(n²)) shows depth.
