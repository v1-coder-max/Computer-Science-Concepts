Number of Connected Components

Count the separate pieces of a graph. Union-Find: start with n components and subtract one every time an edge merges two different sets.

#graphs #union-find #medium #components #dsu #interview

The problem

Given n nodes labelled 0..n-1 and a list of undirected edges, return the number of connected components.

📥 Example

n=5, edges=[[0,1],[1,2],[3,4]] → 2

Brute force

DFS/BFS from each unvisited node, counting how many traversals you start — O(n + e), perfectly fine. Union-Find is the alternative that also supports incremental edge additions.

Optimized approach

Initialize count = n (everyone isolated). For each edge, union its endpoints; if they were in different sets, the merge reduces the component count by 1. After processing all edges, count is the number of components. With path compression and union by rank it’s near-linear.

Time ~O((n + e)·α(n)), Space O(n).

C# solution

C# 
public int CountComponents(int n, int[][] edges)
{
    var parent = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
    int count = n;

    int Find(int x) => parent[x] == x ? x : (parent[x] = Find(parent[x]));

    foreach (var e in edges)
    {
        int a = Find(e[0]), b = Find(e[1]);
        if (a != b) { parent[a] = b; count--; }   // merged two components
    }
    return count;
}

Complexity

⏱️ Complexity

  • Time: ~O((n + e)·α(n)) with path compression.
  • Space: O(n).

Pattern & takeaways

🎯 Remember these three

  • Components via Union-Find: n minus successful merges.
  • A merge only counts when the two roots differ.
  • DFS/BFS counting unvisited starts is the equivalent O(n+e).

Interview tip

🗣️ When this comes up

Offer both DFS-counting and Union-Find. The DSU “n − merges” counter is elegant and extends to streaming edges. It’s the same machinery as Redundant Connection and Graph Valid Tree.
