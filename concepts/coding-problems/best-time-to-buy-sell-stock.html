Best Time to Buy and Sell Stock

Buy on one day, sell on a later day — maximize the profit. One pass, one insight: track the lowest price seen so far and the best profit against it.

#arrays #greedy #one-pass #easy

The problem

Given an array prices where prices[i] is the price on day i, choose a day to buy and a later day to sell to maximize profit. Return the max profit, or 0 if no profit is possible.

📥 Example

[7, 1, 5, 3, 6, 4] → 5  (buy at 1, sell at 6)  ·  [7, 6, 4, 3, 1] → 0

Brute force

Try every buy/sell pair (i < j) and track the best difference. Time O(n²), Space O(1).

Optimized approach

Walk left to right keeping minPrice, the cheapest price seen so far. At each day the best profit if you sell today is price − minPrice; keep the maximum of those. You never need to look back, because the best buy day is always the minimum price before today.

Time O(n), Space O(1). (It’s also a sliding window: minPrice is the window’s left, today is the right.)

C# solution

C# 
public int MaxProfit(int[] prices)
{
    int minPrice = int.MaxValue;   // cheapest buy-in seen so far
    int best = 0;

    foreach (int price in prices)
    {
        if (price < minPrice) minPrice = price;          // new best day to have bought
        else best = Math.Max(best, price - minPrice);    // profit if we sell today
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — a single pass.
  • Space: O(1) — two running integers.

Pattern & takeaways

🎯 Remember these three

  • Track a running minimum (the best buy) and compare today’s price against it for profit.
  • The constraint “sell after buy” is handled for free by scanning left to right.
  • It’s a greedy/one-pass pattern — close cousin of Kadane’s running-state idea.

🗣️ When this comes up
