Middle of the Linked List

Find the middle node in a single pass. Slow and fast pointers: when fast reaches the end, slow sits on the middle.

#linked-list #two-pointers #easy #fast-slow #middle #interview

The problem

Given the head of a singly linked list, return the middle node. If there are two middles, return the second one.

📥 Example

1→2→3→4→5 → node 3  ·  1→2→3→4→5→6 → node 4

Brute force

Count the length in one pass, then walk halfway in a second pass — correct but two passes. Fast/slow pointers do it in one.

Optimized approach

Advance slow by one and fast by two each step. When fast runs off the end (or fast.next is null), slow has traveled half as far and is on the middle node. Starting both at head and looping while fast and fast.next exist returns the second middle for even lengths.

Time O(n), Space O(1).

C# solution

C# 
public ListNode MiddleNode(ListNode head)
{
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null)
    {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • The fast/slow two-pointer finds the midpoint in one pass.
  • Loop guard fast != null && fast.next != null picks the second middle.
  • It’s the first step in palindrome-list and reorder-list.
