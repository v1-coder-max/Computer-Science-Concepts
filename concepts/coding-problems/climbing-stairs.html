Climbing Stairs

You take 1 or 2 steps at a time — how many ways to reach step n? The answer hides a familiar face: Fibonacci. It’s the perfect first dynamic-programming problem.

#dynamic-programming #fibonacci #recurrence #easy

The problem

You’re climbing a staircase of n steps. Each move you climb either 1 or 2 steps. In how many distinct ways can you reach the top?

📥 Example

n = 3 → 3: 1+1+1, 1+2, 2+1

Brute force

Recursively try both moves: ways(n) = ways(n-1) + ways(n-2). Correct, but the naive recursion recomputes the same subproblems and is O(2ⁿ) — exponential. The overlapping subproblems are the cue for dynamic programming.

Optimized approach

To stand on step n, your last move came from step n-1 (a 1-step) or step n-2 (a 2-step). So ways(n) = ways(n-1) + ways(n-2) — the Fibonacci recurrence. Build it bottom-up from ways(1)=1, ways(2)=2. You only ever need the last two values, so keep two rolling variables → O(1) space.

Time O(n), Space O(1).

C# solution

C# — O(1) space DP 
public int ClimbStairs(int n)
{
    if (n <= 2) return n;
    int prev = 1, curr = 2;          // ways(1), ways(2)
    for (int i = 3; i <= n; i++)
    {
        int next = prev + curr;      // ways(i) = ways(i-1) + ways(i-2)
        prev = curr;
        curr = next;
    }
    return curr;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass building up to n.
  • Space: O(1) — two rolling variables instead of a full table.

Pattern & takeaways

🎯 Remember these three

  • Overlapping subproblems + optimal substructure = dynamic programming.
  • Find the recurrence by asking “what was the last move?”
  • If a state depends only on the last k values, collapse the table to O(1) rolling variables.

Interview tip

🗣️ When this comes up

Recognizing this is Fibonacci is the whole point — say it, then derive the recurrence from “the last step was 1 or 2.” Show the O(1)-space rolling version; reducing a DP table to a couple of variables is a reusable trick interviewers love.
