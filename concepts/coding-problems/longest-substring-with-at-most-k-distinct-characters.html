Longest Substring with At Most K Distinct Characters

Grow a window greedily; whenever it holds too many distinct characters, shrink from the left until it is legal again.

#sliding-window #hash-map #strings #medium

The problem

Given a string s and integer k, return the length of the longest substring that contains at most k distinct characters.

📥 Example

s = "eceba", k = 2 → 3 ("ece")

Brute force

Checking all substrings is O(n²) (or worse with a per-substring distinct count). A window with a character-count map answers it in one pass.

Optimized approach

Expand the right edge and update a count map. When the number of distinct characters exceeds k, shrink from the left — decrementing counts and dropping characters that hit zero — until the window is valid. Track the best length.

Time O(n), Space O(k).

C# solution

C# 
public int LengthOfLongestSubstringKDistinct(string s, int k)
{
    var count = new Dictionary<char, int>();
    int left = 0, best = 0;
    for (int right = 0; right < s.Length; right++)
    {
        count[s[right]] = count.GetValueOrDefault(s[right]) + 1;
        while (count.Count > k)
        {
            if (--count[s[left]] == 0) count.Remove(s[left]);
            left++;
        }
        best = Math.Max(best, right - left + 1);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each character enters and leaves once.
  • Space: O(k) — the map holds ≤ k+1 keys.

Pattern & takeaways

🎯 Remember these three

  • A count map tracks distinct characters; remove a key when its count hits zero.
  • Shrink only while the window is invalid (distinct > k).
  • k = 2 is the Fruit Into Baskets special case.
