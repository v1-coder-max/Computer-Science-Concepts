Number Complement

Only the bits up to the highest set bit matter; build a mask of that width and XOR to flip them.

#bit-manipulation #easy

The problem

The complement of an integer flips all its bits within its binary length. Given n, return its complement.

📥 Example

5 (101) → 2 (010)

Brute force

Flipping bit by bit works; constructing a width mask and XOR-ing is cleaner and O(1).

Optimized approach

Find the number of bits in n and build a mask of that many 1s. XOR-ing n with the mask flips exactly those bits: complement = n ^ mask. A neat way to get the mask is to round n up to all-ones via bit smearing, or compute (1 << bitLength) − 1.

Time O(1), Space O(1).

C# solution

C# 
public int FindComplement(int num)
{
    int mask = num;
    mask |= mask >> 1;
    mask |= mask >> 2;
    mask |= mask >> 4;
    mask |= mask >> 8;
    mask |= mask >> 16;   // all 1s up to num's highest bit
    return num ^ mask;
}

Complexity

⏱️ Complexity

  • Time: O(1) — fixed number of bit operations.
  • Space: O(1) — constant.

Pattern & takeaways

🎯 Remember these three

  • Only bits within n's length are flipped.
  • Smear bits down to build an all-ones mask of the right width.
  • complement = n ^ mask.

🗣️ When this comes up

The bit-smearing mask construction is the elegant part — it turns n into all-ones up to its top bit so a single XOR flips exactly the relevant bits.
