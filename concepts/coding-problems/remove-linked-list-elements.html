Remove Linked List Elements

A dummy node in front of the list makes removing the head identical to removing any other node — no special cases.

#linked-list #easy

The problem

Given the head of a linked list and a value, remove all nodes equal to that value and return the new head.

📥 Example

[1,2,6,3,4,5,6], val = 6 → [1,2,3,4,5]

Brute force

You can special-case leading matches, but it is error-prone. A dummy head eliminates the asymmetry entirely.

Optimized approach

Create a dummy node pointing at the head. Walk with a previous pointer; when the next node matches the value, splice it out (prev.next = prev.next.next), otherwise advance. Return dummy.next.

Time O(n), Space O(1).

C# solution

C# 
public ListNode RemoveElements(ListNode head, int val)
{
    var dummy = new ListNode(0, head);
    var prev = dummy;
    while (prev.next != null)
    {
        if (prev.next.val == val) prev.next = prev.next.next;  // unlink
        else prev = prev.next;
    }
    return dummy.next;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — a couple of pointers.

Pattern & takeaways

🎯 Remember these three

  • A dummy head makes head deletion the same as any other.
  • Advance prev only when you don't delete — deleting keeps prev put.
  • Return dummy.next, since the real head may have changed.

Interview tip

🗣️ When this comes up

The dummy-head idiom is the reusable lesson — it removes the leading-node edge case across many linked-list problems. Don't advance prev after a deletion.
