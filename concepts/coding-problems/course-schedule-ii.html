Course Schedule II

Not just “can you finish” — return an actual order. Topological sort emits each course the moment its prerequisites are all done.

#graphs #topological-sort #kahn #medium #ordering #interview

The problem

Given numCourses and prerequisite pairs [course, prereq], return an ordering of courses to finish all of them, or an empty array if impossible (a cycle).

📥 Example

numCourses=4, prereqs=[[1,0],[2,0],[3,1],[3,2]] → [0,1,2,3]

Brute force

Try permutations of courses and check each respects all prerequisites — factorial time, hopeless. The dependency structure is a DAG, so a topological sort produces a valid order directly.

Optimized approach

Build the graph (prereq → course) and each node’s in-degree. Kahn’s algorithm: queue all courses with in-degree 0, and repeatedly dequeue one (append it to the order), decrementing its dependents’ in-degrees and enqueuing any that reach 0. If you output all numCourses courses, that order is valid; if fewer (a cycle remains), return empty.

Time O(V + E), Space O(V + E).

C# solution

C# 
public int[] FindOrder(int numCourses, int[][] prerequisites)
{
    var adj = new List<int>[numCourses];
    for (int i = 0; i < numCourses; i++) adj[i] = new();
    var indeg = new int[numCourses];
    foreach (var p in prerequisites) { adj[p[1]].Add(p[0]); indeg[p[0]]++; }

    var q = new Queue<int>();
    for (int i = 0; i < numCourses; i++) if (indeg[i] == 0) q.Enqueue(i);

    var order = new List<int>();
    while (q.Count > 0)
    {
        int c = q.Dequeue(); order.Add(c);
        foreach (int next in adj[c]) if (--indeg[next] == 0) q.Enqueue(next);
    }
    return order.Count == numCourses ? order.ToArray() : Array.Empty<int>();
}

Complexity

⏱️ Complexity

  • Time: O(V + E).
  • Space: O(V + E) for the graph and queue.

Pattern & takeaways

🎯 Remember these three

  • Ordering with dependencies → topological sort.
  • The dequeue sequence is the answer (vs the boolean in Course Schedule).
  • Fewer than V emitted ⇒ a cycle ⇒ return empty.
