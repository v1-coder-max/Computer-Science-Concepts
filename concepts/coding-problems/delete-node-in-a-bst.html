Delete Node in a BST

Remove a key from a BST and keep it a valid BST. Search down to the node, then handle 0, 1, or 2 children — the tricky case uses the in-order successor.

#trees #bst #recursion #medium #successor #interview

The problem

Given the root of a BST and a key, delete the node with that key (if present) and return the new root, keeping the BST property intact.

📥 Example

root=[5,3,6,2,4,null,7], key=3 → a valid BST without 3 (e.g. 4 replaces it).

Brute force

Flattening to a sorted list, removing the value, and rebuilding loses the structure and is O(n). A targeted BST deletion is O(h) and preserves the tree shape.

Optimized approach

Recurse using the BST ordering: go left if key < node.val, right if greater. When you find the node: if it has no child or one child, return that child (splice it out). If it has two children, find the in-order successor (smallest value in the right subtree), copy its value into the node, then recursively delete the successor from the right subtree. The successor has at most one child, so its removal is simple.

Time O(h), Space O(h).

C# solution

C# 
public TreeNode DeleteNode(TreeNode root, int key)
{
    if (root == null) return null;
    if (key < root.val) root.left = DeleteNode(root.left, key);
    else if (key > root.val) root.right = DeleteNode(root.right, key);
    else
    {
        if (root.left == null) return root.right;     // 0 or 1 child
        if (root.right == null) return root.left;
        TreeNode succ = root.right;                   // in-order successor
        while (succ.left != null) succ = succ.left;
        root.val = succ.val;                          // copy successor value
        root.right = DeleteNode(root.right, succ.val);// delete successor
    }
    return root;
}

Complexity

⏱️ Complexity

  • Time: O(h) — search plus successor walk.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Three deletion cases: 0/1 child (splice) and 2 children (successor).
  • In-order successor = leftmost node of the right subtree.
  • Copy the successor’s value, then delete the successor.
