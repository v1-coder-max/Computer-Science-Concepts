Cheapest Flights Within K Stops

Cheapest flight path with at most k stops. Plain Dijkstra ignores the stop limit — Bellman-Ford run exactly k+1 rounds respects it.

#graphs #bellman-ford #shortest-path #medium #bounded #interview

The problem

Given flights = [from, to, price], n cities, source src, destination dst, and k, return the cheapest price from src to dst using at most k stops, or -1.

📥 Example

n=4, flights=[[0,1,100],[1,2,100],[2,0,100],[1,3,600],[2,3,200]], src=0, dst=3, k=1 → 700

Brute force

DFS every path from src, tracking cost and stops — exponential in the number of paths. The stop limit makes greedy Dijkstra incorrect (the cheapest path may exceed k stops), so we need an edge-count-bounded method.

Optimized approach

Run Bellman-Ford for exactly k + 1 rounds (each round may add one more edge/stop). Crucially, relax against a snapshot of the previous round’s distances so a single round can’t chain multiple new edges. After k + 1 rounds, dist[dst] is the cheapest price within the stop budget.

Time O(k · E), Space O(V).

C# solution

C# 
public int FindCheapestPrice(int n, int[][] flights, int src, int dst, int k)
{
    var dist = new int[n];
    Array.Fill(dist, int.MaxValue);
    dist[src] = 0;

    for (int round = 0; round <= k; round++)
    {
        var snapshot = (int[])dist.Clone();        // freeze last round
        foreach (var f in flights)
        {
            int u = f[0], v = f[1], price = f[2];
            if (snapshot[u] != int.MaxValue && snapshot[u] + price < dist[v])
                dist[v] = snapshot[u] + price;
        }
    }
    return dist[dst] == int.MaxValue ? -1 : dist[dst];
}

Complexity

⏱️ Complexity

  • Time: O(k · E) — k+1 rounds over all edges.
  • Space: O(V) for distances and the snapshot.

Pattern & takeaways

🎯 Remember these three

  • A stop limit means bound the edge count → Bellman-Ford rounds.
  • Relax against a snapshot so one round adds at most one edge.
  • Plain Dijkstra is wrong here because the cheapest path may use too many stops.

Interview tip

🗣️ When this comes up

The snapshot is the subtle, must-mention detail — without it a single round chains multiple edges and breaks the stop bound. Explain why Dijkstra alone fails and Bellman-Ford-by-rounds fits.
