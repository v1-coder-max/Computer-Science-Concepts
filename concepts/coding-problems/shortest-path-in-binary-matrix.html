Shortest Path in Binary Matrix

Shortest clear path corner to corner, moving in 8 directions through 0-cells. Unit steps ⇒ BFS.

#graphs #bfs #grid #medium #shortest-path #interview

The problem

Given an n × n binary grid, return the length (in cells) of the shortest clear path from top-left to bottom-right moving 8-directionally through cells with value 0, or -1 if none exists.

📥 Example

[[0,0,0],[1,1,0],[1,1,0]] → 4

Brute force

DFS over all paths is exponential and doesn’t guarantee shortest. With unit-cost moves, BFS finds the shortest path and visits each cell once.

Optimized approach

If the start or end cell is blocked, return −1. BFS from (0,0), exploring all 8 neighbors, counting path length by level. Mark cells visited (or set them to 1) as you enqueue them so each is processed once. The first time you reach the bottom-right, its level is the answer.

Time O(n²), Space O(n²).

C# solution

C# 
public int ShortestPathBinaryMatrix(int[][] grid)
{
    int n = grid.Length;
    if (grid[0][0] == 1 || grid[n - 1][n - 1] == 1) return -1;

    var q = new Queue<(int r, int c)>();
    q.Enqueue((0, 0)); grid[0][0] = 1;        // mark visited
    int path = 1;
    int[][] dirs = { new[]{0,1}, new[]{1,0}, new[]{0,-1}, new[]{-1,0},
                     new[]{1,1}, new[]{1,-1}, new[]{-1,1}, new[]{-1,-1} };

    while (q.Count > 0)
    {
        for (int i = q.Count; i > 0; i--)
        {
            var (r, c) = q.Dequeue();
            if (r == n - 1 && c == n - 1) return path;
            foreach (var d in dirs)
            {
                int nr = r + d[0], nc = c + d[1];
                if (nr >= 0 && nc >= 0 && nr < n && nc < n && grid[nr][nc] == 0)
                { grid[nr][nc] = 1; q.Enqueue((nr, nc)); }
            }
        }
        path++;
    }
    return -1;
}

Complexity

⏱️ Complexity

  • Time: O(n²) — each cell visited once.
  • Space: O(n²) for the queue.

Pattern & takeaways

🎯 Remember these three

  • Unit-cost grid path → BFS (count levels = path length).
  • 8 directions, not 4 — include the diagonals.
  • Mark cells visited on enqueue to avoid re-processing.

Interview tip

🗣️ When this comes up

Standard grid BFS — the only twists are the 8 directions and counting cells (not edges). For weighted grids you’d switch to Dijkstra or 0-1 BFS; here unit cost keeps it plain BFS.
