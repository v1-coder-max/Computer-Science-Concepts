Add Two Numbers II

The digits arrive most-significant first, but addition needs the least-significant end. Stacks (or reversing) give you that order without modifying the inputs.

#linked-list #stack #math #medium

The problem

Two non-negative integers are stored as linked lists with the most significant digit first. Add them and return the sum as a linked list, again most-significant first.

📥 Example

7243 + 564 → [7,8,0,7]

Brute force

Reversing both lists, adding, and reversing the result works but mutates inputs. Pushing digits onto stacks reads them in reverse without changing the lists.

Optimized approach

Push every digit of each list onto a stack. Pop from both, add with a carry, and prepend each result digit to the output (building it most-significant-first). Continue while either stack has digits or a carry remains.

Time O(n), Space O(n).

C# solution

C# 
public ListNode AddTwoNumbers(ListNode l1, ListNode l2)
{
    var s1 = new Stack<int>();
    var s2 = new Stack<int>();
    for (; l1 != null; l1 = l1.next) s1.Push(l1.val);
    for (; l2 != null; l2 = l2.next) s2.Push(l2.val);
    ListNode head = null;
    int carry = 0;
    while (s1.Count > 0 || s2.Count > 0 || carry > 0)
    {
        int sum = carry;
        if (s1.Count > 0) sum += s1.Pop();
        if (s2.Count > 0) sum += s2.Pop();
        head = new ListNode(sum % 10, head);   // prepend
        carry = sum / 10;
    }
    return head;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass to load, one to add.
  • Space: O(n) — two stacks.

Pattern & takeaways

🎯 Remember these three

  • Stacks read digits least-significant-first without reversing the lists.
  • Prepend result digits to build the answer most-significant-first.
  • Keep going while either stack is non-empty or a carry is pending.

Interview tip

🗣️ When this comes up

Using stacks to avoid reversing the input lists is the elegant move. Prepending to the result (rather than appending) keeps the most-significant-first order for free.
