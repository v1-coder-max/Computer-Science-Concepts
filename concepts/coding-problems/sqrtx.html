Sqrt(x)

The integer square root of x (floor). Binary search for the largest m with m² ≤ x — O(log x).

#binary-search #math #easy #integer #overflow #interview

The problem

Given a non-negative integer x, return the floor of its square root (the largest integer whose square does not exceed x), without using built-in sqrt.

📥 Example

x=8 → 2  ·  x=16 → 4

Brute force

Increment m from 0 while m·m ≤ x — O(√x). Binary search over the candidate range cuts it to O(log x).

Optimized approach

The answer lies in [0, x] and the predicate “m² ≤ x” is monotonic, so binary search for the largest such m. Compare m·m with x using long (or compare m with x / m) to avoid overflow. Keep the best valid m as you narrow the range. (Newton’s method converges even faster.)

Time O(log x), Space O(1).

C# solution

C# 
public int MySqrt(int x)
{
    if (x < 2) return x;
    long lo = 1, hi = x, ans = 1;
    while (lo <= hi)
    {
        long mid = lo + (hi - lo) / 2;
        if (mid * mid <= x) { ans = mid; lo = mid + 1; }   // valid, try larger
        else hi = mid - 1;
    }
    return (int)ans;
}

Complexity

⏱️ Complexity

  • Time: O(log x).
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Binary search the monotonic predicate m² ≤ x.
  • Use long (or x / m) to dodge overflow.
  • Newton’s iteration is the faster numeric alternative.

Interview tip

🗣️ When this comes up

It’s binary-search-on-the-answer in miniature. The only gotcha is overflow in mid*mid — mention the long cast or the division comparison.
