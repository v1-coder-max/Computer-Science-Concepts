Validate Binary Search Tree

A BST isn’t just “left child smaller, right child bigger.” Every node must lie inside the range its ancestors allow. Carry a (min, max) bound down the tree.

#trees #bst #recursion #medium

The problem

Given the root of a binary tree, return true if it is a valid BST: for every node, all keys in its left subtree are strictly less, and all keys in its right subtree are strictly greater.

📥 Example

[2,1,3] → true  ·  [5,1,4,null,null,3,6] → false (4 < 5 but sits in the right subtree)

The naive trap

Comparing each node only to its direct children is wrong. In [5,1,4,null,null,3,6], node 4 is greater than its child 3, yet the tree is invalid because 4 is in 5’s right subtree and must exceed 5. A child can satisfy its parent and still violate a distant ancestor.

Optimized approach

Pass an allowed open interval (low, high) down each path. The root may be anything (-∞, +∞). Going left tightens the upper bound to the current value; going right tightens the lower bound. A node is valid only if low < node.val < high. Alternatively, an in-order traversal of a BST is strictly increasing — verify that.

Time O(n), Space O(h).

C# solution

C# 
public bool IsValidBST(TreeNode root) => Valid(root, long.MinValue, long.MaxValue);

private bool Valid(TreeNode node, long low, long high)
{
    if (node == null) return true;                 // empty subtree is valid
    if (node.val <= low || node.val >= high) return false;
    // left must stay below node.val; right must stay above it
    return Valid(node.left,  low, node.val)
        && Valid(node.right, node.val, high);
}
// (long bounds avoid edge cases when a node holds int.MinValue / int.MaxValue.)

Complexity

⏱️ Complexity

  • Time: O(n) — each node checked once against its bounds.
  • Space: O(h) recursion depth.

Pattern & takeaways

🎯 Remember these three

  • Validity is a range constraint inherited from ancestors, not a parent-child check.
  • Left tightens the max bound; right tightens the min bound.
  • Equivalent check: an in-order traversal must be strictly increasing.

Interview tip

🗣️ When this comes up

The trap answer is comparing only parent and child — call that out and explain the inherited-bounds fix. Mention the in-order alternative and the long sentinel trick for int.MinValue edge cases; that detail catches a lot of buggy submissions.
