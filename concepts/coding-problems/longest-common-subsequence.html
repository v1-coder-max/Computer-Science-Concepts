Longest Common Subsequence

The longest subsequence shared by two strings. A 2-D table where a match extends the diagonal and a mismatch takes the best of dropping one character.

#dynamic-programming #2d-dp #strings #medium #subsequence #interview

The problem

Given strings text1 and text2, return the length of their longest common subsequence (characters in order, not necessarily contiguous).

📥 Example

text1 = "abcde", text2 = "ace" → 3 ("ace")

Brute force

Recursively compare the two current characters: if equal, advance both; else try advancing each and take the max. That’s O(2m+n) with massive overlap. Memoizing on (i, j) already fixes it — the bottom-up table just makes it explicit.

Optimized approach

Let dp[i][j] = LCS of the first i chars of text1 and first j of text2. If the characters match, dp[i][j] = dp[i−1][j−1] + 1 (extend the diagonal). Otherwise dp[i][j] = max(dp[i−1][j], dp[i][j−1]) (drop a char from one side). Build up from the empty-string base of all zeros.

Time O(m·n), Space O(m·n) (reducible to O(min(m,n))).

C# solution

C# 
public int LongestCommonSubsequence(string a, string b)
{
    int m = a.Length, n = b.Length;
    var dp = new int[m + 1, n + 1];        // dp[i,j] over empty-prefixes = 0
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            dp[i, j] = a[i - 1] == b[j - 1]
                ? dp[i - 1, j - 1] + 1                          // match → diagonal
                : Math.Max(dp[i - 1, j], dp[i, j - 1]);         // else best neighbor
    return dp[m, n];
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — fill the table once.
  • Space: O(m·n), or O(min(m,n)) with two rolling rows.

Pattern & takeaways

🎯 Remember these three

  • Match → diagonal + 1; mismatch → max of the two neighbors.
  • Empty-prefix base cases are all zero.
  • It’s the template behind edit distance and diff tools.
