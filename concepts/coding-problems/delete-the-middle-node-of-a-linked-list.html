Delete the Middle Node of a Linked List

Send a fast pointer at double speed; when it reaches the end, slow sits just before the middle, ready to splice it out.

#linked-list #two-pointers #medium

The problem

Given the head of a linked list, delete the middle node (the ⌊n/2⌋-th, 0-indexed) and return the head.

📥 Example

[1,3,4,7,1,2,6] → [1,3,4,1,2,6] (delete 7)

Brute force

Counting the length first, then walking to length/2, takes two passes. Fast/slow finds the middle in one.

Optimized approach

Use a dummy before the head so slow can stop just before the middle. Advance fast by two and slow by one; when fast can't move twice more, slow.next is the middle — unlink it.

Time O(n), Space O(1).

C# solution

C# 
public ListNode DeleteMiddle(ListNode head)
{
    if (head.next == null) return null;
    var dummy = new ListNode(0, head);
    ListNode slow = dummy, fast = head;
    while (fast != null && fast.next != null)
    {
        slow = slow.next;
        fast = fast.next.next;
    }
    slow.next = slow.next.next;   // unlink the middle
    return head;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — two pointers.

Pattern & takeaways

🎯 Remember these three

  • Fast/slow finds the middle in one pass — no length count needed.
  • Start slow at a dummy so it lands just before the middle.
  • Handle the single-node case (returns null) separately.

Interview tip

🗣️ When this comes up

Offsetting slow by one (via a dummy) so it stops before the middle is the key detail that makes the deletion a one-liner. The single-node edge case is the other check.
