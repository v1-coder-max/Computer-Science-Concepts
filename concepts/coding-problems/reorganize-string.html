Reorganize String

Rearrange letters so no two neighbors match. Greedily place the most frequent remaining letter, never reusing the one you just placed — a max-heap drives it.

#heap #greedy #strings #medium #scheduling #interview

The problem

Given a string s, rearrange its characters so that no two adjacent characters are the same. Return any valid rearrangement, or "" if impossible.

📥 Example

"aab" → "aba"  ·  "aaab" → ""

Brute force

Trying permutations until one is valid is factorial. The greedy “always place the most frequent available, different from the last” strategy builds a valid arrangement directly when one exists.

Optimized approach

Count letter frequencies. If any letter occurs more than (n + 1) / 2 times, it’s impossible → "". Otherwise use a max-heap by remaining count. Repeatedly pop the most frequent letter that isn’t the one just placed, append it, decrement its count, and hold it aside until after the next placement (so it can’t be picked twice in a row). Re-add held letters with remaining count. This always succeeds when the feasibility check passes.

Time O(n log k), Space O(k).

C# solution

C# 
public string ReorganizeString(string s)
{
    var count = new Dictionary<char, int>();
    foreach (char c in s) count[c] = count.GetValueOrDefault(c) + 1;

    var heap = new PriorityQueue<char, int>();        // max-heap (negate counts)
    foreach (var (ch, cnt) in count)
    {
        if (cnt > (s.Length + 1) / 2) return "";       // infeasible
        heap.Enqueue(ch, -cnt);
    }

    var sb = new System.Text.StringBuilder();
    (char ch, int cnt)? hold = null;
    while (heap.Count > 0)
    {
        heap.TryDequeue(out char ch, out int negCnt);
        sb.Append(ch);
        if (hold is { cnt: > 0 }) heap.Enqueue(hold.Value.ch, -hold.Value.cnt);
        hold = (ch, -negCnt - 1);                       // hold this letter one round
    }
    return sb.ToString();
}

Complexity

⏱️ Complexity

  • Time: O(n log k) — heap ops per character.
  • Space: O(k) — at most 26 entries.

Pattern & takeaways

🎯 Remember these three

  • Greedy: place the most frequent letter that isn’t the previous.
  • Feasible iff no letter exceeds (n+1)/2.
  • “Hold one round” is the same idea as Task Scheduler’s cooldown.

Interview tip

🗣️ When this comes up

State the feasibility bound first, then the greedy heap with the one-round hold. It’s the string twin of Task Scheduler — spacing out the most frequent element.
