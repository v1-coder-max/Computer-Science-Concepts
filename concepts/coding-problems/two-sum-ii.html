Two Sum II - Input Array Is Sorted

Two numbers in a sorted array that add to a target. Two pointers at the ends move inward — sum too big, lower the right; too small, raise the left.

#two-pointers #arrays #easy #sorted #complement #interview

The problem

Given a 1-indexed sorted array numbers and a target, return the indices of the two numbers that add to target (exactly one solution; no reuse).

📥 Example

numbers = [2,7,11,15], target = 9 → [1,2]

Brute force

Check every pair — O(n²). A hash map gives O(n) time but O(n) space and ignores the sortedness. Since the array is sorted, two pointers achieve O(n) time and O(1) space.

Optimized approach

Place left at the start and right at the end. If numbers[left] + numbers[right] equals the target, return both indices. If the sum is too large, decrement right (the only way to shrink it); if too small, increment left. The pointers converge and you never miss the pair.

Time O(n), Space O(1).

C# solution

C# 
public int[] TwoSum(int[] numbers, int target)
{
    int left = 0, right = numbers.Length - 1;
    while (left < right)
    {
        int sum = numbers[left] + numbers[right];
        if (sum == target) return new[] { left + 1, right + 1 };   // 1-indexed
        if (sum < target) left++;       // need a bigger sum
        else right--;                   // need a smaller sum
    }
    return Array.Empty<int>();
}

Complexity

⏱️ Complexity

  • Time: O(n) — pointers meet after n steps.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • A sorted array unlocks the opposite-end two-pointer sum.
  • Move the side that fixes the sum’s direction — no guessing.
  • Beats the hash-map version on space; it’s the basis for 3Sum/4Sum.
