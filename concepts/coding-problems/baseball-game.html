Baseball Game

A stack of recent scores makes every operation — record, cancel, double, or sum the last two — a constant-time tweak.

#stack #simulation #easy

The problem

Process a list of operations: an integer records that score; "C" cancels the last; "D" records double the last; "+" records the sum of the last two. Return the total of all recorded scores.

📥 Example

["5","2","C","D","+"] → 30 (5, 10, 15)

Brute force

There is no real alternative — the operations reference the most recent scores, which is exactly what a stack provides in O(1).

Optimized approach

Maintain a stack of scores. A number pushes; "C" pops; "D" pushes twice the top; "+" pushes the sum of the top two. The answer is the sum of the stack at the end.

Time O(n), Space O(n).

C# solution

C# 
public int CalPoints(string[] operations)
{
    var stack = new List<int>();
    foreach (var op in operations)
    {
        int n = stack.Count;
        if (op == "C") stack.RemoveAt(n - 1);
        else if (op == "D") stack.Add(2 * stack[n - 1]);
        else if (op == "+") stack.Add(stack[n - 1] + stack[n - 2]);
        else stack.Add(int.Parse(op));
    }
    int sum = 0;
    foreach (int x in stack) sum += x;
    return sum;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass over the operations.
  • Space: O(n) — the score stack.

Pattern & takeaways

🎯 Remember these three

  • A stack naturally supports operations that reference recent values.
  • "+" reads the top two without popping them.
  • Sum the stack at the end for the answer.
