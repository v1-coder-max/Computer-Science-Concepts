Fruit Into Baskets

Pick the longest stretch of trees with at most two fruit types (two baskets). It’s the longest subarray with ≤ 2 distinct values — a sliding window.

#sliding-window #arrays #medium #two-distinct #hashing #interview

The problem

Given fruits where each value is a tree’s fruit type, you have two baskets each holding one type. Return the maximum number of fruits you can collect in a contiguous run.

📥 Example

[1,2,3,2,2] → 4 (the run [2,3,2,2])

Brute force

Check every subarray for ≤ 2 distinct types — O(n²). A sliding window with a frequency map tracks distinct count in O(1) per step for O(n) total.

Optimized approach

Slide a window keeping a count map of fruit types inside it. Expand right, adding the new type. While the map has more than 2 distinct types, shrink from left, decrementing counts and removing a type when its count hits 0. Track the maximum window length.

Time O(n), Space O(1) (≤ 3 map entries).

C# solution

C# 
public int TotalFruit(int[] fruits)
{
    var count = new Dictionary<int, int>();
    int left = 0, best = 0;
    for (int right = 0; right < fruits.Length; right++)
    {
        count[fruits[right]] = count.GetValueOrDefault(fruits[right]) + 1;
        while (count.Count > 2)                  // more than two types
        {
            int f = fruits[left++];
            if (--count[f] == 0) count.Remove(f);
        }
        best = Math.Max(best, right - left + 1);
    }
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(n).
  • Space: O(1) — at most 3 distinct types in the map.

Pattern & takeaways

🎯 Remember these three

  • It’s “longest subarray with at most 2 distinct values.”
  • Shrink when the distinct-type count exceeds 2.
  • Generalizes to “at most k distinct” by changing the bound.
