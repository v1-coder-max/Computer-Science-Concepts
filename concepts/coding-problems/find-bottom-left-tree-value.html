Find Bottom Left Tree Value

The answer is the first node you process on the last level. A left-to-right BFS hands it to you directly.

#trees #bfs #easy

The problem

Given a binary tree, return the leftmost value on its deepest level.

📥 Example

Tree [1,2,3,4,null,5,6,null,null,7] → 7

Brute force

A DFS tracking the maximum depth and the first node seen at each new depth works too, but BFS reads off the answer with no extra bookkeeping.

Optimized approach

BFS level by level, left to right. The first node of the last level processed is the bottom-left value — so just remember the front of each level, and the last one wins.

Time O(n), Space O(w).

C# solution

C# 
public int FindBottomLeftValue(TreeNode root)
{
    var q = new Queue<TreeNode>();
    q.Enqueue(root);
    int leftmost = root.val;
    while (q.Count > 0)
    {
        int size = q.Count;
        for (int i = 0; i < size; i++)
        {
            var node = q.Dequeue();
            if (i == 0) leftmost = node.val;        // front of this level
            if (node.left != null) q.Enqueue(node.left);
            if (node.right != null) q.Enqueue(node.right);
        }
    }
    return leftmost;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node enqueued once.
  • Space: O(w) — w = widest level.

Pattern & takeaways

🎯 Remember these three

  • BFS left-to-right makes the front of the last level the answer.
  • Record the first node of each level; the final one is bottom-left.
  • Enqueue left before right to preserve left-to-right order.

Interview tip

🗣️ When this comes up

Enqueue left child before right and grab the first node of each level — the last level's first node is the answer. It is the cleanest of the BFS family.
