Number of Islands

Count clumps of land in a grid of water and land. Each clump is a connected component — find one cell, flood-fill the whole island, and move on.

#graphs #grid #dfs #medium

The problem

Given an m × n grid of '1' (land) and '0' (water), count the islands. An island is land connected 4-directionally (up/down/left/right), surrounded by water.

📥 Example

11000 / 11000 / 00100 / 00011 → 3 islands.

The idea

Treat the grid as a graph: each land cell is a node with edges to its land neighbors. Scan every cell; when you hit an unvisited '1', that’s a new island, so increment the count and flood the entire connected blob (DFS or BFS), marking each visited land cell so it’s never counted again (overwrite to '0' or use a visited set). Every island is discovered exactly once, from its first-encountered cell.

Time O(m·n), Space O(m·n) worst case (recursion/queue).

C# solution

C# — DFS flood fill 
public int NumIslands(char[][] grid)
{
    int rows = grid.Length, cols = grid[0].Length, count = 0;
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            if (grid[r][c] == '1') { count++; Sink(grid, r, c); }
    return count;
}

private void Sink(char[][] grid, int r, int c)
{
    if (r < 0 || c < 0 || r >= grid.Length || c >= grid[0].Length || grid[r][c] != '1')
        return;                         // off-grid or water/visited
    grid[r][c] = '0';                   // mark visited
    Sink(grid, r + 1, c);  Sink(grid, r - 1, c);
    Sink(grid, r, c + 1);  Sink(grid, r, c - 1);   // 4 directions
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — each cell is visited a constant number of times.
  • Space: O(m·n) worst case for the recursion stack / BFS queue.

Pattern & takeaways

🎯 Remember these three

  • A grid is a graph; “count regions” = count connected components.
  • Mark visited (sink to '0' or a visited set) or the flood will loop forever.
  • DFS and BFS both work; DFS recursion is shortest to write, BFS avoids deep stacks.

Interview tip

🗣️ When this comes up

Frame it as connected components immediately. The single most common bug is forgetting to mark cells visited — state your marking strategy up front. If mutating the input is off-limits, use a separate visited grid and say so.
