Trim a Binary Search Tree

Out-of-range nodes take an entire subtree with them. The BST ordering tells you which side is safe to keep recursing into.

#trees #bst #recursion #medium

The problem

Given a BST and a range [low, high], remove every node outside the range and return the root of the trimmed tree (keeping the relative structure).

📥 Example

Trim [3,0,4,null,2,null,null,1] to [1,3] → [3,2,null,1]

Brute force

Deleting out-of-range nodes one at a time with full BST deletes is correct but slow and fiddly. Recursive trimming handles it in one pass.

Optimized approach

Recurse. If a node is below low, its entire left subtree is also out of range, so return the trimmed right subtree. If above high, return the trimmed left. Otherwise keep the node and trim both children.

Time O(n), Space O(h).

C# solution

C# 
public TreeNode TrimBST(TreeNode root, int low, int high)
{
    if (root == null) return null;
    if (root.val < low) return TrimBST(root.right, low, high);
    if (root.val > high) return TrimBST(root.left, low, high);
    root.left = TrimBST(root.left, low, high);
    root.right = TrimBST(root.right, low, high);
    return root;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) — recursion = height.

Pattern & takeaways

🎯 Remember these three

  • Below low ⇒ skip left subtree (recurse right); above high ⇒ the reverse.
  • The BST property guarantees the discarded side has no in-range nodes.
  • Reassigning root.left/right from the recursion rewires the tree cleanly.

Interview tip

🗣️ When this comes up

The pruning insight — an out-of-range node's near subtree is also out of range — is the BST idea being tested. Returning the recursive result to rewire pointers keeps it tidy.
