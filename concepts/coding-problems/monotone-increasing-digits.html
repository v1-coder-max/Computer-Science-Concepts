Monotone Increasing Digits

Scan for the first place a digit drops below its neighbor, decrement there, and set everything after to 9 to maximize what remains.

#greedy #math #medium

The problem

Given a non-negative integer n, return the largest number ≤ n whose digits are monotone increasing (each digit ≤ the next).

📥 Example

332 → 299; 1234 → 1234

Brute force

Counting down from n and checking each is far too slow. A digit-by-digit greedy is O(number of digits).

Optimized approach

Convert to digits. Find the first index where a digit exceeds the next; decrement that digit and remember the position. After processing, set every digit after that position to 9 (greedily maximizing). Process right-to-left so cascading decrements are handled.

Time O(d), Space O(d).

C# solution

C# 
public int MonotoneIncreasingDigits(int n)
{
    var d = n.ToString().ToCharArray();
    int marker = d.Length;
    for (int i = d.Length - 1; i > 0; i--)
    {
        if (d[i - 1] > d[i])
        {
            d[i - 1]--;        // lower the offending digit
            marker = i;        // 9s start here
        }
    }
    for (int i = marker; i < d.Length; i++) d[i] = '9';
    return int.Parse(new string(d));
}

Complexity

⏱️ Complexity

  • Time: O(d) — d = number of digits.
  • Space: O(d) — the digit array.

Pattern & takeaways

🎯 Remember these three

  • At the first drop, decrement and flood the rest with 9s.
  • Scan right-to-left so a decrement can cascade left.
  • Greedily maximizing the suffix (all 9s) is what keeps the result largest.

Interview tip

🗣️ When this comes up

The cascade is the subtle part: lowering a digit can create a new drop to its left, so scan right-to-left and track where the trailing 9s begin.
