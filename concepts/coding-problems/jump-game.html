Jump Game

Each number is the max jump length from that spot. Can you reach the end? A greedy scan tracking the farthest reachable index answers it in one pass.

#greedy #arrays #dynamic-programming #medium

The problem

Given an array nums where nums[i] is the maximum jump length from index i, return true if you can reach the last index starting from index 0.

📥 Example

[2,3,1,1,4] → true  ·  [3,2,1,0,4] → false (stuck at the 0)

Brute force

Recursively try every jump length from each index — exponential. A DP marking each index reachable is O(n²). Both do more work than needed: a single greedy quantity captures everything.

Optimized approach

Track farthest — the rightmost index reachable so far. Scan left to right; at index i, if i > farthest you can never get here, so return false. Otherwise update farthest = max(farthest, i + nums[i]). If farthest ever reaches the last index, return true. One pass, one variable.

Time O(n), Space O(1).

C# solution

C# — greedy 
public bool CanJump(int[] nums)
{
    int farthest = 0;
    for (int i = 0; i < nums.Length; i++)
    {
        if (i > farthest) return false;               // this index is unreachable
        farthest = Math.Max(farthest, i + nums[i]);   // extend our reach
        if (farthest >= nums.Length - 1) return true; // can already reach the end
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n) — single pass.
  • Space: O(1) — one running maximum.

Pattern & takeaways

🎯 Remember these three

  • Track a single farthest-reachable value instead of enumerating jumps.
  • Failure ⇔ the current index exceeds the farthest reach.
  • A clean greedy invariant often replaces an O(n²) reachability DP.
