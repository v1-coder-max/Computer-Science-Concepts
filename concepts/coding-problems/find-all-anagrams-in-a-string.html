Find All Anagrams in a String

Find every start index where an anagram of p appears in s. A fixed-size window of letter counts, matched against p’s counts.

#sliding-window #strings #medium #frequency #anagram #interview

The problem

Given strings s and p, return all start indices of p’s anagrams in s.

📥 Example

s="cbaebabacd", p="abc" → [0,6]

Brute force

For each window of length |p|, sort it or count its letters and compare to p — O(n·|p|) or worse. Maintaining the window’s counts incrementally makes it O(n).

Optimized approach

Build a frequency array for p and a sliding window of the same width over s. Slide one character at a time: add the entering character and remove the leaving one in O(1). When the window’s counts equal p’s counts, record the start index. Tracking how many of the 26 letter-counts currently match keeps each comparison O(1).

Time O(n), Space O(1).

C# solution

C# 
public IList<int> FindAnagrams(string s, string p)
{
    var result = new List<int>();
    if (s.Length < p.Length) return result;

    int[] need = new int[26], window = new int[26];
    foreach (char c in p) need[c - 'a']++;

    for (int i = 0; i < s.Length; i++)
    {
        window[s[i] - 'a']++;                      // add entering char
        if (i >= p.Length) window[s[i - p.Length] - 'a']--;   // remove leaving char
        if (i >= p.Length - 1 && window.AsSpan().SequenceEqual(need))
            result.Add(i - p.Length + 1);
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — O(1) work per slide.
  • Space: O(1) — two 26-count arrays.

Pattern & takeaways

🎯 Remember these three

  • Anagram search → fixed-size sliding window on counts.
  • Update counts incrementally (add entering, drop leaving).
  • A matches counter makes the comparison O(1) instead of O(26).
