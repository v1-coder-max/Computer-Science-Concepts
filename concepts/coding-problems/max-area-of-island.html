Max Area of Island

Number of Islands, but instead of counting blobs you measure them — flood-fill each and keep the biggest area.

#graphs #dfs #matrix #medium

The problem

Given a binary grid, return the area (cell count) of the largest island, where an island is a group of 1s connected horizontally or vertically.

📥 Example

A grid of scattered 1s → largest island area 4

Brute force

There is no avoiding visiting each cell, but a naive re-scan per island would repeat work. A single DFS/BFS that marks visited cells keeps it linear in the grid size.

Optimized approach

Scan the grid; at each unvisited land cell, flood-fill (DFS or BFS) its whole connected region, counting cells and marking them visited so they aren't recounted. Track the maximum region size.

Time O(m·n), Space O(m·n).

C# solution

C# 
public int MaxAreaOfIsland(int[][] grid)
{
    int rows = grid.Length, cols = grid[0].Length, best = 0;
    int Dfs(int r, int c)
    {
        if (r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] == 0) return 0;
        grid[r][c] = 0;   // mark visited
        return 1 + Dfs(r + 1, c) + Dfs(r - 1, c) + Dfs(r, c + 1) + Dfs(r, c - 1);
    }
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            if (grid[r][c] == 1) best = Math.Max(best, Dfs(r, c));
    return best;
}

Complexity

⏱️ Complexity

  • Time: O(m·n) — each cell visited once.
  • Space: O(m·n) — recursion/visited worst case.

Pattern & takeaways

🎯 Remember these three

  • Flood-fill counts an island's area; track the running maximum.
  • Mark cells visited (sink them to 0) to avoid recounting.
  • DFS returns 1 + the areas of its four neighbors — clean and recursive.
