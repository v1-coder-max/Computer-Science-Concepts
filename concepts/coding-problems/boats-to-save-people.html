Boats to Save People

Sort the weights, then try to seat the lightest person with the heaviest. If they fit together, both go; if not, the heaviest goes alone.

#greedy #two-pointers #sorting #medium

The problem

Each boat carries at most two people within a weight limit. Given everyone's weights, return the minimum number of boats needed.

📥 Example

people = [3,2,2,1], limit = 3 → 3

Brute force

Trying all pairings is exponential. Sorting plus a greedy two-pointer pairing is optimal and O(n log n).

Optimized approach

Sort the weights. Put pointers at the lightest and heaviest. If they fit together within the limit, board both (advance both pointers); otherwise the heaviest takes a boat alone (advance only the right pointer). Each step uses one boat.

Time O(n log n), Space O(1).

C# solution

C# 
public int NumRescueBoats(int[] people, int limit)
{
    Array.Sort(people);
    int l = 0, r = people.Length - 1, boats = 0;
    while (l <= r)
    {
        if (people[l] + people[r] <= limit) l++;  // lightest boards too
        r--;                                       // heaviest always boards
        boats++;
    }
    return boats;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — dominated by the sort.
  • Space: O(1) — in-place sort.

Pattern & takeaways

🎯 Remember these three

  • Pair the lightest with the heaviest after sorting.
  • The heaviest always boards; the lightest joins only if they fit together.
  • Each iteration consumes exactly one boat.
