Minimum Path Sum

Cheapest path through a grid moving only right or down. Each cell’s best cost is its value plus the smaller of the top and left neighbors.

#dynamic-programming #2d-dp #grid #medium #paths #interview

The problem

Given an m × n grid of non-negative numbers, find a path from top-left to bottom-right that minimizes the sum of values along it, moving only right or down. Return that sum.

📥 Example

[[1,3,1],[1,5,1],[4,2,1]] → 7 (1→3→1→1→1)

Brute force

Recursively branch right and down, taking the min — exponential and re-solving the same cells. The overlapping subproblems collapse into a grid DP.

Optimized approach

Let dp[r][c] = minimum cost to reach that cell. It equals grid[r][c] + min(dp[r−1][c], dp[r][c−1]), with the first row/column accumulating along the only path into them. Fill row by row; the answer is dp[m−1][n−1]. Since each row depends only on the previous, a single rolling row gives O(n) space (or update the grid in place).

Time O(m·n), Space O(n).

C# solution

C# 
public int MinPathSum(int[][] grid)
{
    int m = grid.Length, n = grid[0].Length;
    var dp = new int[n];
    dp[0] = grid[0][0];
    for (int c = 1; c < n; c++) dp[c] = dp[c - 1] + grid[0][c];   // first row

    for (int r = 1; r < m; r++)
    {
        dp[0] += grid[r][0];                                       // first column
        for (int c = 1; c < n; c++)
            dp[c] = grid[r][c] + Math.Min(dp[c], dp[c - 1]);       // top vs left
    }
    return dp[n - 1];
}

Complexity

⏱️ Complexity

  • Time: O(m·n).
  • Space: O(n) with a rolling row (O(1) in place).

Pattern & takeaways

🎯 Remember these three

  • Grid min-cost path: value + min(top, left).
  • Initialize the first row/column as cumulative sums.
  • Collapse to one rolling row for O(n) space.
