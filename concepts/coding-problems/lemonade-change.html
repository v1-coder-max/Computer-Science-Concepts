Lemonade Change

Every drink costs $5. The only question is whether you always have the right bills on hand — and the greedy rule is to hoard your $5s.

#greedy #arrays #simulation #easy

The problem

Customers pay with a $5, $10, or $20 bill for a $5 lemonade, in order. Return true if you can give every customer correct change starting with no money.

📥 Example

[5,5,5,10,20] → true

Brute force

There is no search space worth exploring — but the greedy choice (prefer giving a $10 over two $5s when making $15) is the non-obvious part that makes it always work.

Optimized approach

Track how many $5 and $10 bills you hold. A $10 needs one $5 back; a $20 prefers a $10 + $5, falling back to three $5s. Spending $10s first keeps the flexible $5s in reserve.

Time O(n), Space O(1).

C# solution

C# 
public bool LemonadeChange(int[] bills)
{
    int five = 0, ten = 0;
    foreach (int b in bills)
    {
        if (b == 5) five++;
        else if (b == 10) { if (five == 0) return false; five--; ten++; }
        else
        {
            if (ten > 0 && five > 0) { ten--; five--; }
            else if (five >= 3) five -= 3;
            else return false;
        }
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass over the customers.
  • Space: O(1) — two bill counters.

Pattern & takeaways

🎯 Remember these three

  • Greedy works because $5 bills are the most flexible — spend bigger bills first.
  • For $20, prefer $10+$5 over three $5s to preserve change-making power.
  • You never need to count $20 bills; they are never given as change.

Interview tip

🗣️ When this comes up

Justify the greedy: handing back a $10 before two $5s keeps maximum flexibility. Interviewers want the argument, not just the simulation.
