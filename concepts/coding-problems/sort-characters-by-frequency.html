Sort Characters By Frequency

Count each character, then emit them most-frequent first — a bucket indexed by frequency does it in linear time.

#hash-map #heap #sorting #medium

The problem

Given a string, return it with characters sorted in decreasing order of frequency (ties in any order).

📥 Example

"tree" → "eert" (or "eetr")

Brute force

Sorting characters by count is O(n log n). Bucketing by frequency achieves O(n).

Optimized approach

Count each character's frequency. Place characters into buckets indexed by frequency, then read buckets from highest frequency down, appending each character that many times. (A max-heap by count is the O(n log k) alternative.)

Time O(n), Space O(n).

C# solution

C# 
public string FrequencySort(string s)
{
    var count = new Dictionary<char, int>();
    foreach (char c in s) count[c] = count.GetValueOrDefault(c) + 1;
    var buckets = new List<char>[s.Length + 1];
    foreach (var kv in count)
        (buckets[kv.Value] ??= new List<char>()).Add(kv.Key);
    var sb = new StringBuilder();
    for (int f = s.Length; f >= 1; f--)
        if (buckets[f] != null)
            foreach (char c in buckets[f]) sb.Append(c, f);
    return sb.ToString();
}

Complexity

⏱️ Complexity

  • Time: O(n) — count + bucket read.
  • Space: O(n) — count map and buckets.

Pattern & takeaways

🎯 Remember these three

  • Bucket by frequency for an O(n) sort (frequencies are bounded by n).
  • Read buckets high-to-low to order by descending count.
  • A max-heap by count is the standard O(n log k) alternative.
