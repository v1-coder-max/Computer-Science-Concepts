Basic Calculator II

Evaluate 3+2*2 respecting precedence, no parentheses. A stack applies * and / immediately and defers +/− as signed terms.

#stack #strings #medium #parsing #precedence #interview

The problem

Given a string expression of non-negative integers and the operators + − * / (integer division truncates toward zero, no parentheses), evaluate it.

📥 Example

"3+2*2" → 7  ·  " 3/2 " → 1  ·  "3+5/2" → 5

Brute force

Two passes (resolve *// first, then +/−) work but require an intermediate structure. A single stack pass with a “previous operator” handles precedence cleanly.

Optimized approach

Scan, building each number. When you hit an operator (or the end), act on the previous operator: for + push the number, for − push its negation, for * or / pop the top and push the product/quotient. Higher-precedence *// thus combine with the immediate top, while +/− terms just accumulate. The answer is the sum of the stack.

Time O(n), Space O(n).

C# solution

C# 
public int Calculate(string s)
{
    var stack = new Stack<int>();
    int num = 0;
    char op = '+';
    for (int i = 0; i < s.Length; i++)
    {
        char c = s[i];
        if (char.IsDigit(c)) num = num * 10 + (c - '0');
        if ((!char.IsDigit(c) && c != ' ') || i == s.Length - 1)
        {
            switch (op)
            {
                case '+': stack.Push(num); break;
                case '-': stack.Push(-num); break;
                case '*': stack.Push(stack.Pop() * num); break;
                case '/': stack.Push(stack.Pop() / num); break;
            }
            op = c; num = 0;
        }
    }
    return stack.Sum();
}

Complexity

⏱️ Complexity

  • Time: O(n) — one pass.
  • Space: O(n) for the stack.

Pattern & takeaways

🎯 Remember these three

  • Act on the previous operator when the next one appears.
  • *// combine with the stack top immediately; +/− defer.
  • The result is the sum of the stack.

Interview tip

🗣️ When this comes up

The “act on the previous operator” framing is what makes precedence fall out without a second pass. With parentheses (Basic Calculator I), you add a recursion or a second stack for the running result and sign.
