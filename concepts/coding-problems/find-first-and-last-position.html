Find First and Last Position of Element in Sorted Array

The first and last index of a target in a sorted array with duplicates. Run two boundary binary searches — one biased left, one biased right.

#binary-search #arrays #medium #boundaries #duplicates #interview

The problem

Given a sorted array nums and a target, return the starting and ending indices of target, or [-1, -1] if absent. Must be O(log n).

📥 Example

nums=[5,7,7,8,8,10], target=8 → [3,4]  ·  target=6 → [-1,-1]

Brute force

Find the target with one binary search, then expand linearly to both ends — O(log n + count), which is O(n) if the target fills the array. Two boundary searches keep it strictly O(log n).

Optimized approach

Do two binary searches. For the leftmost occurrence, when nums[mid] == target keep searching left (hi = mid − 1) while remembering the index. For the rightmost, on a match search right (lo = mid + 1). Each is a standard binary search with the equality case biased toward one side.

Time O(log n), Space O(1).

C# solution

C# 
public int[] SearchRange(int[] nums, int target)
{
    return new[] { Bound(true), Bound(false) };

    int Bound(bool leftmost)
    {
        int lo = 0, hi = nums.Length - 1, idx = -1;
        while (lo <= hi)
        {
            int mid = lo + (hi - lo) / 2;
            if (nums[mid] == target)
            {
                idx = mid;
                if (leftmost) hi = mid - 1; else lo = mid + 1;   // bias the search
            }
            else if (nums[mid] < target) lo = mid + 1;
            else hi = mid - 1;
        }
        return idx;
    }
}

Complexity

⏱️ Complexity

  • Time: O(log n) — two binary searches.
  • Space: O(1).

Pattern & takeaways

🎯 Remember these three

  • Find boundaries with two biased binary searches.
  • On a match, keep going left (first) or right (last), recording the index.
  • Strictly O(log n) — don’t expand linearly from a found index.
