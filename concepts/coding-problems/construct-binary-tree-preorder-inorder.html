Construct Binary Tree from Preorder and Inorder

Preorder hands you each root first; inorder tells you the left/right split. Together they pin down a unique tree.

#trees #recursion #divide-and-conquer #medium #traversal #interview

The problem

Given two arrays preorder and inorder of a binary tree’s traversals (unique values), construct and return the tree.

📥 Example

preorder = [3,9,20,15,7], inorder = [9,3,15,20,7] → the tree [3,9,20,null,null,15,7]

The idea

The first element of preorder is always the current root. Find that value’s index in inorder: everything to its left is the left subtree, everything to its right is the right subtree, and the count of left elements tells you how to split the remaining preorder. Recurse on each side. A hash map from value → inorder index makes the split O(1) instead of a scan.

Time O(n), Space O(n).

C# solution

C# 
public TreeNode BuildTree(int[] preorder, int[] inorder)
{
    var idx = new Dictionary<int, int>();
    for (int i = 0; i < inorder.Length; i++) idx[inorder[i]] = i;
    int pre = 0;
    return Build(0, inorder.Length - 1);

    TreeNode Build(int lo, int hi)
    {
        if (lo > hi) return null;
        int rootVal = preorder[pre++];           // next preorder value is the root
        var node = new TreeNode { val = rootVal };
        int mid = idx[rootVal];                  // split point in inorder
        node.left  = Build(lo, mid - 1);         // build left first (preorder order)
        node.right = Build(mid + 1, hi);
        return node;
    }
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node built once, O(1) split via the map.
  • Space: O(n) — the index map plus recursion.

Pattern & takeaways

🎯 Remember these three

  • Preorder → roots; inorder → left/right boundary.
  • Build the left subtree first to consume preorder in the right order.
  • A value→inorder-index map turns the split from O(n) to O(1).

Interview tip

🗣️ When this comes up

State the two roles clearly: preorder yields roots in order, inorder locates the split. The shared pre pointer and left-first recursion are the bits people get wrong — and the map is the optimization they look for.
