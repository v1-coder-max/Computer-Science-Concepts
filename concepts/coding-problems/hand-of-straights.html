Hand of Straights

Can the hand split into groups of w consecutive cards? Greedily start each group at the smallest remaining card and consume upward.

#greedy #hash-map #sorting #medium #consecutive #interview

The problem

Given hand and a group size groupSize, return true if the cards can be rearranged into groups of groupSize consecutive values.

📥 Example

hand = [1,2,3,6,2,3,4,7,8], groupSize = 3 → true ([1,2,3],[2,3,4],[6,7,8])

The idea

The smallest remaining card must begin a group (nothing smaller can include it). So count card frequencies, then repeatedly take the minimum present card x and try to remove one each of x, x+1, …, x+groupSize−1. If any is missing, it’s impossible. A sorted structure (or a min-heap / SortedDictionary) gives the smallest card quickly.

Time O(n log n), Space O(n).

C# solution

C# 
public bool IsNStraightHand(int[] hand, int groupSize)
{
    if (hand.Length % groupSize != 0) return false;

    var count = new SortedDictionary<int, int>();
    foreach (int c in hand) count[c] = count.GetValueOrDefault(c) + 1;

    while (count.Count > 0)
    {
        int start = count.Keys.First();          // smallest remaining card
        for (int x = start; x < start + groupSize; x++)
        {
            if (!count.TryGetValue(x, out int cnt)) return false;
            if (cnt == 1) count.Remove(x);
            else count[x] = cnt - 1;
        }
    }
    return true;
}

Complexity

⏱️ Complexity

  • Time: O(n log n) — sorted map operations across n cards.
  • Space: O(n) — the count map.

Pattern & takeaways

🎯 Remember these three

  • The smallest remaining card must start a group — that forces the greedy.
  • Track counts in a sorted map / min-heap to grab the minimum fast.
  • Early-out if n % groupSize != 0.

🗣️ When this comes up

The greedy proof is one line: the minimum card can only be the start of a group, so its group is forced. From there it’s bookkeeping. The same idea solves “Divide Array in Sets of K Consecutive.”
