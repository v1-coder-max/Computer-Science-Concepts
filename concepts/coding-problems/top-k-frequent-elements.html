Top K Frequent Elements

Return the k values that appear most often. Count once, then either bucket by frequency for O(n) or keep a size-k heap for O(n log k).

#hashing #bucket-sort #heap #medium

The problem

Given an integer array nums and an integer k, return the k most frequent elements (in any order).

📥 Example

nums = [1,1,1,2,2,3], k = 2 → [1,2]

Brute force

Count frequencies and sort the distinct values by count — O(n log n). Fine, but sorting is more than we need: frequencies are bounded by n, which lets us avoid a comparison sort entirely.

Optimized approach

Build a frequency map in O(n). Then bucket sort: create an array indexed by frequency (0…n), and drop each value into the bucket for its count. Walk buckets from highest frequency down, collecting values until you have k. Because there are at most n possible frequencies, this is O(n). (A size-k min-heap is the O(n log k) alternative — better when k is tiny relative to the number of distinct values.)

Time O(n), Space O(n).

C# solution

C# — bucket sort 
public int[] TopKFrequent(int[] nums, int k)
{
    var freq = new Dictionary<int, int>();
    foreach (int n in nums)
        freq[n] = freq.GetValueOrDefault(n) + 1;

    // buckets[f] = values that occur exactly f times
    var buckets = new List<int>[nums.Length + 1];
    foreach (var (val, f) in freq)
        (buckets[f] ??= new List<int>()).Add(val);

    var result = new List<int>();
    for (int f = buckets.Length - 1; f >= 0 && result.Count < k; f--)
        if (buckets[f] != null)
            result.AddRange(buckets[f]);

    return result.Take(k).ToArray();
}

Complexity

⏱️ Complexity

  • Time: O(n) bucket sort (O(n log k) for the heap variant).
  • Space: O(n) — frequency map plus buckets.

Pattern & takeaways

🎯 Remember these three

  • Count then organize: a hash map followed by bucketing or a heap.
  • Bucket sort beats comparison sort when the key range (here, frequency) is bounded by n.
  • Use a size-k heap instead when k ≪ distinct count or data streams in.

Interview tip

🗣️ When this comes up

Offer both: the O(n log k) heap (simple, classic top-k) and the O(n) bucket sort (the “can you beat n log n?” answer). Knowing when each wins — small k vs. bounded key range — is what impresses.
