Copy List with Random Pointer

Deep-copy a list where each node also has a random pointer anywhere in the list. A map from original → clone makes wiring the randoms trivial.

#linked-list #hash-map #medium #deep-copy #pointers #interview

The problem

Each node has next and a random pointer (to any node or null). Return a deep copy: new nodes with the same next and random structure, sharing nothing with the original.

📥 Example

A list of nodes with tangled random pointers → an identical independent copy.

Brute force

You can’t set a clone’s random until that target’s clone exists — naively that means repeated scans to locate clones, O(n²). A hash map (or the interleaving trick) gives O(1) clone lookup.

Optimized approach

Two passes with a hash map. Pass 1: create a clone for every node and store original → clone. Pass 2: for each original, set clone.next = map[orig.next] and clone.random = map[orig.random]. The map turns “find this node’s clone” into O(1). An O(1)-space variant interleaves each clone right after its original, copies randoms via orig.next, then unzips the two lists.

Time O(n), Space O(n) (O(1) with interleaving).

C# solution

C# 
public Node CopyRandomList(Node head)
{
    if (head == null) return null;
    var map = new Dictionary<Node, Node>();

    for (var cur = head; cur != null; cur = cur.next)   // pass 1: clones
        map[cur] = new Node(cur.val);

    for (var cur = head; cur != null; cur = cur.next)   // pass 2: wire pointers
    {
        map[cur].next   = cur.next   != null ? map[cur.next]   : null;
        map[cur].random = cur.random != null ? map[cur.random] : null;
    }
    return map[head];
}
// class Node { public int val; public Node next, random; public Node(int v){ val = v; } }

Complexity

⏱️ Complexity

  • Time: O(n) — two linear passes.
  • Space: O(n) for the map (O(1) with interleaving).

Pattern & takeaways

🎯 Remember these three

  • A original→clone map resolves random targets in O(1).
  • Clone all nodes first, then wire next and random.
  • The interleave-then-unzip trick removes the map for O(1) space.
