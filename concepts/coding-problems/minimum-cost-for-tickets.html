Minimum Cost For Tickets

Walk forward day by day; on a travel day pick whichever pass (1-, 7-, or 30-day) ending here is cheapest, looking back the right number of days.

#dynamic-programming #medium

The problem

Given the days you travel and the costs of 1-, 7-, and 30-day passes, return the minimum total cost to cover all travel days.

📥 Example

days = [1,4,6,7,8,20], costs = [2,7,15] → 11

Brute force

Trying all pass combinations is exponential. A DP over calendar days is linear in the last travel day.

Optimized approach

Let dp[d] be the min cost through day d. On a non-travel day, dp[d] = dp[d−1]. On a travel day, dp[d] = min(dp[d−1]+cost1, dp[d−7]+cost7, dp[d−30]+cost30) — each option covers day d with a pass bought the corresponding span earlier.

Time O(maxDay), Space O(maxDay).

C# solution

C# 
public int MincostTickets(int[] days, int[] costs)
{
    var travel = new HashSet<int>(days);
    int last = days[^1];
    var dp = new int[last + 1];
    for (int d = 1; d <= last; d++)
    {
        if (!travel.Contains(d)) { dp[d] = dp[d - 1]; continue; }
        dp[d] = Math.Min(dp[d - 1] + costs[0],
                Math.Min(dp[Math.Max(0, d - 7)] + costs[1],
                         dp[Math.Max(0, d - 30)] + costs[2]));
    }
    return dp[last];
}

Complexity

⏱️ Complexity

  • Time: O(maxDay) — one pass over the calendar.
  • Space: O(maxDay) — the DP array.

Pattern & takeaways

🎯 Remember these three

  • Index DP by calendar day, not by travel-day position.
  • On a travel day, look back 1/7/30 days for each pass option.
  • Non-travel days just carry the previous cost forward.

Interview tip

🗣️ When this comes up

Indexing by day (not by travel-day index) is the cleanest formulation. Each travel day takes the min over the three look-back distances; non-travel days carry forward.
