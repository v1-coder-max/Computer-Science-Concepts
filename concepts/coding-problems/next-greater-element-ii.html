Next Greater Element II

For each element in a circular array, find the next greater one. A monotonic stack — iterated twice around — resolves each in O(n).

#stack #monotonic-stack #arrays #medium #circular #interview

The problem

Given a circular integer array nums, return an array where each position holds the next greater number going forward (wrapping around), or -1 if none exists.

📥 Example

[1,2,1] → [2,-1,2]

Brute force

For each element, scan forward (wrapping) until a greater one appears — O(n²). A monotonic stack resolves each element exactly once for O(n).

Optimized approach

Keep a stack of indices whose values are awaiting a next-greater element, decreasing down the stack. Iterate i from 0 to 2n−1, using nums[i % n] to simulate the circular wrap. While the current value is greater than the value at the stack’s top index, pop and set that index’s answer to the current value. Push indices only during the first pass (i < n); the second pass just resolves leftovers via the wrap.

Time O(n), Space O(n).

C# solution

C# 
public int[] NextGreaterElements(int[] nums)
{
    int n = nums.Length;
    var result = new int[n];
    Array.Fill(result, -1);
    var stack = new Stack<int>();        // indices, decreasing values

    for (int i = 0; i < 2 * n; i++)
    {
        int val = nums[i % n];
        while (stack.Count > 0 && nums[stack.Peek()] < val)
            result[stack.Pop()] = val;   // current is the next greater
        if (i < n) stack.Push(i);         // push only on the first pass
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each index pushed/popped once.
  • Space: O(n) for the stack and result.

Pattern & takeaways

🎯 Remember these three

  • “Next greater” → monotonic stack of indices.
  • Handle the circle by iterating 2n with i % n.
  • Push only during the first pass; the second resolves wrap-arounds.
