Encode and Decode Strings

Pack a list of strings into one string and recover it exactly — even if strings contain your delimiter. Length-prefix each one and parsing is unambiguous.

#strings #design #medium #serialization #length-prefix #interview

The problem

Design Encode(list) → a single string and Decode(s) → the original list, so encoding then decoding round-trips any list of strings (including ones with special characters).

📥 Example

["lint","code"] → "4#lint4#code" → ["lint","code"]

Brute force

Joining with a delimiter like , breaks if a string contains that delimiter. Escaping delimiters works but is fiddly. Length-prefixing sidesteps the problem entirely.

Optimized approach

Encode each string as length + a delimiter + the string (e.g. "4#lint"). To decode, read digits up to the # to get the length L, then take exactly the next L characters — regardless of what they are — and repeat. Because you always know how many characters to read, the content can contain any character, including #.

Time O(total length), Space O(total length).

C# solution

C# 
public string Encode(IList<string> strs)
{
    var sb = new System.Text.StringBuilder();
    foreach (var s in strs) sb.Append(s.Length).Append('#').Append(s);
    return sb.ToString();
}

public IList<string> Decode(string s)
{
    var result = new List<string>();
    int i = 0;
    while (i < s.Length)
    {
        int j = i;
        while (s[j] != '#') j++;                 // read the length
        int len = int.Parse(s.Substring(i, j - i));
        result.Add(s.Substring(j + 1, len));     // take exactly len chars
        i = j + 1 + len;
    }
    return result;
}

Complexity

⏱️ Complexity

  • Time: O(total length) to encode and decode.
  • Space: O(total length).

Pattern & takeaways

🎯 Remember these three

  • Length-prefix beats delimiter-joining — no escaping needed.
  • Decode reads the length, then takes exactly that many characters.
  • Robust to any character in the payload, including the delimiter.
