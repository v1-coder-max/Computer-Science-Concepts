Flatten Binary Tree to Linked List

Flatten a tree into a right-leaning linked list following preorder, in place. A reverse-preorder walk prepends each node for O(1) extra space.

#trees #recursion #medium #in-place #preorder #interview

The problem

Given the root of a binary tree, flatten it in place into a linked list: each node’s right points to the next preorder node and left is null.

📥 Example

[1,2,5,3,4,null,6] → 1→2→3→4→5→6 (all via right pointers)

Brute force

Collect nodes via a preorder traversal into a list, then relink them — O(n) time and O(n) space. The reverse-preorder trick flattens in place with only recursion stack space.

Optimized approach

Traverse in reverse preorder (right, then left, then node) while keeping a prev pointer to the previously processed node. At each node, set node.right = prev, node.left = null, then update prev = node. Because you visit nodes in the reverse of preorder, each node ends up pointing to its correct preorder successor. (A Morris-style O(1) version moves left subtrees to the right inline.)

Time O(n), Space O(h).

C# solution

C# 
private TreeNode prev = null;

public void Flatten(TreeNode root)
{
    if (root == null) return;
    Flatten(root.right);          // reverse preorder: right first
    Flatten(root.left);
    root.right = prev;            // link to the previously seen node
    root.left = null;
    prev = root;
}

Complexity

⏱️ Complexity

  • Time: O(n) — each node visited once.
  • Space: O(h) recursion (O(1) with the Morris variant).

Pattern & takeaways

🎯 Remember these three

  • Reverse-preorder + a prev pointer flattens in place.
  • Set right = prev, null out left, then advance prev.
  • The Morris-style splice gives true O(1) extra space.
